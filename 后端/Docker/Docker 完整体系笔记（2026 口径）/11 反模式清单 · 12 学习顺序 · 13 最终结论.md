# §11 反模式清单（高频踩坑）

> [!important]
> 
> 以下是 Docker 项目中**最常见、最危险的反模式**。每一条都有实际踩坑案例。如果你在自己的项目中发现了其中任何一条，应当立即修正。

|编号|反模式|风险|正确做法|
|---|---|---|---|
|1|**一个巨大容器塞下所有服务**|无法独立扩缩容、更新、调试|按职责拆分为独立服务|
|2|**Compose 文件混乱复制，失去单一真相源**|配置漂移、不一致|1 主 compose.yaml + 场景覆盖文件|
|3|**所有服务** `**0.0.0.0**` **暴露端口**|攻击面暴增|仅入口层暴露，绑定 `127.0.0.1`|
|4|**生产环境 bind mount 整个项目目录**|容器可读写宿主机所有项目文件|生产用镜像内代码 + volume 数据|
|5|**用** `**ARG**` **/** `**ENV**` **传 secret**|机密写入镜像层/元数据|构建期 `--mount=type=secret`，运行期 Compose `secrets`|
|6|**长期靠** `**docker exec -it**` **手工维护**|状态不可复现|容器可删除可重建，配置和数据外置|
|7|**用** `**depends_on**` **误以为服务已 ready**|启动失败、竞态条件|`depends_on` • `healthcheck` • `condition: service_healthy`|
|8|**用 host 网络掩盖网络拓扑理解不足**|隔离丧失、端口冲突|用户自定义 bridge + 显式端口发布|
|9|**用** `**privileged: true**` **解决权限问题**|容器获得宿主机几乎全部权限|按需 `cap_add` 特定 capabilities|
|10|**普通业务容器挂载 Docker socket**|容器可控制宿主机所有容器|受控代理或专用 API|
|11|**不写** `**.dockerignore**`|build context 臃肿、缓存失效|配置 `.dockerignore` 排除无关文件|
|12|**Dockerfile 不做多阶段构建**|镜像臃肿、攻击面大|builder + runtime 分离|
|13|**不区分构建时依赖和运行时依赖**|运行时镜像包含不需要的编译工具|多阶段构建，runtime 只含产物|

# §12 当前最值得优先掌握的能力排序

> [!important]
> 
> **工程判断**：以下按**投入产出比**排列。第一层是任何 Docker 用户必须掌握的基础；第二层是工程化项目必备；第三层是高阶可选。

## 第一层：必须会

1. Image / Container / Volume / Network 基本模型

1. Dockerfile 基础指令（FROM / COPY / RUN / USER / ENTRYPOINT / CMD）

1. Compose 基本服务编排（services / networks / volumes）

1. Bind Mount vs Volume vs tmpfs 选型

1. bridge 网络与端口发布

1. `.dockerignore` + 多阶段构建

## 第二层：工程化必会

1. healthcheck + depends_on + service_healthy

1. profiles / 多文件 merge

1. 非 root / read_only / cap_drop / no-new-privileges 安全基线

1. BuildKit cache / secret / ssh mount

1. Bake 多镜像编排

1. 多平台构建（amd64 + arm64）

## 第三层：高阶可选

1. Rootless Docker / userns-remap

1. overlay / 多主机网络

1. macvlan / ipvlan

1. 远程 daemon / docker context

1. Compose `include` 模块化

1. Compose `models` AI 模型声明

# §13 最终结论

## 当前 Docker 项目主线

> [!important]
> 
> `Dockerfile` 负责**单镜像构建**，`Compose` 负责**多服务应用模型**。`compose.yaml` 为主文件名，Compose Specification 为准，不再按旧版文件格式思维设计。

## 当前最稳的工程化组合

**多阶段 Dockerfile** + **BuildKit** + `**.dockerignore**` + `**compose.yaml**` + **多文件 merge / profiles** + **用户自定义 bridge 网络** + **volume 持久化** + **最小权限安全基线**

## 当前最常见的正确默认值

|维度|正确默认值|
|---|---|
|**持久化**|volume|
|**开发源码**|bind mount|
|**网络**|user-defined bridge|
|**多机互联**|overlay（有刚需时才用）|
|**host / macvlan / ipvlan**|明确需求时才使用|
|**端口暴露**|仅必要端口，绑定 `127.0.0.1`|
|**机密**|不进镜像，用 secret mount / Compose secrets|
|**运行用户**|非 root|
|**文件系统**|只读优先|
|**权限**|最小权限（cap_drop ALL + no-new-privileges）|
