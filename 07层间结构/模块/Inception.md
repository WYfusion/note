Inception 模块由 **GoogLeNet（Inception-v1）**（2014, Szegedy et al.）提出。

---

## 一、核心思想

将**不同尺寸的卷积核**通过**并行分支**组合，在**通道维度拼接**输出，实现多尺度特征提取。

---

## 二、设计动机

### 2.1 多尺度感受野

不同尺寸的卷积核具有不同的感受野：
- **1×1 卷积**：局部点特征
- **3×3 卷积**：小范围空间特征
- **5×5 卷积**：大范围空间特征

> **关键洞察**：网络越深，不同卷积核的感受野差异越大，多尺度特征的优势越明显。

### 2.2 感受野计算

对于堆叠的卷积层，感受野递推公式为：

$$RF_l = RF_{l-1} + (k_l - 1) \times \prod_{i=1}^{l-1} s_i$$

其中 $k_l$ 是第 $l$ 层卷积核大小，$s_i$ 是第 $i$ 层步长。

## 三、结构演进

### 3.1 朴素版本（Naive Inception）

```
       输入
    ┌───┼───┬───┐
    │   │   │   │
  1×1  3×3 5×5 MaxPool
    │   │   │   │
    └───┴───┴───┘
      Concat（通道拼接）
```

**问题**：5×5 卷积计算量过大。

### 3.2 优化版本（带降维）

在 3×3 和 5×5 卷积前添加 **1×1 卷积进行降维**：

```
            输入
    ┌───┬───┼───┬───┐
    │   │   │   │   │
  1×1  1×1 1×1  MaxPool
    │   │   │   │
    │  3×3 5×5  1×1
    │   │   │   │
    └───┴───┴───┘
      Concat（通道拼接）
```

## 四、1×1 卷积的作用

| **作用**           | **说明**                                           |
| ------------------ | -------------------------------------------------- |
| **降维**           | 减少通道数，降低后续 3×3/5×5 卷积的计算量          |
| **增加非线性**     | 每个 1×1 卷积后接 ReLU，增加网络深度和表达能力     |
| **跨通道交互**     | 实现通道间信息融合（类似全连接层作用于通道维度）   |

### 4.1 计算量对比

设输入通道 $C_{in}=256$，输出通道 $C_{out}=256$，空间尺寸 $H \times W$：

**不使用 1×1 降维（5×5 卷积）**：
$$\text{MACs} = H \times W \times C_{in} \times C_{out} \times 5^2 = 1638400 \cdot HW$$

**使用 1×1 降维（先降至 64 通道）**：
$$\text{MACs} = HW \times (256 \times 64 + 64 \times 256 \times 25) = 425984 \cdot HW$$

计算量减少约 **74%**。

## 五、Inception 变体

| **版本**       | **主要改进**                                       |
| -------------- | -------------------------------------------------- |
| Inception-v1   | 基础多分支结构 + 1×1 降维                          |
| Inception-v2   | 5×5 卷积分解为两个 3×3 卷积                        |
| Inception-v3   | 非对称卷积分解（n×n → 1×n + n×1）                  |
| Inception-v4   | 结合残差连接                                       |

---

## 六、优势总结

| **优势**           | **说明**                                       |
| ------------------ | ---------------------------------------------- |
| **多尺度特征提取** | 同时捕获不同尺度的空间信息                     |
| **计算高效**       | 1×1 降维显著减少计算量                         |
| **宽度扩展**       | 通过增加并行分支扩展网络宽度                   |
| **防止过拟合**     | 多分支结构具有一定的正则化效果                 |
