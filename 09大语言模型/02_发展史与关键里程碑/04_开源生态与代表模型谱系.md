# 开源生态与代表模型谱系

2023 年被称为“大模型元年”，很大程度上归功于开源社区的爆发。本章将梳理当前主流的开源模型家族及其背后的生态系统。

## 1. 闭源巨头 vs 开源先锋

*   **闭源 (Closed Source)**: 以 OpenAI (GPT-4), Anthropic (Claude 3), Google (Gemini) 为代表。它们提供 API，性能最强，但不公开权重。
*   **开源 (Open Weights)**: 以 Meta (Llama), Mistral, Alibaba (Qwen) 为代表。它们公开模型权重，允许开发者在本地部署和微调。

## 2. 代表性开源模型家族

### 2.1 Llama 系列 (Meta) —— 开源基石
Meta 的 Llama 系列是开源 LLM 的里程碑，定义了许多标准（如架构、数据格式）。
*   **Llama 1 (2023.2)**: 证明了在更多数据上训练的小模型（7B/13B/65B）可以战胜大模型（GPT-3 175B）。
*   **Llama 2 (2023.7)**: 允许商用，增加了 RLHF 对齐，上下文扩展到 4k。
*   **Llama 3 (2024.4)**: 性能大幅提升，词表扩大到 128k，训练数据量达到 15T Tokens。
    *   **架构特点**: [[Multi-query attention(多查询注意力)、Grouped-query attention(分组查询注意力)#^21f310|GQA]]GQA (Grouped-Query Attention), RoPE, [[SwiGLU]]。

### 2.2 Mistral 系列 (Mistral AI) —— 欧洲之光
由 DeepMind 前员工创立的法国公司。
*   **Mistral 7B**: 在小参数下性能极强，使用了 Sliding Window Attention。
*   **Mixtral 8x7B**: 普及了 **MoE (Mixture of Experts)** 架构。
    *   **MoE 原理**: 模型包含 8 个专家网络，每次推理只激活其中 2 个。
    *   **优势**: 拥有 47B 的参数量（知识容量），但推理计算量仅相当于 13B 模型（速度快）。

### 2.3 Qwen (通义千问) 系列 (Alibaba)
目前中文能力最强、综合能力全球领先的开源模型之一。
*   **特点**: 优秀的数学和代码能力，多语言支持好。
*   **Qwen-1.5 / Qwen-2**: 提供了从 0.5B 到 72B 的全尺寸模型，甚至包括 MoE 版本。

### 2.4 其他重要玩家
*   **DeepSeek (深度求索)**: 国内黑马，在 Coding 和 Math 领域表现惊人，推崇 MoE 架构 (DeepSeek-V2)。
*   **Yi (零一万物)**: 李开复团队，主打长上下文 (200k) 和强大的基座能力。
*   **Gemma (Google)**: Google 发布的轻量级开源模型，与 Gemini 同源。

### 2.5 语音大模型 (Audio LLMs)
随着多模态的发展，语音模型也迎来了爆发。
*   **Whisper (OpenAI)**: 鲁棒性极强的 ASR 模型，使用 68万小时的多语言音频训练。虽然不是 LLM，但它是语音理解的基石。
*   **VALL-E (Microsoft)**: 神经编解码语言模型。只需 3 秒参考音频，就能合成具有同样音色和情感的语音（Zero-shot TTS）。
*   **Qwen-Audio (Alibaba)**: 通用音频理解模型。能处理语音、音乐、音效等多种音频，支持语音问答。
*   **GPT-4o (OpenAI)**: 原生多模态模型（Omni）。音频不再是外挂，而是与文本、图像同等的一等公民，实现了端到端的实时语音交互。

## 3. 开源生态工具栈

有了模型权重，还需要工具来运行和微调它们。

### 3.1 模型库与托管
*   **Hugging Face (HF)**: AI 界的 GitHub。几乎所有开源模型都托管在这里。
*   **ModelScope (魔搭)**: 阿里推出的社区，国内下载速度更快。

### 3.2 推理框架 (Inference)
如何让模型跑得快、省显存？
*   **vLLM**: 目前最流行的生产级推理框架。核心技术是 **PagedAttention**（受操作系统虚拟内存启发），极大提高了吞吐量。
*   **llama.cpp**: 专注于 CPU/Mac 推理。通过 **GGUF** 格式和量化技术，让大模型能在普通笔记本甚至手机上运行。
*   **Ollama**: 基于 llama.cpp 的封装，提供了极简的命令行工具，适合小白用户。

### 3.3 微调框架 (Fine-tuning)
*   **PEFT (Parameter-Efficient Fine-Tuning)**: HF 的库，支持 LoRA, QLoRA, P-Tuning 等技术。
*   **LLaMA-Factory**: 一个可视化的微调 WebUI，集成了各种微调算法，无需写代码即可训练自己的模型。
*   **DeepSpeed / Megatron-LM**: 用于多卡/多机分布式训练的底层库。

### 3.4 量化技术 (Quantization)
将模型参数从 FP16 (16-bit) 压缩到 INT4 (4-bit)，显存占用减少 75%，性能损失微乎其微。
*   **GPTQ**: Post-Training Quantization，需要少量校准数据。
*   **AWQ**: Activation-aware Weight Quantization，比 GPTQ 更快更稳。

---

## 4. 总结：如何选择？

*   **想在本地玩玩**: 下载 Ollama + Llama 3 8B / Qwen 7B。
*   **显存有限但要高性能**: 使用 Mixtral 8x7B (MoE) 或 QLoRA 微调。
*   **生产环境部署**: 推荐 vLLM + AWQ 量化。
*   **特定领域任务**: 寻找在该领域微调过的模型（如 CodeLlama, MedLLaMA）。
