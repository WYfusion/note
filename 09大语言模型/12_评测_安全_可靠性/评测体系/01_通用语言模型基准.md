# 通用语言模型评测基准 (General LLM Benchmarks)

随着语言模型从 BERT 时代的 NLU（理解）向 GPT 时代的 NLG（生成）和 AGI（通用智能）演进，评测基准也经历了巨大的变革。

---

## 1. NLU 时代：理解能力的试金石
在 BERT、RoBERTa、T5 时期，评测主要关注模型对自然语言的**理解**能力（分类、匹配、标注）。

### **GLUE (General Language Understanding Evaluation)**
*   **发布时间**: 2018年。
*   **组成**: 9 个任务，包括情感分析 (SST-2)、文本蕴含 (MNLI, RTE)、相似度判断 (STS-B, QQP) 等。
*   **地位**: 推动了预训练模型（Pre-trained Models）的爆发。
*   **局限**: 随着模型变强，GLUE 很快被“刷爆”（超过人类水平），区分度降低。

### **SuperGLUE**
*   **发布时间**: 2019年。
*   **特点**: 针对 GLUE 饱和问题提出的更难版本。
*   **任务**: 包含 BoolQ (布尔问答)、MultiRC (多跳阅读理解)、WiC (词义消歧) 等更考验推理和常识的任务。

---

## 2. LLM 时代：知识与推理的全面评估
当模型参数突破 10B/100B，涌现出 Few-shot/Zero-shot 能力后，评测重点转向了**世界知识**、**逻辑推理**和**生成质量**。

### **MMLU (Massive Multitask Language Understanding)**
*   **简介**: 目前评估 LLM **综合知识能力**的首选基准。
*   **内容**: 涵盖 STEM、人文、社科等 **57 个学科**。
*   **形式**: 4 选 1 的多项选择题。
*   **意义**: 测试模型是否具备类似人类的广泛世界知识和解决问题的能力。
*   **现状**: GPT-4 等模型在 MMLU 上已达到 86%+ 的准确率（人类专家水平约为 89.8%）。

### **BIG-Bench (Beyond the Imitation Game Benchmark)**
*   **简介**: 由 Google 发起，旨在探索 LLM 能力边界的**超大规模**协作基准。
*   **特点**:
    *   **多样性**: 包含 200+ 个任务，从简单的算术到复杂的类比推理、甚至下棋。
    *   **BIG-Bench Hard (BBH)**: 从中筛选出 LLM 表现尚不如人类的 23 个高难度任务，常用于测试强推理模型（如 CoT 思维链）。

### **HELM (Holistic Evaluation of Language Models)**
*   **发布方**: 斯坦福大学 (CRFM)。
*   **核心理念**: **全方位 (Holistic)** 评估，不仅仅看准确率。
*   **维度**: 包含 **Accuracy (准确率)**, **Calibration (校准度)**, **Robustness (鲁棒性)**, **Fairness (公平性)**, **Bias (偏见)**, **Toxicity (毒性)**, **Efficiency (效率)** 等 7 大维度。
*   **意义**: 它是目前最严谨、最透明的评测体系之一，强调模型的安全性与可靠性。

## 3. 垂类能力基准

### **推理与数学 (Reasoning & Math)**
*   **GSM8K**: 8.5k 个高质量小学数学应用题。测试模型的多步推理能力，是评估 **CoT (Chain-of-Thought)** 能力的标准数据集。
*   **MATH**: 包含代数、几何、微积分等高中/竞赛级数学题，难度远高于 GSM8K。

### **代码生成 (Coding)**
*   **HumanEval**: OpenAI 发布的 Python 编程任务，要求模型根据函数签名和文档字符串补全代码。
*   **MBPP (Mostly Basic Python Problems)**: 谷歌发布的入门级 Python 编程题。

---

## 4. 现代化评测：Chatbot Arena
随着模型微调（SFT / RLHF）技术的发展，静态的客观题已难以完全反映人类对“好模型”的感知。

### **LMSYS Chatbot Arena**
*   **机制**: **Elo Rating (天梯积分)** 系统。
*   **方法**: **Side-by-Side (SBS)** 盲测。用户输入一个 Prompt，两个匿名模型分别生成回答，用户投票选出更好的一个。
*   **地位**: 目前公认最能反映**真实用户体验**的排行榜。

---

## 5. 总结：如何选择基准？

| 你的目标 | 推荐基准 |
| :--- | :--- |
| **测试综合知识** | MMLU, C-Eval (中文), CMMLU (中文) |
| **测试逻辑推理** | GSM8K, BBH, MATH |
| **测试代码能力** | HumanEval, MBPP |
| **测试安全性/偏见** | HELM, RealToxicityPrompts |
| **测试用户体验** | Chatbot Arena (Elo) |
*   $I$: 插入 (Insertions)
*   $N$: 参考文本的总词数

### 3.2 生成质量指标
*   **FAD (Fréchet Audio Distance)**: 越低越好。计算生成音频与真实音频在 Embedding 空间（通常使用 VGGish 或 CLAP）的分布距离。
*   **KL Divergence**: 衡量生成音频的类别分布与真实分布的差异。
*   **Inception Score (IS)**: 衡量生成样本的多样性和清晰度（在音频领域较少用，逐渐被 FAD 取代）。
