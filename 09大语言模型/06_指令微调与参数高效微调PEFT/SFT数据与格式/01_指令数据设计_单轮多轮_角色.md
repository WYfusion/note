# 指令数据设计：单轮、多轮与角色扮演

高质量的指令数据（Instruction Data）是 SFT 成功的关键。对于语音大模型，数据设计还需要考虑模态的特殊性。

## 1. 数据结构基础

一条标准的指令数据通常包含三个部分：
*   **Instruction (指令)**: 告诉模型要做什么。
*   **Input (输入)**: (可选) 任务的具体上下文或素材。
*   **Output (输出)**: 期望的模型回答。

### 1.1 JSON 格式示例
```json
{
  "instruction": "请将下面的句子翻译成英文。",
  "input": "今天天气真好。",
  "output": "The weather is very good today."
}
```

## 2. 单轮 vs 多轮对话

### 2.1 单轮对话 (Single-turn)
*   **结构**: User -> Assistant。
*   **适用**: 问答、翻译、摘要等一次性任务。

### 2.2 多轮对话 (Multi-turn)
*   **结构**: User -> Assistant -> User -> Assistant ...
*   **挑战**: 模型需要维护上下文历史（Context History）。
*   **训练方式**:
    *   将整个对话拼接成一个长序列。
    *   **Masking**: 只对 Assistant 的回复部分计算 Loss，User 的发言部分不计算 Loss。

## 3. 语音指令数据设计

语音大模型的指令数据比纯文本更复杂，因为 Input 可能包含音频路径或特殊的 Audio Token。

### 3.1 显式音频标记
在 Instruction 或 Input 中使用特殊标记（如 `<Audio>`）来占位。

**示例：语音问答 (ASR + QA)**
```json
{
  "instruction": "请回答这段录音中提到的问题。",
  "input": "<Audio>path/to/question.wav</Audio>",
  "output": "录音中询问了珠穆朗玛峰的高度，答案是 8848.86 米。"
}
```

### 3.2 语音描述与分析
**示例：语音情感分析**
```json
{
  "instruction": "分析这段语音中说话人的情绪。",
  "input": "<Audio>path/to/angry_speech.wav</Audio>",
  "output": "说话人的语速较快，音调较高，且使用了愤怒的语气词，推测其情绪为愤怒。"
}
```

### 3.3 跨模态生成 (TTS 指令)
**示例：风格化语音合成**
```json
{
  "instruction": "请用悲伤的语气朗读这句话。",
  "input": "Text: 我把钱包弄丢了。\nReference Audio: <Audio>path/to/sad_ref.wav</Audio>",
  "output": "<Audio>path/to/generated_sad.wav</Audio>"
}
```
*注意：在训练生成模型时，Output 通常是音频的离散 Token 序列。*

## 4. 角色扮演 (Role Playing)

通过 System Prompt 或特定的 Instruction 设定模型的人设。

### 4.1 文本人设
"你是一个资深的 Python 程序员..."

### 4.2 语音人设
对于语音模型，人设不仅包含知识，还包含**声音特征**。
*   **Instruction**: "你是一个老年的讲故事的人，声音沙哑但亲切。"
*   **实现**: 除了文本 Prompt，可能还需要输入一段对应的 Speaker Embedding 或 Prompt Audio 作为条件。
