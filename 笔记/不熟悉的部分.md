### 批量安装软件包

```bash
pip install -r .\requirements.txt
```

.\requirements.txt是当前目录下的requirements.txt文件，该文件里面包含了要求的软件包（一列）

### 删除指定的环境

```bash
 # 列出所有环境（可选）
conda env list
 # 删除名为 "myenv" 的环境
conda remove --name myenv --all
```

### **导出环境为 YAML 文件**：

使用 `conda env export` 命令将当前激活的环境导出为一个 YAML 格式的文件。例如，输入 `conda env export > environment.yml`，这将创建一个名为 `environment.yml` 的文件，其中包含了环境的所有依赖包及其版本信息。

```bash
conda env export > environment.yml
```



### **在目标主机上创建环境**：

将 `environment.yml` 文件复制到目标主机上，然后在目标主机的 Conda 终端中，使用 `conda env create -f environment.yml` 命令来根据 YAML 文件创建环境。这将重建与源主机上相同的环境。

```bash
conda env create -f environment.yml
```

### 3. 初始化本地项目

在命令行中导航到你的项目目录，执行以下命令来初始化 Git 仓库：

```bash
cd /path/to/your/project  # 替换为你的项目路径
git init
```

### 4. 添加项目文件到 Git

使用以下命令将项目文件添加到 Git 暂存区：

```bash
git add .
```

### 5. 提交更改

提交更改并添加提交信息：

```bash
git commit -m "Initial commit"  # 替换为适合的提交信息
```

### 6.查看仓库状态和历史

1. **查看仓库状态**：
   - 使用命令`git status`来查看当前工作目录和暂存区的状态。
2. **查看修改内容**：
   - 使用命令`git diff`来查看工作目录与暂存区之间或暂存区与最近一次提交之间的差异。
3. **查看提交历史**：
   - 使用命令`git log`来查看所有提交的历史记录。

### 7.管理分支

1. **创建分支**：
   - 使用命令`git branch <new-branch>`来创建一个新的分支 只需要写`<>`内的部分就行了。
2. **切换分支**：
   - 使用命令`git checkout <branch-name>`来切换到指定的分支。
   - 使用命令`git checkout -b <new-branch>`来创建并切换到一个新的分支。
3. **合并分支**：
   - 使用命令`git merge <branch-name>`来将指定分支的更改合并到当前分支。

### 8. 连接到 GitHub 仓库

将本地仓库与 GitHub 仓库连接起来。使用以下命令（将 `your_username` 和 `your_repository` 替换为你的 GitHub 用户名和仓库名称）：

```bash
git remote add origin https://github.com/your_username/your_repository.git
```

### 9. 推送代码到 GitHub

使用以下命令将本地代码推送到 GitHub 仓库：

```bash
git push -u origin master   # 如果你在本地使用 master 分支
git push -u origin main		# 如果你在本地使用 main 分支
git pull					# 在
```

### 10.解决冲突和回滚版本

1. **解决冲突**：
   - 当推送或拉取更改时出现冲突，需要手动解决冲突文件中的内容。
   - 使用`git add <resolved-file>`来标记冲突已解决的文件。
2. **回滚版本**：
   - 使用命令`git reset --hard <commit-id>`来将仓库回滚到指定的提交。
   - 注意，回滚版本会丢失当前版本与指定版本之间的所有更改，因此需要谨慎使用。
3. **查看命令历史**：
   - 使用命令`git reflog`来查看所有Git命令的历史记录，以便在需要时恢复到某个状态。

## 切换远程仓库

### 一、查看当前远程仓库

首先，使用以下命令查看当前关联的远程仓库：

```bash
git remote -v
```

这个命令会列出所有远程仓库的URL和对应的别名（通常是`origin`）。

### 二、移除当前远程仓库（如果需要）

如果你想要切换到一个新的远程仓库，并且不再需要当前的远程仓库，可以使用以下命令移除它：

```bash
git remote remove origin
```

这里的`origin`是远程仓库的别名，如果你的远程仓库使用了不同的别名，请替换为相应的别名。

### 三、添加新的远程仓库

接下来，使用以下命令添加新的远程仓库：

```bash
git remote add origin <新的远程仓库地址>
```

请将`<新的远程仓库地址>`替换为新的远程仓库的URL。这样，你就将本地仓库与新的远程仓库关联起来了。

### 四、验证新的远程仓库

为了确认新的远程仓库已经成功添加，你可以再次使用以下命令查看远程仓库：

```bash
git remote -v
```

此时，你应该能看到新的远程仓库地址。

### 五、推送本地更改到新的远程仓库

如果你需要将本地的更改推送到新的远程仓库，可以使用以下命令：

```bash
git push -u origin <分支名>
```

请将`<分支名>`替换为你要推送的分支名称。如果你想要推送所有分支，可以使用`--all`选项，但请注意这可能会覆盖远程仓库中的分支。
