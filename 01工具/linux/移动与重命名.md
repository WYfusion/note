# 移动与重命名

本节介绍如何移动文件/目录，以及如何对它们进行重命名。在 Linux 中，这两个操作都使用同一个命令：`mv`。

> **前置场景**：假设我们当前位于 `/home/user/project_alpha/` 目录下。
> - `src/main.py` 存在。
> - `data/raw.csv` 存在。

## `mv` - 移动或重命名
**M**o**v**e。

### 1. 重命名文件
将源文件移动到同一目录下的新名称，即为重命名。

- **示例**：将 `src/main.py` 重命名为 `src/app.py`。
  ```bash
  mv src/main.py src/app.py
  ```

### 2. 移动文件
将文件移动到另一个目录中。

- **示例**：将 `data/raw.csv` 移动到 `docs/` 目录下。
  ```bash
  mv data/raw.csv docs/
  ```

- **移动并重命名**：
  将文件移动到新目录的同时更改文件名。
  ```bash
  # 将 config.py 移动到 docs 目录并改名为 configuration.py
  mv src/config.py docs/configuration.py
  ```

### 3. 移动目录
将一个目录及其所有内容移动到另一个位置。

- **示例**：将 `logs` 目录移动到 `data` 目录下。
  ```bash
  mv logs data/
  # 结果：/home/user/project_alpha/data/logs/
  ```

### 4. 常用选项

- **交互模式 (`-i`)**：
  如果目标位置已存在同名文件，提示是否覆盖（防止误操作）。
  ```bash
  mv -i notes.md docs/
  # 如果 docs/notes.md 已存在，会提示：mv: overwrite 'docs/notes.md'?
  ```

- **强制覆盖 (`-f`)**：
  不提示，直接覆盖（慎用）。
  ```bash
  mv -f update.log logs/current.log
  ```

- **不覆盖 (`-n`)**：
  如果目标文件已存在，则不进行移动。
  ```bash
  mv -n important.txt backup/
  ```
