## 是什么
基于检索到的证据片段，由 LLM 生成最终的回答。这是用户直接看到的输出环节。
## 为什么重要
- 前面所有环节的工作最终都体现在生成质量上
- 需要在**忠实性**（只说有证据的）和**流畅性**（读起来通顺）之间平衡
## 核心设计要素
### Prompt 设计
提示词决定生成行为。典型结构：
`[系统指令] + [检索到的证据] + [用户问题] → LLM 生成回答`
关键指令包括：
- 只基于提供的证据回答
- 如果证据不足则明确说明
- 引用证据来源

### Answer Synthesis（答案合成）
当多个片段包含部分答案时，需要将信息整合为一个完整、连贯的回答。
### Grounded Generation（有据生成）
强制模型的每句话都有对应的证据支撑，减少幻觉。
### Streaming Generation（流式生成）
边生成边返回，降低用户感知延迟。

## 常见问题
- **幻觉**：生成了证据中不存在的信息 → 需要 Verification 模块
- **遗漏**：忽略了某些相关证据 → 需要优化 Prompt 和上下文排列
- **矛盾**：证据之间冲突时生成了矛盾回答 → 需要冲突检测策略

## 与其他模块的关系
- **输入** ← 接收 Context Compression 处理后的精炼上下文
- **输出** → 将生成结果传递给 Verification 进行校验
