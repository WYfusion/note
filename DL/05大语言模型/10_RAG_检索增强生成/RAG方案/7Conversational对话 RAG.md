## 是什么
带有对话记忆的 RAG 方案。在多轮对话中，利用历史消息重写当前查询，使检索能理解上下文中的指代和省略。

## 核心流程
`历史消息 + 当前问题 → query rewrite（消解指代）→ 检索 → 生成`

## 为什么需要对话记忆
多轮对话中用户经常使用指代词和省略：

- 第 1 轮：`"RAG 是什么？"` → 正常检索
- 第 2 轮：`"它和微调有什么区别？"` → **「它」指什么？**
- 第 3 轮：`"哪个更适合企业用？"` → **「哪个」指什么？**

不做 query rewrite，第 2、3 轮的检索会完全失效。

## 工作原理
1. 维护对话历史（最近 N 轮消息）
2. 将历史 + 当前问题送入 LLM，生成一个**独立的、自包含的**检索查询
3. 用重写后的查询进行检索
4. 基于检索结果生成回答

## Query Rewrite 示例
历史：`"RAG 是什么？" → "它和微调有什么区别？"`
重写：`"RAG 与模型微调（fine-tuning）的区别是什么？"`

## 记忆管理
- **窗口策略**：只保留最近 N 轮
- **摘要策略**：用 LLM 将历史压缩为摘要
- **混合策略**：最近几轮完整保留 + 更早的做摘要

## 适用场景
- 客服对话
- 知识库多轮问答
- 任何需要连续追问的交互场景
