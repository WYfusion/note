## 定位

- **不是近似 attention，是把 exact attention 分布式做出来**

- 当单卡放不下超长上下文时的 exact 路线核心方案

- 常与 FlashAttention、activation checkpointing 联合使用

---

## 两大方向

### 4.1 Ring Attention / Blockwise Exact Attention

**核心思想**：

- 将长序列按 block 分布到 $P$ 个设备

- 每个设备持有自己的 $Q$ 块，$K, V$ 块在设备间以 ring 方式传递

- 在等待通信的同时做当前块的 attention 计算 → **通信-计算重叠**

- 使用 online softmax（同 [[1 FlashAttention 系列|FlashAttention]]）增量更新输出

**关键公式**：

- 每个设备 $p$ 持有 $Q_p$，接收 $K_j, V_j$（$j$ 轮转）

- 输出：$O_p = \sum_j \text{diag}(\alpha_{p,j}) \cdot \text{softmax}(Q_p K_j^T) V_j$

- 其中 $\alpha_{p,j}$ 是 online softmax 的 rescaling 系数

**代表**：

- Ring Attention（Liu et al., 2023）

- Striped Attention（Brandon et al., 2023）— 解决 causal mask 下的负载不均

**Striped Attention 改进**：

- Causal attention 中，后面的 token attend 更多 → 三角形计算量不均

- Striped 将 token 按条纹分布（而非连续块），使每个设备工作量均衡

---

### 4.2 Context / Sequence Parallelism

**核心**：

- 按序列维度切分激活与计算

- 降低单卡激活显存，使更长序列可训练

- 常与 TP/PP/ZeRO/FSDP 联用

**工程定位**：

- 这是 **训练系统优化**，不是新的注意力数学

- 长序列训练几乎绑定使用

- 常与 FlashAttention + activation checkpointing + ring-like communication 一起出现

---

## 工程判断

|方案|本质|是否近似|核心收益|
|---|---|---|---|
|Ring Attention|跨设备 blockwise exact attention|❌ Exact|突破单卡显存限制|
|Striped Attention|Ring + 负载均衡|❌ Exact|causal 场景吞吐提升|
|Sequence Parallelism|序列维度切分激活|❌ Exact|降单卡激活显存|
