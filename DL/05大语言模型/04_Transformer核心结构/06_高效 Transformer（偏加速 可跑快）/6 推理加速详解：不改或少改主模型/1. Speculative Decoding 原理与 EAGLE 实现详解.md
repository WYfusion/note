## 概述

Speculative Decoding 通过小 draft 模型并行猜测多个 token、大模型一次验证的方式，减少自回归解码步数。本页详述数学原理、rejection sampling 保证、以及 EAGLE 系列的工程实现。

---

## 1. 经典 Speculative Decoding

### 1.1 问题：自回归瓶颈

- LLM decode 每步生成 1 个 token

- 每步需完整前向传播（memory-bound）

- 生成 $T$ 个 token 需 $T$ 次前向 → **串行瓶颈**

### 1.2 算法流程

**给定**：大模型 $M_p$（target），小模型 $M_q$（draft），猜测长度 $\gamma$

**Step 1：Draft 阶段**

- $M_q$ 自回归生成 $\gamma$ 个候选 token：$hat{x}_1, hat{x}_2, dots, hat{x}_gamma$

- 记录每步的 draft 分布 $q(x_t | x_{<t})$

**Step 2：Verify 阶段**

- $M_p$ 对前缀 + 所有候选 token **一次前向**

- 得到 target 分布 $p(x_t | x_{<t})$，$t = 1, dots, gamma$

**Step 3：Rejection Sampling**

- 对每个位置 $t$，以概率 $\min(1, \frac{p(\hat{x}_t)}{q(\hat{x}_t)})$ 接受

- 遇到第一个拒绝就停止

- 从拒绝位置用修正分布采样一个新 token

### 1.3 数学保证

**定理**：经过 rejection sampling，输出的 token 序列严格服从 $M_p$ 的分布。

$$p_{\text{accept}}(x) = \min\left(1, \frac{p(x)}{q(x)}\right) \cdot q(x)$$

被拒绝时从修正分布采样：

$$p_{\text{resample}}(x) = \text{norm}\left(\max(0, p(x) - q(x))\right)$$

**关键**：输出分布 = $p_{\text{accept}} + p_{\text{reject}} \cdot p_{\text{resample}} = p(x)$

### 1.4 期望加速比

设每个位置的平均接受率为 $alpha$，则：

- 期望接受 token 数 = $\frac{1 - \alpha^{\gamma+1}}{1 - \alpha}$

- 加速比 ≈ $\frac{\text{期望接受数}}{1 + \gamma \cdot c}$

其中 $c = \frac{\text{draft 前向时间}}{\text{target 前向时间}}$

## 2. EAGLE：Feature-Level Draft

### 2.1 核心洞察

- **Token-level** 自回归预测很难（draft 模型太小）

- **Feature-level**（倒数第二层隐状态）自回归更容易

- 因为 feature 比 token 更 "平滑"、更可预测

### 2.2 架构

```JavaScript
┌─────────────────────────────────────┐
│  Target LLM (frozen)                │
│  ┌──────┐  ┌──────┐  ┌──────┐      │
│  │Layer1│→ │......│→ │LayerN│→ LMHead │
│  └──────┘  └──────┘  └──┬───┘      │
│                          │ feature  │
│                          ↓          │
│  ┌────────────────────────┐         │
│  │  EAGLE Draft Head      │         │
│  │  (轻量 Transformer)     │         │
│  │  输入: feature + embed  │         │
│  │  输出: next feature     │         │
│  └──────────┬─────────────┘         │
│             ↓ → LMHead → draft token│
└─────────────────────────────────────┘
```

- Draft head 输入：target 模型倒数第二层的 feature + token embedding

- Draft head 自回归预测下一个 feature

- 通过共享 LM head 映射回 token

### 2.3 优势

- **接受率高**：feature-level 预测比 token-level 更准

- **训练简单**：只需训练轻量 draft head，target 模型不动

- **无需独立 draft 模型**：与 target 模型紧耦合

### 2.4 Tree-based Verification

EAGLE 使用 **tree 结构** 而非线性链：

- Draft head 在每个位置生成 top-k 候选

- 形成候选树

- Target 模型一次前向验证整棵树

- 从树中选择最长的有效路径

```JavaScript
     root
    / | \
   a  b  c       (位置 1 的 top-3)
  /|  |  |\
 d e  f  g h     (位置 2 的 top-k)
...
```

## 3. Python 参考实现

```Python
import torch
import torch.nn.functional as F

def speculative_decode(
    target_model,   # 大模型
    draft_model,    # 小模型
    input_ids,      # 已有序列
    gamma: int = 5, # 猜测长度
    temperature: float = 1.0
):
    """经典 Speculative Decoding 参考实现"""
    device = input_ids.device
    
    # Step 1: Draft 阶段 - 小模型生成 gamma 个 token
    draft_tokens = []
    draft_probs = []
    current_ids = input_ids.clone()
    
    for _ in range(gamma):
        with torch.no_grad():
            logits = draft_model(current_ids).logits[:, -1]
            probs = F.softmax(logits / temperature, dim=-1)
            token = torch.multinomial(probs, num_samples=1)
            draft_tokens.append(token)
            draft_probs.append(probs)
            current_ids = torch.cat([current_ids, token], dim=-1)
    
    draft_tokens = torch.cat(draft_tokens, dim=-1)  # (B, gamma)
    
    # Step 2: Verify 阶段 - 大模型一次前向
    verify_ids = torch.cat([input_ids, draft_tokens], dim=-1)
    with torch.no_grad():
        target_logits = target_model(verify_ids).logits
    
    # Step 3: Rejection Sampling
    n_accepted = 0
    accepted_tokens = []
    
    for t in range(gamma):
        pos = input_ids.shape[-1] + t - 1
        target_probs = F.softmax(
            target_logits[:, pos] / temperature, dim=-1
        )
        draft_prob = draft_probs[t]
        token = draft_tokens[:, t:t+1]
        
        # 接受概率
        p = target_probs.gather(-1, token)
        q = draft_prob.gather(-1, token)
        accept_prob = torch.min(
            torch.ones_like(p), p / (q + 1e-10)
        )
        
        # 随机决定是否接受
        if torch.rand(1, device=device) < accept_prob:
            accepted_tokens.append(token)
            n_accepted += 1
        else:
            # 从修正分布采样
            residual = F.relu(target_probs - draft_prob)
            residual = residual / (residual.sum(dim=-1, keepdim=True) + 1e-10)
            new_token = torch.multinomial(residual, num_samples=1)
            accepted_tokens.append(new_token)
            break
    else:
        # 所有 gamma 个都被接受，额外从 target 采样一个
        last_probs = F.softmax(
            target_logits[:, -1] / temperature, dim=-1
        )
        bonus_token = torch.multinomial(last_probs, num_samples=1)
        accepted_tokens.append(bonus_token)
    
    result = torch.cat(accepted_tokens, dim=-1)
    print(f"接受 {n_accepted}/{gamma} 个 draft token")
    return torch.cat([input_ids, result], dim=-1)
```

## 4. Medusa vs EAGLE 对比

|维度|Medusa|EAGLE|
|---|---|---|
|**Draft 方式**|多个独立 LM head 并行预测|feature-level 自回归|
|**是否自回归**|否（各位置独立）|是（feature 依赖前一步）|
|**接受率**|较低（位置独立限制）|**更高**（自回归建模依赖）|
|**额外参数**|$K$ 个 LM head|1 个轻量 Transformer 层|
|**训练数据**|target 模型生成的 token|target 模型的 hidden states|
|**典型加速**|2-3x|**3-5x**|

## 5. 工程实践要点

> [!important]
> 
> - **Draft 质量是核心**：接受率决定一切
> 
> - $\gamma$ **的选择**：通常 3-7，太大浪费 draft 计算，太小收益不足
> 
> - **Batch 场景**：不同请求接受长度不同，需要 padding / dynamic batching
> 
> - **高熵场景收益低**：创意写作、代码生成等高不确定性任务
> 
> - **低熵场景收益高**：翻译、摘要、格式化输出等
