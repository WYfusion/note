## 概述

本页涵盖三类稀疏注意力的详细原理与实现：Block Sparse（Sparse Transformer）、Local+Global+Random（BigBird）、Hash Routing（Reformer）。

---

## 1. Block Sparse Attention（Sparse Transformer）

### 1.1 核心思想

- 将序列切成大小为 $B$ 的块

- 注意力仅在选定的块对之间计算

- **块结构** 让内存访问模式规则，适合 GPU Tensor Core

### 1.2 两种经典稀疏模式

**① Strided Pattern**

- 每个块 attend 自身 + 固定步长间隔的远块

- 例：块 $i$ attend 块 $\{i, i-s, i-2s, \dots\}$

- 适合周期性依赖（如图像像素）

**② Fixed Pattern**

- 局部块 + 一个汇聚块（summary block）

- 汇聚块收集全局信息

- 适合文本序列

### 1.3 复杂度

若每个块平均 attend $s$ 个块：

$$\text{计算} = O(n \cdot s \cdot B \cdot d) = O(n \cdot s \cdot d) \quad (B \text{ 为常数})$$

### 1.4 GPU Kernel 要点

```Python
# Block Sparse Attention 伪代码

    Q, K, V,         # (n, d)
    block_size: int,
    sparse_layout    # 哪些块对需要计算

    for i in range(n_blocks):
        Qi = Q[i*B : (i+1)*B]  # 当前 Q 块
        
        # 只遍历 sparse_layout 中指定的 KV 块
        attend_blocks = sparse_layout[i]
        for j in attend_blocks:
            Kj = K[j*B : (j+1)*B]
            Vj = V[j*B : (j+1)*B]
            # 计算块级 attention 并累积
            ...
    
    return O
```

**关键工程考量**：

- 块大小 $B$ 应为 Tensor Core tile 大小的倍数（通常 16/32/64）

- 稀疏模式应固定且可预测，避免动态 gather

- 使用 `triton.ops.blocksparse` 或 DeepSpeed Sparse Attention

## 2. BigBird：Local + Global + Random

### 2.1 三种注意力组合

BigBird 的注意力矩阵 $A$ 是三个稀疏模式的并集：

$$A = A_{\text{local}} \cup A_{\text{global}} \cup A_{\text{random}}$$

**① Local（滑动窗口）**

- 每个 token attend 左右 $w/2$ 个邻居

- 提供局部上下文

**② Global Token**

- $g$ 个 token attend 全部序列，全部序列也 attend 它们

- **ITC 模式**（Internal）：选已有 token 作 global

- **ETC 模式**（Extended）：加额外 `[CLS]` 等 global token

**③ Random**

- 每个 token 随机 attend $r$ 个位置

- 数学上保证图的直径小 → **任意两 token 间路径短**

### 2.2 理论保证

- **通用近似器**：BigBird 的稀疏 attention 仍然是 sequence-to-sequence 函数的通用近似器

- **图灵完备**：保持了全 attention Transformer 的图灵完备性

- 关键定理依赖：图连通性 + Erdős–Rényi 随机图 + expander graph 理论

### 2.3 复杂度

$$O(n(w + g + r) \cdot d)$$

当 $w, g, r$ 均为常数时，总复杂度 **线性** $O(n)$。

### 2.4 实现

```Python
import torch
import math

def bigbird_attention_mask(
    n: int, 
    window_size: int = 3,
    num_global: int = 2,
    num_random: int = 3
) -> torch.Tensor:
    """生成 BigBird 风格的稀疏注意力 mask"""
    mask = torch.zeros(n, n, dtype=torch.bool)
    
    # 1. Local: 滑动窗口
    for i in range(n):
        j_start = max(0, i - window_size // 2)
        j_end = min(n, i + window_size // 2 + 1)
        mask[i, j_start:j_end] = True
    
    # 2. Global: 前 num_global 个 token
    mask[:num_global, :] = True  # global attend all
    mask[:, :num_global] = True  # all attend global
    
    # 3. Random: 每行随机选 num_random 个位置
    for i in range(n):
        rand_idx = torch.randperm(n)[:num_random]
        mask[i, rand_idx] = True
    
    return mask

# 可视化
mask = bigbird_attention_mask(16, window_size=3, num_global=2, num_random=2)
print(mask.int())  # 稀疏 attention mask
```

## 3. Reformer：LSH Attention

### 3.1 核心思想

- 相似的 Q, K 向量哈希到同一个桶

- 只在桶内做 attention

- 避免计算不相关 token 对的注意力

### 3.2 LSH（Locality-Sensitive Hashing）

给定随机投影矩阵 $R in mathbb{R}^{d times b}$：

$$h(x) = \arg\max([xR; -xR])$$

- 相似向量大概率哈希到同一桶

- 多轮哈希提高召回率

### 3.3 算法流程

1. 对所有 Q, K 计算哈希值

1. 按哈希值排序

1. 在同一桶（或相邻桶）内做 attention

1. 重复多轮，取平均

### 3.4 复杂度

$$O(n \log n \cdot d) \quad (\text{理论})$$

### 3.5 工程问题

- **排序开销**：GPU 上全局排序不便宜

- **负载不均**：不同桶大小不同

- **训练不稳定**：哈希分桶引入离散操作

- **kernel 生态差**：缺乏成熟的高效 kernel

### 3.6 为什么不再主流？

- 实践中 FlashAttention + 滑窗通常更快更稳

- LSH 的随机性增加了训练方差

- 现代 GPU 的 Tensor Core 更适合规则块结构

## 4. DeepSpeed Sparse Attention 使用示例

```Python
# 使用 DeepSpeed 的 Block Sparse Attention
from deepspeed.ops.sparse_attention import (
    SparseSelfAttention,
    FixedSparsityConfig
)

config = FixedSparsityConfig(
    num_heads=8,
    block=64,          # 块大小
    different_layout_per_head=True,
    num_local_blocks=4,
    num_global_blocks=1,
    num_random_blocks=1,
    attention='bidirectional'
)

sparse_attn = SparseSelfAttention(
    sparsity_config=config,
    max_seq_length=4096,
    attn_mask_mode='add'
)

# 前向计算
Q = torch.randn(2, 8, 4096, 64, device='cuda')
K = torch.randn(2, 8, 4096, 64, device='cuda')
V = torch.randn(2, 8, 4096, 64, device='cuda')
out = sparse_attn(Q, K, V)
```

## 5. 工程选型总结

> [!important]
> 
> - **结构化稀疏 > 任意稀疏**：块结构适合 GPU，随机稀疏性能差
> 
> - **Sliding Window > Block Sparse > BigBird > LSH**：按工程成熟度和可落地性排序
> 
> - 真正落地时关注：**块大小是否适配 Tensor Core、稀疏模式是否便于并行、是否减少 kernel 碎片化**
