## 定位

- **改连接模式**：每个 token 不再 attend 参与全部序列，只看子集

- 复杂度从 $O(n^2)$ 降至近线性

- 与 FlashAttention 互补：FlashAttention 优化 IO，稀疏注意力减少计算量

---

## 四大子类

### 2.1 Local / Sliding Window Attention

- 每个 token 只看附近窗口 $w$，复杂度 $O(nwd)$

- 局部归纳偏置强，适合邻近依赖明显的文本/语音/视频

- **代表**：Longformer、各类 sliding-window causal attention、SWAT

- **最实用的稀疏方案之一**：窗口结构最容易写高效 kernel

### 2.2 Block Sparse Attention

- 把序列切成固定大小的块，注意力按块选择

- 若平均访问 $s$ 个块，复杂度 $O(nsd)$

- **代表**：Sparse Transformer、各类 blockwise kernels

- 硬件友好度高于任意随机稀疏

- 关键：块大小适配 Tensor Core、稀疏模式便于并行

### 2.3 Local + Global + Random Sparse

- local 保邻域 + global 保汇聚 + random 补图连通性

- 在近线性成本下保留全局可达性

- **代表**：BigBird

- 理论性质强（通用近似器 + 图灵完备），工程实现比纯滑窗复杂

### 2.4 Hash / Routing Sparse

- 先将 token 映射/聚类到桶，桶内做注意力

- 复杂度近 $O(n \log n)$

- **代表**：Reformer（LSH attention）

- 经典但工业落地受限：训练稳定性/哈希开销/kernel 生态

## 工程判断总览

|方案|复杂度|硬件友好|工程成熟度|典型用途|
|---|---|---|---|---|
|Sliding Window|$O(nwd)$|⭐⭐⭐|⭐⭐⭐|超长序列训练/推理首选稀疏方案|
|Block Sparse|$O(nsd)$|⭐⭐⭐|⭐⭐|长序列 + 需跨块依赖|
|BigBird|$O(n)$|⭐⭐|⭐⭐|需局部+全局保证的任务|
|Reformer (LSH)|$O(n \log n)$|⭐|⭐|研究探索|
