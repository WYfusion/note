## 概述

FlashAttention-2 和 FlashAttention-3 是对初代算法的架构级优化。FA-2 重点是提升 GPU 利用率（A100）；FA-3 针对 Hopper 架构（H100）利用异步执行和 FP8 达到 85% 利用率。

---

## 1. FlashAttention-2 改进

### 1.1 三大改进方向

**① 减少非 matmul FLOPs**

- FA-1 中 online softmax 的 rescaling 操作在 warp 之间产生大量非 matmul 矩阵乘法运算

- FA-2 将 rescaling 延迟到整个 KV 遍历结束后一次性做

- 好处：matmul 占比提高 → Tensor Core 利用率提升

**② 改并行维度**

- FA-1：外循环遍历 KV，内循环遍历 Q → Q 块之间有依赖

- FA-2：**外循环遍历 Q，内循环遍历 KV** → Q 块之间完全独立 → 更好并行

- 直接后果：并行度从 `batch × heads` 提升到 `batch × heads × ceil(n/Br)`

**③ 更好的 warp 分工**

- FA-1：4 个 warp 分别计算 $QK^T$ 的不同行，需要跨 warp 通信来做 softmax reduction

- FA-2：每个 warp 计算 **全部** $QK^T$ 行的一部分列，然后独立做 softmax → **消除跨 warp 通信**

### 1.2 性能提升

|指标|FA-1|FA-2|
|---|---|---|
|A100 前向 TFLOPS|~120|**~230** (~73% 理论峰值)|
|前向+反向 TFLOPS|~80|**~220** (~70%)|
|相比 FA-1 加速|1x|**~2x**|

### 1.3 关键实现细节

```Python
# FA-2 的循环结构（伪代码）
# 外循环：遍历 Q 块 (可完全并行)
for i in parallel(range(0, n, BLOCK_M)):  
    Qi = load_from_hbm(Q[i:i+BLOCK_M])
    # 初始化 running stats
    m_i, ell_i, O_i = -inf, 0, zeros
    
    # 内循环：遍历 KV 块 (串行)
    for j in range(0, n, BLOCK_N):
        Kj = load_from_hbm(K[j:j+BLOCK_N])
        Vj = load_from_hbm(V[j:j+BLOCK_N])
        
        S = Qi @ Kj.T * scale
        m_new = max(m_i, row_max(S))
        P = exp(S - m_new)
        
        # 延迟 rescale：不立即归一化 O_i
        O_i = O_i * exp(m_i - m_new) + P @ Vj
        ell_i = ell_i * exp(m_i - m_new) + row_sum(P)
        m_i = m_new
    
    # 最后一次性归一化
    O_i = O_i / ell_i
    store_to_hbm(O[i:i+BLOCK_M], O_i)
```

## 2. FlashAttention-3 改进（Hopper 架构）

### 2.1 Hopper 新硬件特性

- **TMA（Tensor Memory Accelerator）**：硬件异步数据搬运，无需占用计算 warp

- **Warp Group 专用化**：不同 warp group 可专门做计算或数据搬运

- **FP8 Tensor Core**：2x 于 FP16 的吞吐

- **异步执行**：TMA 搬运与 Tensor Core 计算可完全重叠

### 2.2 三大技术

**① Warp Specialization + 异步重叠**

- 将 warp groups 分为 **producer**（数据搬运）和 **consumer**（计算）

- Producer 用 TMA 异步加载下一块 K, V 到 SRAM

- Consumer 同时对当前块做 matmul

- **通信-计算完全重叠**

```JavaScript
时间线：
Producer:  [Load K1] [Load V1] [Load K2] [Load V2] ...
Consumer:  [idle]    [QK1^T]   [P1*V1]   [QK2^T]  ...
                     ↑ 重叠区间
```

**② Ping-pong Scheduling**

- 两组 SRAM buffer 交替使用

- 当一组在计算时，另一组在加载

- 进一步隐藏数据搬运延迟

**③ FP8 + 非相干处理（Incoherent Processing）**

- 直接用 FP8 做 $QK^T$ 和 $PV$ 的 matmul

- **问题**：FP8 量化误差大（相干误差会累积）

- **解法**：对 Q, K 做随机正交变换使其 "非相干"，降低系统性误差

- 结果：FP8 FlashAttention-3 的数值误差比朴素 FP8 attention **低 2.6x**

### 2.3 性能数据

|配置|TFLOPS|利用率|
|---|---|---|
|FA-3 BF16 (H100)|**840**|**85%**|
|FA-3 FP8 (H100)|**~1300** (1.3 PFLOPS)|~66%|
|FA-2 FP16 (H100)|~350|~35%|

- FA-3 BF16 比 FA-2 **快 1.5-2.0x**

- FA-3 FP8 比 FA-3 BF16 再快 **~1.5x**

## 3. 实际使用指南

### 3.1 PyTorch 集成

```Python
import torch
from torch.nn.functional import scaled_dot_product_attention

# PyTorch 2.0+ 自动选择 FlashAttention 后端
Q = torch.randn(2, 8, 1024, 64, device='cuda', dtype=torch.float16)
K = torch.randn(2, 8, 1024, 64, device='cuda', dtype=torch.float16)
V = torch.randn(2, 8, 1024, 64, device='cuda', dtype=torch.float16)

# 自动使用 FlashAttention（如果硬件支持）
with torch.backends.cuda.sdp_kernel(
    enable_flash=True, 
    enable_math=False, 
    enable_mem_efficient=False
):
    out = scaled_dot_product_attention(Q, K, V, is_causal=True)
```

### 3.2 flash-attn 库直接调用

```Python
from flash_attn import flash_attn_func

# Q, K, V: (batch, seqlen, nheads, headdim)
out = flash_attn_func(
    Q, K, V, 
    causal=True,
    softmax_scale=1.0 / (64 ** 0.5)
)
```

### 3.3 选择建议

- **head_dim ≤ 256**：FlashAttention 支持良好

- **A100/A800**：FA-2 最佳

- **H100/H800**：FA-3 最佳（需 CUDA 12.3+）

- **Turing (T4, 2080)**：用 FA-1 或 memory-efficient attention

## 4. 反向传播的 Recomputation 策略

### 核心权衡

- **不存储**：$S = QK^T$ 和 $P = \text{softmax}(S)$

- **只存储**：$O$（输出）、$L = m + log(\ell)$（log $\sum$ \exp）

- **反向时**：从 $Q, K, V, O, L$ 重新计算 $S, P$

- **代价**：多一次前向计算量（~33% overhead）

- **收益**：显存从 $O(n^2)$ 降到 $O(n)$

### 为什么这是值得的？

- 对于 $n = 8192, d = 64$：$n^2 = 67M$ floats ≈ 256 MB/head

- 存储 $O + L$ 只需 ≈ 1 MB/head

- **显存节省 256x**，代价仅为 ~33% 额外计算

- 在 memory-bound 场景下，总 wallclock 反而更快
