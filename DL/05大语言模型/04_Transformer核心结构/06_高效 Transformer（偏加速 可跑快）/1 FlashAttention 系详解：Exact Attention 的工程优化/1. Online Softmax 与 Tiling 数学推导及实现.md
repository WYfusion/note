## 概述

FlashAttention 的两大核心技术：**Online Softmax**（增量计算 softmax，不需 materialize 完整 $QK^T$）和 **Tiling**（将大矩阵切块放入 SRAM）。本页给出完整的数学推导和 Python/Triton 实现。

---

## 1. 标准 Softmax 的问题

### 1.1 朴素 Softmax

给定向量 $x = [x_1, x_2, \dots, x_n]$：

$$\text{softmax}(x_i) = \frac{e^{x_i}}{\sum_\limits {j=1}^{n} e^{x_j}}$$

**问题**：需要两遍扫描——先算 $\sum e^{x_j}$，再算每个 $\frac{e^{x_i}}{\sum}$。

### 1.2 Safe Softmax

为避免数值溢出，减去最大值 $m = \max_j x_j$：

$$\text{softmax}(x_i) = \frac{e^{x_i - m}}{\sum_\limits {j=1}^{n} e^{x_j - m}}$$

**问题**：需要三遍扫描——先求 $m$，再算 $\sum e^{x_j - m}$，再算输出。每遍都要读写 HBM。

---

## 2. Online Softmax 推导

### 2.1 核心思想

**将三遍扫描合并为一遍**。在遍历过程中增量维护 $(m, d)$，其中 $m$ 是当前最大值，$d$ 是当前归一化因子。

### 2.2 增量更新公式

设已处理前 $k$ 个元素，维护状态 $(m_k, d_k)$：

$$m_k = \max(x_1, \dots, x_k)$$

$$d_k = \sum_{j=1}^{k} e^{x_j - m_k}$$

当新元素 $x_{k+1}$ 到来时：

$$m_{k+1} = \max(m_k, x_{k+1})$$

$$d_{k+1} = d_k \cdot e^{m_k - m_{k+1}} + e^{x_{k+1} - m_{k+1}}$$

**关键洞察**：旧的归一化因子 $d_k$ 通过乘以 $e^{m_k - m_{k+1}}$ 来 "rescale"，适应新的最大值。

### 2.3 扩展到矩阵（Attention 场景）

对于 attention 输出 $O = \text{softmax}(QK^T) V$，需要同时维护：

- $m_i$：第 $i$ 行的 running max

- $ell_i$：第 $i$ 行的 running $\sum$（归一化因子）

- $O_i$：第 $i$ 行的 running output（未归一化）

当处理新的 $K, V$ 块 $j$ 时：

$$m_i^{\text{new}} = \max(m_i^{\text{old}}, \max_j(Q_i K_j^T))$$

$$\ell_i^{\text{new}} = \ell_i^{\text{old}} \cdot e^{m_i^{\text{old}} - m_i^{\text{new}}} + \sum_j e^{Q_i K_j^T - m_i^{\text{new}}}$$

$$O_i^{\text{new}} = O_i^{\text{old}} \cdot \frac{\ell_i^{\text{old}} \cdot e^{m_i^{\text{old}} - m_i^{\text{new}}}}{\ell_i^{\text{new}}} + \frac{\text{softmax}(Q_i K_j^T) \cdot V_j}{\ell_i^{\text{new}}}$$

## 3. Tiling 平铺策略

### 3.1 GPU 存储层次

|存储层级|容量（典型）|带宽|特点|
|---|---|---|---|
|**HBM**（显存）|40-80 GB|~2 TB/s (A100)|大但慢|
|**SRAM**（片上缓存）|~20 MB (A100)|~19 TB/s|小但极快|
|**寄存器**|~几 MB|最快|每线程私有|

### 3.2 Tiling 方案

将 $Q, K, V \in \mathbb{R}^{n \times d}$ 切成块：

- $Q$ 切为 $T_r$ 个行块，每块 $B_r$ 行

- $K, V$ 切为 $T_c$ 个行块，每块 $B_c$ 行

- 块大小选择使得 $B_r \times d$ 和 $B_c \times d$ 可装入 SRAM

**外层循环**：遍历 $K, V$ 的块 $j = 1, \dots, T_c$

**内层循环**：遍历 $Q$ 的块 $i = 1, \dots, T_r$

每步：

1. 从 HBM 加载 $Q_i, K_j, V_j$ 到 SRAM

1. 在 SRAM 内计算 $S_{ij} = Q_i K_j^T$（局部注意力分数）

1. 用 online softmax 更新 $m_i, \ell_i, O_i$

1. **中间** $n \times n$ **矩阵从不写入 HBM**

### 3.3 IO 复杂度分析

- 朴素 attention：$O(n^2 + nd)$ HBM 读写

- FlashAttention：$O(n^2 d^2 / M)$ HBM 读写（$M$ = SRAM 大小）

- 当 $M > d^2$ 时（通常成立），IO 显著降低

## 4. Python 示例：Online Softmax

```Python
import torch

def online_softmax(x: torch.Tensor) -> torch.Tensor:
    """单遍扫描实现 softmax（数值稳定）"""
    n = x.shape[-1]
    m = torch.tensor(float('-inf'))  # running max
    d = torch.tensor(0.0)            # running \sum
    
    # 第一遍：计算 max 和归一化因子
    for j in range(n):
        m_new = torch.max(m, x[j])
        d = d * torch.exp(m - m_new) + torch.exp(x[j] - m_new)
        m = m_new
    
    # 第二遍：计算输出
    out = torch.exp(x - m) / d
    return out

# 验证
x = torch.randn(128)
assert torch.allclose(
    online_softmax(x), 
    torch.softmax(x, dim=-1), 
    atol=1e-6
)
```

## 5. Python 示例：FlashAttention 前向（简化版）

```Python
import torch
import math

def flash_attention_forward(
    Q: torch.Tensor,  # (n, d)
    K: torch.Tensor,  # (n, d)
    V: torch.Tensor,  # (n, d)
    block_size: int = 64
) -> torch.Tensor:
    """FlashAttention 前向的 Python 参考实现（简化，忽略 mask/dropout）"""
    n, d = Q.shape
    scale = 1.0 / math.sqrt(d)
    
    O = torch.zeros_like(Q)         # 输出累积
    m = torch.full((n,), float('-inf'))  # 每行 running max
    ell = torch.zeros(n)            # 每行 running \sum
    
    # 外层：遍历 K, V 块
    for j_start in range(0, n, block_size):
        j_end = min(j_start + block_size, n)
        Kj = K[j_start:j_end]  # (Bc, d)
        Vj = V[j_start:j_end]  # (Bc, d)
        
        # 内层：遍历 Q 块
        for i_start in range(0, n, block_size):
            i_end = min(i_start + block_size, n)
            Qi = Q[i_start:i_end]  # (Br, d)
            
            # 1) 计算局部注意力分数
            S = (Qi @ Kj.T) * scale  # (Br, Bc)
            
            # 2) Online softmax 更新
            m_old = m[i_start:i_end].clone()
            m_new = torch.max(m_old, S.max(dim=-1).values)
            
            # rescale 旧的累积
            alpha = torch.exp(m_old - m_new)
            # 当前块的 exp
            P = torch.exp(S - m_new.unsqueeze(-1))
            
            ell_old = ell[i_start:i_end].clone()
            ell_new = ell_old * alpha + P.\sum(dim=-1)
            
            # 3) 更新输出
            O_old = O[i_start:i_end]
            O[i_start:i_end] = (
                O_old * (ell_old * alpha).unsqueeze(-1) 
                + P @ Vj
            ) / ell_new.unsqueeze(-1)
            
            m[i_start:i_end] = m_new
            ell[i_start:i_end] = ell_new
    
    return O

# 验证
n, d = 256, 64
Q = torch.randn(n, d)
K = torch.randn(n, d)
V = torch.randn(n, d)

# 标准 attention
scale = 1.0 / math.sqrt(d)
attn = torch.softmax(Q @ K.T * scale, dim=-1) @ V

# FlashAttention
flash_out = flash_attention_forward(Q, K, V, block_size=32)

print(f"Max error: {(attn - flash_out).abs().max():.2e}")
# 应接近 1e-6 级别
```

## 6. Triton 实现要点

### 6.1 Kernel 结构

- 每个 kernel job 负责一个 $Q$ 块（$B_r$ 行）

- 遍历所有 $K, V$ 块

- 用 `tl.load` / `tl.store` 显式管理 SRAM

- 块内 matmul 用 `tl.dot`（映射到 Tensor Core）

### 6.2 关键优化点

- **Causal mask**：跳过不需要计算的 $K, V$ 块

- **块大小选择**：$B_r, B_c$ 需要平衡 SRAM 利用率与并行度

- **反向传播**：不存储 $S$ 矩阵，反向时重新计算（recomputation），只需额外存储 $m, \ell$

### 6.3 Triton 伪代码

```Python
# Triton kernel 骨架（伪代码）
@triton.jit
def flash_attn_fwd_kernel(
    Q_ptr, K_ptr, V_ptr, O_ptr,
    stride_qn, stride_kn, ...,
    BLOCK_M: tl.constexpr,  # Q 块大小
    BLOCK_N: tl.constexpr,  # K 块大小
):
    # 获取当前 program 负责的 Q 块索引
    pid = tl.program_id(0)
    
    # 加载 Q 块到 SRAM
    q = tl.load(Q_ptr + offsets_q)
    
    # 初始化 running stats
    m_i = tl.full([BLOCK_M], float('-inf'))
    ell_i = tl.zeros([BLOCK_M])
    o_i = tl.zeros([BLOCK_M, HEAD_DIM])
    
    # 遍历所有 K, V 块
    for j in range(0, N, BLOCK_N):
        k = tl.load(K_ptr + offsets_k)  # 加载 K 块
        v = tl.load(V_ptr + offsets_v)  # 加载 V 块
        
        s = tl.dot(q, tl.trans(k))  # QK^T
        s *= scale
        
        # Online softmax
        m_new = tl.maximum(m_i, tl.max(s, axis=1))
        alpha = tl.exp(m_i - m_new)
        p = tl.exp(s - m_new[:, None])
        ell_new = ell_i * alpha + tl.\sum(p, axis=1)
        
        # 更新输出
        o_i = o_i * (ell_i * alpha)[:, None] + tl.dot(p, v)
        o_i /= ell_new[:, None]
        
        m_i = m_new
        ell_i = ell_new
    
    tl.store(O_ptr + offsets_o, o_i)
```

## 7. 反向传播：Recomputation 策略

### 关键设计

- **不存储** $S = QK^T$ 和 $P = \text{softmax}(S)$（节省 $O(n^2)$ 显存）

- 只存储 $O$（输出）、$m$（行最大值）、$ell$（行归一化因子）

- 反向时从 $Q, K, V, O, m, \ell$ **重新计算** $S$ 和 $P$

- 额外计算量约 1 次前向，但显存从 $O(n^2)$ 降到 $O(n)$
