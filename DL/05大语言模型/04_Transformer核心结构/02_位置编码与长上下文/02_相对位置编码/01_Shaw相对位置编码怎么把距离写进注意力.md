# Shaw 相对位置编码怎么把距离写进注意力

## 问题定义

Shaw 等早期相对位置方法，是如何把“距离”直接写进 attention 关系里的？

## 直觉解释

它不是先给每个 token 单独编号，而是为“相差几步”准备位置向量，让 attention 在比较两个 token 时，同时看到它们的内容和距离。

## 形式化推导

核心思想是在 key/value 侧加入与相对距离相关的项，使 attention 打分与聚合都对相对位移敏感，而不是只依赖内容向量。

## 工程意义

Shaw 的意义在于第一次清楚展示：位置不必只加在输入 embedding 上，也可以直接进入 token 间关系建模。

## 常见误解

> [!warning] 常见误解
> - “相对位置只能加 bias。” 不对，Shaw 就是更直接的表示写入。

## 例子或反例

“前一个词”和“前十个词”即使内容相似，Shaw 方案也能通过不同相对距离项让它们的打分模式不同。
