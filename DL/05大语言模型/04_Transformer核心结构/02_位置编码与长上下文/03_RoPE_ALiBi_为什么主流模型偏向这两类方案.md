# RoPE 与 ALiBi：为什么主流模型偏向这两类方案

> [!abstract] 摘要
> RoPE 和 ALiBi 流行的原因，不是它们“理论最先进”，而是它们在表达能力、外推行为、实现复杂度和推理兼容性之间形成了很强的现实平衡。

## 这页先讲清什么

- RoPE 与 ALiBi 各自把位置信息写进什么地方。
- 为什么两者都比简单绝对位置更适合现代长上下文 LLM。
- 为什么“外推更稳”和“表达更强”不一定由同一方案同时满足。

## 关键结论

- RoPE 把相对位移写进 Q/K 的几何关系里。
- ALiBi 用距离线性偏置直接修改 logits，结构更简单、外推更清晰。
- 主流模型常选 RoPE，是因为它在表达与兼容性之间更均衡；但极端追求外推时，ALiBi 也很有吸引力。

## 最短闭环解释

RoPE 的思路是：不要只给 token 一张位置名片，而是直接旋转 Query / Key，让位置进入它们的内积几何。这样 attention 在比较内容时，天然会同时感知相对位移。

ALiBi 更直接：它不改向量几何，而是在 logits 上对远距离施加线性惩罚。于是远处默认更难被选中，外推行为也更清楚。

两者共同点是都比绝对位置更接近“关系建模”；区别在于 RoPE 表达更柔和、更几何化，ALiBi 更硬、更偏规则化。因此它们对应的优势也不同。
