# 为什么主流 LLM 常在 RoPE 与 ALiBi 之间选择

## 问题定义

为什么现代大模型不太回到简单绝对位置，而更常在 RoPE 和 ALiBi 两类方案里做取舍？

## 直觉解释

因为主流 LLM 既要保留表达力，又要考虑扩窗、推理兼容性和实现成本。这两类方案刚好覆盖了最常见的权衡两端。

## 形式化推导

RoPE 偏向“把位置写进几何”，ALiBi 偏向“把距离写成偏置规则”。两者都比绝对位置更接近关系建模，也都比许多复杂相对位置方案更适合大规模生成模型。

## 工程意义

主流 LLM 常选 RoPE，是因为它在效果、生态、实现上更均衡；当项目更强调极长外推、实现简单或稳定性时，也会优先评估 ALiBi。

## 常见误解

> [!warning] 常见误解
> - “主流选择就一定是理论最优。” 不对，往往是综合成本最优。

## 例子或反例

同样做长上下文产品，若团队已有成熟 RoPE 扩窗方案，继续沿 RoPE 演化更现实；若从零出发且更看重规则清晰，ALiBi 也可能是更顺手的起点。
