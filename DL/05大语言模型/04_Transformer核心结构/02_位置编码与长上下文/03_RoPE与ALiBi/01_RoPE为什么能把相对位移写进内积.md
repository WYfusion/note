# RoPE 为什么能把相对位移写进内积

## 问题定义

RoPE 为什么不是“又一种位置 embedding”，而是直接把位移关系写进了 Query / Key 的内积？

## 直觉解释

RoPE 通过对不同位置的 Q/K 做角度旋转，让两个位置之间的内积天然携带相对相位差。

## 形式化推导

把二维子空间写成复数形式，位置 $m$ 的 query 近似变成 $q e^{im\theta}$，位置 $n$ 的 key 变成 $k e^{in\theta}$。两者内积中的相位差会依赖 $(m-n)$，于是相对位移进入打分。

## 工程意义

RoPE 的好处是：不必额外维护大位置表，也不只是加一个偏置，而是直接让位置进入 attention 几何，因此与标准 attention 主干兼容性很好。

## 常见误解

> [!warning] 常见误解
> - “RoPE 只是复杂版 sinusoidal。” 不准确。它不是只加到输入，而是改 Q/K 几何。

## 例子或反例

同样内容的两个 token，只要相对位置差不同，旋转后的内积就不同；这使模型能直接感知距离变化。
