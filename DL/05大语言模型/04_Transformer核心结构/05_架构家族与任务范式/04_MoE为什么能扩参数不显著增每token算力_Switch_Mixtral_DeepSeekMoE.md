# MoE 为什么能扩参数而不显著增每 token 算力：Switch、Mixtral 与 DeepSeekMoE

> [!abstract] 摘要
> MoE 的关键不是“模型里有更多参数”，而是“每个 token 只激活其中一小部分参数”。因此它扩的是总容量，而不是所有 token 都必须走完的全路径计算。

## 这页先讲清什么

- MoE 为什么能把总参数量和单 token 激活算力拆开。
- 路由器到底在决定什么。
- 为什么 MoE 的难点往往在负载均衡和并行系统，而不只是模型结构。

## 关键结论

- MoE 扩的是总容量，不是每 token 的全激活成本。
- Router 质量和负载均衡决定了 MoE 能否把理论优势兑现到实践。
- MoE 成功与否，常常是结构设计和系统并行共同决定的。

## 最短闭环解释

密集模型扩参数时，每个 token 基本都要经过所有 FFN。MoE 把 FFN 换成很多专家网络，再让 router 为每个 token 选择 top-k 专家，于是大多数参数只对部分 token 激活。

这让总容量可以大幅提升，但代价并没有消失，而是转移到了路由质量、专家负载均衡、跨卡通信和部署复杂度上。

所以 MoE 不是“白拿更多参数”，而是“用更复杂的系统换更大的稀疏容量”。
