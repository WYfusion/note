# MoE 路由器到底在学什么

## 问题定义

Router 给每个 token 选专家时，究竟在学什么？

## 直觉解释

它本质上在学“这个 token 更该交给哪类专家处理”，也就是在学一种内容到专家的动态分工。

## 形式化推导

路由器通常输出每个 token 对各专家的打分，再取 top-k 专家激活。这个过程定义了稀疏计算路径。

## 工程意义

Router 质量直接决定 MoE 的容量利用率和专家分工是否有效。

## 常见误解

> [!warning] 常见误解
> - “专家自己会平均分工，不需要管。” 不对，路由和负载均衡至关重要。

## 例子或反例

若某类 token 总被几个专家垄断，其余专家闲置，MoE 的理论容量就很难兑现。
