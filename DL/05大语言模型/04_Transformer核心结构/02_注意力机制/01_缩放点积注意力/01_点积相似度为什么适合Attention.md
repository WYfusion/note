# 点积相似度为什么适合 Attention

## 问题定义

为什么 attention 常用点积而不是欧氏距离、余弦相似度或一个更复杂的 MLP 打分器？

## 直觉解释

点积同时回答了两件事：方向是否对齐、幅度是否足够强。更重要的是，它可以直接写成矩阵乘法，于是非常适合大规模并行。

## 形式化推导

对单个 query 和 key：

$$
s(q, k) = q^\top k
$$

把所有 query 和 key 叠起来，就得到：

$$
S = QK^\top
$$

这一步天然是 GEMM。若改用加性注意力或复杂 MLP 打分，虽然表达也可能足够强，但更难像 `QK^T` 一样直接映射到高效矩阵乘法。

## 工程意义

点积的真正优势是“表达够用 + 实现极友好”。它与线性投影、批处理、Tensor Core、高效 kernel 都天然兼容，因此在工程上几乎成了默认基线。

## 常见误解

> [!warning] 常见误解
> - “点积一定是最强相似度。” 不对。它是综合权衡后的默认最优，而不是理论唯一正确。
> - “余弦一定更合理，因为有归一化。” 不完整。余弦去掉了范数信息，也会改变优化几何。

## 例子或反例

若两个 key 与 query 方向一致，但一个范数更大，点积会给更高分；这意味着模型可以把“更重要”也编码进向量尺度中，而不只是编码角度相似性。
