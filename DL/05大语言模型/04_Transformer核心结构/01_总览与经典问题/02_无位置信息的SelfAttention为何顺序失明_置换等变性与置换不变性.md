# 无位置信息的 Self-Attention 为何顺序失明：置换等变性与置换不变性

> [!abstract] 摘要
> 无位置信息的 self-attention 并不是严格意义上的序列算子，而更像一个对 token 集合做内容交互的算子。它对输入重排满足置换等变性；如果再做 pooling 或 CLS 聚合，就会进一步表现为置换不变性，于是顺序信息被彻底洗掉。

## 这页先讲清什么

- “顺序失明”不是比喻，而是可以用置换矩阵精确表述的结构性质。
- token 级 self-attention 是置换等变，不是置换不变；真正让顺序彻底丢掉的，往往是后续聚合。
- 位置编码的作用不是给 token 编号而已，而是打破这种纯内容集合算子的对称性。

## 关键结论

- 若没有位置机制，输入怎样重排，输出就怎样同步重排，模型本身并没有学到“先后”。
- 一旦再对 token 维做平均池化、求和池化或依赖一个位置无语义锚点的聚合器，输出就会对置换不敏感。
- 这就是为什么 `dog bites man` 和 `man bites dog` 在“无位置 + pooling”的结构下会被看得过于相似。

## 最短闭环解释

设输入序列是一个矩阵 $X$。如果我们只对每个 token 做线性投影，然后用内容相似度计算 attention，那么把输入行顺序打乱，相当于左乘一个置换矩阵 $P$。这时 Query、Key、Value 都会同步被置换，最后 attention 输出也只会同步被置换，而不会产生新的顺序差异。

这就是置换等变：输入怎么排，输出就怎么跟着排。注意，这还不是“输出不变”。如果你仍保留每个位置的输出向量，那么模型至少知道“第 3 行输出现在跑到第 7 行了”。真正更强的对称性出现在你把整段序列再压成一个向量时，比如平均池化、求和池化，或者没有明确位置信号的集合式聚合。这些操作对重排本来就不敏感，于是顺序信息被整体抹平。

所以，顺序问题不是 attention 的一个小缺陷，而是它默认对输入排列具有对称性。位置编码、相对位置 bias、RoPE 这些设计，都是在显式打破这个对称性，让模型不再只看“内容集合”，而是真正区分“哪个内容出现在什么相对位置上”。
