# 位置编码到底补回了什么

## 问题定义

位置编码究竟是在“给每个 token 编号”，还是在更深层地改变 attention 的对称性与几何？

## 直觉解释

没有位置机制时，模型只知道内容像不像，不知道谁在前谁在后。位置编码的作用，就是把“相同内容但不同位置”变成模型可区分的状态。

## 形式化推导

最简单的绝对位置编码是把位置向量 $p_i$ 加到 token 表示 $x_i$ 上，得到 $\tilde{x}_i = x_i + p_i$。这样即使两个 token 内容相同，只要位置不同，Query/Key 也会不同。于是输入置换不再只是一个简单的左乘 $P$，因为位置向量不会跟着内容自动保持同构。

相对位置偏置和 RoPE 则更进一步，它们不是只修改输入表示，而是直接修改 attention 打分或 Q/K 几何，因此补回的不是单纯“编号”，而是“距离、方向、相位”这类关系信息。

## 工程意义

位置机制补回的核心不是一个整数索引，而是让模型可以表达：

- 谁在谁前面。
- 两者相距多远。
- 局部关系与远距离关系是否不同。

这也是为什么不同任务会偏好不同位置方案：有的更关心绝对锚点，有的更关心相对距离，有的更关心长距离外推。

## 常见误解

> [!warning] 常见误解
> - “位置编码只是输入特征增强。” 不完整。它会改变整个注意力算子的对称性。
> - “有 causal mask 就不需要位置编码。” 不对。mask 只给偏序，不给距离和方向。

## 例子或反例

“上海到北京”和“北京到上海”里的词集合高度相似，但先后关系不同。没有位置机制时，模型很难区分这种方向变化；位置进入后，这种关系才会被显式建模。
