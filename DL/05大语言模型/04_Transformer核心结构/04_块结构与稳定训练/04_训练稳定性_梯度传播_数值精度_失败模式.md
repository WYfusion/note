# 训练稳定性：梯度传播、数值精度与失败模式

> [!abstract] 摘要
> 大模型训练失败通常不是单一原因。梯度路径、激活尺度、低精度数值、mask 填充值、位置外推、cache 与 kernel 实现，都会共同决定模型是在稳定收敛，还是出现 NaN、loss 抖动和长上下文退化。

## 这页先讲清什么

- 梯度为什么会不稳。
- 低精度训练经常在哪些地方先出问题。
- 实际排障时应如何区分结构问题、数值问题和实现问题。

## 关键结论

- “没有 NaN”不等于训练稳定，很多退化是静默发生的。
- 低精度出问题常常先出在 softmax、mask、归一化和大范围累加。
- 稳定训练是结构、优化器、精度和 kernel 的联合作品。

## 最短闭环解释

同样表现为“loss 不对”，背后的根因可能完全不同。Norm 摆放不对会伤梯度主路，残差尺度不稳会让激活漂移，低精度设置不当会让 softmax 或归一化直接溢出，长上下文退化还可能来自位置几何失真。

所以排障不能只盯着一个症状。正确做法是沿着“结构 -> 学习率与优化 -> 数值精度 -> kernel 与实现 -> 长上下文验证”这条链路逐层定位。

稳定训练不是最后补几条经验规则，而是整个模型设计是否协同一致的结果。
