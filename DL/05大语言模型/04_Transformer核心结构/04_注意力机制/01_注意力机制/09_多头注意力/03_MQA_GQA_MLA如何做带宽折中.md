# MQA、GQA、MLA 如何做带宽折中

> [!abstract]
> 这三个名字都在回答同一个工程问题：  
> 当上下文越来越长时，真正把推理拖慢的往往不是当步 Query 的计算，而是历史 Key / Value 的缓存与读取。

## 为什么优化重点落在 K/V，而不是 Q

在自回归解码中：

- 当前步的 Query 只计算一次
- 历史 Key / Value 会被整个前缀反复读取

因此随着序列变长，系统压力主要来自两件事：

1. KV cache 的存储量
2. 每步读取历史 K/V 的内存带宽

这就是 [[05_多查询注意力MQA|MQA]]、[[06_分组注意力GQA|GQA]] 和 [[08_多头潜变量注意力MLA|MLA]] 的共同出发点。

## 四种机制放在一起看

| 机制 | Query | Key / Value | 优化思路 | 主要代价 |
| --- | --- | --- | --- | --- |
| MHA | 多头 | 每头独立 | 不压缩，表达最直接 | KV cache 最重 |
| [[05_多查询注意力MQA|MQA]] | 多头 | 所有头共享一组 K/V | 极限减少显式 K/V 份数 | K/V 多样性压得最狠 |
| GQA | 多头 | 多个 Query 头共享一组 K/V | 质量与缓存折中 | 仍然缓存显式 K/V |
| MLA | 多头 | 缓存 K/V 的潜表示 | 重写 K/V 参数化与缓存对象 | 结构和实现更复杂 |

## 从 cache 规模直觉理解

设：

- 头数为 $h$
- 每头维度为 $d_h$
- GQA 的 K/V 组数为 $g$
- MLA 的 K/V 潜维度为 $d_c$

那么每层、每个 token 的缓存量级可以粗略理解为：

| 机制 | 每 token 缓存量级 |
| --- | --- |
| MHA | $O(2 h d_h)$ |
| [[05_多查询注意力MQA|MQA]] | $O(2 d_h)$ |
| GQA | $O(2 g d_h)$ |
| MLA | 约为 $O(d_c + d_h^R)$ |

> [!tip]
> 最值得记住的不是公式本身，而是方向：
> - [[05_多查询注意力MQA|MQA]] / GQA：减少显式 K/V 的份数
> - MLA：连“完整显式 K/V”都不再作为主要缓存对象

## 三种折中到底分别怎么做

### [[05_多查询注意力MQA|MQA]]：把所有头压到同一组 K/V

[[05_多查询注意力MQA|MQA]] 的做法最激进：  
所有 Query 头共享同一组 Key / Value。

优点很直接：

- cache 最省
- 带宽最轻

代价也很直接：

- 不同头能够读取的 K/V 通道被压得最狠

### GQA：按组共享 K/V

GQA 不把 K/V 压成一组，而是压成若干组。  
因此它通常是工程上的中间解：

- 比 MHA 省很多
- 比 MQA 更稳

这也是为什么很多现代 LLM 更常把 GQA 作为默认折中点。

### MLA：不是“少几组”，而是“换一种存法”

MLA 再往前走一步。  
它不是只减少显式 K/V 的头数，而是改写：

$$
W^K h_t,\; W^V h_t
\quad \Longrightarrow \quad
W^{UK} W^{DKV} h_t,\; W^{UV} W^{DKV} h_t
$$

并把真正缓存的对象改成共享潜表示：

$$
c_t^{KV} = W^{DKV} h_t
$$

所以 MLA 压缩的不是“头的份数”，而是**历史状态本身的表示形式**。

## MLA 与 MQA/GQA 的本质差异

> [!important]
> 这三者都在做带宽折中，但 MLA 的层级更深。

### [[05_多查询注意力MQA|MQA]] / GQA 在做什么

MQA / GQA 的思想是：

- 仍然保留显式 Key / Value
- 只是减少需要缓存多少组 K/V

因此它们的核心问题仍然是“显式 K/V 该共享到什么程度”。

### MLA 在做什么

MLA 的思想是：

- 不把完整内容 Key / Value 当作主要缓存对象
- 改为缓存一个共享潜表示 $c_t^{KV}$
- 需要时再由上投影矩阵恢复内容 Key / Value

如果你卡在“为什么这叫低秩压缩”，直接看：

- [[08_多头潜变量注意力MLA#低秩分解为何成立|MLA：低秩分解为何成立]]

如果你卡在“为什么 MLA 还必须引入 decoupled RoPE”，直接看：

- [[08_多头潜变量注意力MLA#为什么普通 RoPE 会破坏这个结构|MLA：为什么普通 RoPE 会破坏这个结构]]

## 一句话比较

| 机制 | 最短记忆句 |
| --- | --- |
| [[05_多查询注意力MQA|MQA]] | 所有头共用一组显式 K/V |
| GQA | 多个头共用一组显式 K/V |
| MLA | 不直接缓存完整 K/V，而是缓存其低维潜表示 |

## 什么时候优先想到哪种方案

| 场景 | 更常见的优先选择 |
| --- | --- |
| 极度看重 cache 和带宽极限 | [[05_多查询注意力MQA|MQA]] |
| 想要更稳的质量与成本折中 | GQA |
| 想进一步压缩历史状态，并接受更复杂实现 | MLA |
| 更关心表达直接性、推理预算足够 | MHA |
