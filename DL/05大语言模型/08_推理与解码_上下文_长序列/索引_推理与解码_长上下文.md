# 08_推理与解码、上下文与长序列

本模块深入探讨 LLM 推理阶段的核心技术，包括解码策略、KV Cache 优化以及长上下文处理方法。特别针对语音大模型（Audio LLM）序列长度远超文本的特性，提供了针对性的优化思路。

## 核心内容

### 1. 推理基础
- **[01_推理阶段张量与复杂度.md](01_推理阶段张量与复杂度.md)**
  - **Prefill vs Decode**：计算密集型 vs 访存密集型。
  - **复杂度分析**：Attention 的 $O(S^2)$ 时间复杂度。
  - **语音特殊性**：Audio Token 序列长度是文本的 ~20 倍，KV Cache 显存压力剧增。

### 2. 解码策略
- **[解码策略](索引_解码策略.md)**
  - **[01_Greedy与Beam Search](01_Greedy_BeamSearch.md)**：基础解码方法，ASR 中的 Beam Search 应用。
  - **[02_采样策略](02_TopK_TopP_温度_重复惩罚.md)**：Top-K, Top-P, Temperature 以及语音生成中的"结巴"问题与动态温度调节。

### 3. KV Cache 与加速
- **[KVCache与加速](索引_KVCache与加速.md)**
  - **[01_KVCache原理](01_KVCache_原理与显存.md)**：显存占用计算公式，Audio LLM 的显存挑战（1分钟音频 ~1GB KV Cache）。
  - **[02_PagedAttention](02_PagedAttention_vLLM思路.md)**：vLLM 的分页机制，变长音频请求的高效处理。

### 4. 长上下文方法
- **[长上下文方法](索引_长上下文方法.md)**
  - **[01_RoPE扩展](01_RoPE扩展_NTK_Scaling.md)**：位置插值 (PI)、NTK-Aware Scaling，流式语音的 StreamingLLM。
  - **[02_RAG与长上下文](02_检索增强_RAG与长上下文的关系.md)**：Audio RAG 的应用场景（播客问答、会议助手）。

## 学习路径建议
1.  理解 **Prefill/Decode 阶段**，明确推理瓶颈所在（Decode 阶段是显存瓶颈）。
2.  掌握 **KV Cache 计算公式**，这是优化 Audio LLM 推理的关键。
3.  了解 **PagedAttention**，它是高并发语音服务的核心技术。
4.  学习 **RoPE 扩展**，解决长音频处理中的位置外推问题。
