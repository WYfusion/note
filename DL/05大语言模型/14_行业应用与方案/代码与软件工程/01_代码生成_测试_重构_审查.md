# 代码生成、测试与重构

AI 辅助编程（AI-Assisted Programming）已成为开发者的标配。除了键盘输入，语音编程（Voice Coding）正在成为一种新兴的交互方式，特别是对于无障碍需求或预防 RSI（重复性劳损）的开发者。

## 1. 语音编程 (Voice Coding)

通过语音指令编写代码，解放双手。

### 1.1 传统语音编程 vs LLM 语音编程
*   **传统工具 (如 Talon, Dragon)**: 依赖严格的语法映射。
    *   User: "camel case get user id open paren close paren"
    *   Code: `getUserId()`
    *   缺点: 学习曲线陡峭，需要背诵大量指令。
*   **LLM 驱动 (如 Serenade, Copilot Voice)**: 支持自然语言意图。
    *   User: "Create a Python function to calculate Fibonacci numbers."
    *   Code: LLM 直接生成完整的函数代码。
    *   User: "Change the variable name 'n' to 'index'."
    *   Code: LLM 理解上下文并执行重构。

### 1.2 核心挑战
*   **同音词歧义**: `for` (循环) vs `four` (数字 4)，`write` vs `right`。
*   **符号输入**: 编程涉及大量特殊符号 (`{`, `}`, `[`, `]`, `$`, `_`)，口语表达效率低。
*   **命名规范**: 需要智能判断使用 `camelCase`, `snake_case` 还是 `PascalCase`。

## 2. 辅助测试与审查

### 2.1 语音驱动的测试生成
*   **场景**: 开发者口述测试用例的逻辑，LLM 生成 Jest/PyTest 代码。
*   **User**: "Test that the login function throws an error if the password is less than 8 characters."
*   **AI**: Generates `expect(login('user', '123')).toThrow(...)`.

### 2.2 代码审查 (Code Review)
*   **语音批注**: Reviewer 可以直接录音发表评论，LLM 将其转录并总结为结构化的 Review Comments，甚至自动生成修复建议的代码 Patch。
