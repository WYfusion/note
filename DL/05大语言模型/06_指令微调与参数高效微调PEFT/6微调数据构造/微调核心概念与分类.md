# 微调核心概念与分类

本节梳理 LLM 微调的基础概念体系，帮助理解后续各方案的定位与区别。

---

## 什么是微调（Fine-tuning）

在一个已经预训练好的大模型基础上，用**特定任务或领域的数据**继续训练，使模型获得新能力或提升特定表现。

- **核心目标**：保留底座通用能力 + 注入任务/领域专有能力
- **与预训练的区别**：预训练从零学语言知识；微调在此基础上做"精修"

---

## 微调的四种本质操作

| 操作类型 | 说明 | 代表方法 |
| --- | --- | --- |
| 更新全部参数 | 所有权重都参与梯度更新 | Full Fine-tuning |
| 更新部分参数 | 冻结大部分层，只训练选定层 | 冻结主干 + 训练头部/连接器 |
| 附加新参数 | 在原模型旁插入小模块，只训练新参数 | LoRA、Adapter、Prefix Tuning |
| 连续预训练 | 用领域语料以预训练方式继续训练 | CPT / DAPT / TAPT |

---

## 微调技术主线分类

### 1. Full Fine-tuning（全参数微调）

更新模型所有参数。效果上限最高，但需要大量 GPU 显存和计算资源。

### 2. PEFT（Parameter-Efficient Fine-Tuning，参数高效微调）

只训练极少量参数（通常 <1% 原模型），大幅降低资源需求。包括 LoRA、Adapter、Prompt Tuning 等一大族方法。

### 3. Continued Pretraining / Domain Adaptation

用领域语料以**语言建模**目标继续训练，注入领域知识。常作为 SFT 的前置步骤。

- **CPT**（Continued Pre-Training）：通用继续预训练
- **DAPT**（Domain-Adaptive Pre-Training）：领域自适应预训练
- **TAPT**（Task-Adaptive Pre-Training）：任务自适应预训练

### 4. Instruction / Task SFT（监督微调）

用 `(指令, 输出)` 配对数据做监督学习，让模型学会遵循指令、完成任务。

### 5. Multimodal Adapter Tuning（多模态适配器微调）

在视觉/音频编码器与 LLM 之间插入连接器（projector），分阶段训练使 LLM 理解新模态。

### 6. Speech / Audio Joint Tuning（语音联合微调）

音频编码器 + 适配器 + LLM 联合训练，实现语音理解与生成。

### 7. Embedding / Reranker Contrastive Tuning

用对比学习（contrastive learning）或排序损失训练向量模型/重排模型，优化检索和排序能力。

---

## 按模型类型的微调路线

| 模型类型 | 典型微调路线 |
| --- | --- |
| 文本 LLM | CPT/DAPT/TAPT → SFT → 蒸馏/压缩 |
| 多模态 LLM（图文/视频文） | 视觉编码器 / 连接器 / LLM 分阶段微调 |
| 语音/音频 LLM | 音频编码器 / 离散单元 / 适配器 / LLM 联合微调 |
| 向量模型 | 对比学习 / 三元组 / hard negative 微调 |
| 重排模型 | cross-encoder / bi-encoder reranker tuning |
