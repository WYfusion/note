# 微调数据构造

**数据质量决定微调效果的上限**。不同微调目标需要不同格式和策略的数据。

---

## 数据类型总览

| 数据类型 | 格式 | 适用微调方式 |
| --- | --- | --- |
| 指令样本 | `{instruction, input, output}` | Instruction SFT |
| 问答样本 | `{question, answer}` | SFT / 领域 QA |
| CoT / 长输出 | `{question, reasoning_chain, answer}` | 推理增强 SFT |
| 领域无标注文本 | 纯文本 | CPT / DAPT / TAPT |
| 检索对 | `{query, positive, negatives}` | 向量模型 / Reranker |
| 多模态对 | `{image/video/audio, text}` | 多模态 / 语音 LLM |

---

## 关键数据技巧

### Hard Negative Mining

"简单负样本"模型早已区分，学不到东西。**Hard negatives**（与 query 表面相似但实际不相关的文档）才能有效提升模型能力。

常见挖掘方法：

- BM25 / 向量模型召回 top-K 中的假阳性
- Cross-encoder 打分后取中间分数段
- LLM 生成迷惑性负样本

### 合成数据

- **Teacher 生成**：用强模型生成训练数据给弱模型
- **Self-Instruct**：模型自己生成指令-回答对
- **回译**：翻译到另一种语言再翻回来增加多样性
- **ASR/TTS 合成**：语音-文本互转生成配对数据

### Curriculum / Mixture Weighting

- **课程学习**：先简单后困难，模型学习更稳定
- **混合权重**：多数据源时按比例/策略混合

---

## 数据质量检查清单

- [ ]  去重（精确去重 + 近似去重）
- [ ]  格式统一（统一模板、字段名）
- [ ]  质量过滤（长度、语言、有害内容）
- [ ]  标签一致性检查
- [ ]  多模态对齐检查（图文/音文配对是否正确）
