# 数据来源、许可与合规

高质量数据是模型性能的天花板。对于语音大模型，数据来源比文本更加复杂，涉及版权和隐私的双重挑战。

## 1. 开源数据集 (Open Source Datasets)

### 1.1 英文主导
*   **LibriSpeech**: 960小时，有声书读取。干净，ASR 的“MNIST”。
*   **GigaSpeech**: 10,000小时，播客/YouTube。包含更多真实场景噪声。
*   **Common Voice**: Mozilla 众包项目，多语言，口音丰富。

### 1.2 中文数据集
*   **WenetSpeech**: 10,000+小时，从互联网挖掘。
*   **AISHELL 系列**: 经典的中文语音识别数据集。

### 1.3 多模态/多语言
*   **Multilingual LibriSpeech (MLS)**: 扩展到多种欧洲语言。
*   **VoxPopuli**: 欧洲议会演讲录音。

## 2. 网络挖掘数据 (Web-Crawled Data)

像 Whisper 这样的模型使用了 680,000 小时的私有数据，主要来源是互联网。

### 2.1 来源渠道
*   **视频平台**: YouTube (自动字幕 vs 上传者字幕)。
*   **播客 (Podcasts)**: 包含大量的对话、访谈，非常适合训练对话模型。
*   **会议录音**: 适合长音频转录任务。

### 2.2 挖掘流程
1.  爬取音视频文件。
2.  下载对应的字幕/元数据。
3.  使用 VAD 和对齐工具进行切分和清洗。

## 3. 许可与合规 (Licensing & Compliance)

### 3.1 文本/代码
*   **MIT / Apache 2.0**: 商业友好。
*   **GPL**: 传染性，需谨慎。
*   **CC-BY-NC**: 不可商用。

### 3.2 语音特有的伦理问题
*   **声纹隐私 (Biometric Privacy)**: 声音是生物特征。欧盟 GDPR 对此有严格规定。
*   **声音克隆 (Voice Cloning)**: 训练 TTS 模型（如 VALL-E）可能被用于 Deepfake 诈骗。
    *   **对策**: 许多开源 TTS 模型会添加不可见的水印，或者限制生成特定名人的声音。
*   **版权**: 有声书、歌曲通常受严格版权保护。用于训练是否属于 "Fair Use" 目前在法律上仍有争议。
