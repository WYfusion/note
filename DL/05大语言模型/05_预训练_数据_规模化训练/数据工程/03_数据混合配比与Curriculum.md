# 数据混合配比与课程学习 (Curriculum Learning)

单一来源的数据往往不足以训练出通用的模型。我们需要混合多种数据源，并精心设计训练顺序。

## 1. 数据混合 (Data Mixing)

### 1.1 文本混合策略
*   **Llama 策略**:
    *   CommonCrawl (67%): 基础语言能力。
    *   C4 (15%): 清洗过的网页。
    *   Github (4.5%): 代码推理能力。
    *   Wikipedia (4.5%): 世界知识。
    *   Books (4.5%): 长文本建模。
*   **重采样 (Upsampling)**: 高质量数据（如 Wiki, Books）会被重复训练多次（Epochs > 1），而低质量网页数据只看一次。

### 1.2 语音混合策略
语音任务对数据类型的敏感度极高。
*   **ASR (识别) 混合**:
    *   **多样性优先**: 混合干净语音 (LibriSpeech) 和 噪声语音 (GigaSpeech)。
    *   **比例**: 通常噪声数据占比要大，以提高鲁棒性。
*   **TTS (合成) 混合**:
    *   **音质优先**: 主要使用高保真录音棚数据 (Hi-Fi TTS)。
    *   **多说话人**: 混合大量不同说话人的数据以学习音色泛化。

## 2. 课程学习 (Curriculum Learning)

由易到难的训练策略。

### 2.1 序列长度课程
*   **文本**: 先在短序列（如 128）上训练，快速收敛；再逐渐增加到长序列（2048/4096）。
*   **语音**:
    *   **初期**: 使用短音频（< 10秒），Batch Size 可以很大，梯度稳定。
    *   **后期**: 引入长音频（30秒甚至更长），训练模型处理长距离依赖和上下文的能力。

### 2.2 噪声课程
*   先用干净语音训练，让模型学会基本的发音规则。
*   逐渐加入背景噪声、混响、音乐干扰，提升模型的抗噪能力。

### 2.3 跨模态课程 (Speech-LLM)
对于像 Qwen-Audio 这样的模型：
1.  **Stage 1 (ASR Pre-training)**: 仅训练 Speech Encoder，目标是纯 ASR 任务。
2.  **Stage 2 (Alignment)**: 冻结 LLM，训练 Adapter，让语音特征对齐到文本空间。
3.  **Stage 3 (Instruction Tuning)**: 解冻部分参数，使用复杂的语音指令数据进行微调。
