# 预训练全流程概览

预训练（Pre-training）是大语言模型（LLM）获得通用知识和推理能力的基石。对于语音大模型（Audio LLM），流程在数据处理和目标函数上有所不同，但核心逻辑一致。

## 1. 标准 LLM 预训练流水线

### 1.1 数据准备 (Data Preparation)
*   **收集**: Common Crawl, GitHub, Wikipedia, 书籍。
*   **清洗**: 去重 (Dedup)、过滤 (Filtering)、PII 移除。
*   **Tokenization**: BPE / Byte-level BPE。将文本转为整数序列。

### 1.2 模型架构 (Architecture)
*   通常采用 **Decoder-Only Transformer** (如 Llama, GPT)。
*   参数量级：7B, 13B, 70B, 175B+。

### 1.3 训练配置 (Training Setup)
*   **目标**: Next Token Prediction (Causal Language Modeling)。
*   **优化器**: AdamW ($\beta_1=0.9, \beta_2=0.95$)。
*   **精度**: BF16 / FP16 混合精度。
*   **并行**: 3D 并行 (Data + Tensor + Pipeline)。

## 2. 语音大模型 (Audio LLM) 的特殊性

语音模型的预训练流程根据任务类型（理解 vs 生成）有显著差异。

### 2.1 数据形态
*   **输入**: 原始波形 (Raw Waveform) 或 频谱图 (Spectrogram)。
*   **离散化**:
    *   **理解任务**: 通常直接输入连续特征 (Log-Mel)。
    *   **生成任务**: 使用 Neural Audio Codec (如 EnCodec) 将音频转为离散 Token。

### 2.2 预训练范式对比

| 范式 | 代表模型 | 核心任务 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **Encoder-Only** | Wav2Vec 2.0, HuBERT | Masked Prediction (对比学习/聚类分类) | 语音识别, 说话人识别, 情感分析 |
| **Encoder-Decoder** | Whisper | Supervised Multitask (ASR, Trans, VAD) | 通用语音转录与翻译 |
| **Decoder-Only** | VALL-E, AudioLM | Next Token Prediction (Audio Tokens) | 语音合成 (TTS), 语音续写 |
| **Multimodal** | Qwen-Audio, Speech-LLaMA | Speech Encoder -> LLM Adapter -> LLM | 语音问答, 复杂指令遵循 |

## 3. 关键挑战

### 3.1 序列长度 (Sequence Length)
*   文本: 2048 Tokens $\approx$ 1500 单词。
*   语音: 10秒音频 $\approx$ 500 帧 (理解) 或 750 Tokens (生成)。
*   **对策**: 下采样 (Subsampling)、FlashAttention、滑动窗口。

### 3.2 数据稀缺性
*   高质量文本数据：数万亿 (Trillions) Token。
*   高质量转录语音：仅数万小时 (如 LibriSpeech 960h)。
*   **对策**: 利用无标注音频 (Self-Supervised Learning) 或 弱监督数据 (Whisper 680k hours)。

## 4. 评估指标 (Evaluation)

*   **文本**: Perplexity (PPL), MMLU, GSM8K。
*   **语音理解**: WER (Word Error Rate), CER (Character Error Rate)。
*   **语音生成**: SIM-WER (说话人相似度), MOS (主观评分)。
