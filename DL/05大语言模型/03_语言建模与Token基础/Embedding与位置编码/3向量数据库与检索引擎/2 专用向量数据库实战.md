# 专用向量数据库实战

> [!abstract] 模块定位
> 专用向量数据库从底层为向量检索而设计，把 ANN 索引、元数据过滤、集合管理、扩缩容和运维接口做成独立服务。相对 [[4 pgvector 关系数据库向量扩展|PostgreSQL + pgvector]]，它更像“检索系统本身就是主角”的方案。

## 什么时候优先考虑专用向量数据库

> [!tip] 典型信号
> - 向量召回是核心链路，而不是数据库里的附属查询
> - 单集合规模、写入频率、多租户隔离要求已经逼近独立检索服务
> - 需要副本、分片、集合隔离、命名向量、独立运维面板等服务化能力
> - 计划把召回层抽成平台，而不是继续耦合在业务数据库里

## 主流产品矩阵

| 产品 | 核心定位 | 强项 | 适合场景 |
| --- | --- | --- | --- |
| **Qdrant** | 开源、向量优先的服务端引擎 | payload filter、named vectors、多租户体验好 | 自部署知识库、SaaS 检索 |
| **Milvus** | 大规模分布式向量平台 | 多索引类型、工程生态、十亿级规模 | 独立召回平台、海量数据 |
| **Weaviate** | 偏应用层的向量数据库 | hybrid、多模态、schema 体验好 | 语义搜索、多模态知识库 |
| **Pinecone** | 全托管向量服务 | 运维最省、上线快 | 小团队、快速上线 |
| **Chroma** | 本地优先的轻量方案 | 原型简单、零门槛 | Demo、实验、个人项目 |

## 工程上真正要比较的能力

| 维度 | 专用向量数据库关注点 | 与 [[4 pgvector 关系数据库向量扩展|PostgreSQL + pgvector]] 的差别 |
| --- | --- | --- |
| 索引能力 | HNSW、IVF、PQ、量化、命名向量 | 更偏“检索优先”，参数更丰富 |
| 过滤能力 | payload / scalar filter | `pgvector` 依赖 SQL 过滤，表达力强但要自己调优 |
| 服务能力 | collection、namespace、replica、shard | PostgreSQL 路线更像“一套业务库内扩展” |
| 运维能力 | 独立监控、独立扩容、独立备份 | 需要新增系统组件 |
| 集成能力 | SDK 丰富，常带 rerank / hybrid 组件 | 数据一致性通常需额外设计 |

> [!quote] 官方文档里的共性
> Qdrant 把 `payload` 过滤作为核心能力之一；Milvus 将过滤进一步区分为 standard filtering 与 iterative filtering，用于平衡过滤表达式复杂度和延迟。
> 参考：[Qdrant Filtering](https://qdrant.tech/documentation/concepts/filtering/)、[Milvus Filtered Search](https://milvus.io/docs/filtered-search.md)

## 最小实战：Qdrant 的向量检索 + metadata 过滤

```python
from qdrant_client import QdrantClient
from qdrant_client.models import Distance, FieldCondition, Filter, MatchValue, PointStruct, VectorParams
import numpy as np

client = QdrantClient(":memory:")

client.create_collection(
    collection_name="docs",
    vectors_config=VectorParams(size=768, distance=Distance.COSINE),
)

points = [
    PointStruct(
        id=i,
        vector=np.random.randn(768).tolist(),
        payload={"tenant_id": 7, "category": "tech", "lang": "zh"},
    )
    for i in range(100)
]
client.upsert(collection_name="docs", points=points)

results = client.search(
    collection_name="docs",
    query_vector=np.random.randn(768).tolist(),
    query_filter=Filter(
        must=[
            FieldCondition(key="tenant_id", match=MatchValue(value=7)),
            FieldCondition(key="lang", match=MatchValue(value="zh")),
        ]
    ),
    limit=5,
)

for hit in results:
    print(hit.id, hit.score, hit.payload)
```

## 最小实战：Milvus 的过滤检索

```python
from pymilvus import MilvusClient
import numpy as np

client = MilvusClient("milvus_demo.db")

client.create_collection(
    collection_name="docs",
    dimension=768,
    metric_type="COSINE",
)

data = [
    {
        "id": i,
        "vector": np.random.randn(768).tolist(),
        "tenant_id": 7,
        "category": "tech",
    }
    for i in range(100)
]
client.insert(collection_name="docs", data=data)

results = client.search(
    collection_name="docs",
    data=[np.random.randn(768).tolist()],
    filter='tenant_id == 7 and category == "tech"',
    limit=5,
    output_fields=["tenant_id", "category"],
)

for hits in results:
    for hit in hits:
        print(hit["id"], hit["distance"], hit["entity"])
```

## 与其他路线的边界

| 路线 | 你得到什么 | 你失去什么 |
| --- | --- | --- |
| [[4 pgvector 关系数据库向量扩展|PostgreSQL + pgvector]] | SQL、事务、`JOIN`、全文检索一体化 | 分布式和检索平台能力较弱 |
| [[3 搜索引擎向量扩展|搜索引擎向量扩展]] | BM25 / aggregation / ranking 与向量统一 | 向量优先能力通常不如专用引擎丰富 |
| [[5 Faiss 算法库实战|Faiss]] | 极强控制力和实验自由度 | 需要自己补齐服务化、存储和过滤 |
