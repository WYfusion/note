# 中文分词与多语言挑战

> [!abstract] 摘要
> 中文作为无空格语言，其分词具有独特挑战。本节深入分析中文分词的特殊性、多语言模型的支持策略、低资源语言的解决方案，以及跨文化背景下的大语言模型适配问题。

## 0. 统一概念：中文分词的特殊性

> [!important] 中文语言特性 #关键定义
> **中文分词**的挑战源于其**无空格分隔**的特性：
- 字是最小书写单位，但词是语义单位
- 词边界不明确，需要智能判断
- 存在大量歧义和未登录词
- 不同领域有不同的分词需求

### 中文语言的层次结构

| 层次 | 单位 | 特点 | 示例 |
|------|------|------|------|
| **字级** | 单个汉字 | 最小书写单位 | "你"、"好"、"世"、"界" |
| **词级** | 词组 | 语义完整单位 | "你好"、"世界"、"人工智能" |
| **句级** | 句子 | 完整表达 | "你好，世界！" |
| **篇章级** | 段落/文章 | 完整语义 | 整段文字 |

## 1. 中文分词的核心挑战

### 1.1 分词歧义

> [!warning] 歧义类型
> 中文分词面临多种歧义：

#### 1. 交叉歧义

```
"发展中国家"
- 正确：/发展/中国/家/
- 错误：/发展/中/国家/

"乒乓球拍子"
- 正确：/乒乓球/拍子/
- 错误：/乒乓/球拍/子/
```

#### 2. 组合歧义

```
"美国会通过法案"
- 正确：/美国/会/通过/法案/
- 错误：/美/国会/通过/法案/

"研究生命起源"
- 正确：/研究/生命/起源/
- 错误：/研究生/命/起源/
```

#### 3. 新词发现

```
"人工智能"
- 传统词表中不存在
- 需要识别为新词

"元宇宙"
- 新兴概念
- 网络用语
```

> [!tip] 歧义解决策略
> - **基于统计**：使用 n-gram 概率
> - **基于规则**：词典匹配
> - **基于深度学习**：序列标注模型

### 1.2 未登录词问题

> [!important] 未登录词类型
> **OOV（未登录词）**在中文中尤为严重：

| 类型 | 示例 | 处理难度 |
|------|------|----------|
| **专有名词** | "ChatGPT", "元宇宙" | 需要特殊词典 |
| **新词** | "内卷", "躺平" | 需要动态更新 |
| **领域术语** | "量子纠缠", "区块链" | 需要领域知识 |
| **混用词** | "AI智能", "互联网+" | 需要模式识别 |

### 1.3 多音字问题

```python
# 多音字示例
"银行"  # yín háng - 金融机构
"行走"  # xíng zǒu - 步行
"行业"  # háng yè - 职业

"长大"  # zhǎng dà - 成长
"长官"  # zhǎng guān - 军官
```

> [!warning] 多音字影响
> 多音字的存在增加了分词的复杂度，需要上下文判断。

## 2. 中文分词算法演进

### 2.1 基于词典的方法

> [!important] 词典分词原理
> 使用预定义词典进行最大匹配：

```python
def max_match(text, dictionary):
    result = []
    i = 0
    n = len(text)

    while i < n:
        # 从长到短匹配
        for j in range(n, i, -1):
            word = text[i:j]
            if word in dictionary:
                result.append(word)
                i = j
                break
        else:
            # 未找到匹配，按字符切分
            result.append(text[i])
            i += 1

    return result
```

#### 词典类型

| 词典类型 | 特点 | 适用场景 |
|----------|------|----------|
| **基础词典** | 3-5万常用词 | 通用分词 |
| **领域词典** | 专业术语 | 医疗、法律 |
| **新词词典** | 最新词汇 | 社交媒体 |
| **用户词典** | 自定义词 | 企业应用 |

### 2.2 基于统计的方法

>[[01_语言建模目标_MLE与交叉熵困惑度|语言建模]]] 在分词中的应用

```python
def statistical_segment(text, model):
    # 使用 Viterbi 算法找最优路径
    n = len(text)
    dp = [float('inf')] * (n + 1)
    path = [[]] * (n + 1)
    dp[0] = 0

    for i in range(1, n + 1):
        for j in range(max(0, i - 5), i):
            word = text[j:i]
            if word in model.vocab:
                score = dp[j] - log(model.prob(word))
                if score < dp[i]:
                    dp[i] = score
                    path[i] = path[j] + [word]

    return path[n]
```

### 2.3 基于深度学习的方法

#### 1. BiLSTM-CRF

```python
class BiLSTM_CRF(nn.Module):
    def __init__(self, vocab_size, embedding_dim, hidden_dim, num_tags):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embedding_dim)
        self.lstm = nn.LSTM(embedding_dim, hidden_dim // 2,
                          bidirectional=True, batch_first=True)
        self.crf = CRF(num_tags)

    def forward(self, x):
        embeddings = self.embedding(x)
        sequence_output, _ = self.lstm(embeddings)
        return self.crf(sequence_output)
```

#### 2. BERT+CRF

```python
class BERT_CRF(nn.Module):
    def __init__(self, bert_model, num_tags):
        super().__init__()
        self.bert = bert_model
        self.dropout = nn.Dropout(0.1)
        self.classifier = nn.Linear(768, num_tags)
        self.crf = CRF(num_tags)

    def forward(self, input_ids, attention_mask):
        outputs = self.bert(input_ids, attention_mask=attention_mask)
        sequence_output = self.dropout(outputs[0])
        logits = self.classifier(sequence_output)
        return self.crf.decode(logits)
```

## 3. 多语言挑战

### 3.1 语言特性差异

| 语言特性 | 分词挑战 | 代表语言 |
|----------|----------|----------|
| **无空格语言** | 需要智能分词 | 中文、日文、泰文 |
| **空格分隔** | 基于空格分割 | 英文、德文、法文 |
| **黏着语** | 词形变化复杂 | 土耳其语、芬兰语 |
| **声调语言** | 音调影响语义 | 中文、泰语、越南语 |
| **从右到左** | 文本方向处理 | 阿拉伯语、希伯来语 |

> [!note] RTL 语言处理
> 阿拉伯语、希伯来语等 RTL 语言需要特殊处理：
- 文本方向反转
- 连字符处理
- 排版规则

### 3.2 多语言模型架构

> [!important] 多语言模型设计

#### 共享参数架构

```python
class MultilingualModel(nn.Module):
    def __init__(self, vocab_sizes, hidden_dim):
        super().__init__()
        # 共享嵌入层
        self.shared_encoder = TransformerEncoder(hidden_dim)

        # 语言特定层
        self.lang_specific_layers = nn.ModuleDict({
            'zh': LanguageLayer(hidden_dim),
            'en': LanguageLayer(hidden_dim),
            'ja': LanguageLayer(hidden_dim),
            'ar': LanguageLayer(hidden_dim)  # RTL 支持
        })

        # 语言分类器
        self.lang_classifier = LanguageClassifier(hidden_dim)

    def forward(self, input_ids, lang_id):
        # 共同编码
        encoded = self.shared_encoder(input_ids)

        # 语言特定处理
        lang_output = self.lang_specific_layers[lang](encoded)

        # 语言分类
        lang_pred = self.lang_classifier(encoded)

        return lang_output, lang_pred
```

### 4.3 深度学习分词

```python
class ChineseSegmenter:
    def __init__(self, model_path):
        self.tokenizer = AutoTokenizer.from_pretrained(model_path)
        self.model = AutoModelForTokenClassification.from_pretrained(model_path)

        # 标签映射
        self.label_map = {
            'B': '词开始',
            'M': '词中间',
            'E': '词结束',
            'S': '单独成词'
        }

    def segment(self, text):
        # 分词
        inputs = self.tokenizer(text, return_tensors="pt")

        # 预测
        outputs = self.model(**inputs)
        predictions = torch.argmax(outputs.logits, dim=-1)

        # 解析结果
        tokens = self.tokenizer.convert_ids_to_tokens(inputs["input_ids"][0])
        labels = [self.label_map[p] for p in predictions[0]]

        # 组合成词
        words = []
        current_word = []

        for token, label in zip(tokens[1:-1], labels[1:-1]):  # 去掉 [CLS] 和 [SEP]
            if label == 'S':
                if current_word:
                    words.append("".join(current_word))
                    current_word = []
                words.append(token)
            elif label == 'B':
                if current_word:
                    words.append("".join(current_word))
                current_word = [token]
            elif label == 'M':
                current_word.append(token)
            elif label == 'E':
                current_word.append(token)
                words.append("".join(current_word))
                current_word = []

        if current_word:
            words.append("".join(current_word))

        return words
```

## 5. 数字与代码的处理

### 5.1 数字分割

> [!important] 数字分割策略
> **问题**：早期 Tokenizer 将 "12345" 作为一个 Token，模型无法理解数值关系。

**现代做法**：
```python
# 数字按位切分
def split_digits(number_str):
    return [c for c in number_str]

# 示例
"12345" → ["1", "2", "3", "4", "5"]

# 不同模型策略
{
    "Llama 2": "123" → ["1", "2", "3"],
    "GPT-4": "123" → ["123"],  # 保持完整
    "Qwen": "123" → ["123"]    # 中文数字
}
```

### 5.2 代码中的空格

> [!warning] 代码空格处理
> Python 依赖缩进，必须精确保留：

```python
# SentencePiece 处理方式
sp = spm.SentencePieceProcessor(model_file='code.model')

# 空格被转换为 ▁
code = "def hello():"
tokens = sp.encode(code, out_type=str)
# ['▁def', '▁hello', '(', ')', ':']

# 反转时恢复空格
text = sp.decode(tokens)
# "def hello():"
```

## 6. 主流模型对比

| 模型 | 词表大小 | 中文处理 | 特点 | 压缩比  |
|------|----------|----------|------|----------------|
| **GPT-4 (cl100k)** | ~100k | 一般 | 字节级 BPE | 中文 1-3 Token/字 |
| **Llama 3** | 128k | 较好 | 多语言平衡 | 中文 1-2 Token/字 |
| **Qwen (千问)** | ~152k | **极好** | 中文深度优化 | 中文 <1 Token/字 |
| **Baichuan** | ~64k | 好 | 中文优化 | 中文 1-2 Token/字 |
| **XLM-R** | 250K | 多语言 | 跨语言能力强 | 多语言均衡 |

> [!important] 压缩率定义
> **压缩率 (Compression Ratio)**：
> $$ \text{Ratio} = \frac{\text{Bytes of Text}}{\text{Number of Tokens}} $$
> 比值越大，说明每个 Token 承载的信息越多，推理效率越高。

qwen_vocab = {
    '常用词': 10000,      # 高频词保持完整

    '网络用语': 20000,    # 新兴词汇

def qwen_tokenize(text):
    # 优先识别已知词

            return [word] + qwen_tokenize(text[len(word):])

### 7.2 压缩率平衡

> [!important] 重采样策略
> 解决语言不平衡问题：

```python
def balanced_training():
    # 原始分布
    original_dist = {
        'en': 0.9,
        'zh': 0.05,
        'ja': 0.03,
        'other': 0.02
    }

    # 重采样后分布
    balanced_dist = {
        'en': 0.5,
        'zh': 0.2,
        'ja': 0.15,
        'other': 0.15
    }

    # 重采样因子
    resample_factor = {
        'en': 0.5 / 0.9,
        'zh': 0.2 / 0.05,
        'ja': 0.15 / 0.03,
        'other': 0.15 / 0.02
    }

    return resample_factor
```

### 8.3 实时自适应分词

```python
class AdaptiveTokenizer:
    def __init__(self):
        self.models = {
            'general': load_model('general'),
            'medical': load_model('medical'),
            'tech': load_model('tech')
        }
        self.usage_stats = defaultdict(int)

    def tokenize(self, text, domain=None):
        # 根据领域选择模型
        if domain and domain in self.models:
            model = self.models[domain]
            self.usage_stats[domain] += 1
        else:
            model = self.models['general']
            self.usage_stats['general'] += 1

        # 分词
        tokens = model.tokenize(text)

        # 收集反馈
        self._collect_feedback(text, tokens)

        # 优化模型
        if self.usage_stats[domain] % 1000 == 0:
            self._optimize_model(domain)

        return tokens
```
