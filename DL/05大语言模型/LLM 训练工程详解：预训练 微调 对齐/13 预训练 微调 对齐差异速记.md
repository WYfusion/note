# 13. 预训练 / 微调 / 对齐差异速记

三个阶段共享底层工程规则，但各自有不同的关注重点和易爆点。

---

## 差异对照表

| 维度 | 🏗️ 预训练 | 🎯 微调 | ⚖️ 对齐 |
| --- | --- | --- | --- |
| **首要关注** | 吞吐、稳定、长程曲线、ckpt 成本 | 过拟合、灾难遗忘、数据配比 | reward/KL/rollout + 采样分布漂移 |
| **数据量级** | 万亿 token | 万~百万样本 | 万~十万样本 |
| **LR 量级** | 1e-4 ~ 3e-4 | 1e-5 ~ 5e-5 | 1e-6 ~ 5e-6 |
| **训练时长** | 数周~数月 | 数小时~数天 | 数小时~数天 |
| **Checkpoint 频率** | 高频（token 成本高） | epoch 粒度即可 | rollout 前后各存 |
| **主要风险** | 脏 batch / 混精 overflow / 硬件故障 | 过拟合 / 数据配比失衡 / 灾难遗忘 | reward hacking / KL 崩塌 / 采样漂移 |
| **监控重点** | loss + grad norm + tokens/sec + MFU | train/val gap + 任务 metric | reward / KL / advantage / rollout stats |

---

## 预训练特有关注

- **吞吐是命脉**：万亿 token 下 10% 的吞吐提升 = 节省数天训练时间
- **长程稳定**：需要持续数十万 step 不崩溃
- **Checkpoint 成本**：模型 + optimizer 几十到几百 GB
- **数据去重与配比**：多源数据的混合比例直接影响模型质量

---

## 微调特有关注

- **过拟合**：数据量小，极易在 2~3 epoch 后过拟合
- **灾难遗忘**：学新知识同时丢失预训练能力
- **数据配比**：多任务微调时各任务比例需要调优
- **评估**：需要 task-specific metric，不能只看 loss

---

## 对齐特有关注

<aside>
⚠️

对齐阶段在共有工程规则基础上，额外引入以下挑战：

- **reward / KL / advantage / rollout 统计**：需要独立监控面板
- **on-policy 数据漂移**：模型变化导致生成分布变化，进而影响 reward 分布
- **采样温度与长度分布**：温度过高生成随机，过低缺乏多样性
- **reward hacking**：模型找到 reward model 的漏洞而非真正提升质量
</aside>
