# 10. 监控机制与告警体系

监控是训练的「仪表盘」——没有监控的训练等于蒙眼开车。好的监控体系 = 实时面板 + 自动告警 + 异常留证。

---

## 实时面板（最少要有）

### 训练指标

- `train loss` / `val loss`
- `lr`
- `grad norm` / `clip ratio`
- `overflow / nonfinite count`

### 吞吐指标

- `tokens/sec` / `samples/sec`
- `MFU / HFU`（若有）
- `step time 分解`：data / forward / backward / optimizer / communication

### 系统指标

- `GPU util` / `GPU mem`
- `data loader wait`
- `checkpoint save time`

---

## 告警规则

| 告警项 | 触发条件 | 严重级别 |
| --- | --- | --- |
| loss spike | loss > rolling_mean + k*std | 🔴 Critical |
| grad norm 突变 | 突增 >3x 或突降至 ~0 | 🔴 Critical |
| tokens/sec 下降 | 下降 >10%~20% | 🟡 Warning |
| dataloader wait 占比过高 | data wait > 20% step time | 🟡 Warning |
| GPU util 长时间低 | util < 60% 持续 >5min | 🟡 Warning |
| checkpoint 时间变长 | save time 持续增长 | 🟠 Info |
| val loss 连续恶化 | 连续 N 次 eval 变差 | 🔴 Critical |

---

## 异常留证

<aside>
📸

告警触发时自动保存：

- 前后 **N 个 step** 的 batch 元信息
- **profiler trace**（torch.profiler / Nsight）
- 当前 **grad norm per-layer dump**
- 相关 **rank 日志片段**
