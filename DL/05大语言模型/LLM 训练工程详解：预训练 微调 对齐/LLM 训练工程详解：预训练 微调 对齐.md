# LLM 训练工程详解：预训练 / 微调 / 对齐

本文系统梳理大模型训练的 **工程实战细节**，涵盖预训练、微调（SFT / LoRA）、对齐（RLHF / DPO / PPO）三大阶段。底层工程规则大体相同，核心主线：

> **稳定性 + 可回滚 + 可诊断 + 吞吐 + 可复现**

对齐阶段额外关注：`reward / KL / advantage / rollout 统计`、`on-policy 数据漂移`、`采样温度与长度分布`。

## 总原则
⚡**先保证可回滚，再追求更高吞吐** · **先证明不是数据坏，再调超参** · **看趋势，不迷信单个 step** · **所有异常都必须能定位到 step、batch、layer、rank**

### 三阶段共性 vs 差异

| 维度 | 预训练 | 微调 | 对齐 |
| --- | --- | --- | --- |
| **首要关注** | 吞吐、稳定、长程曲线、ckpt 成本 | 过拟合、灾难遗忘、数据配比 | reward/KL/rollout + 采样漂移 |
| **Checkpoint** | 大量 token → 高频存 | epoch 粒度存即可 | rollout 前后各存 |
| **LR 量级** | 1e-4 ~ 3e-4（典型 7B） | 1e-5 ~ 5e-5 | 1e-6 ~ 5e-6（PPO actor） |
| **易爆点** | 脏 batch / 混精 overflow | 数据配比失衡 / 遗忘 | reward hacking / KL 崩塌 |
