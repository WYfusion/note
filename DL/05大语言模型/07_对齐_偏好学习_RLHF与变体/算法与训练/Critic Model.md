Critic Model评价模型的作用就是计算 **优势函数 (Advantage Function)**，从而**减少策略梯度估计的方差**，使训练更稳定、高效。
- **Actor (做题人):** 在生成 Token 时，它只能根据当前的上下文去**猜**下一个词。它当时是不知道结果好坏的。
- **Critic (估分人):** 在训练 Critic 时，**整个对话已经结束了**。Critic 手里拿着的是“整张卷子”以及最重要的——**Reward Model 打出的最终分数**。
#### Critic 是怎么变聪明的？
Critic 的训练目标是：**最小化预测误差（MSE Loss）**。
- **场景：** 假设 Actor 犯傻了，在“你好”后面接了“坏”（烂 Token）。
- **结果：** 整个句子生成完后，Reward Model (RM) 看到这句话很不通顺，给了一个**极低的分数（比如 -10 分）**。
此时**Critic 的训练：**
1. PPO 算法会对 Critic 说：“嘿，你之前在 $t=2$ 看到‘坏’字的时候，预测将来的得分是 +5 分，但实际上最后的真实得分是 -10 分。**你预测错了！**”
2. **Critic 更新参数：** 为了减少这个误差，Critic 会修改自己的参数（权重），记住这个教训：**“下次再看到上下文里出现‘你好 坏’，我就要把价值（Value）预测得非常低。”**

**所以，Critic 的智慧来自于 Reward Model 的打分反馈。** 它是通过不断地看“这个状态”和“最终得分”之间的关系，强行拟合出来的判断力。相当于考完试后另一个学生拿着老师给的分数来检查评估哪里有问题。

### 举个通俗的例子
想象 Actor 和 Critic 是一对双胞胎（参数相同），他们一起去参加一场迷宫游戏。
1. **探索阶段（Sampling）：**
    - **Actor（哥哥）** 负责走路。他走到一个岔路口，瞎选了一条路（输出了一个烂 Token），结果掉进了坑里（Reward Model 给了低分）。
    - **Critic（弟弟）** 只是跟在后面看着，当时他也不知情，可能还觉得哥哥走得挺对。
2. **复盘/训练阶段（Training）：**
    - 游戏结束，大家都知道结果是“掉坑里了”。
    - **训练 Critic：** 教练（算法）指着那个岔路口告诉弟弟：“你看，当时在这个路口，只要往那边走就是坑。你当时没看出来吧？现在记住了吗？这个路口很危险，价值极低！” —— **弟弟（Critic）学会了识别坏动作。**
    - **训练 Actor：** 弟弟学会后，转头告诉哥哥：“下次再到这种路口，千万别往那边走了，我刚学过，那边是死路。” —— **哥哥（Actor）通过 Advantage 信号学会了不输出烂 Token。**

通常 Actor 和 Critic 共享同一个 Transformer 的底座（Backbone），只是最后分出了两个头（Head）：
- **Actor Head:** 输出词表大小（vocab size）的 logits。
- **Critic Head:** 输出 1 个标量（Scalar Value）。
即便底座参数一样，它们也会“懂”得分辨好坏，原因在于：

#### 1. 不同的 Loss Function 塑造了不同的能力
- **Actor 的 Loss:** 它的目标是“**提高高分动作的概率**”。它在尝试探索哪里有高回报。
- **Critic 的 Loss:** 它的目标是“**准确预测当前的局面值多少分**”。它不负责探索，只负责客观评价。
即使是同一个神经网络底座，通过**梯度反向传播**，Reward Model 的判别能力会通过 Critic 的 Loss “蒸馏”进这个网络里。

#### 2. 这里的“好坏”标准是谁定的？

一定要记住，**RLHF 中真正的上帝是 Reward Model (RM)**，而不是 Critic。
- Critic 只是 RM 的**代理人（Proxy）**。
- RM 告诉 Critic 什么是好的，Critic 学会后，再手把手教 Actor 怎么做。

$V(s_{t+1})$ 的计算不是通过数学公式推导出来的，而是通过 **神经网络的前向传播（Inference）** 得到的。
假设当前时刻是 $t$，模型刚刚生成了动作 $a_t$（即第 $t$ 个 token）。
现在的状态 $s_{t+1}$ 实际上等于：“Prompt + 之前生成的 $t-1$ 个 token + 刚刚生成的 $a_t$”。
Critic 模型会接收这个完整的序列作为输入：

1. **输入：** 完整的 token 序列 $sequence = [Prompt, a_1, a_2, ..., a_t]$。
2. **Transformer 处理：** 经过多层 Transformer 运算，拿到最后一个 token 位置对应的 **Hidden State（隐藏层向量）**，记为 $h_{t+1}$。
3. 线性层映射： 将 $h_{t+1}$ 输入到一个线性层（Linear Layer），映射为一个标量。$$V(s_{t+1}) = \text{Linear}(h_{t+1})$$
    

#### 3. 它的含义
这个标量代表了 Critic 模型对 **“当前话说到这个份上，未来能拿多少分”** 的预估。
注意边界情况： 如果 $t=T$（已经是最后一个 token 了），这就意味着游戏结束了（Terminal State）。在计算优势函数（Advantage）时，我们通常设定：$$V(s_{Final+1}) = 0$$
因为结束之后就没有未来的奖励了。


我们看一个具体的 **GAE (Generalized Advantage Estimation)** 计算流：假设 Prompt 是 "你好"，模型回答 "很高兴" (3个token: 很, 高, 兴)。
这里 $T=3$。

1. **t=1 ("很"):**
    - 即时奖励 $r_1$: 只有 KL 惩罚（微小的负数）。
    - $V(s_1)$: Critic 看了 "你好 很"，预测一个分。
    - $V(s_2)$: Critic 看了 "你好 很 高"，预测一个分。
    - **计算优势:** $A_1 \approx r_1 + V(s_2) - V(s_1)$ (Critic 告诉 Actor: 下一步状态价值变高了，说明"很"选对了；如果选了其他的，那就低了，说明选错了)。

2. **t=2 ("高"):**
    - 即时奖励 $r_2$: 只有 KL 惩罚。
    - $V(s_2)$: 上一步算过了。
    - $V(s_3)$: Critic 看了 "你好 很 高 兴"，预测一个分。
    - **计算优势:** $A_2 \approx r_2 + V(s_3) - V(s_2)$。

3. **t=3 ("兴", T=3, 最后一个Token):**
    - **关键点来了！**
    - 即时奖励 $r_3$: **$R_{model}(\text{“你好很高兴”}) - KL$**。这里加上了真正的“大奖”。
    - $V(s_3)$: 上一步算过了。
    - $V(s_4)$: 游戏结束，价值为 0。
    - **计算优势:** $A_3 = r_3 + 0 - V(s_3)$。
