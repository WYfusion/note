**对齐蒸馏**蒸的不是"知识本身"，而是**偏好、价值取向、安全边界、judge/reward 信号**。本质是把 teacher/judge 的"好坏判断函数"迁移给 student。

> [!info] 与 SFT 蒸馏的区别

> SFT 蒸馏学的是"怎么做"；对齐蒸馏学的是"什么更好 / 什么不该做"。

---

## 1. 经典方案

|方案|核心流程|特点|
|---|---|---|
|**RLHF**|SFT → preference data → reward model → PPO/RL|最经典对齐流水线，训练复杂但效果验证充分|
|**RLAIF / Constitutional AI**|用 AI feedback 替代部分人工偏好；规则/宪法驱动 critique → revision → preference|大幅降低人工标注成本|

详见 → [[1. RLHF 与 RLAIF / Constitutional AI]]

## 2. 当前主流方案：偏好优化族

|方案|核心思想|需要 reference model?|需要 reward model?|
|---|---|---|---|
|**DPO**|不显式训练 reward model，直接用 preference pairs 优化|✅|❌|
|**ORPO**|在 odds-space 中优化，SFT + preference 合并训练|❌|❌|
|**SimPO**|reference-free，用序列平均 log probability 做隐式奖励|❌|❌|
|**KTO**|从 prospect-theoretic utility 出发，仅需二元好/坏信号|❌|❌|

详见 → [[2. DPO / ORPO / SimPO / KTO 偏好优化族]]

## 4. 适用场景

- 安全对齐、拒答策略

- helpfulness / harmlessness / honesty

- 品牌语气、企业规范、内部政策

- 真实产品风格一致性

---

## 5. 实务推荐

> [!important] 推荐路线

> 小中模型优先 **SFT + DPO/ORPO/SimPO**

**高安全要求时增加**：

- AI judge 打分

- constitution / rulebook 规则约束

- adversarial prompts 对抗测试

- refusal calibration 拒答校准

> [!tip] 不要只蒸"赢者答案"

> 还要蒸：rejected answers、critique、refusal boundary、safe completion style。完整的对齐需要正反两面信号。
