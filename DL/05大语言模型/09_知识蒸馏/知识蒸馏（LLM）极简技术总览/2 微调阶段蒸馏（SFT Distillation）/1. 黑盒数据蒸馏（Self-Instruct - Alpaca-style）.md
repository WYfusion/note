黑盒数据蒸馏是 LLM 蒸馏中最工程友好的方式：不需要教师权重，只需 API 调用获取教师文本输出。

---

## 1. Self-Instruct

> [!info] 核心思想

> 模型自举生成 instruction → input → output 三元组，经过过滤后回训自己或训练更小的模型。

**Pipeline**：

1. 从少量种子 instructions（~175条）出发

1. 模型生成新 instructions

1. 模型为每条 instruction 生成 input/output

1. 过滤：去重、格式检查、质量筛选

1. 积累足够数据后做 SFT

**过滤策略**：

- ROUGE-L 去重（与已有 instructions 相似度过高则丢弃）

- 格式合规检查（是否符合 instruction/input/output 结构）

- 长度过滤

- 可选：用强模型做质量打分

详见 → [[1. Self-Instruct Pipeline 实现与数据过滤]]

## 2. Alpaca-style Distillation

> [!info] 核心思想

> 用强闭源 teacher（如 GPT-4）批量生成高质量 instruction-response 数据，再用开源小模型做 SFT。

**典型流程**：

1. 准备 instruction 集合（人工 + Self-Instruct 混合）

1. 调用强 teacher API 生成 response

1. 质量过滤

1. 对开源小模型做 SFT

**代表项目**：

|项目|教师|学生|数据量|
|---|---|---|---|
|Alpaca|text-davinci-003|LLaMA-7B|52K|
|Vicuna|ChatGPT（ShareGPT 对话）|LLaMA-13B|70K|
|WizardLM|ChatGPT（Evol-Instruct）|LLaMA|250K|
|Orca|GPT-4|LLaMA-13B|5M|

## 3. 数据质量工程

> [!important] 数据质量决定蒸馏上限

**关键实践**：

- **多样性**：instruction 类型覆盖广（open QA、code、math、reasoning、roleplay...）

- **难度分层**：easy → medium → hard curriculum

- **Self-consistency 筛选**：同一 prompt 多次采样，取一致性高的

- **Best-of-N**：采样 N 个，选最佳

- **AI judge 打分**：用强模型给 response 打分/排名

- **去污染**：确保不泄露评测集数据
