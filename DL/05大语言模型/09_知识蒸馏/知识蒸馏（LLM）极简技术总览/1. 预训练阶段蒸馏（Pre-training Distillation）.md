**预训练蒸馏**是在继续预训练或从头预训练时，把普通语言模型损失与蒸馏损失联合训练，让学生模型从教师模型处获得**通用语言表征能力**。

> [!info] 核心公式

> $L = L_{LM} + \lambda_1 L_{KD} + \lambda_2 L_{repr}$

> 其中 $L_{LM}$ 为标准语言模型损失，$L_{KD}$ 为 logits 蒸馏损失，$L_{repr}$ 为表征对齐损失。

---

## 1. 常见蒸馏对象

- **token-level soft targets**：教师在每个 token 位置的概率分布

- **hidden states**：教师中间层的隐藏表示

- **self-attention / relation matrix**：教师注意力权重或层间关系矩阵

- **layer mapping**：教师多层 → 学生少层的映射策略（如 teacher 12层 → student 4层，取 3/6/9/12 层对齐）

---

## 2. 经典方案

|方案|核心思想|蒸馏对象|年代|
|---|---|---|---|
|**Hinton KD**|soft targets + temperature scaling，蒸馏范式源头|logits（soft labels）|2015|
|**DistilBERT**|预训练阶段联合 LM loss + distillation loss + cosine loss|logits + hidden|2019|
|**TinyBERT**|两阶段蒸馏（预训练 + 任务），hidden/attention 层面对齐|hidden + attention + logits|2020|
|**MiniLM**|重点蒸馏 self-attention 关系（Q-K relation），压缩效率高|self-attention relations|2020|

## 3. 新/流行方案：LLM 预训练蒸馏（PD）

随着开源 LLM 的繁荣，预训练蒸馏从"小 BERT 压缩"扩展到"通用 LLM 继续预训练压缩"：

- **offline logits**：预先缓存教师 logits 到磁盘，训练时加载

- **online logits**：训练时实时推理教师获取 logits

- **KL 散度选择**：forward KL / reverse KL / mixed KL，不同选择对学生生成质量影响显著

- **logits 裁剪**：top-k / top-p logits 蒸馏，减少存储和噪声

- **温度缩放**：动态/静态 temperature 策略

- **Scaling law**：更关注 teacher-student size ratio 和 teacher quality 上限

> [!tip] 趋势

> - 从单纯 token KD 扩展到 **token + representation + calibration** 联合优化

> - Meta 最新研究（2025）表明蒸馏预训练可能损害 ICL 能力，需权衡

详见 → [[2. LLM 预训练蒸馏新范式（PD）]]

## 4. 适用场景

- 想做**基础模型压缩**（如 70B → 7B）

- 想保留**通用语言能力**而不是只学某个任务

- 有白盒教师或可离线保存 logits

---

## 5. 实务推荐

> [!important] 工程首选

> `LM loss + KD(logits) + 少量 representation loss`

- **数据**：尽量覆盖学生未来使用域

- **注意事项**：

    - teacher 太强不一定线性提升 student（容量瓶颈）

    - student 太小会出现表达力不足

    - 先做 tokenizer / vocab / architecture 对齐决策

    - DDK（NeurIPS 2024）提出按领域动态调整蒸馏数据比例
