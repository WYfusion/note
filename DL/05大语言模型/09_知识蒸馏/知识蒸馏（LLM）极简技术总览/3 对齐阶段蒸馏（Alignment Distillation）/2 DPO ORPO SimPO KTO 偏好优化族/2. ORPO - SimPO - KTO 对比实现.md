## 1. ORPO（Odds Ratio Preference Optimization）

### 1.1 核心思想

将 SFT 和偏好优化合并为单次训练，不需要 reference model。

**损失函数**：

$$L_{ORPO} = L_{SFT} + \lambda \cdot L_{OR}$$

其中 Odds Ratio 损失：

$$L_{OR} = -\log \sigma\left(\log \frac{\text{odds}_\theta(y_w|x)}{\text{odds}_\theta(y_l|x)}\right)$$

$$\text{odds}_\theta(y|x) = \frac{P_\theta(y|x)}{1 - P_\theta(y|x)}$$

```Python
import torch
import torch.nn.functional as F

def orpo_loss(
    policy_chosen_logps: torch.Tensor,
    policy_rejected_logps: torch.Tensor,
    sft_loss: torch.Tensor,
    lambda_weight: float = 1.0,
) -> torch.Tensor:
    """ORPO loss: SFT + odds ratio preference."""
    # Convert log probs to odds
    chosen_odds = policy_chosen_logps.exp() / (1 - policy_chosen_logps.exp() + 1e-8)
    rejected_odds = policy_rejected_logps.exp() / (1 - policy_rejected_logps.exp() + 1e-8)

    # Log odds ratio
    log_odds_ratio = torch.log(chosen_odds / (rejected_odds + 1e-8) + 1e-8)
    or_loss = -F.logsigmoid(log_odds_ratio).mean()

    return sft_loss + lambda_weight * or_loss
```

> [!tip] ORPO 优势

> 一次训练完成 SFT + 偏好对齐，无需 reference model，显存减半。

## 2. SimPO（Simple Preference Optimization）

### 2.1 核心思想

用平均 log probability 作为隐式 reward，完全无需 reference model。

**SimPO reward**：

$$r_{SimPO}(x, y) = \frac{\beta}{|y|} \log \pi_\theta(y|x)$$

**损失函数**：

$$L_{SimPO} = -\log \sigma\left(\frac{\beta}{|y_w|} \log \pi_\theta(y_w|x) - \frac{\beta}{|y_l|} \log \pi_\theta(y_l|x) - \gamma\right)$$

其中 $\gamma$ 为 target reward margin。

```Python
def simpo_loss(
    chosen_logps: torch.Tensor,      # sum log P(y_w|x)
    rejected_logps: torch.Tensor,    # sum log P(y_l|x)
    chosen_lengths: torch.Tensor,    # |y_w|
    rejected_lengths: torch.Tensor,  # |y_l|
    beta: float = 2.5,
    gamma: float = 0.5,
) -> torch.Tensor:
    """SimPO: reference-free, length-normalized."""
    chosen_reward = beta * chosen_logps / chosen_lengths
    rejected_reward = beta * rejected_logps / rejected_lengths
    logits = chosen_reward - rejected_reward - gamma
    return -F.logsigmoid(logits).mean()
```

> [!important] SimPO 创新点

> 长度归一化解决 DPO 偏向长回复的问题；$gamma$ margin 增强区分度。

## 3. KTO（Kahneman-Tversky Optimization）

### 3.1 核心思想

基于前景理论的效用函数，只需二值信号（好/坏），不需要成对偏好。

**损失函数**：

$$L_{KTO} = \mathbb{E}_{(x,y) \sim D^+}[w(y)(1 - v_\theta^+)] + \mathbb{E}_{(x,y) \sim D^-}[w(y) \cdot v_\theta^-]$$

其中：

$$v_\theta^+ = \sigma(\beta(r_\theta(x,y) - z_0)), \quad v_\theta^- = \sigma(\beta(z_0 - r_\theta(x,y)))$$

```Python
def kto_loss(
    policy_logps: torch.Tensor,
    ref_logps: torch.Tensor,
    is_desirable: torch.Tensor,  # bool: True=good, False=bad
    beta: float = 0.1,
) -> torch.Tensor:
    """KTO: binary utility signal, no pairwise data needed."""
    # Implicit reward
    rewards = beta * (policy_logps - ref_logps)

    # Reference point (mean reward)
    z0 = rewards.detach().mean()

    desirable_mask = is_desirable.float()
    undesirable_mask = 1 - desirable_mask

    # Value functions
    v_pos = torch.sigmoid(rewards - z0)
    v_neg = torch.sigmoid(z0 - rewards)

    loss_pos = (desirable_mask * (1 - v_pos)).sum() / desirable_mask.sum().clamp(min=1)
    loss_neg = (undesirable_mask * v_neg).sum() / undesirable_mask.sum().clamp(min=1)

    return loss_pos + loss_neg
```

> [!tip] KTO 最大优势

> 只需 thumbs-up / thumbs-down 信号，数据构造成本极低。
