DPO 证明 RLHF 的 reward model + PPO 可等价转化为直接在偏好数据上的分类问题。

---

## 1. 从 RLHF 到 DPO 的推导

**RLHF 目标**：

$$\max_\pi \mathbb{E}[r(x,y)] - \beta \cdot KL(\pi \| \pi_{ref})$$

**最优解**：

$$\pi^*(y|x) = \frac{1}{Z(x)} \pi_{ref}(y|x) \exp\left(\frac{r(x,y)}{\beta}\right)$$

**反解出隐式 reward**：

$$r(x,y) = \beta \log \frac{\pi^*(y|x)}{\pi_{ref}(y|x)} + \beta \log Z(x)$$

代入 Bradley-Terry 模型，$Z(x)$ 消去，得到 **DPO 损失**：

$$L_{DPO} = -\mathbb{E}\left[\log \sigma\left(\beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right)\right]$$

> [!important] 核心洞察

> DPO 把 reward model 完全集成到了 policy 中，不需要显式训练 RM，也不需要 PPO。

## 2. Python 实现

```Python
import torch
import torch.nn.functional as F

def dpo_loss(
    policy_chosen_logps: torch.Tensor,   # log P_theta(y_w|x)
    policy_rejected_logps: torch.Tensor, # log P_theta(y_l|x)
    ref_chosen_logps: torch.Tensor,      # log P_ref(y_w|x)
    ref_rejected_logps: torch.Tensor,    # log P_ref(y_l|x)
    beta: float = 0.1,
) -> torch.Tensor:
    """DPO loss implementation."""
    chosen_rewards = beta * (policy_chosen_logps - ref_chosen_logps)
    rejected_rewards = beta * (policy_rejected_logps - ref_rejected_logps)
    logits = chosen_rewards - rejected_rewards
    loss = -F.logsigmoid(logits).mean()

    # Metrics
    with torch.no_grad():
        chosen_reward = chosen_rewards.mean()
        rejected_reward = rejected_rewards.mean()
        accuracy = (logits > 0).float().mean()

    return loss

def compute_log_probs(model, input_ids, labels, attention_mask):
    """Compute per-sequence log probabilities."""
    outputs = model(input_ids=input_ids, attention_mask=attention_mask)
    logits = outputs.logits[:, :-1, :]
    labels = labels[:, 1:]
    per_token = F.cross_entropy(
        logits.reshape(-1, logits.size(-1)),
        labels.reshape(-1),
        reduction='none'
    ).reshape(labels.shape)
    mask = (labels != -100).float()
    return -(per_token * mask).sum(-1)  # negative CE = log prob
```

## 3. 训练流程

```Python
class DPOTrainer:
    def __init__(self, policy, ref_policy, tokenizer, beta=0.1, lr=5e-7):
        self.policy = policy
        self.ref = ref_policy
        self.tokenizer = tokenizer
        self.beta = beta
        self.optimizer = torch.optim.AdamW(policy.parameters(), lr=lr)
        for p in self.ref.parameters():
            p.requires_grad = False

    def train_step(self, batch):
        # Compute log probs for chosen and rejected
        pi_chosen = compute_log_probs(self.policy, batch['chosen_ids'],
                                       batch['chosen_labels'], batch['chosen_mask'])
        pi_rejected = compute_log_probs(self.policy, batch['rejected_ids'],
                                         batch['rejected_labels'], batch['rejected_mask'])
        with torch.no_grad():
            ref_chosen = compute_log_probs(self.ref, batch['chosen_ids'],
                                            batch['chosen_labels'], batch['chosen_mask'])
            ref_rejected = compute_log_probs(self.ref, batch['rejected_ids'],
                                              batch['rejected_labels'], batch['rejected_mask'])

        loss = dpo_loss(pi_chosen, pi_rejected, ref_chosen, ref_rejected, self.beta)
        loss.backward()
        self.optimizer.step()
        self.optimizer.zero_grad()
        return loss.item()
```

## 4. 关键超参数

|参数|推荐|说明|
|---|---|---|
|$\beta$|0.05~0.5|越大 → 偏好差距越明显；太大会不稳定|
|lr|1e-7 ~ 5e-6|比 SFT 低 1~2 个数量级|
|epochs|1~3|过多会过拟合偏好数据|
