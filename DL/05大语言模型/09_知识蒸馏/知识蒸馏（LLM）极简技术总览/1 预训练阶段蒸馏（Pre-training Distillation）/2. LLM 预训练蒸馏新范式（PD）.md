## 1. Offline vs Online Logits

|模式|机制|优点|缺点|
|---|---|---|---|
|**Offline**|预先缓存 teacher logits 到磁盘|训练时无需 teacher 推理，显存友好|存储成本高（全词表 logits）；需 top-k 裁剪|
|**Online**|训练时实时推理 teacher|logits 精确，支持动态 curriculum|训练吞吐降低；需额外 GPU 给 teacher|

> [!tip] 工程选择

> 数据量大 + 存储充足 → offline 更经济；需要动态调整或 teacher 更新 → online。

## 2. KL 散度选择

这是 LLM PD 中影响最大的设计选择之一：

|KL 类型|公式|行为|适用场景|
|---|---|---|---|
|**Forward KL**|$KL(p_T \\| p_S)$|mean-seeking：student 覆盖 teacher 全部模式|保留多样性、通用能力|
|**Reverse KL**|$KL(p_S \\| p_T)$|mode-seeking：student 集中在 teacher 高概率区|生成质量优先、避免幻觉|
|**Mixed KL**|$\beta \cdot KL_f + (1-\beta) \cdot KL_r$|平衡覆盖度与集中度|通用推荐|

> [!important] MiniLLM（ICLR 2024）发现

> reverse KL 在 LLM 生成任务中显著优于 forward KL，因为它避免 student 在 teacher 低概率区浪费概率质量。

## 3. Logits 裁剪与 Top-k/Top-p 蒸馏

**问题**：全词表 logits（32k~128k）存储和计算成本极高。

**解决方案**：

- **Top-k 裁剪**：只保留 teacher 概率最高的 k 个 token 的 logits，其余置零/忽略

- **Top-p 裁剪**：按累积概率阈值截断

- **温度缩放**：$q = text{softmax}(z/T)$，配合裁剪使用

> [!tip] 实践经验

> - offline 存储：top-k=128~512 通常足够

> - 裁剪后需重归一化

> - 温度 T 和 k 值需联合调参

## 4. Scaling Law 与 Teacher-Student Size Ratio

**关键发现**：

- teacher 太强不一定线性提升 student（容量瓶颈 / capacity gap）

- 最优 teacher-student size ratio 通常在 2x~10x

- DDK（NeurIPS 2024）发现：按领域动态调整蒸馏数据组成可提升效果

- Meta（2025）发现：蒸馏预训练可能损害 in-context learning 能力

> [!faq] 什么时候蒸馏比直接训练更好？

> 当 student 能力不足以从原始数据中学到足够信号时，teacher 的软标签提供了更高效的学习信号。但如果 teacher-student gap 过大，teacher 的信息反而会成为噪声。

## 5. 从 token KD 到联合优化

**趋势**：不再只蒸 token-level logits，而是联合多个目标：

- **Token-level KD**：logits 对齐（基础）

- **Representation alignment**：hidden states / attention map 对齐

- **Calibration**：让 student 的置信度分布接近 teacher

$$L_{total} = L_{LM} + \lambda_1 L_{KD}^{token} + \lambda_2 L_{repr} + \lambda_3 L_{calib}$$

实务上 $lambda_1 gg lambda_2 > lambda_3$，token KD 仍是主导项。
