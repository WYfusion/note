本页面详细介绍 LLM 预训练蒸馏中 logits 裁剪的理论与实现。

---

## 1. 问题背景

**全词表 logits 的成本**：

- 词表大小 $V$ 通常为 32K~128K

- 每个 token 位置存储 $V$ 个 float16 值 = 64KB~256KB

- 1B tokens × 128K vocab × 2 bytes = **256 TB**（不可接受）

> [!important] 核心矛盾

> 全量 logits 信息最丰富但存储不可行；裁剪后需保证蒸馏效果不显著下降。

---

## 2. Top-k Logits 裁剪

**策略**：只保留 teacher 概率最高的 $k$ 个 token 的 logits。

$$q_i^{topk} = \begin{cases}
q_i^{(T)} / Z_k & \text{if } i \in \text{top-k}(q^{(T)}) \\
0 & \text{otherwise}
\end{cases}$$

其中 $Z_k = \sum_{i \in topk} q_i^{(T)}$ 为重归一化常数。

**存储节省**：

- $k = 128$：存储降为全量的 $128/32000 \approx 0.4\%$

- $k = 512$：存储降为 $512/32000 \approx 1.6\%$

---

## 3. Top-p Logits 裁剪

**策略**：按概率从大到小排序，取累积概率达到 $p$ 的最小 token 集合。

$$S_p = \min \{S \subseteq V : \sum_{i \in S} q_i^{(T)} \geq p\}$$

**优势**：自适应——teacher 确定性高时保留少数 token，不确定时保留更多。

---

## 4. Python 实现

```Python
import torch
import torch.nn.functional as F
from typing import Tuple


def topk_logits_distillation(
    student_logits: torch.Tensor,  # (B, L, V)
    teacher_logits: torch.Tensor,  # (B, L, V)
    k: int = 128,
    temperature: float = 2.0,
) -> torch.Tensor:
    """Top-k logits distillation loss."""
    # Get teacher probs at temperature
    teacher_probs = F.softmax(teacher_logits / temperature, dim=-1)

    # Get top-k indices from teacher
    topk_vals, topk_indices = teacher_probs.topk(k, dim=-1)  # (B, L, k)

    # Renormalize teacher top-k probs
    teacher_topk = topk_vals / topk_vals.sum(dim=-1, keepdim=True)

    # Gather student logits at top-k positions
    student_topk_logits = student_logits.gather(-1, topk_indices)  # (B, L, k)
    student_topk_log_probs = F.log_softmax(
        student_topk_logits / temperature, dim=-1
    )

    # KL divergence on top-k subset
    loss = F.kl_div(
        student_topk_log_probs,
        teacher_topk,
        reduction="batchmean",
    ) * (temperature ** 2)

    return loss


def topp_logits_distillation(
    student_logits: torch.Tensor,  # (B, V)
    teacher_logits: torch.Tensor,  # (B, V)
    p: float = 0.95,
    temperature: float = 2.0,
) -> torch.Tensor:
    """Top-p (nucleus) logits distillation loss."""
    teacher_probs = F.softmax(teacher_logits / temperature, dim=-1)

    # Sort by probability (descending)
    sorted_probs, sorted_indices = teacher_probs.sort(dim=-1, descending=True)
    cumsum = sorted_probs.cumsum(dim=-1)

    # Create mask: keep tokens until cumulative prob >= p
    mask = cumsum - sorted_probs < p  # shift by one to include the p-crossing token

    # Mask out low-probability tokens
    sorted_probs = sorted_probs * mask.float()

    # Renormalize
    sorted_probs = sorted_probs / sorted_probs.sum(dim=-1, keepdim=True).clamp(min=1e-8)

    # Get student probs in same order
    student_sorted_logits = student_logits.gather(-1, sorted_indices)
    student_log_probs = F.log_softmax(
        student_sorted_logits / temperature, dim=-1
    )
    student_log_probs = student_log_probs * mask.float()

    # KL on filtered set
    loss = (sorted_probs * (sorted_probs.log().clamp(min=-100) - student_log_probs)).sum(-1).mean()
    return loss * (temperature ** 2)


# --- Offline storage helper ---
def save_topk_logits(
    teacher_logits: torch.Tensor,  # (L, V)
    k: int = 256,
) -> Tuple[torch.Tensor, torch.Tensor]:
    """Save top-k logits for offline distillation.
    Returns (values, indices) for compact storage.
    """
    topk_vals, topk_idx = teacher_logits.topk(k, dim=-1)
    return topk_vals.half(), topk_idx.int()  # fp16 + int32


# --- Usage ---
B, L, V = 4, 512, 32000
student = torch.randn(B, L, V)
teacher = torch.randn(B, L, V)

loss_topk = topk_logits_distillation(student, teacher, k=128, temperature=2.0)
print(f"Top-k loss: {loss_topk.item():.4f}")
```

---

## 5. 裁剪参数选择

|参数|推荐值|说明|
|---|---|---|
|Top-k（offline 存储）|128~512|k=256 通常是最佳平衡点|
|Top-p（online 裁剪）|0.90~0.99|p=0.95 为常用默认值|
|Temperature|1.0~4.0|与 k/p 联合调参效果最佳|

> [!tip] 工程经验

> - 裁剪后**必须重归一化**，否则 KL 散度计算不正确

> - offline 存储推荐 fp16 values + int32 indices

> - top-k=256 + temperature=2.0 是稳定的起点配置
