全词表 logits 存储不可行（1B tokens × 128K vocab × 2B = 256TB），必须裁剪。

---

## 1. Top-k Logits 裁剪

只保留 teacher 概率最高的 $k$ 个 token 的 logits，其余置零，重归一化。

- $k=128$：存储降为全量的 $\sim 0.4\%$

- $k=256$：最佳平衡点

## 2. Top-p 裁剪

按累积概率阈值 $p$ 截断，自适应不同的 teacher 确定性。

---

## 3. Python 实现

```Python
import torch
import torch.nn.functional as F
from typing import Tuple

def topk_logits_distillation(
    student_logits: torch.Tensor,  # (B, L, V)
    teacher_logits: torch.Tensor,  # (B, L, V)
    k: int = 128,
    temperature: float = 2.0,
) -> torch.Tensor:
    teacher_probs = F.softmax(teacher_logits / temperature, dim=-1)
    topk_vals, topk_indices = teacher_probs.topk(k, dim=-1)
    teacher_topk = topk_vals / topk_vals.sum(dim=-1, keepdim=True)
    student_topk_logits = student_logits.gather(-1, topk_indices)
    student_topk_log_probs = F.log_softmax(
        student_topk_logits / temperature, dim=-1
    )
    loss = F.kl_div(
        student_topk_log_probs, teacher_topk, reduction="batchmean"
    ) * (temperature ** 2)
    return loss

def save_topk_logits(
    teacher_logits: torch.Tensor, k: int = 256
) -> Tuple[torch.Tensor, torch.Tensor]:
    """Save top-k logits for offline distillation."""
    topk_vals, topk_idx = teacher_logits.topk(k, dim=-1)
    return topk_vals.half(), topk_idx.int()  # fp16 + int32
```

## 4. 参数选择

|参数|推荐值|说明|
|---|---|---|
|Top-k（offline）|128~512|k=256 通常最佳平衡|
|Top-p（online）|0.90~0.99|p=0.95 常用默认|
|Temperature|1.0~4.0|与 k/p 联合调参|

> [!tip] 裁剪后必须重归一化，offline 推荐 fp16 values + int32 indices。
