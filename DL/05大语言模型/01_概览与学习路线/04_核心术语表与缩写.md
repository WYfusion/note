# 核心术语表与缩写

本表整理了 LLM 领域最常见的术语和缩写，按功能分类，方便查阅。

## 1. 基础概念 (Basics)

| 术语/缩写 | 全称 | 解释 |
| :--- | :--- | :--- |
| **LLM** | Large Language Model | 大语言模型。参数量巨大（通常 >10B）的深度学习模型，用于理解和生成人类语言。 |
| **Token** | Token | 词元。LLM 处理文本的最小单位，可以是一个词、一个字或词的一部分（如 "ing"）。 |
| **Context Window** | Context Window | 上下文窗口。模型一次推理能处理的最大 Token 数量（输入 Prompt + 输出 Completion）。 |
| **Parameters** | Parameters | 参数量。衡量模型规模的指标，如 7B (70亿), 175B (1750亿)。通常参数越多，能力越强。 |
| **Perplexity (PPL)** | Perplexity | 困惑度。衡量语言模型预测下一个词的不确定性，数值越低代表模型效果越好。 |
| **Scaling Laws** | Scaling Laws | 缩放定律。描述模型性能随参数量、数据量和计算量增加而呈幂律提升的经验公式。 |
| **Emergent Ability** | Emergent Ability | 涌现能力。小模型不具备，当模型规模突破临界点后突然出现的能力（如推理、编码）。 |
| **Grounding** | Grounding | 接地/事实一致性。模型生成内容基于真实世界信息或检索到的上下文，而非凭空捏造。 |

## 2. 模型架构与组件 (Architecture & Components)

| 术语/缩写 | 全称 | 解释 |
| :--- | :--- | :--- |
| **Transformer** | Transformer | 现代 LLM 的基础架构，完全基于自注意力机制（Self-Attention）的深度神经网络。 |
| **MoE** | Mixture of Experts | 混合专家模型。将大模型拆分为多个小专家网络，每次推理仅激活部分专家，实现“大参数量、低计算量”。 |
| **RoPE** | Rotary Positional Embeddings | 旋转位置编码。目前最流行的相对位置编码方式，能更好地处理长序列。 |
| **FlashAttention** | FlashAttention | 闪电注意力。一种通过优化 GPU 显存读写（IO）来大幅加速注意力计算并节省显存的算法。 |
| **[[06_分组注意力|GQA]]** | Grouped-Query Attention | 分组查询注意力。介于多头注意力 (MHA) 和多查询注意力 ([[05_多查询注意力MQA|MQA]]) 之间的变体，平衡了推理速度和模型质量。 |
| **KV Cache** | Key-Value Cache | 键值缓存。推理加速技术，缓存 Attention 机制中的 K 和 V 矩阵，避免生成每个 Token 时重复计算历史信息。 |

## 3. 提示工程 (Prompt Engineering)

| 术语/缩写 | 全称 | 解释 |
| :--- | :--- | :--- |
| **Prompt** | Prompt | 提示词。用户输入给模型的文本指令，用于引导模型生成特定的输出。 |
| **Zero-shot** | Zero-shot Learning | 零样本学习。不给示例，直接让模型完成任务（如“翻译这句话”）。 |
| **Few-shot** | Few-shot Learning | 少样本学习。在 Prompt 中提供少量示例（Demonstrations）供模型参考（“照猫画虎”）。 |
| **ICL** | In-Context Learning | 上下文学习。利用 Prompt 中的上下文信息进行学习的能力，无需更新模型参数。 |
| **CoT** | Chain-of-Thought | 思维链。引导模型通过生成中间推理步骤（“让我们一步步思考”）来得出最终答案的技术。 |
| **System Prompt** | System Prompt | 系统提示词。设定模型角色、行为规范、边界条件的初始指令（如“你是一个有用的助手”）。 |
| **Prompt Injection** | Prompt Injection | 提示注入攻击。用户通过精心设计的输入，诱导模型忽略系统指令并执行恶意操作。 |

## 4. 训练与微调 (Training & Fine-tuning)

| 术语/缩写            | 全称                                         | 解释                                                         |
| :--------------- | :----------------------------------------- | :--------------------------------------------------------- |
| **Pre-training** | Pre-training                               | 预训练。在海量无标注数据（Corpus）上训练模型，使其掌握语言规律和世界知识。                   |
| **SFT**          | Supervised Fine-Tuning                     | 有监督微调（指令微调）。使用高质量的“指令-回复”对数据微调模型，使其学会遵循指令。                 |
| **PEFT**         | Parameter-Efficient Fine-Tuning            | 参数高效微调。只微调少量参数（或增加少量参数），冻结大部分预训练参数，降低训练成本。                 |
| **LoRA**         | Low-Rank Adaptation                        | 低秩自适应。最流行的 PEFT 方法，通过低秩矩阵分解来减少可训练参数。                       |
| **QLoRA**        | Quantized LoRA                             | 量化 LoRA。结合 4-bit 量化和 LoRA，进一步降低显存需求，使消费级显卡也能微调大模型。         |
| **Alignment**    | Alignment                                  | 对齐。使模型的输出符合人类的价值观（有用 Helpful、诚实 Honest、无害 Harmless，简称 3H）。 |
| **RLHF**         | Reinforcement Learning from Human Feedback | 基于人类反馈的强化学习。利用人类偏好数据训练奖励模型 (Reward Model)，再用 PPO 算法优化 LLM。 |
| **DPO**          | Direct Preference Optimization             | 直接偏好优化。一种无需强化学习（无 Reward Model）的对齐算法，直接在偏好数据上优化，更稳定高效。     |

## 5. 推理与解码 (Inference & Decoding)

| 术语/缩写 | 全称 | 解释 |
| :--- | :--- | :--- |
| **Temperature** | Temperature | 温度。控制生成随机性的参数。越高越随机（有创造力），越低越确定（保守）。 |
| **Top-P** | Top-P (Nucleus Sampling) | 核采样。从累积概率达到 P 的最小集合中采样 Token，动态调整候选池大小。 |
| **Top-K** | Top-K Sampling | 从概率最高的 K 个 Token 中采样。 |
| **Hallucination** | Hallucination | 幻觉。模型生成看似合理但实际上错误、捏造或与事实不符的内容。 |
| **Quantization** | Quantization | 量化。降低参数精度（如从 FP16 到 INT4）以减少显存占用和加速推理。常见方法有 GPTQ, AWQ。 |
| **Speculative Decoding** | Speculative Decoding | 投机解码。使用小模型快速生成草稿，大模型进行验证和修正，以加速推理过程。 |
| **vLLM / PagedAttention** | vLLM | 一个高吞吐量的推理框架，核心技术是 PagedAttention，像操作系统管理内存一样管理 KV Cache。 |

## 6. 评测与基准 (Evaluation & Benchmarks)

| 术语/缩写 | 全称 | 解释 |
| :--- | :--- | :--- |
| **MMLU** | Massive Multitask Language Understanding | 大规模多任务语言理解。涵盖 STEM、人文社科等 57 个学科的评测基准，衡量模型的综合知识能力。 |
| **GSM8K** | Grade School Math 8K | 小学数学应用题评测集。包含 8500 道高质量数学题，主要测试模型的多步推理能力。 |
| **HumanEval** | HumanEval | 代码生成评测集。OpenAI 推出，通过单元测试来衡量模型编写 Python 代码的能力。 |
| **Leaderboard** | Leaderboard | 排行榜。如 Hugging Face Open LLM Leaderboard，展示各模型在公开基准上的排名。 |
| **LMSYS Arena** | Chatbot Arena | 竞技场。基于 Elo 等级分系统，通过人类盲测（Side-by-side）来评估模型的真实对话体验。 |

## 7. 应用架构 (Application Architecture)

| 术语/缩写 | 全称 | 解释 |
| :--- | :--- | :--- |
| **RAG** | Retrieval-Augmented Generation | 检索增强生成。先从外部知识库检索相关信息，再结合 Prompt 输入给模型，以减少幻觉和补充私有/最新知识。 |
| **Vector DB** | Vector Database | 向量数据库。用于存储和检索文本 Embedding（向量）的数据库（如 Milvus, Chroma），是 RAG 的核心组件。 |
| **Embedding** | Embedding | 嵌入/向量化。将文本转换为高维实数向量，使计算机能计算文本之间的语义相似度。 |
| **Agent** | Agent | 智能体。能够感知环境、进行规划（Planning）、记忆（Memory）并调用工具（Tools）来完成复杂任务的系统。 |
| **Function Calling** | Function Calling | 函数调用。LLM 输出特定格式的 JSON 参数以调用外部 API 的能力，是 Agent 的基础。 |
| **LangChain** | LangChain | 一个流行的 LLM 应用开发框架，提供了链（Chains）、代理（Agents）和回调（Callbacks）等组件。 |

## 8. 语音与多模态 (Audio & Multimodal)

| 术语/缩写 | 全称 | 解释 |
| :--- | :--- | :--- |
| **ASR** | Automatic Speech Recognition | 自动语音识别。将语音信号转换为文本的技术（Speech-to-Text）。 |
| **TTS** | Text-to-Speech | 语音合成。将文本转换为语音信号的技术。 |
| **S2ST** | Speech-to-Speech Translation | 语音到语音翻译。直接将一种语言的语音翻译为另一种语言的语音。 |
| **Mel-spectrogram** | Mel-spectrogram | 梅尔频谱图。一种模拟人耳听觉特性的音频频域特征表示，常作为语音模型的输入。 |
| **Vocoder** | Vocoder | 声码器。将声学特征（如频谱图）还原为连续音频波形的模型（如 HiFi-GAN）。 |
| **WER** | Word Error Rate | 词错误率。衡量 ASR 系统性能的核心指标，越低越好。计算公式：$(S+D+I)/N$。 |
| **Audio Tokenizer** | Audio Tokenizer | 音频分词器。将连续音频离散化为 Token 序列的模型（如 EnCodec），使 LLM 能处理音频。 |
