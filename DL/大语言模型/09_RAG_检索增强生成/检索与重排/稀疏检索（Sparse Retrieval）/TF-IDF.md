## 一句话概括

TF-IDF（Term Frequency–Inverse Document Frequency）是最基础的文本权重计算方法，用「一个词在文档中出现多频繁」×「这个词在整个语料库中多稀有」来衡量词对文档的重要性。

## 核心原理

### 直觉

- 一个词在某篇文档中频繁出现 → 可能跟这篇文档很相关（TF）
- 一个词在所有文档中都频繁出现 → 没有区分力，不重要（IDF）
- 两者相乘 → 既频繁出现又有区分力的词，权重最高

### 数学公式

**TF（词频）**

$TF(t,d) = \frac{\text{词 } t \text{ 在文档 } d \text{ 中的出现次数}}{\text{文档 } d \text{ 的总词数}}$

**IDF（逆文档频率）**

$IDF(t) = \ln\frac{N}{n(t)}$

- $N$：语料库中的文档总数
- $n(t)$：包含词 $t$ 的文档数

**TF-IDF 得分**

$TF\text{-}IDF(t,d) = TF(t,d) \times IDF(t)$

### 举例

假设有 10,000 篇文档：

- 「的」出现在 9,800 篇 → $IDF = ln(10000/9800) ≈ 0.02$，权重极低
- 「ColBERT」出现在 3 篇 → $IDF = ln(10000/3) ≈ 8.1$，权重极高

## 文档向量化

每篇文档可以表示为一个向量，维度 = 词汇表大小，每个维度的值 = 该词的 TF-IDF 分数。

`文档 → [0, 0, 0.32, 0, 0, 0.15, 0, ...]`（大部分为 0 → 稀疏向量）

查询时，将 query 也转为 TF-IDF 向量，计算 cosine 相似度即可排序。

## 与 BM25 的关系
[[BM25 (Best Matching 25)]]
TF-IDF 是 BM25 的前身。BM25 在此基础上做了两个关键改进：

1. **TF 饱和**：词出现 100 次不应该比出现 10 次重要 10 倍，BM25 引入饱和函数
2. **文档长度归一化**：长文档天然词多，BM25 对此做了惩罚

实践中 **BM25 几乎全面优于 TF-IDF**，TF-IDF 更多作为理解基础。

## 适用场景

- 理解信息检索基础原理的入门
- 特征工程中的文本特征提取
- 简单的关键词提取和文档摘要
- scikit-learn 等工具中的快速文本向量化

## 工程实现

- **scikit-learn**：`TfidfVectorizer`，最常用的 Python 实现
- **Gensim**：`TfidfModel`
- **Spark MLlib**：分布式大规模 TF-IDF
