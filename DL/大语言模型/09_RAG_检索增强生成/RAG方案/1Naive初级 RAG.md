## 是什么
最基础的 RAG 方案，也是所有进阶方案的起点。流程极简：用户提问 → 检索相关文档 → 拼接为上下文 → LLM 生成回答。

## 适用场景
- 快速原型验证
- 简单的文档问答场景
- 作为基线方案进行对比评测

## 核心流程
`query → dense/BM25 检索 top-k → 拼接上下文 → LLM 生成`

## 工作原理
1. 用户输入问题
2. 用 Dense Embedding 或 BM25 从文档库中检索 top-k 个最相关的片段
3. 将检索到的片段拼接成上下文，连同原始问题一起送入 LLM
4. LLM 基于上下文生成回答

## 优点
- 实现简单，快速搭建
- 理解门槛低，适合原型验证
- 依赖组件少

## 局限性
- 只用单路检索（Dense 或 Sparse），召回率有上限
- 没有精排，噪声多
- 没有上下文压缩，可能超出窗口或引入干扰
- 没有引用和校验机制

## 演进路径
- 加入 Sparse → **Hybrid RAG**
- 加入 Reranker → **Two-Stage RAG**
- 加入多查询 → **Multi-Query RAG**
- 加入记忆 → **Conversational RAG**
