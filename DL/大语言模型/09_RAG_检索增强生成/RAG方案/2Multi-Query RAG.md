## 是什么
将用户的原始问题扩展为多个不同角度的子查询，分别检索后合并结果。解决单一查询视角不足的问题。

## 适用场景
- 开放性问题、对比类问题
- 用户问题涉及多个方面
- 需要全面覆盖的分析型问答

## 核心流程
`原问题 → LLM 生成多个子查询 → 各自并行召回 → 结果合并去重 → 生成`

## 工作原理
1. 用户提出一个问题
2. LLM 将其拆解为 3~5 个不同角度的子查询
3. 每个子查询独立检索
4. 将所有检索结果合并、去重
5. 送入生成模型

## 示例
原问题：`"RAG 和微调各自的优缺点是什么？"`
扩展为：

- `"RAG 的优势和适用场景"`
- `"模型微调的优势和适用场景"`
- `"RAG 与微调的对比分析"`
- `"RAG 和微调的局限性"`

## 优点
- 覆盖面广，减少遗漏
- 解决原始查询的歧义和片面性
- 与 Hybrid Retrieval 可叠加使用

## 注意事项
- 子查询太多会增加检索延迟和成本
- 需要有效的结果去重和合并策略
- 建议 3~5 个子查询为宜
