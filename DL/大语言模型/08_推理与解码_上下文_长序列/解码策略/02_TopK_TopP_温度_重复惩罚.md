# 采样策略：Top-K, Top-P, Temperature 与重复惩罚

## 1. 随机采样 (Sampling)
与 Greedy/Beam Search 不同，采样策略根据概率分布随机选择下一个 Token，赋予模型创造力。

### 1.1 温度 (Temperature)
通过调整 Softmax 前的 Logits 来控制**分布**的平滑程度：
$$ P_i = \frac{\exp(z_i / T)}{\sum_j \exp(z_j / T)} $$
- **$T < 1$**：分布变尖（Sharpen），高概率 Token 被放大，生成更保守、确定。
- **$T > 1$**：分布变平（Flatten），低概率 Token 机会增加，生成更多样、随机。

### 1.2 Top-K 采样
仅从概率最高的 $K$ 个 Token 中进行重归一化采样。
- **作用**：切断长尾（Long Tail），防止生成极低概率的离谱内容。

### 1.3 Top-P (Nucleus) 采样
从累积概率达到 $P$（如 0.9）的**最小 Token 集合**中采样。
- **优势**：相比 Top-K，Top-P 的候选集合大小是动态的。在确定性高时候选少，不确定时候选多。

## 2. 惩罚机制

### 2.1 重复惩罚 (Repetition Penalty)
如果一个 Token 已经生成过，这就对其 Logit 施加惩罚：
$$ z_i' = \begin{cases} z_i / \theta & \text{if } z_i > 0 \\ z_i \cdot \theta & \text{if } z_i < 0 \end{cases} $$
其中 $\theta > 1$。

## 3. 语音生成中的特殊考量

### 3.1 语音生成的"结巴"问题
Audio LLM（如 VALL-E）在自回归生成 Audio Token 时，极易出现重复循环（Infinite Loop）或静音片段。
- **原因**：音频的局部相关性极强，模型容易陷入局部模式。
- **对策**：
  - **强重复惩罚**：在语音生成中，Repetition Penalty 通常设置得比文本生成更高。
  - **N-gram Blocking**：禁止生成重复的 N-gram 序列。

### 3.2 温度对语音风格的影响
- **低温度 ($T \to 0$)**：
  - **效果**：生成的语音非常稳定，发音清晰，但可能缺乏情感起伏，听起来像"机器音"。
  - **适用**：新闻播报、朗读。
- **高温度 ($T \to 1$)**：
  - **效果**：情感丰富，语调变化大，但容易出现发音错误（哑音、破音）或不稳定的伪影。
  - **适用**：情感对话、戏剧配音。

### 3.3 动态温度调节
在生成长语音时，为了兼顾稳定性与多样性，可以采用动态策略：
- 在句首使用高温度（决定整体风格）。
- 在句中使用低温度（保持发音准确）。
