# 长上下文方法

本章节探讨了如何让模型处理超出训练长度的序列，包括 RoPE 位置编码扩展技术以及 RAG 检索增强生成。针对 Audio LLM 的长音频处理场景提供了专门的解决方案。

## 目录

### [01_RoPE扩展_NTK_Scaling.md](./01_RoPE扩展_NTK_Scaling.md)
- **RoPE 回顾**：旋转位置编码的核心公式与相对位置性质。
- **长度外推问题**：训练长度 2048，推理长度 16384 时的 PPL 飙升。
- **扩展方法**：
  - **位置插值 (PI)**：线性压缩位置索引。
  - **NTK-Aware Scaling**：区分高频/低频分量，保留局部精度。
- **语音应用**：
  - Audio Token 序列远超训练长度的处理。
  - 流式场景的 StreamingLLM（滑动窗口 + Sink Token）。

### [02_检索增强_RAG与长上下文的关系.md](./02_检索增强_RAG与长上下文的关系.md)
- **RAG 核心流程**：Retriever + Generator 架构。
- **RAG vs 长上下文**：适用场景对比。
- **Audio RAG**：
  - **CLAP**：音频-文本共享 Embedding 空间。
  - **播客问答**：长音频分段索引 + 按需检索。
  - **会议助手**：实时索引 + 说话人定位。
