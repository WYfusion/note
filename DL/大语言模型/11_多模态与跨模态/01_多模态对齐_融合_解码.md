# 多模态对齐、融合与解码

多模态大模型（Multimodal LLM, MLLM）旨在打破单一文本模态的限制，实现对图像、音频、视频的统一理解与生成。本节重点探讨音频模态如何与文本模态进行对齐、融合与联合解码。

## 1. 多模态对齐 (Alignment)

对齐的目标是将不同模态的数据映射到同一个语义空间，使得相似语义的音频和文本在向量空间中距离更近。

### 1.1 对比学习 (Contrastive Learning)
类似于 CLIP (Contrastive Language-Image Pre-training)，音频领域有 **CLAP (Contrastive Language-Audio Pretraining)**。

*   **原理**:
    *   双塔结构：Audio Encoder (CNN/Transformer) + Text Encoder (BERT/RoBERTa)。
    *   正样本：(音频, 对应的描述文本)。
    *   负样本：Batch 内的其他配对。
    *   Loss: InfoNCE Loss，最大化正样本相似度，最小化负样本相似度。
*   **应用**: 零样本音频分类、文本到音频检索 (Text-to-Audio Retrieval)。

### 1.2 联合嵌入空间 (Joint Embedding Space)
**ImageBind (Meta)** 展示了如何以图像为锚点，将音频、文本、深度图、热力图等绑定在一起。
*   **Audio-Image Alignment**: 狗叫声的 Embedding 应该靠近狗的图片的 Embedding。
*   **意义**: 实现了跨模态检索（用声音搜图片）和算术运算（鸟的图片 + 歌声 = 唱歌的鸟）。

## 2. 多模态融合 (Fusion)

如何将音频特征输入到 LLM 中？

### 2.1 早期融合 (Early Fusion) / Tokenization
将音频离散化为 Token，与文本 Token 混合输入。
*   **Audio Tokenizer**: 使用 VQ-VAE 或 EnCodec 将连续音频波形压缩为离散的 Codebook Indices。
*   **Input Sequence**: `[TEXT_TOKENS] [AUDIO_TOKENS] [TEXT_TOKENS]`
*   **优点**: LLM 可以像处理文本一样处理音频，实现统一建模。
*   **代表模型**: AudioLM, VALL-E.

### 2.2 适配器融合 (Adapter / Projection)
保持音频为连续特征，通过适配器映射到 LLM 的输入空间。
*   **流程**:
    1.  Audio Encoder (如 Whisper Encoder) 提取特征 $F_{audio}$。
    2.  Projection Layer (Linear/MLP) 将 $F_{audio}$ 变换为 $E_{audio}$，维度与 LLM 的 Text Embedding 一致。
    3.  LLM Input: `[System Prompt] <audio_embedding> [User Prompt]`
*   **优点**: 保留了音频的细粒度信息（如情感、语调），且无需重新训练 LLM 主干。
*   **代表模型**: Qwen-Audio, SALMONN.

## 3. 多模态解码 (Decoding)

LLM 如何输出音频？

### 3.1 级联生成 (Cascaded Generation)
LLM 输出文本或控制信号，再调用 TTS/MusicGen 生成音频。
*   **路径**: LLM -> Text/JSON -> TTS -> Audio。
*   **缺点**: 丢失了 LLM 直接控制声学细节（如笑声、停顿）的能力。

### 3.2 原生生成 (Native Generation)
LLM 直接预测音频 Token。
*   **自回归生成**: $P(token_t | token_{<t}, \text{context})$。
*   **词表扩展**: LLM 的词表扩充，包含 Text Tokens (0-50k) 和 Audio Tokens (50k-51k)。
*   **解码**: 采样得到的 Audio Tokens 序列通过 Audio Decoder (如 EnCodec Decoder) 还原为波形。
*   **优势**: 支持 Speech-to-Speech，能够生成包含情感、背景音的富媒体音频。
*   **代表模型**: GPT-4o (Omni), SpeechGPT.

---

## 4. Any-to-Any 范式

未来的趋势是任意模态互转。

*   **Speech-to-Speech (S2S)**: 实时翻译、语音助手。
*   **Image-to-Audio (I2A)**: 看图配乐。
*   **Audio-to-Image (A2I)**: 根据声音画图（如听到海浪声画海滩）。

$$
\text{Unified Model} = \text{Encoder}_{\{T, A, I\}} \rightarrow \text{LLM Core} \rightarrow \text{Decoder}_{\{T, A, I\}}
$$
