从检索系统的角度看，向量表示可以分为五类，从简单到复杂，精度逐步提升，成本也逐步增加。

---

## 1. Dense 单向量

- **原理**：每个文档编码为 1 个稠密向量（通常 768~3072 维）
- **优点**：简单、高效、生态成熟
- **缺点**：单一向量可能丢失细节信息
- **代表**：E5、BGE、OpenAI embeddings

```python
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("BAAI/bge-base-en-v1.5")

# 每个文档 → 1 个向量
docs = ["Machine learning is a branch of AI.", "Python is a programming language."]
embeddings = model.encode(docs, normalize_embeddings=True)
print(f"每个文档 1 个向量, 维度: {embeddings.shape}")  # (2, 768)
```

## 2. Sparse 稀疏向量

- **原理**：每个文档表示为稀疏向量，大部分维度为 0，非零维度对应具体词项
- **优点**：可解释性强（能看到哪些词被激活）、精确匹配能力好
- **缺点**：语义泛化弱（同义词难匹配）
- **代表**：BM25（经典）、SPLADE（学习型）

```python
# SPLADE 稀疏嵌入示例
from transformers import AutoModelForMaskedLM, AutoTokenizer
import torch

tokenizer = AutoTokenizer.from_pretrained("naver/splade-cocondenser-ensembledistil")
model = AutoModelForMaskedLM.from_pretrained("naver/splade-cocondenser-ensembledistil")

def splade_encode(text):
    tokens = tokenizer(text, return_tensors="pt", padding=True, truncation=True)
    with torch.no_grad():
        output = model(**tokens)
    # SPLADE: ReLU + log(1+x) 激活，再取 max pooling
    sparse_vec = torch.max(
        torch.log1p(torch.relu(output.logits)) * tokens["attention_mask"].unsqueeze(-1),
        dim=1
    ).values.squeeze()
    return sparse_vec

vec = splade_encode("What is machine learning?")

# 查看激活的 top 词项
nonzero = torch.nonzero(vec).squeeze()
top_indices = vec[nonzero].argsort(descending=True)[:10]
for idx in top_indices:
    token_id = nonzero[idx].item()
    weight = vec[token_id].item()
    token = tokenizer.decode([token_id])
    print(f"  {token}: {weight:.3f}")
```

## 3. Hybrid 稠密+稀疏

- **原理**：同时使用 dense 和 sparse 检索，融合两路结果
- **优点**：兼具语义泛化和精确匹配，**当前最推荐的方案**
- **缺点**：需要维护两套索引，融合权重需调优
- **融合方式**：线性加权、RRF（Reciprocal Rank Fusion）

```python
def reciprocal_rank_fusion(ranked_lists, k=60):
    """
    RRF 融合多路召回结果
    ranked_lists: list of lists, 每个内部 list 是 (doc_id, score) 按分数降序
    k: 平滑参数（默认 60）
    """
    fused_scores = {}
    for ranked_list in ranked_lists:
        for rank, (doc_id, _) in enumerate(ranked_list):
            if doc_id not in fused_scores:
                fused_scores[doc_id] = 0
            fused_scores[doc_id] += 1.0 / (k + rank + 1)
    
    # 按融合分数降序排序
    return sorted(fused_scores.items(), key=lambda x: -x[1])

# 示例：dense 和 sparse 两路召回结果
dense_results = [("doc_A", 0.95), ("doc_B", 0.90), ("doc_C", 0.85), ("doc_D", 0.80)]
sparse_results = [("doc_B", 12.5), ("doc_E", 11.0), ("doc_A", 10.5), ("doc_F", 9.0)]

fused = reciprocal_rank_fusion([dense_results, sparse_results])
print("RRF 融合结果:")
for doc_id, score in fused[:5]:
    print(f"  {doc_id}: {score:.4f}")
```

## 4. Multi-vector

- **原理**：每个文档不是 1 个向量，而是多个向量（每个 token 一个）
- **优点**：保留细粒度信息，精度显著高于单向量
- **缺点**：存储 ×N 倍，索引更复杂
- **代表**：ColBERT、BGE-M3

```python
# ColBERT 多向量编码概念
import torch
from transformers import AutoTokenizer, AutoModel

tokenizer = AutoTokenizer.from_pretrained("colbert-ir/colbertv2.0")
model = AutoModel.from_pretrained("colbert-ir/colbertv2.0")

def colbert_encode(text, max_length=128):
    inputs = tokenizer(text, return_tensors="pt", max_length=max_length,
                       padding="max_length", truncation=True)
    with torch.no_grad():
        outputs = model(**inputs)
    # 每个 token 一个向量
    token_embeds = outputs.last_hidden_state  # [1, seq_len, dim]
    # 只保留非 padding token
    mask = inputs["attention_mask"].bool()
    token_embeds = token_embeds[mask]  # [n_tokens, dim]
    # 归一化
    token_embeds = torch.nn.functional.normalize(token_embeds, dim=-1)
    return token_embeds

q_tokens = colbert_encode("What is machine learning?")
d_tokens = colbert_encode("Machine learning is a branch of artificial intelligence.")
print(f"Query: {q_tokens.shape[0]} 个 token 向量")
print(f"Doc: {d_tokens.shape[0]} 个 token 向量")

# MaxSim 打分
sim = q_tokens @ d_tokens.T  # [n_q, n_d]
score = sim.max(dim=1).values.sum()
print(f"MaxSim Score: {score:.4f}")
```

## 5. Contextual / Late-chunked Chunk Embeddings

- **原理**：先用长上下文模型处理整篇文档，再切分为 chunk 并取对应向量
- **优点**：每个 chunk 的向量包含全文上下文信息，解决"孤立 chunk"问题
- **缺点**：需要长上下文模型，处理成本更高
- **代表**：Late Chunking、Contextual Retrieval（Anthropic）

```python
# Late Chunking 概念实现
def late_chunking(model, tokenizer, full_text, chunk_boundaries):
    """
    1. 将整篇文档送入长上下文模型，获取所有 token 的上下文表示
    2. 按 chunk 边界切分 token 表示
    3. 对每个 chunk 的 token 做 mean pooling
    """
    # Step 1: 编码整篇文档
    inputs = tokenizer(full_text, return_tensors="pt", truncation=True, max_length=8192)
    with torch.no_grad():
        outputs = model(**inputs)
    all_token_embeds = outputs.last_hidden_state[0]  # [seq_len, dim]
    
    # Step 2 & 3: 按边界切分并 pooling
    chunk_embeddings = []
    for start, end in chunk_boundaries:
        chunk_tokens = all_token_embeds[start:end]
        chunk_embed = chunk_tokens.mean(dim=0)
        chunk_embed = torch.nn.functional.normalize(chunk_embed, dim=0)
        chunk_embeddings.append(chunk_embed)
    
    return torch.stack(chunk_embeddings)

# 对比：传统 chunking vs Late Chunking
print("传统 Chunking: 每个 chunk 独立编码 → 丢失上下文")
print("Late Chunking: 全文编码后切分 → 保留上下文")
print("\n效果：Late Chunking 在长文检索中通常优于传统方式 5-15%")
```

## 五类表示对比

| 类型 | 存储/索引 | 精度 | 适用场景 |
| --- | --- | --- | --- |
| Dense 单向量 | 低 | 中 | 通用检索、快速落地 |
| Sparse 稀疏 | 中 | 中 | 精确匹配、可解释性 |
| Hybrid | 中 | 高 | **最推荐的生产方案** |
| Multi-vector | 高 | 很高 | 高精度要求 |
| Contextual chunk | 中-高 | 高 | 长文档检索 |
