嵌入模型的质量取决于训练方式。当前主流训练范式从对比学习出发，衍生出多种架构和优化策略。

## 对比学习（Contrastive Learning）

**核心思想**：正样本（语义相关）拉近，负样本（语义无关）拉远。

- **InfoNCE Loss**：给定 query，在一批候选中识别出正样本
- **Hard Negatives**：选择难以区分的负样本，能显著提升模型质量
- **In-batch Negatives**：同一 batch 中其他样本的正样本互为负样本，节省采样开销

```python
import torch
import torch.nn.functional as F

def info_nce_loss(query_embeds, positive_embeds, temperature=0.05):
    """
    InfoNCE 对比损失
    query_embeds: [batch_size, dim]
    positive_embeds: [batch_size, dim]
    """
    # 归一化
    q = F.normalize(query_embeds, dim=-1)
    p = F.normalize(positive_embeds, dim=-1)
    
    # 计算相似度矩阵 [batch, batch]
    # 对角线是正样本对，其余是 in-batch negatives
    similarity = q @ p.T / temperature
    
    # 标签：每个 query 的正样本是对角线位置
    labels = torch.arange(len(q), device=q.device)
    
    # 交叉熵损失
    loss = F.cross_entropy(similarity, labels)
    return loss

# 示例
batch_size, dim = 32, 768
q_emb = torch.randn(batch_size, dim)
p_emb = torch.randn(batch_size, dim)
loss = info_nce_loss(q_emb, p_emb)
print(f"InfoNCE Loss: {loss.item():.4f}")
```

## 双塔 / Bi-Encoder

**核心思想**：query 和 document 各自独立编码，检索时只需计算向量相似度。

- **优点**：document 可预先编码并缓存，检索时只需编码 query，速度极快
- **缺点**：query 和 document 没有交互，可能丢失细粒度匹配信息
- **代表**：SBERT、DPR、E5、BGE

```python
from sentence_transformers import SentenceTransformer

# Bi-Encoder：query 和 doc 独立编码
model = SentenceTransformer("BAAI/bge-base-en-v1.5")

queries = ["What is machine learning?"]
docs = [
    "Machine learning is a branch of AI...",
    "The weather today is sunny...",
    "Deep learning uses neural networks...",
]

# 分别编码（可离线批量编码 docs）
q_embeds = model.encode(queries, normalize_embeddings=True)
d_embeds = model.encode(docs, normalize_embeddings=True)

# 计算相似度
import numpy as np
scores = q_embeds @ d_embeds.T
print("Bi-Encoder 相似度:", scores[0])
```

## 交叉编码 / Cross-Encoder

**核心思想**：query-doc 拼接后联合打分，精度更高但速度更慢。

- **优点**：query 和 document 充分交互，精度显著高于 Bi-Encoder
- **缺点**：每对 query-doc 都要过一次模型，无法预计算，不适合大规模召回
- **用途**：**Rerank**——先用 Bi-Encoder 召回 top-100，再用 Cross-Encoder 精排 top-10

```python
from sentence_transformers import CrossEncoder

# Cross-Encoder：query-doc 联合打分
reranker = CrossEncoder("cross-encoder/ms-marco-MiniLM-L-6-v2")

query = "What is machine learning?"
candidates = [
    "Machine learning is a branch of AI that enables systems to learn from data.",
    "The weather today is sunny and warm.",
    "Deep learning is a subset of machine learning using neural networks.",
]

# 打分（注意：每对都要单独计算）
pairs = [[query, doc] for doc in candidates]
scores = reranker.predict(pairs)

# 按分数排序
ranked = sorted(zip(candidates, scores), key=lambda x: -x[1])
for doc, score in ranked:
    print(f"Score: {score:.4f} | {doc[:60]}...")
```

## Late Interaction

**核心思想**：保留 token 粒度的向量表示，在检索时做 token 级别的交互匹配。

- **代表**：ColBERT
- **优点**：精度接近 Cross-Encoder，但 document 表示可预计算
- **缺点**：存储更大（每个 token 一个向量），索引更复杂
- **原理**：MaxSim——每个 query token 找到最匹配的 document token，取最大相似度之和

```python
import torch
import torch.nn.functional as F

def maxsim_score(q_tokens, d_tokens):
    """
    ColBERT MaxSim 计算
    q_tokens: [n_query_tokens, dim]
    d_tokens: [n_doc_tokens, dim]
    """
    # 计算每个 query token 与所有 doc tokens 的相似度
    sim_matrix = q_tokens @ d_tokens.T  # [n_q, n_d]
    
    # 每个 query token 取与 doc tokens 的最大相似度
    max_sims = sim_matrix.max(dim=1).values  # [n_q]
    
    # 求和得到总分
    return max_sims.sum().item()

# 模拟：query 5 个 token，doc 20 个 token，维度 128
q = F.normalize(torch.randn(5, 128), dim=-1)
d = F.normalize(torch.randn(20, 128), dim=-1)

score = maxsim_score(q, d)
print(f"ColBERT MaxSim Score: {score:.4f}")
```

## 当前质量提升主线

| 技术 | 说明 |
| --- | --- |
| 弱监督数据 | 利用点击日志、标题-正文对等自然配对 |
| 合成数据 | 用 LLM 生成 query-passage 对 |
| 指令微调 | 在 embedding 时加入 task instruction |
| 知识蒸馏 | 用 Cross-Encoder 教 Bi-Encoder |
| Hard Negative Mining | 用模型自身找难区分的负样本反复训练 |

## 架构对比

| 维度 | Bi-Encoder | Cross-Encoder | Late Interaction |
| --- | --- | --- | --- |
| 编码方式 | 独立编码 | 联合编码 | 独立编码+token交互 |
| 检索速度 | 极快 | 极慢 | 中等 |
| 精度 | 中 | 最高 | 高 |
| 可预计算 | ✅ | ❌ | ✅（doc端） |
| 典型用途 | 大规模召回 | Rerank | 高质量召回 |
| 存储 | 低（1向量/doc） | 无需存储 | 高（多向量/doc） |
