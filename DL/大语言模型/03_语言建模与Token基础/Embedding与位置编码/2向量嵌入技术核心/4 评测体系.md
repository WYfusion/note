嵌入模型和检索系统的质量需要通过系统化的评测来衡量，分为离线指标和在线指标两大类。

<aside>
🔑

**核心原则**：检索质量 ≠ 生成质量，必须分开测。好的检索是好的生成的前提，但两者需要独立评估。

</aside>

## 离线评测指标

### Recall@k（召回率）

- **定义**：在返回的 top-k 结果中，包含了多少真正相关的文档
- **用途**：衡量召回阶段的覆盖度
- **常用值**：Recall@10, Recall@100

### MRR（Mean Reciprocal Rank）

- **定义**：第一个正确结果的排名的倒数的平均值
- **用途**：衡量"第一个正确答案出现得多快"

### nDCG@k（Normalized Discounted Cumulative Gain）

- **定义**：考虑排名位置的加权相关性得分
- **用途**：衡量排序质量，适合多级相关性标注

### MAP（Mean Average Precision）

- **定义**：所有相关文档位置的精确率的平均值
- **用途**：综合衡量检索质量

## Python 实战：评测指标计算

```python
import numpy as np

def recall_at_k(relevant_docs, retrieved_docs, k):
    """Recall@k: top-k 中包含的相关文档比例"""
    retrieved_k = set(retrieved_docs[:k])
    relevant = set(relevant_docs)
    return len(retrieved_k & relevant) / len(relevant) if relevant else 0

def mrr(relevant_docs, retrieved_docs):
    """MRR: 第一个相关文档的排名倒数"""
    relevant = set(relevant_docs)
    for rank, doc in enumerate(retrieved_docs, 1):
        if doc in relevant:
            return 1.0 / rank
    return 0.0

def ndcg_at_k(relevance_scores, k):
    """
    nDCG@k: 归一化折损累计增益
    relevance_scores: 按检索排序的相关性分数列表
    """
    dcg = sum(rel / np.log2(i + 2) for i, rel in enumerate(relevance_scores[:k]))
    ideal = sorted(relevance_scores, reverse=True)[:k]
    idcg = sum(rel / np.log2(i + 2) for i, rel in enumerate(ideal))
    return dcg / idcg if idcg > 0 else 0

def average_precision(relevant_docs, retrieved_docs):
    """AP: 平均精确率"""
    relevant = set(relevant_docs)
    hits = 0
    sum_precision = 0
    for rank, doc in enumerate(retrieved_docs, 1):
        if doc in relevant:
            hits += 1
            sum_precision += hits / rank
    return sum_precision / len(relevant) if relevant else 0

# --- 使用示例 ---
relevant = ["doc_A", "doc_C", "doc_E"]
retrieved = ["doc_A", "doc_B", "doc_C", "doc_D", "doc_E", "doc_F"]

print(f"Recall@3: {recall_at_k(relevant, retrieved, 3):.2f}")  # 2/3
print(f"Recall@5: {recall_at_k(relevant, retrieved, 5):.2f}")  # 3/3
print(f"MRR: {mrr(relevant, retrieved):.2f}")  # 1/1
print(f"AP: {average_precision(relevant, retrieved):.4f}")

# nDCG 示例（多级相关性）
rel_scores = [3, 0, 2, 0, 1, 0]  # 对应 retrieved 的相关性
print(f"nDCG@3: {ndcg_at_k(rel_scores, 3):.4f}")
print(f"nDCG@5: {ndcg_at_k(rel_scores, 5):.4f}")
```

## 标准评测基准

### MTEB（Massive Text Embedding Benchmark）

- **内容**：涵盖分类、聚类、检索、重排等 8 大任务，100+ 数据集
- **用途**：嵌入模型的综合排行榜
- **地址**：[huggingface.co/spaces/mteb/leaderboard](http://huggingface.co/spaces/mteb/leaderboard)

### BEIR（Benchmarking IR）

- **内容**：18 个多样化检索数据集
- **用途**：衡量嵌入模型的零样本检索泛化能力

```python
# 使用 MTEB 评测嵌入模型
from mteb import MTEB
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("BAAI/bge-base-en-v1.5")

# 在特定任务上评测
evaluation = MTEB(tasks=["NFCorpus"])  # 选择一个检索任务
results = evaluation.run(model, output_folder="results/bge-base")

# 查看结果
for task_result in results:
    print(f"Task: {task_result.task_name}")
    print(f"  nDCG@10: {task_result.scores['test'][0]['ndcg_at_10']:.4f}")
```

## 在线评测指标

| 指标 | 说明 |
| --- | --- |
| 点击率 (CTR) | 用户点击了检索结果的比例 |
| 首条命中率 | 第一条结果就是用户想要的比例 |
| 人工相关性标注 | 请标注员判断结果相关性 |
| 延迟 (Latency) | 从 query 到返回结果的时间 |
| 吞吐 (QPS) | 每秒处理的查询数 |
| 成本 | 每百万次查询的费用 |

## 评测最佳实践

<aside>
⚠️

**常见误区**：只看离线 Recall@k，不做 rerank 评测，不做端到端生成质量评测。

</aside>

- 检索评测和生成评测要分开
- 同时测 Recall（召回覆盖度）和 nDCG（排序质量）
- 在目标领域的数据上测，而非只看通用 benchmark
- 要测延迟和成本，不只测质量
- 做 A/B 测试验证在线效果
