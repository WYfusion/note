## 概述

训练时 GPU 显存由 7 项主要成分组成。理解每一项是 **选择并行策略和显存优化方案** 的基础。

---

## 七项显存公式

$$M_{train} = M_{weight} + M_{master} + M_{grad} + M_{opt} + M_{act} + M_{temp} + M_{frag}$$

---

## 逐项详解

### 1. $M_{weight}$：模型权重

$$M_{weight} = N \times \frac{b_w}{8}$$

- BF16 训练：$M_{weight} = 2N$ bytes

- 7B 模型：$2 times 7 times 10^9 = 14$ GB

### 2. $M_{master}$：FP32 主权重副本

混合精度训练时需维护 FP32 主权重：

$$M_{master} = N \times 4 \text{ bytes}$$

- 7B 模型：28 GB

> [!important]
> 
> 某些框架（如 DeepSpeed ZeRO）将 master weight 归入优化器状态。本文分开列出以便理解。

### 3. $M_{grad}$：梯度

$$M_{grad} = N \times \frac{b_g}{8}$$

- BF16 梯度：$M_{grad} = 2N$ bytes

- 7B 模型：14 GB

### 4. $M_{opt}$：优化器状态

AdamW 维护两个动量：

$$M_{opt} = N \times (4 + 4) = 8N \text{ bytes}$$

- $m$（一阶矩）：FP32，$4N$ bytes

- $v$（二阶矩）：FP32，$4N$ bytes

- 7B 模型：56 GB

> [!important]
> 
> **优化器状态是显存占用最大的单项**。AdamW 的 $m, v$ 占 $8N$ bytes，远大于模型权重的 $2N$ bytes（BF16）。

### 5. $M_{act}$：中间激活

激活显存取决于 batch size $B$、序列长度 $S$、模型结构：

$$M_{act} \approx L \times B \times S \times (34d + 5 \cdot \frac{S \cdot h}{\text{TP}}) \times \frac{b_a}{8}$$

简化版（无 FlashAttention，BF16）：

$$M_{act} \approx L \times B \times S \times d \times 34 \times 2 \text{ bytes}$$

> [!important]
> 
> **FlashAttention 的关键作用**：将 attention 激活从 $O(B \times S^2 \times h)$ 降为 $O(B times S times h)$（不存储完整 attention 矩阵），长序列训练的必备条件。

### 6. $M_{temp}$：临时缓冲

- All-gather buffer（FSDP/ZeRO）

- Reduce-scatter buffer（TP）

- Kernel workspace（cuBLAS、FlashAttention 等）

- 通常占总显存的 5-15%

### 7. $M_{frag}$：碎片与框架开销

- CUDA 内存碎片

- PyTorch 缓存分配器预留

- NCCL 通信缓冲区

- 通常 10-20% overhead

---

## 参数相关显存速算

> [!important]
> 
> **不含激活时的参数相关显存**（BF16 AMP + AdamW）：

$$M_{param} = M_{weight} + M_{master} + M_{grad} + M_{opt} = 2N + 4N + 2N + 8N = 16N \text{ bytes}$$

|模型大小|$16N$ (bytes)|单卡 80GB 能否装下（仅参数相关）|
|---|---|---|
|1B|16 GB|✅ 富余|
|3B|48 GB|✅ 勉强（加激活可能溢出）|
|7B|112 GB|❌ 超出|
|13B|208 GB|❌ 远超|
|70B|1.12 TB|❌ 需 14+ 张 80GB GPU|

---

## ZeRO 各 Stage 对显存的影响

|Stage|分片内容|每卡显存（$G$ 卡，不含激活）|通信量|
|---|---|---|---|
|无 ZeRO|无|$16N$|All-reduce $2N$|
|**Stage 1**|优化器状态|$4N + \frac{12N}{G}$|All-reduce $2N$|
|**Stage 2**|优化器 + 梯度|$2N + \frac{14N}{G}$|All-reduce $2N$|
|**Stage 3**|优化器 + 梯度 + 权重|$\frac{16N}{G}$|All-gather $N$ • reduce-scatter $N$（每步 2x）|

> [!important]
> 
> **Stage 3 权衡**：显存最省但通信量增 ~1.5x。实际使用时通常 Stage 2 + TP 足以覆盖大多数场景。

## Python 显存估算器

```Python
def estimate_training_memory(
    N: float,           # 参数量 (e.g., 7e9)
    B: int = 1,         # micro batch size
    S: int = 2048,      # 序列长度
    L: int = 32,        # 层数
    d: int = 4096,      # 隐藏维
    zero_stage: int = 0,
    n_gpus: int = 1,
    activation_ckpt: bool = False,
) -> dict:
    """估算 BF16 AMP + AdamW 训练显存 (GB)"""
    G = n_gpus
    
    # 参数相关 (bytes)
    if zero_stage == 0:
        param_mem = 16 * N
    elif zero_stage == 1:
        param_mem = 4 * N + 12 * N / G
    elif zero_stage == 2:
        param_mem = 2 * N + 14 * N / G
    else:  # stage 3
        param_mem = 16 * N / G
    
    # 激活 (简化估算)
    act_layers = L if not activation_ckpt else L ** 0.5  # sqrt(L) for ckpt
    act_mem = act_layers * B * S * d * 34 * 2  # bytes
    
    # 临时 + 碎片 (~20%)
    overhead = 0.2 * (param_mem + act_mem)
    
    total = param_mem + act_mem + overhead
    
    return {
        "param_mem_GB": param_mem / 1e9,
        "act_mem_GB": act_mem / 1e9,
        "overhead_GB": overhead / 1e9,
        "total_GB": total / 1e9,
    }

# 7B 模型，单卡，ZeRO-2
print(estimate_training_memory(N=7e9, B=1, S=2048, L=32, d=4096, zero_stage=2, n_gpus=8))
```
