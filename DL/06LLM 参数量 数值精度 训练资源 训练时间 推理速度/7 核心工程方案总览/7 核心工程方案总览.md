## 概述

本页按 **结构层 → 并行层 → 显存层 → kernel/attention 层 → 微调层 → 量化层 → 解码加速层** 7 个维度，梳理经典→最新→流行的核心工程方案。

---

## 8.1 参数/结构层

|方案|核心思路|优势|代价|
|---|---|---|---|
|**Dense Transformer**|所有参数每 token 全参与|最稳、最通用、训练行为最可控|FLOPs 与 N 线性增长|
|**MoE / Switch**|路由选择 top-k expert|提高 $N_{total}$ 而不按比例增加每 token FLOPs|路由稳定性、负载均衡、通信复杂度|

---

## 8.2 训练并行层

|策略|切分维度|核心解决问题|通信模式|
|---|---|---|---|
|**DDP**|数据|最基础数据并行|All-reduce 梯度|
|**TP（Tensor Parallel）**|层内张量|模型太大单卡放不下|All-reduce / reduce-scatter（高频）|
|**PP（Pipeline Parallel）**|层间|深模型装载|Point-to-point（bubble 问题）|
|**FSDP / ZeRO**|参数/梯度/优化器|副本冗余显存浪费|All-gather / reduce-scatter|
|**CP（Context Parallel）**|序列维度|长上下文训练|Ring / all-to-all attention|
|**EP（Expert Parallel）**|expert 维度|MoE 必备|All-to-all dispatch/combine|

> [!important]
> 
> **组合策略示例**（DeepSeek-V3）：TP=1（靠 FP8 GEMM 单卡放下），PP=16, EP=64, DP=2, 无 CP。不同模型/硬件组合策略不同。

---

## 8.3 显存优化层

- **Activation checkpointing** — 用重算换显存（时间换空间，额外 ~33% FLOPs）

- **Offload** — 用 CPU/NVMe 换 GPU 显存（适合微调，训练吞吐降幅大）

- **Optimizer/state sharding** — ZeRO Stage 1/2/3 / FSDP 主战场

- **Low-rank / low-state training** — LoRA / QLoRA / GaLore（大幅降低可训练参数 + 优化器状态）

---

## 8.4 Kernel / Attention 层

|方案|核心思路|关键收益|
|---|---|---|
|**FlashAttention / FA-2 / FA-3**|IO-aware tiling，减少 HBM 读写|训练/推理长序列极关键；2-4x attention 加速|
|**FlashMLA**|Multi-head Latent Attention 优化 kernel|DeepSeek MLA 架构专用；进一步降 KV 开销|
|**GQA / MQA**|减少 KV head 数|直接降低 KV cache 与 decode 带宽压力|
|**PagedAttention / vLLM**|KV cache 虚拟内存分页管理|解决 KV 内存碎片，高吞吐 serving 主流|

---

## 8.5 微调层

|方案|可训练参数|显存需求|适用场景|
|---|---|---|---|
|**Full fine-tuning**|100%|$\sim 16N$ bytes|资源充足、效果最优|
|**LoRA**|$< 1\%$|权重 + LoRA 适配器|高效多任务部署|
|**QLoRA**|$< 1\%$|4-bit base + LoRA|**单机低成本微调主流**|

---

## 8.6 量化层

|方案|精度|量化对象|定位|
|---|---|---|---|
|**[LLM.int](http://LLM.int)8()**|INT8|权重 + 激活（混合分解）|早期成熟 8-bit 推理路线|
|**SmoothQuant**|W8A8|权重 + 激活（通道迁移）|W8A8 代表方案|
|**GPTQ**|3/4-bit|权重|高质量后训练权重量化|
|**AWQ**|4-bit|权重（保护显著通道）|工程化友好 weight-only|
|**FP8**|8-bit|权重 + 激活 + 梯度|训练/推理前沿主力|
|**NVFP4 / MXFP4**|4-bit|权重（+ 部分激活）|2025+ 硬件原生超低精度前沿|

---

## 8.7 解码加速层

|方案|核心思路|关键收益|
|---|---|---|
|**Speculative Decoding**|小 draft model 猜 + 大 target model 并行验证|减少串行 decode 壁垒；2-3x 加速|
|**Continuous Batching**|请求粒度的动态 batch|提升 GPU 利用率与系统吞吐|
|**Prefix / KV 共享**|多请求共享公共前缀 KV|system prompt 重复场景收益显著|
