## Codec LM 核心思想

将语音编码为多层离散 codec token（通过 RVQ），然后用语言模型逐 token 生成：

```mermaid
graph LR
    TXT["📝 Text + Prompt"] --> AR["AR Model<br>第 1 层 token"]
    AR --> NAR["NAR Model<br>第 2~8 层 token"]
    NAR --> DEC["Codec Decoder"]
    DEC --> WAV["🎵 Waveform"]
```

### Residual Vector Quantization (RVQ)

Encodec / SoundStream 使用 RVQ 将连续音频编码为多层离散 token：

$$\hat{x} = \sum_{q=1}^{Q} \text{Codebook}_q\left(\arg\min_{c} \| r_{q} - c \|\right), \quad r_{q+1} = r_q - \hat{r}_q$$

- 第 1 层捕获**粗粒度**信息（韵律、内容）

- 后续层捕获**细粒度**残差（音质细节）

## VALL-E 架构

|**组件**|**说明**|
|---|---|
|**音频编码器**|Encodec (8 层 RVQ, 75Hz)|
|**AR 模型**|Transformer Decoder, 生成第 1 层 token|
|**NAR 模型**|Transformer, 并行生成第 2~8 层 token|
|**训练数据**|LibriLight 60K 小时|

### 生成过程

$$p(\mathbf{c} | \text{text}, \text{prompt}) = \underbrace{\prod_{t} p(c_{t,1} | c_{<t,1}, \text{text}, \text{prompt})}_{\text{AR: 第1层}} \cdot \underbrace{\prod_{q=2}^{Q} p(\mathbf{c}_{:,q} | \mathbf{c}_{:,<q}, \text{text})}_{\text{NAR: 第2~Q层}}$$

## 局限性分析

> [!important]
> 
> **Codec LM 范式的核心局限**（也是 CosyVoice 选择 Hybrid 路线的原因）：

1. **语义信息不足**：Encodec token 是无监督学习的，缺乏显式文本对齐 → 内容一致性差

1. **多层建模复杂**：8 层 RVQ 需要 AR+NAR 两个模型 → 架构复杂

1. **推理速度慢**：AR 生成 75Hz × 序列长度的 token → 长语音推理慢

1. **音色-内容耦合**：codec token 同时编码了音色和内容 → 零样本克隆困难

### 与 CosyVoice 的关键差异

|维度|VALL-E (Codec LM)|CosyVoice (Hybrid)|
|---|---|---|
|Token 类型|无监督 codec (8层 RVQ)|监督语义 (单层)|
|Token Rate|75 Hz × 8层|25 Hz × 1层|
|语义对齐|❌ 无|✅ ASR 监督|
|声学生成|Codec Decoder|Flow Matching|
