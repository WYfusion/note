> [!important]
> 
> **一句话定位**：连续特征上的非自回归扩散/流匹配生成，以 Voicebox 和 F5-TTS 为代表。

---

## 核心思想

直接在连续声学特征空间（Mel 频谱图或潜在表示）上进行生成，无需离散化步骤：

$$dx_t = v_\theta(x_t, t, \text{cond}) \, dt, \quad x_0 \sim \mathcal{N}(0, I), \quad x_1 \approx \text{Mel}$$

## Voicebox (Meta, 2023)

**特点**：基于 Non-autoregressive Flow Matching，使用 **音素级对齐** 作为条件。

|**组件**|**说明**|
|---|---|
|**文本表示**|音素序列 + 强制对齐 duration|
|**生成模型**|Conditional Flow Matching (Transformer)|
|**条件方式**|Masked infilling（随机 mask 部分 Mel，预测完整 Mel）|
|**训练数据**|60K 小时英文|

### Masked Infilling 训练

$$\mathcal{L} = \mathbb{E}_{t, m} \left\| v_\theta(x_t, t, x_{\text{masked}}, \text{text}) - (x_1 - x_0) \right\|^2$$

其中 $m$ 是随机 mask，训练时随机遮蔽 70% 的 Mel 帧。

## F5-TTS (2024)

**特点**：**无需音素对齐**，直接使用字符级文本，大幅简化流水线。

- 使用 **DiT (Diffusion Transformer)** 架构

- 文本填充（text infilling）：将文本 token 和语音 token 拼接

- 无需 G2P、forced alignment 等预处理

- 开源、可复现

## 局限性

> [!important]
> 
> Feature Diffusion 范式的主要局限：

1. **韵律建模弱**：非自回归生成难以建模长程韵律依赖（停顿、语调变化）

2. **缺乏粗粒度控制**：没有离散 token 层，难以实现 token 级的细粒度控制

3. **流式困难**：扩散/流匹配需要全局生成，天然不支持流式

4. **依赖对齐**（Voicebox）：需要音素级强制对齐，增加预处理复杂度

### 与 CosyVoice 的定位差异

CosyVoice 通过 Hybrid 路线解决了这些问题——用 LM 负责韵律建模，FM 负责声学生成，取两者之长。
