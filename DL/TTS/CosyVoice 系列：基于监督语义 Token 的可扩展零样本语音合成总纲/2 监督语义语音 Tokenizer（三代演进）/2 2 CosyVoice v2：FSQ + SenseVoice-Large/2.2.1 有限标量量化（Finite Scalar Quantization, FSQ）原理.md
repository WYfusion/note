> [!important]
> 
> **一句话定位**：低秩投影 + bounded round + (2K+1)-ary 索引的数学推导。

---

## FSQ 完整数学推导

### Step 1: 低秩投影

将高维编码器输出 $H \in \mathbb{R}^{T \times D}$ 投影到低维空间：

$$\tilde{H} = H \cdot W_{\text{down}} \in \mathbb{R}^{T \times d}, \quad W_{\text{down}} \in \mathbb{R}^{D \times d}$$

其中 $d \ll D$（例如 $D=512, d=4$）。

### Step 2: Bounded Rounding

对每个维度进行有界取整，若每一个维度的可选范围都限制在 $[-K, K]$ 范围内：

$$\bar{h}_i = \text{clamp}\left(\text{round}(\tilde{h}_i), -K, K\right), \quad i = 1, \ldots, d$$

### Step 3: 上投影恢复

将量化后的低维向量投影回高维空间：

$$\hat{H} = \bar{H} \cdot W_{\text{up}} \in \mathbb{R}^{T \times D}, \quad W_{\text{up}} \in \mathbb{R}^{d \times D}$$

### Step 4: Token 索引计算

每个 $d$ 维量化向量可映射为唯一整数索引（$(2K+1)$ 进制编码）：

$$\text{idx} = \sum_{i=1}^{d} (\bar{h}_i + K) \cdot (2K+1)^{i-1}$$
#### 进一步的细致展开的说：
降维和取整后，你其实只拿到了一个**多维的离散坐标** $[x_0,x_1,...,x_{d-1}]$ ，这里的每一个维度的可选值范围 $L_i$ 可以自己定，一般是精心设计好了，比如 $x_0 \in [-2, -1, 0, 1, 2]$，$x_1 \in [-3, -2, -1, 0, 1, 2, 3]$。但大语言模型（LLM）的 Embedding 层不认识多维坐标，它只认识一个**一维的整数 Token ID**（比如第 3450 号词）。

将“多维离散坐标”转化为“一维整数 Token ID”的过程，本质上就是一个**多进制（混合进制）转十进制**的数学计算。在 FSQ 的代码实现中，这只需要一次极其简单的**点积**（[[Dot-product]]）运算。

下面为你一步步拆解这个计算过程：

##### 1. 设定网格层级 (Levels)

在定义 FSQ 时，我们需要为降维后的每一个维度设定一个“级别数（Levels）”。

假设我们降维到了 $D=4$ 维，并设定每个维度的级别数分别为 $L = [8, 5, 5, 5]$。

这意味着：

- 第 1 维有 8 个离散值（例如 $-3, -2, -1, 0, 1, 2, 3, 4$）
- 第 2、3、4 维各有 5 个离散值（例如 $-2, -1, 0, 1, 2$）
- 此时，大模型的总词表大小（Vocabulary Size）就是笛卡尔积：$8 \times 5 \times 5 \times 5 = 1000$。Token 的索引范围就是 $0$ 到 $999$。

##### 2. 第一步：坐标平移（Shift to Zero-based）

取整之后，网络输出的可能是有正有负的坐标，比如 $z = [1, -2, 0, 2]$。

为了计算索引，我们首先要把它们平移成从 $0$ 开始的正整数。

- 第 1 维原本范围是 $[-3, 4]$，全部加上 $3$，变成 $[0, 7]$。
- 第 2、3、4 维原本范围是 $[-2, 2]$，全部加上 $2$，变成 $[0, 4]$。
    假设平移后，我们的坐标向量变成了 $X = [x_1, x_2, x_3, x_4]$，确保每一个 $x_i \in [0, L_i - 1]$。

##### 3. 第二步：计算“步长基底”（Strides / Basis）

这就好比十进制里的“个位、十位、百位”，因为每个维度的进制不同，我们需要提前计算好一个固定的**基底向量（Basis Vector）**。

计算规则是累乘前一个维度的级别数：

- $B_1 = 1$ （类似于个位）
- $B_2 = L_1 = 8$ （逢 8 进 1）
- $B_3 = L_1 \times L_2 = 8 \times 5 = 40$
- $B_4 = L_1 \times L_2 \times L_3 = 8 \times 5 \times 5 = 200$

所以，我们的基底向量固定为 $B = [1, 8, 40, 200]$。

##### 4. 第三步：点积得出最终 Token ID

计算最终的 Token ID 极其简单，直接将平移后的坐标向量 $X$ 与基底向量 $B$ 求**点积**即可：

$$I = \sum_{i=1}^{D} x_i \cdot B_i$$

**举个具体的例子：**

假设平移后的取整特征向量是 $X = [3, 1, 4, 2]$。

它的 Token Index 计算如下：

$$I = (3 \times 1) + (1 \times 8) + (4 \times 40) + (2 \times 200) = 3 + 8 + 160 + 400 = 571$$

这个 `571` 就是最终喂给大语言模型的 Token ID！

---

#### 反向过程：LLM 预测出 ID 后，怎么变回坐标？

当后端的 LLM 进行自回归生成，预测出下一个 Token ID 是 `571` 时，它是如何还原回多维坐标，并送给升维层（Up-projection）的呢？

只需要用连续的**整除**（`//`）和**取余**（`%`）操作就能完美逆向解构：

1. `571 % 8 = 3`，`571 // 8 = 71` $\rightarrow$ 第 1 维是 3
2. `71 % 5 = 1`，`71 // 5 = 14` $\rightarrow$ 第 2 维是 1
3. `14 % 5 = 4`，`14 // 5 = 2` $\rightarrow$ 第 3 维是 4
4. `2 % 5 = 2`，`2 // 5 = 0` $\rightarrow$ 第 4 维是 2

得到 $[3, 1, 4, 2]$ 后，再反向减去之前的平移量，就完美还原出了网格上的物理坐标。
## 为什么 FSQ 不会 Collapse？

> [!important]
> 
> **根本原因**：FSQ 没有可学习的 codebook 向量。码本是由所有可能的整数组合隐式定义的，每个码本天然存在且不需要被"激活"。

|机制|VQ|FSQ|
|---|---|---|
|码本定义|可学习向量|隐式整数网格|
|码本更新|EMA / 梯度|不需要（投影层学习）|
|码本利用|受限于初始化和训练动态|**所有码本都可被使用**|

## 梯度传递

FSQ 同样使用 STE 进行梯度传递：

$$\frac{\partial \mathcal{L}}{\partial \tilde{h}_i} \approx \frac{\partial \mathcal{L}}{\partial \bar{h}_i} \quad \text{(round 操作的 STE)}$$

但与 VQ 不同，FSQ 的投影层 $W_{\text{down}}, W_{\text{up}}$ 是完全可微分的，梯度信号更稳定。


## FSQ 词表设计原则
在 FSQ (Finite Scalar Quantization) 的设计中，确定每一个维度的可选数值范围（通常称为 **Levels** 或 **Bins**，比如之前例子中的 `[8, 5, 5, 5]`），并不是靠数学推导算出来的唯一解，而是基于一个**极其精妙的工程与经验折中（Trade-off）**。

这个折中主要围绕三个核心目标来展开。我们一层层来剖析：

### 1. 核心目标一：凑出一个合适的“总词表大小”（Vocabulary Size）

我们在前面提到过，FSQ 的总词表大小 $|V|$ 是各个维度级别的**笛卡尔积（累乘）**：

$$|V| = \prod_{i=1}^{d} L_i$$

- **如果 $|V|$ 太小（比如只有 256）：** 表达能力太弱，同一个 Token 会被迫代表太多不同的声音细节，导致合成的音频发糊、发音含混不清。
- **如果 $|V|$ 太大（比如超过 10 万）：** 会给后端的 LLM 分类预测带来极大的计算负担（Softmax 层会非常庞大），并且数据稀疏，模型很难训练充分。

在语音大模型中，通常希望这个词表大小落在 **$1000$ 到 $65000$** 之间。

- 例如，研究人员想要一个大概 1000 大小的词表，他们就会去拆解因数：$8 \times 5 \times 5 \times 5 = 1000$。
- 如果想要一个大一点的，比如大概 1.5 万的词表：$8 \times 8 \times 8 \times 6 \times 5 = 15360$。

### 2. 核心目标二：兼顾直通估计器（STE）的“梯度健康”

既然想要 1000 的词表，为什么不直接降维到 $1$ 维，然后给这 1 个维度设置 $1000$ 个级别？或者降维到 $10$ 维，每个维度只设 $2$ 个级别（二进制量化）？

这就涉及到了神经网络反向传播的命门：**直通估计器（STE）**。

我们知道，取整操作 `round()` 是不可导的，STE 是强行把梯度复制过去。这种做法天然带有**梯度误差**。

- **维度太少、级别太多（例如 1 维，1000 个级别）：** 相邻两个级别之间的跨度非常小。这意味着模型稍微有一点波动，就会跳到另一个词上，导致特征对齐非常困难，梯度误差被急剧放大。
- **维度太多、级别太少（例如 10 维，全是 2 级别）：** 虽然量化极度粗糙，但维度太高，模型很难在这么高维度的离散空间里建立起有效的语义拓扑映射，相当于在玩一个有 1024 种变化的拼图，极易陷入局部最优。

**经验上的“甜点区（Sweet Spot）”：** FSQ 的原作者（来自 Google 团队）通过大量的消融实验发现，将降维层输出的**维度 $d$ 控制在 4 到 6 维之间**，并且**每个维度的级别数 $L_i$ 控制在 4 到 8 之间**，网络收敛得最快，STE 的梯度误差也最小，特征还原度最高。
### 3. 核心目标三：为何钟爱奇数（比如 5 或 7）？—— 零的特殊语义

仔细观察业界常用的 FSQ 配置，你会发现它们极其偏爱奇数，比如 $L = [8, 5, 5, 5]$ 或者 $[7, 5, 5, 5, 5]$。

为什么是奇数？**因为奇数天然包含一个绝对中心点：0。**

- 如果一个维度的范围是 5 级：`[-2, -1, 0, 1, 2]`。
- 如果一个维度的范围是 7 级：`[-3, -2, -1, 0, 1, 2, 3]`。

在语音处理中，经常会遇到**绝对静音（Silence）、无语音活动区间（Padding）或者背景白噪声**。

当网络遇到这些“空信息”时，降维层最自然的反应就是输出一个趋近于 $0$ 的向量。如果网格恰好在 $0$ 点有一个离散坐标（即所有的奇数级别维度都会在 $0$ 处交汇），网络就能非常轻松、稳定地用坐标 `[0, 0, 0, 0]` 来表示“这里什么声音都没有”。

如果强制使用偶数（比如 4 级：`[-2, -1, 1, 2]`，没有 `0`），网络遇到静音时，就会在 `-1` 和 `1` 之间疯狂左右横跳（因为它输出趋近于 $0$，但网格强制它必须归类为 $-1$ 或 $1$），这会导致背景生成莫名其妙的底噪。

_(注：有时保留一个 8 是为了让整体词表大小凑整，或者用特定的偏移逻辑处理。)_
