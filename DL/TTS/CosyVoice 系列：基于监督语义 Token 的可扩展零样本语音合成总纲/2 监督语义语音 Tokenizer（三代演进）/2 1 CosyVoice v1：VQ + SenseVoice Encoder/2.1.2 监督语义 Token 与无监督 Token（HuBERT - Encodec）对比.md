## 三类语音 Token 对比

|**维度**|**HuBERT (自监督)**|**Encodec (codec)**|**CosyVoice (监督)**|
|---|---|---|---|
|**训练目标**|掩码预测 (BERT-style)|音频重建|**ASR 语音识别**|
|**语义信息**|中等（隐式学习）|弱（重建为主）|**强（显式对齐文本）**|
|**说话人信息**|部分保留|完全保留|**最少（信息瓶颈）**|
|**声学细节**|丢失较多|保留完整|丢失（由 FM 负责）|
|**Token Rate**|50 Hz|75 Hz × 8层|**25 Hz × 1层**|
|**下游 TTS**|VEVO 等|VALL-E 等|**CosyVoice**|

## 监督 Token 的核心优势

### 1. 内容一致性更强

ASR 监督训练迫使 token 编码与文本对齐的语义信息：

$$\text{MI}(\text{Token}; \text{Text}) \gg \text{MI}(\text{HuBERT Token}; \text{Text})$$

实验证据：CosyVoice 的 CER 显著低于使用 HuBERT token 的系统。

### 2. 说话人无关性更好

VQ/FSQ 作为信息瓶颈，迫使编码器丢弃与 ASR 无关的信息（包括说话人特征）：

$$\text{MI}(\text{Token}; \text{Speaker}) \to \text{minimal}$$

这使得零样本克隆更容易：音色完全由 prompt 和 FM 控制，token 不会干扰。

### 3. 更低的 Token Rate

25Hz × 1层 vs 75Hz × 8层（Encodec），序列长度大幅缩短：

- **LM 建模更高效**：更短的序列，更快的训练和推理

- **韵律建模更自然**：每个 token 覆盖更长的时间跨度

> [!important]
> 
> **核心洞察**：CosyVoice 的监督语义 token 实现了一种优雅的信息分配——token 专注于"说什么"（内容+韵律），FM 负责"怎么说"（音色+音质）。这种解耦是 CosyVoice 三代共享的设计基石。
