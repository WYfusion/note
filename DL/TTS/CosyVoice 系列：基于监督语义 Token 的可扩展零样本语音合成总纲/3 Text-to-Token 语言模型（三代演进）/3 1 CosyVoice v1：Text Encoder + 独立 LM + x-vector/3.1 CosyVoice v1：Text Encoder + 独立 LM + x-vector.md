### 输入序列结构

#### 怎么训练的：
![[Pasted image 20260412100911.png]]
$$[S, \mathbf{v}, \text{text\_tokens}, T, \text{speech\_tokens}, E]$$
是一个通用的模板：
- $S$: 序列开始标记

- $\mathbf{v}$: x-vector 说话人嵌入

- ${text_{tokens}}$: Text Encoder 编码的文本，也就是“这段语音在说什么”。

- $T$: 文本-语音分隔符

- ${speech_{tokens}}$: 目标语义 token 序列

- $E$: 序列结束标记
也就是说，训练时 $v、{text_{tokens}}、{speech_{tokens}}$ 这三者通常来自**同一个训练样本**，但承担的是**不同信息角色**。

[[CosyVoice- A Scalable Multilingual Zero-shot Text-to-speech Synthesizer Using Supervised Semantic Tokens#^c9a4bb|语义、说话人、韵律]]
- **Text Encoder 编码的文本**：对应**文本侧 semantic 条件**
- **x-vector 说话人嵌入**：对应 **speaker**
- **目标语义 token 序列**：对应 **semantic 的主要承载体**，同时也包含部分 **prosody**
- **prosody 本身**：不是图里某一个单独输入块，而是主要由 **LLM 对 token 序列的建模**来体现，并在 **flow matching** 中继续落实到实际语音上
##### `speech_tokens`：Tokenizer 从真实语音中提取的离散语义 token

这是最容易混淆的部分。`speech_tokens` 是监督语义 Tokenizer 从真实语音波形中提取的离散 token 序列，不是最终输出的波形，即a子图中的Speech Tokenizer的输出。

它的角色取决于**训练还是推理**：

| 阶段      | `speech_tokens` 的角色                                                             |
| ------- | ------------------------------------------------------------------------------- |
| **训练时** | Ground Truth——从训练集语音中用 Tokenizer 提取的真实语义 token，作为 LM 的**预测目标**（Teacher Forcing） |
| **推理时** | LM **自回归生成**的输出——逐个 token 生成，直到遇到 $E$（结束符）                                      |

#### 在零样本推理的完整序列中：
$$[S,v,\bar Y_{prompt}​,\bar Y_{target}​,T,\mu_{prompt}​,\hat\mu_{target},E]$$

其中：

- $\mathbf v$：从参考音频里用预训练声纹模型抽出来的**speaker embedding**。它主要负责“像谁说”，也就是说话人身份、音色等稳定 speaker 特征。
- $\bar Y_{prompt}$：参考音频内容的文本转写，经 BPE + Text Encoder 后得到的编码
- $\bar Y_{target}$：你真正想让模型说出来的目标文本，经同样编码得到
- $\mu_{prompt}$：从参考音频提取的 speech tokens，**作为已知前缀**
- $\hat\mu_{target}$：LM 在这个前缀后面**一步一步生成**的 target speech tokens所以 $\hat\mu_{target}$​ 可以理解为对 $\mu_{prompt}$ 的**continuation / continuation generation**。论文原文就是把 prompt tokens 当作 pre-generated，然后让 LM 继续预测后续 tokens，CosyVoice 的 speech tokens 是以 semantic 为主的 supervised semantic tokens。续写的是 **speech token 序列**，其中不仅包含内容信息，也可能带有一部分 prosody。[[3.1.1 序列构造与 In-Context Learning 机制|ICL机制]]
- $T$：只出现一次，用来把“文本条件段”和“speech token 段”分开
- $E$：目标 speech token 生成结束标记。
不要混淆这里的 `参考音频` 与 `自回归出来的音频` 认为这两个本来就不该是一个说话人！！这里做的是零样本语音合成，生成出来的就是和参考的语音很像才对，所以说用的是同一个说话人嵌入 $\mathbf v$，训练的时候用的就是同一个说话人嵌入的。


## 训练策略

- **Teacher Forcing**：训练时使用 ground truth token 作为上下文

- **损失函数**：交叉熵损失

$$\mathcal{L}_{\text{LM}} = -\sum_{t=1}^{T} \log p_\theta(s_t | s_{<t}, \text{text}, \mathbf{v})$$

## v1 的局限

1. **信息泄漏**：x-vector 直接注入音色信息，可能干扰 token 生成的韵律自然度

2. **Text Encoder 冗余**：独立训练的 TE 与 LM 可能存在表征失配

3. **不支持流式**：AR 生成完整序列后才能传给 FM
