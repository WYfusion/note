## v1 序列构造

CosyVoice v1 将所有信息编码为一个线性序列：

$$\text{Seq} = [\underbrace{S}_{\text{开始}}, \underbrace{\mathbf{v}}_{\text{x-vector}}, \underbrace{t_1, \ldots, t_N}_{\text{文本 token}}, \underbrace{T}_{\text{分隔}}, \underbrace{s_1, \ldots, s_M}_{\text{语义 token}}, \underbrace{E}_{\text{结束}}]$$

### 各组件说明

| **符号**             | **来源**       | **作用**            |
| ------------------ | ------------ | ----------------- |
| $S$                | 特殊 token     | 序列开始标记            |
| $\mathbf{v}$       | x-vector 提取器 | 注入目标说话人音色         |
| $t_1, \ldots, t_N$ | Text Encoder | 文本语义表示            |
| $T$                | 特殊 token     | 文本-语音分隔符          |
| $s_1, \ldots, s_M$ | Tokenizer 编码 | 目标语义 token（AR 生成） |
| $E$                | 特殊 token     | 序列结束标记            |

## In-Context Learning (ICL)

零样本合成时，将 prompt 的文本和语音 token 作为前缀拼接：

$$[\ldots, t^{\text{prompt}}, T, s^{\text{prompt}}, t^{\text{target}}, T, \underbrace{s^{\text{target}}}_{\text{AR 生成}}, E]$$

LM 通过 self-attention 机制访问 prompt 的 token，学习其韵律和风格模式，实现零样本音色克隆。

### 跨语言 Voice Cloning

ICL 机制天然支持跨语言克隆：

```JavaScript
Prompt: [中文文本 tokens] + [中文语音 tokens]
Target: [英文文本 tokens] + [?? 英文语音 tokens]  ← AR 生成
```

LM 从 prompt 中学习音色，同时将其应用到不同语言的文本上。
