## 首包延迟模型

流式 TTS 的用户体验由**首包延迟**（First Packet Latency）决定：

$$L_{\text{TTS}} = M \cdot (d_{\text{lm}} + d_{\text{fm}} + d_{\text{voc}})$$

### 各项含义

|**符号**|**含义**|**典型值**|
|---|---|---|
|$M$|每 chunk 的语音 token 数|~4 tokens|
|$d_{\text{lm}}$|LM 生成单个 token 的延迟|~10ms|
|$d_{\text{fm}}$|FM 生成对应 Mel 的延迟|~20ms|
|$d_{\text{voc}}$|Vocoder 生成波形的延迟|~5ms|

### 计算示例

$$L = 4 \times (10 + 20 + 5) = 140\text{ms}$$

CosyVoice v2 实际测量首包延迟约 **150ms**，与理论值吻合。

## 延迟-质量权衡

|**FM Mask**|**首包延迟**|**质量**|
|---|---|---|
|Full-causal|~100ms（最低）|略有下降|
|Chunk-M|~150ms|接近离线|
|Chunk-2M|~200ms|几乎无损|
|Non-causal（离线）|全句延迟|最优|

> [!important]
> 
> **工程判断**：150ms 首包延迟对实时对话场景已足够流畅（人类感知阈值约 200ms），且合成质量与离线模式几乎无差异。
