## N:M 交错机制

v2 的核心创新：文本 token 和语音 token **交替排列**，而非像 v1 那样先文本后语音：

```JavaScript
v1: [text₁...textₙ | speech₁...speechₘ]  ← 全部文本 → 全部语音
v2: [text₁...textₙ, speech₁...speechₘ, textₙ₊₁...text₂ₙ, speechₘ₊₁...speech₂ₘ, ...]
```

### 交错参数

- **N**：每组文本 token 数量

- **M**：每组语音 token 数量

- 比例 N:M 控制文本-语音的交织密度

### Filling Token 机制

双向流式场景中，文本可能尚未完全到达。用 **filling token** 占位：

```JavaScript
已到达: [text₁...textₙ, speech₁...speechₘ]
未到达: [FILL...FILL, speech_{m+1}...speech_{2m}]  ← filling token 占位
到达后: [text_{n+1}...text_{2n}, speech_{m+1}...speech_{2m}]  ← 替换
```
