## 核心设计哲学

CosyVoice 系列的核心范式可以用一个公式概括：

$$\text{Speech} = \underbrace{\text{LM}(\text{Text} \to \text{Semantic Tokens})}_{\text{Stage 1: 粗粒度语义建模}} + \underbrace{\text{FM}(\text{Tokens} \to \text{Mel})}_{\text{Stage 2: 细粒度声学生成}}$$

### 两阶段解耦设计

![[2026-04-21 20.54.38.excalidraw|1200]]

- **Stage 1（语义建模）**：自回归 LM 将文本转换为离散语义 token 序列，捕获韵律、节奏、停顿等高层信息

- **Stage 2（声学生成）**：条件流匹配模型将离散 token 解码为连续 Mel 频谱图，BigVGAN 声码器重建波形

> [!important]
> 
> **核心差异化**：CosyVoice 使用 **监督语义 token**（由 ASR 编码器训练得到），而非无监督 codec token（如 Encodec / SoundStream）。这使 token 具有更好的**内容一致性**和**说话人无关性**，是三代共享的核心设计原则。

## 关键创新点一览

1. **监督语义 Token**：将 VQ 插入 ASR 编码器中间层，用语音识别损失监督训练 → 内容对齐 + 说话人无关

2. **FSQ 替代 VQ**（v2）：有限标量量化，codebook 利用率 23% → 100%，消除 codebook collapse

3. **LLM Backbone**（v2）：移除 Text Encoder 和 Speaker Embedding，直接复用预训练 LLM，简化架构

4. **统一流式/非流式**（v2）：N:M token 交错 + chunk-aware causal FM → 150ms 首包延迟

5. **DiffRO 后训练**（v3）：可微分奖励优化 + Gumbel Softmax + token 级 KL 约束

6. **规模化验证**（v3）：1.5B 参数 + 1M 小时数据 + 9 语言 + 18 方言
