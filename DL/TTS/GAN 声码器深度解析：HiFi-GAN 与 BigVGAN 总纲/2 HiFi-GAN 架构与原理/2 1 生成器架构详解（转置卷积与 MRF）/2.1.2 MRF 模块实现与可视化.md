## 1. MRF 数学定义

$$\text{MRF}(x) = \sum_{n=1}^{|k_r|} \text{ResBlock}_{k_r[n], D_r[n]}(x)$$

HiFi-GAN V1: $k_r = [3, 7, 11]$，$D_r = [[1,3,5], [1,3,5], [1,3,5]]$

### 1.1 每个 ResBlock 的感受野

|**ResBlock**|**kernel** $k$|**dilations**|**单层感受野**|**堆叠感受野**|1|3|[1,3,5]|3, 7, 11|21|
|---|---|---|---|---|---|---|---|---|---|
|2|7|[1,3,5]|7, 19, 31|57|3|11|[1,3,5]|11, 31, 51|93|

并行求和后，MRF 同时具备 **21~93** 个采样点的多尺度感受野。

## 2. 完整 PyTorch 实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
from torch.nn.utils import weight_norm

class ResBlock(nn.Module):
    """HiFi-GAN ResBlock: 多层膨胀卷积 + 残差连接
    
    每层结构: LeakyReLU -> DilatedConv -> LeakyReLU -> Conv1x1
    """
    def __init__(self, channels, kernel_size=3, dilations=(1, 3, 5)):
        super().__init__()
        self.convs1 = nn.ModuleList()  # 膨胀卷积
        self.convs2 = nn.ModuleList()  # 1x1 卷积
        for d in dilations:
            self.convs1.append(weight_norm(nn.Conv1d(
                channels, channels, kernel_size,
                dilation=d,
                padding=self._get_padding(kernel_size, d)
            )))
            self.convs2.append(weight_norm(nn.Conv1d(
                channels, channels, kernel_size,
                dilation=1,
                padding=self._get_padding(kernel_size, 1)
            )))
    
    @staticmethod
    def _get_padding(kernel_size, dilation):
        """same padding: 保持序列长度不变"""
        return (kernel_size * dilation - dilation) // 2
    
    def forward(self, x):
        for c1, c2 in zip(self.convs1, self.convs2):
            xt = F.leaky_relu(x, 0.1)
            xt = c1(xt)                 # 膨胀卷积
            xt = F.leaky_relu(xt, 0.1)
            xt = c2(xt)                 # 1x1 卷积
            x = xt + x                  # 残差
        return x


class MRF(nn.Module):
    """多感受野融合模块: 并行多个不同 kernel size 的 ResBlock
    
    核心思想: 不同 kernel size 的 ResBlock 提供不同感受野,
    求和融合后生成器可以同时感知多种尺度的音频模式。
    """
    def __init__(self, channels, kernel_sizes=(3, 7, 11),
                 dilations=((1,3,5), (1,3,5), (1,3,5))):
        super().__init__()
        self.resblocks = nn.ModuleList([
            ResBlock(channels, k, d)
            for k, d in zip(kernel_sizes, dilations)
        ])
    
    def forward(self, x):
        # 并行处理 + 求和融合
        out = self.resblocks[0](x)
        for rb in self.resblocks[1:]:
            out = out + rb(x)
        return out
```

> [!important]
> 
> **思辨：为什么是求和而非 concat？**
> 
> 求和保持通道数不变，不会增加后续层的计算量。而 concat 会将通道数乘以 $|k_r|$（如 3×），显著增加参数量和计算量。消融实验也显示求和和 concat 的 MOS 差异很小，但求和远更高效。
