## 0. 定位

> Feature Matching 的感知学习原理、与 Perceptual Loss 的关系、实现细节

---

## 1. 数学定义

$$\mathcal{L}_{FM}(G; D) = \mathbb{E}_{(x,s)} \sum_{i=1}^{L} \frac{1}{N_i} \| D^{(i)}(x) - D^{(i)}(G(s)) \|_1$$

其中 $D^{(i)}$ 是判别器第 $i$ 层的中间特征，$N_i$ 是该层特征的元素数。

---

## 2. 与 Perceptual Loss 的关系

> [!important]
> 
> Feature Matching 本质上是一种**感知损失（Perceptual Loss）**——不是在像素级别比较波形，而是在判别器学到的高级特征空间中比较。判别器的中间层编码了人耳敏感的音频特征（如频谱结构、周期性、连续性），提供比原始 L1/L2 损失更丰富的监督信号。

---

## 3. 关键实现细节

```python
def feature_matching_loss(fmap_real, fmap_gen):
    """
    fmap_real: 判别器对真实音频的中间特征
    fmap_gen:  判别器对生成音频的中间特征
    注意: r.detach() 阻止梯度传播到判别器
    """
    loss = 0
    for fr_list, fg_list in zip(fmap_real, fmap_gen):
        for fr, fg in zip(fr_list, fg_list):
            loss += torch.mean(torch.abs(fr.detach() - fg))
    return loss * 2  # λ_fm = 2
```

> [!important]
> 
> `**detach()**` **的关键作用**：对真实样本的特征 `fr` 调用 `detach()`，阻止 FM 损失的梯度反传到判别器。这确保 FM 损失只优化生成器——如果不 detach，判别器会被误导向最小化自己的特征差异，导致训练崩溃。
