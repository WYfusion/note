## 1. STFT 数学定义

短时傅里叶变换（Short-Time Fourier Transform）将信号分为短帧后做 DFT：

$$X(m, k) = \sum_{n=0}^{N-1} x(n + mH) \cdot w(n) \cdot e^{-j2\pi kn/N}$$

其中：

- $m$ 为帧索引，$k$ 为频率 bin 索引

- $H$ 为 hop length（帧移），$N$ 为 FFT 点数（窗长）

- $w(n)$ 为窗函数（通常为 Hann 窗）

STFT 产生的复数矩阵 $X \in \mathbb{C}^{T \times F}$，其中 $T = \lfloor(L-N)/H\rfloor + 1$，$F = N/2+1$。

**幅度谱**与**相位谱**：

$$|X(m,k)| = \sqrt{\text{Re}^2 + \text{Im}^2}, \quad \phi(m,k) = \text{atan2}(\text{Im}, \text{Re})$$

```python
import torch
import torchaudio

def compute_stft(waveform, n_fft=1024, hop_length=256, win_length=1024):
    """计算 STFT，返回幅度谱和相位谱"""
    window = torch.hann_window(win_length)
    # stft_complex: [B, F, T] complex tensor
    stft_complex = torch.stft(
        waveform, n_fft=n_fft,
        hop_length=hop_length, win_length=win_length,
        window=window, return_complex=True
    )
    magnitude = stft_complex.abs()    # 幅度谱
    phase = stft_complex.angle()      # 相位谱
    return magnitude, phase

# 示例：22kHz 音频，n_fft=1024
# 输出：magnitude [1, 513, T], phase [1, 513, T]
# 其中 513 = 1024/2+1 个频率 bin
```

> [!important]
> 
> **思辨：窗长与 hop length 的时频分辨率权衡。**
> 
> 这是 Heisenberg 不确定性原理在信号处理中的体现：$\Delta t \cdot \Delta f \geq \frac{1}{4\pi}$。窗长 $N$ 越大，频率分辨率越高（$\Delta f = f_s/N$），但时间分辨率越低。声码器常用 $N=1024, H=256$，这是语音基频（80–400Hz）和时间精度（11.6ms 帧移）之间的经验性平衡。

## 2. Mel 滤波器组

### 2.1 从线性频率到 Mel 尺度

人耳对频率的感知是**对数式**的：1000Hz 到 2000Hz 的感知差异 ≈ 4000Hz 到 8000Hz。Mel 尺度将线性频率映射为人耳感知的等距离刻度：

$$m = 2595 \cdot \log_{10}\left(1 + \frac{f}{700}\right)$$

$$f = 700 \cdot \left(10^{m/2595} - 1\right)$$

### 2.2 滤波器组构造

构造步骤：

1. 在 Mel 尺度上等距立 $M+2$ 个中心点（含左右边界）

1. 将中心点转回线性频率，对应到 FFT bin 索引

1. 每个滤波器是三角窗，相邻滤波器重叠

```python
import librosa
import numpy as np
import matplotlib.pyplot as plt

def mel_spectrogram(audio, sr=22050, n_fft=1024, hop=256, n_mels=80):
    """计算 Log-Mel 频谱图"""
    # 1. 计算功率谱
    S = np.abs(librosa.stft(audio, n_fft=n_fft, hop_length=hop))**2
    # 2. 应用 Mel 滤波器组
    mel_basis = librosa.filters.mel(sr=sr, n_fft=n_fft, n_mels=n_mels,
                                     fmin=0, fmax=sr//2)
    mel_spec = mel_basis @ S  # [n_mels, T]
    # 3. 取对数（加 floor 避免 log(0)）
    log_mel = np.log(np.maximum(mel_spec, 1e-5))
    return log_mel

# 可视化
# plt.imshow(log_mel, aspect='auto', origin='lower')
# plt.colorbar(); plt.xlabel('Frame'); plt.ylabel('Mel Band')
```

> [!important]
> 
> **思辨：Mel 滤波器组的信息损失。** Mel 频谱图是**有损压缩**：513 个线性频率 bin 压缩到 80 个 Mel band，且**相位信息完全丢失**。这意味着声码器必须从 80 维输入中「幻视」出 513 维幅度 + 513 维相位——这是声码器任务困难的根源。SoundStream 的端到端学习编码器正是为了绕过这个损失。

## 3. 相位谱的重要性

相位 $\phi(m,k)$ 携带了音频的**时间细节**信息。没有相位，仅有幅度谱的 Griffin-Lim 重建会产生金属质感的伪影。

不同声码器处理相位的方式：

|**方法**|**相位处理**|**优劣**|
|---|---|---|
|Griffin-Lim|迭代估计相位|慢、质量差|
|HiFi-GAN|时域直接生成（隐式相位）|快、但棋盘格伪影|
|Vocos|单位圆 cos/sin 显式建模|最快、无伪影|
