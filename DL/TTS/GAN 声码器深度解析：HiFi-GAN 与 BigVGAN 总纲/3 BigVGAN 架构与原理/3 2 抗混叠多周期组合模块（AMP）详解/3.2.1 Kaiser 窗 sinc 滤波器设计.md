## 0. 定位

> AMP 中 Kaiser 窗低通滤波器的设计原理、参数选择与 PyTorch 实现

---

## 1. 理想低通滤波器

理想低通滤波器的脉冲响应为 sinc 函数：

$$h_{\text{ideal}}[n] = 2f_c \cdot \text{sinc}(2f_c n)$$

其中 $f_c$ 是归一化截止频率。

---

## 2. Kaiser 窗加权

无限长 sinc 必须截断，截断会产生 **Gibbs 现象**（旁瓣振荡）。Kaiser 窗以可调参数 $\beta$ 控制主瓣宽度和旁瓣衰减的权衡：

$$h[n] = h_{\text{ideal}}[n] \cdot w_{\text{Kaiser}}[n; \beta]$$

|$\beta$|**旁瓣衰减**|**主瓣宽度**|**用途**|
|---|---|---|---|
|**14**|**高**|**宽**|**BigVGAN 选择，低旁瓣优先**|

---

## 3. PyTorch 实现

```python
import torch
import numpy as np
from scipy.signal import kaiser

def design_lowpass_filter(cutoff_freq=0.25, kernel_size=12, beta=14.0):
    """设计 Kaiser 窗 sinc 低通滤波器
    
    Args:
        cutoff_freq: 归一化截止频率 (0.25 = Nyquist/2)
        kernel_size: 滤波器长度
        beta: Kaiser 窗参数
    Returns:
        [kernel_size] 的滤波器核
    """
    n = torch.arange(kernel_size) - (kernel_size - 1) / 2
    # sinc 核
    h = 2 * cutoff_freq * torch.sinc(2 * cutoff_freq * n)
    # Kaiser 窗
    w = torch.tensor(kaiser(kernel_size, beta), dtype=torch.float32)
    h = h * w
    return h / h.sum()  # 归一化保证直流增益为 1
```

> [!important]
> 
> **为什么选 $beta=14$？** 高 $\beta$ 意味着更强的旁瓣抑制，代价是截止带更宽（过渡带更大）。对于声码器场景，抑制混叠伪影比保留截止频率附近的细节更重要，因此选择较大的 $beta$。
