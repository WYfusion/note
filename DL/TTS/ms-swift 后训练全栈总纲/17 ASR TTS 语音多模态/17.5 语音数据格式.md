## 0. 定位

> 一句话：5 套语音数据格式（ASR / TTS / Speech LLM / 偏好 / 强化学习）的字段约定与示例。

## 1. ASR 数据

```json
{
  "audio": "/data/asr/zh/clip_0001.wav",
  "transcript": "今天北京天气怎么样",
  "language": "zh",
  "timestamp": [[0.0, 0.5, "今天"], [0.5, 1.2, "北京"], [1.2, 2.0, "天气怎么样"]],
  "speaker": "spk_023",
  "domain": "daily",
  "noise_level": "clean",
  "sampling_rate": 16000
}
```

**关键字段**：audio / transcript / language / timestamp / speaker / domain / noise_level / sampling_rate。

## 2. TTS 数据

```json
{
  "text": "欢迎使用 Notion AI",
  "audio": "/data/tts/spk001/0042.wav",
  "speaker_id": "spk001",
  "emotion": "neutral",
  "style": "news",
  "prosody": {"pause": [3], "stress": [5]},
  "language": "zh",
  "sampling_rate": 24000
}
```

**关键字段**：text / audio / speaker_id / emotion / style / prosody / language / sampling_rate。

## 3. Speech LLM 数据（ms-swift 主格式）

```json
{
  "messages": [
    {"role": "system", "content": "你是一个语音助手。"},
    {"role": "user", "content": "<audio>请帮我总结这段录音</audio>"},
    {"role": "assistant", "content": "录音中讨论了 Q3 营收增长 23%，主要驱动来自……"}

```

## 6. 校验脚本

```python
import json, librosa, os
from pathlib import Path

REQUIRED_KEYS = {'messages', 'audios'}
ALLOWED_SR = {16000, 24000, 48000}

def validate(jsonl_path):
    bad, total = 0, 0
    for ln, line in enumerate(open(jsonl_path), 1):
        total += 1
        try:
            row = json.loads(line)
        except json.JSONDecodeError as e:
            print(f'[L{ln}] JSON 解析失败: {e}')
            bad += 1
            continue
        # 必填
        miss = REQUIRED_KEYS - set(row)
        if miss:
            print(f'[L{ln}] 缺失字段 {miss}')
            bad += 1
            continue
        # 占位符与文件数一致
        n_tokens = sum(m.get('content', '').count('<audio>') for m in row['messages'])
        if n_tokens != len(row['audios']):
            print(f'[L{ln}] <audio> 占位符 {n_tokens} 个，audios 列表 {len(row["audios"])} 个')
            bad += 1
            continue
        # 文件存在 + 采样率
        for p in row['audios']:
            if not Path(p).exists():
                print(f'[L{ln}] 音频不存在 {p}')
                bad += 1
                break
            sr = librosa.get_samplerate(p)
            if sr not in ALLOWED_SR:
                print(f'[L{ln}] 异常采样率 {sr} ({p})')
                bad += 1
                break
    print(f'总计 {total} 行 / 异常 {bad} 行 / 通过率 {(total-bad)/total:.2%}')

if __name__ == '__main__':
    validate('speech-llm.jsonl')
```

> [!important]
> 采样率不一致是最常见的灾难性错误，训练前用脚本统一。
> - speaker / language / domain 字段缺失会让后续的分层评测无法做。
> - 多模态数据中 **<audio> 占位符与 audios 列表必须一一对应**，错位会引起token 错位。
