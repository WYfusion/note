## 0. 定位

> 一句话：明确 Genshin 台词 ASR 的 3 种落地形态与各自需求差异。

---

## 1. 三种产品形态

![[2.1 业务场景与产品形态 - 1. 三种产品形态.excalidraw|800]]

|形态|输入时长|延迟要求|必需字段|部署形式|
|---|---|---|---|---|
|1 实时转写|chunk ≤ 5 s|< 800 ms 首词|transcription, speaker|vLLM 流式|
|2 字幕生成|整集 ≤ 30 s|无严格|transcription, speaker, emotion (可选)|vLLM 批量|
|3 全文索引|5–20 s|无|transcription, tags (可选)|离线 one-shot|

---

## 2. 与另两种方案的折中

> [!important]
> 
> **优先级**：先做形态 2（字幕生成）证明价值 → 迭代 1（实时转写）补充延迟优化 → 最后补全 3（索引）。别一上来直接冲实时转写，实时化是工程坑最多的组件。

---

## 3. 输出模板定错

```json
{
  "transcription": "哦呵呵呵，这可是往生堂的大生意呢。",
  "speaker": "胡桃",
  "emotion": "兴奋",
  "language": "zh"
}
```

> [!important]
> 
> **误区**：「让模型自由写 markdown」—— 错。下游解析会炸，必须锁死 JSON Schema 并在 prompt 中示例。
