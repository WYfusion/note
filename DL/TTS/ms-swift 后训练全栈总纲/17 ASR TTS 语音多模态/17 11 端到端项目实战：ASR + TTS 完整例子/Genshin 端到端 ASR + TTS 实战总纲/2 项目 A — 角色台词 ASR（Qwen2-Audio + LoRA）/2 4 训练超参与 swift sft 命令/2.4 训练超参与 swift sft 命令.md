## 0. 定位

> 一句话：给出 Genshin ASR 项目一键训练脚本，逐参解释其选型依据。

---

## 1. 主训练脚本

```bash
#!/usr/bin/env bash
# scripts/train_asr.sh
set -euo pipefail

NPROC_PER_NODE=4 \
CUDA_VISIBLE_DEVICES=0,1,2,3 \
swift sft \
  --model Qwen/Qwen2-Audio-7B-Instruct \
  --dataset data/genshin_asr_train.jsonl \
            speech_asr/speech_asr_aishell1_trainsets:validation#3000 \
  --split_dataset_ratio 0.02 \
  --train_type lora \
  --lora_rank 16 \
  --lora_alpha 32 \
  --target_modules all-linear \
  --freeze_vit true \
  --torch_dtype bfloat16 \
  --attn_impl flash_attn \
  --num_train_epochs 2 \
  --per_device_train_batch_size 1 \
  --gradient_accumulation_steps 8 \
  --learning_rate 1e-4 \
  --lr_scheduler_type cosine \
  --warmup_ratio 0.05 \
  --max_length 2048 \
  --eval_steps 200 \
  --save_steps 200 \
  --save_total_limit 5 \
  --logging_steps 10 \
  --output_dir out/genshin-asr-qwen2audio \
  --report_to wandb \
  --seed 42
```

---

## 2. 超参选型依据

|参数|取值|依据|
|---|---|---|
|lora_rank|16|中等任务复杂度，<8 欠拟合，>32 迭代增益不明显|
|lora_alpha|32|alpha/rank=2 为常用缩放倍|
|lr|1e-4|LoRA 经验值，全参该用 1e-5|
|per_device_bs|1|音频序列长，单卡 80G 只摆下 1 样本|
|grad_accum|8|有效 BS = 1×8×4 = 32|
|epochs|2|超过 3 在领域外 dev 上表现下降|
|warmup_ratio|0.05|多模态训练需较长 warmup 防梯度爆炸|

## 3. 烟囱测试

```bash
# scripts/smoke_test.sh
swift sft \
  --model Qwen/Qwen2-Audio-7B-Instruct \
  --dataset data/genshin_asr_train.jsonl \
  --train_type lora --lora_rank 16 --target_modules all-linear \
  --max_steps 20 --per_device_train_batch_size 1 \
  --logging_steps 1 --output_dir /tmp/smoke
```

> [!important]
> 
> **质量门**：20 step 内 train_loss 必须从 >2.0 下降到 <1.5，否则检查 prompt / 数据路径。
