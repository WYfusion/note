## 0. 定位

> 一句话：Flash Attention 2 通过 IO-aware tiling 与反向重计算 attention，避免 $O(N^2)$ 中间矩阵落盘，训练使用一个开关即可。

---

## 1. 原理几理

标准 attention：$O = \text{softmax}(QK^T / \sqrt{d}) V$，需临时存 $N \times N$ 的 attention matrix 到 HBM。Flash Attention 采用「在快存上分块计算 + 在快存上交叉 softmax 重标准化」：

$$m_i^{new} = \max(m_i^{old}, \tilde m_i)\,, \quad \ell_i^{new} = e^{m_i^{old} - m_i^{new}} \ell_i^{old} + e^{\tilde m_i - m_i^{new}} \tilde \ell_i$$

从而使内存从 $O(N^2)$ 返为 $O(N)$，并以 IO-aware kernel 获得 2–3× 运行加速。

## 3. 开关与安装

```bash
# 1. 安装预编译 wheel
pip install flash-attn==2.6.3 --no-build-isolation

# 2. 在 swift sft 中使用
swift sft \
  --model Qwen/Qwen2-Audio-7B-Instruct \
  --attn_impl flash_attn \
  ...
```

## 4. 验证是否生效

```python
# debug/check_flash.py
import torch
from transformers import AutoModelForCausalLM

m = AutoModelForCausalLM.from_pretrained(
    'Qwen/Qwen2-Audio-7B-Instruct',
    torch_dtype=torch.bfloat16,
    attn_implementation='flash_attention_2',
).cuda()
print('attn impl:', m.config._attn_implementation)
assert m.config._attn_implementation == 'flash_attention_2'
```

## 5. 常见坍坑

> [!important]
> 
> 1. **CUDA 与 wheel 不匹配** → 请明确 `nvcc --version`，选对应 wheel。 
> 
> 1. **fp32 训练会报错** → Flash Attention 2 仅支持 fp16 / bf16。 
> 
> 1. **变长序列必须同时启 packing** → 否则 padding token 会参与 attention 计算。
