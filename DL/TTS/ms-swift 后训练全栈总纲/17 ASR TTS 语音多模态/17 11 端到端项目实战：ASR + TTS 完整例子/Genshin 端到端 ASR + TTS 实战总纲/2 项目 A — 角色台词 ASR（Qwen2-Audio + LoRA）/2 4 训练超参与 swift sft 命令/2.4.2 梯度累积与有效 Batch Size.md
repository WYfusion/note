## 0. 定位

> 一句话：梯度累积（gradient accumulation）让你在显存不墟的情况下达到任意大的有效 batch。

---

## 1. 与超参的数学关系

设单卡 micro batch size $b$，梯度累积步数 $g$，卡数 $n$，有效 batch size：

$$\text{BS}_{eff} = b \cdot g \cdot n$$

设训练 $T$ tokens、epochs $E$、序列长 $L$，有效 step 数：

$$N_{step} = \frac{T \cdot E}{\text{BS}_{eff} \cdot L}$$

---

## 2. 选型准则

> [!important]
> 
> **准则 1**：LoRA 下，$\text{BS}_{eff} \in [16, 64]$ 为佳。
> 
> **准则 2**：多模态训练 $b=1$ 是硬限，调 $g$ 到 $\text{BS}_{eff}$ 达标。
> 
> **准则 3**：学习率需随 $\text{BS}_{eff}$ 平方根调整：$lr \propto \sqrt{\text{BS}_{eff}}$。

---

## 3. 手动实现参考

```python
# train/manual_grad_accum.py
import torch
from torch.optim import AdamW

MODEL = ...   # 训练中的 model (LoRA)
LOADER = ... # DataLoader化训练集
OPT = AdamW(MODEL.parameters(), lr=1e-4)
GRAD_ACCUM = 8
SCALER = torch.amp.GradScaler('cuda')

MODEL.train()
for step, batch in enumerate(LOADER):
    with torch.amp.autocast('cuda', dtype=torch.bfloat16):
        loss = MODEL(**batch).loss / GRAD_ACCUM
    SCALER.scale(loss).backward()
    if (step + 1) % GRAD_ACCUM == 0:
        SCALER.unscale_(OPT)
        torch.nn.utils.clip_grad_norm_(MODEL.parameters(), 1.0)
        SCALER.step(OPT)
        SCALER.update()
        OPT.zero_grad(set_to_none=True)
```

## 4. 质量验证

> [!important]
> 
> 梯度累积是否等价于「真大 batch」？ 
> 
> 理论上是的，但实际存在三个差异点： 
> 
> 1. **BatchNorm** 表现不同（LLM 多为 LayerNorm，影响小）。 
> 
> 1. **随机 Dropout** 取样粒度不同。 
> 
> 1. **远程 sync 频率减少**，反而获得 5~15% 训练加速。
