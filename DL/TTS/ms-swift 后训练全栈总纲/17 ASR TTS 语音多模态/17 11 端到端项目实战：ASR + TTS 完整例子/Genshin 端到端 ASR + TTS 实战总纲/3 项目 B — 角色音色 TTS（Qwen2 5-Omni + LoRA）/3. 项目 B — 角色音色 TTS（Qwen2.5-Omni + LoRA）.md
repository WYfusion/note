## 0. 定位

> 一句话：基于 **Qwen2.5-Omni-7B**（原生 text↔speech 双向 Omni LLM）在 Genshin 单角色 + 风格标注子集上做 LoRA SFT，目标是「指定角色 + 情感 + 语速」三维 prompt 控制下的高自然度合成。

---

## 1. 任务定义

|维度|规格|
|---|---|
|**输入**|system+user 文本：风格、语速、文本内容|
|**输出**|音频 token 流（24 kHz，单声道）|
|**风格类别**|正式 / 亲切 / 兴奋（三选一）|
|**语速控制**|0.8 / 1.0 / 1.2 三档|
|**目标**|WER（ASR 反推）<5%、SECS>0.75、MOS≥4.0|

---

## 2. 为什么用 Qwen2.5-Omni 而不是 CosyVoice

![[3. 项目 B — 角色音色 TTS（Qwen2.5-Omni + LoRA） - 2. 为什么用 Qwen2.5-Omni 而不是 CosyVoice.excalidraw|800]]

| 维度     | Qwen2.5-Omni      | CosyVoice 2                                          |
| ------ | ----------------- | ---------------------------------------------------- |
| 训练框架   | **ms-swift 原生支持** | 官方仓库 examples/libritts/cosyvoice2，**不在 ms-swift 范围** |
| 指令式控制  | 原生 system prompt  | 需自行扩展                                                |
| 音质上限   | 中等偏上              | 更高（专业声学）                                             |
| 长文本稳定性 | 偏弱                | 更稳                                                   |
| 跨模态对话  | 原生                | 仅 TTS                                                |

> [!important]
> 
> **判断**：教学/研究/原型迭代 → Qwen2.5-Omni；生产高保真直播 → CosyVoice 2 或两者折中（Omni 控语义，Cosy 出声学）。本项目演示 Omni 全链路。
