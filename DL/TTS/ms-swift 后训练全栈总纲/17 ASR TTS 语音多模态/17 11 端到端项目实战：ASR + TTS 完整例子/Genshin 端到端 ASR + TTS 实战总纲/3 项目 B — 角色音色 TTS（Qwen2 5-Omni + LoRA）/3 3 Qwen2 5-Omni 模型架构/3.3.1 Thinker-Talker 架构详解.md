## 0. 定位

## 2. 隐状态交接

Talker 接收 Thinker 最后一层 hidden state 作为 cross-attention key/value。设 Thinker 输出 hidden state $H_T \in \mathbb{R}^{L \times d_T}$，Talker 生成 step $t$ 的概率：

$$P(a_t \mid a_{<t}, H_T) = \text{softmax}\big(\text{TalkerLayer}(a_{<t}; H_T)\big)$$

其中 $a_t$ 为音频 token。Talker 仅输出音频 token，不输出文本 token。

> [!important]
> 
> 为什么不让 Thinker 直接输出音频 token？ 
> 
> 1. 音频 token 与文本 token 的采样购买差异大。 
> 
> 1. 音频 token 闭门在一个独立 codebook，避免与文本词表原生混杂。 
> 
> 1. 以 Talker 为「考量者」可使用不同采样策略（temperature, top-k）。

---

## 3. 训练 loss 拆解

设文本损失 $L_{txt}$，音频损失 $L_{aud}$，总损失：

$$L = \lambda_t L_{txt} + \lambda_a L_{aud}$$

官方训练采用 $\lambda_t = 1, \lambda_a = 1$，为避免 Talker 折昵 Thinker，其梯度会被 clip 到 [-1, 1] 区间。
