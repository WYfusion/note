## 0. 定位

> 一句话：qwen-tts-tokenizer 是一个能运行在流式场景下的 24 kHz codec，负责「音频 ↔ token」双向编码。

---

## 1. 设计指标

|指标|值|备注|
|---|---|---|
|采样率|24 kHz|与原生语音一致|
|token rate|~25 token/s|接近人耳感知上限|
|codebook 大小|16,384|4 层 RVQ|
|首包延迟|< 200 ms|流式场景需求|
|WER反推损失|< 0.5pp|越低越好|

## 3. [[RVQ 数学原理与训练目标|RVQ]] 公式

设输入 latent $z$，逸位词典 $\{C_l\}_{l=1}^L$，逐层量化：

$$z_l = z - \sum_{l'=1}^{l-1} q_{l'}, \quad q_l = \arg\min_{c \in C_l} \|z_l - c\|^2$$

重建 $\hat z = \sum_l q_l$。损失：

$$L = \|x - \text{Dec}(\hat z)\|_1 + \beta \sum_l \|z_l - \text{sg}(q_l)\|^2 + \gamma L_{adv}$$

其中 $\text{sg}$ 为 stop-gradient，$L_{adv}$ 为多尺度判别器损失。

## 4. 代码调用示例

```python
# infer/codec_demo.py
import torch, torchaudio
from transformers import AutoFeatureExtractor, AutoModel

tok = AutoFeatureExtractor.from_pretrained('Qwen/Qwen2.5-Omni-7B', subfolder='audio_tokenizer')
codec = AutoModel.from_pretrained('Qwen/Qwen2.5-Omni-7B', subfolder='audio_tokenizer').cuda().eval()

wav, sr = torchaudio.load('test.wav')
if sr != 24_000:
    wav = torchaudio.functional.resample(wav, sr, 24_000); sr = 24_000
feat = tok(wav.mean(0).numpy(), sampling_rate=sr, return_tensors='pt').to('cuda')

with torch.inference_mode():
    code = codec.encode(**feat)        # (4, T_token)
    rec  = codec.decode(code)          # (1, T_sample)

torchaudio.save('rec.wav', rec.cpu(), sr)
print('audio token rate:', code.shape[-1] / wav.shape[-1] * sr, 'token/s')
```
