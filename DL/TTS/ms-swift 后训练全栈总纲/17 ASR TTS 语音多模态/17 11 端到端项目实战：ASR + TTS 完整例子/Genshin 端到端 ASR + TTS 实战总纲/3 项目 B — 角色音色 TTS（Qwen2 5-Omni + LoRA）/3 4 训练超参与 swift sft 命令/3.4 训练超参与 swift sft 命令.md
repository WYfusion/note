## 0. 定位

> 一句话：给出 Genshin TTS（Qwen2.5-Omni 7B）训练足本，并说明为什么需要 packing 与 expandable_segments。

---

## 1. 主训练脚本

```bash
#!/usr/bin/env bash
# scripts/train_tts.sh
set -euo pipefail

export PYTORCH_CUDA_ALLOC_CONF='expandable_segments:True'
export NCCL_P2P_DISABLE=0

NPROC_PER_NODE=4 \
CUDA_VISIBLE_DEVICES=0,1,2,3 \
swift sft \
  --model Qwen/Qwen2.5-Omni-7B \
  --model_type qwen2_5_omni \
  --dataset data/genshin_tts_train.jsonl \
            data/aishell3_train.jsonl#5000 \
  --split_dataset_ratio 0.01 \
  --train_type lora \
  --lora_rank 8 \
  --lora_alpha 32 \
  --target_modules all-linear \
  --freeze_vit true \
  --freeze_aligner false \
  --torch_dtype bfloat16 \
  --attn_impl flash_attn \
  --padding_free true \
  --packing true \
  --num_train_epochs 3 \
  --per_device_train_batch_size 1 \
  --gradient_accumulation_steps 16 \
  --learning_rate 1e-4 \
  --lr_scheduler_type cosine \
  --warmup_ratio 0.03 \
  --max_length 4096 \
  --eval_steps 200 --save_steps 200 \
  --output_dir out/genshin-tts-omni \
  --seed 42
```

---

## 2. 主要超参说明

|参数|取值|原因|
|---|---|---|
|lora_rank|**8**|TTS LoRA 一般取 ASR 的一半，防过拟合|
|lora_alpha|32|较大 alpha 补偿较小 rank|
|padding_free + packing|true|变长音频序列拼接消除 pad，吞吐+30%～50%|
|freeze_aligner|**false**|TTS 有设上结，aligner 需调|
|max_length|4096|音频 token 较长，取 ASR 的 2×|
