## 0. 定位

> 一句话：padding_free 不补 pad token，packing 拼接多个变长样本为一个序列，在音频序列 var-length 场景下吞吐提升 30~50%。

---

## 1. 传统 padding 问题

![[3.4.1 padding_free 与 packing 原理 - 1. 传统 padding 问题 - 图 01.excalidraw|800]]

Genshin TTS 训练集中序列长度分布为：$\mu = 1100$，$\sigma = 800$。传统 padding 点灯量下可能出现 60% 计算在 pad token 上，浪费 GPU。

---

## 2. packing 机制

![[3.4.1 padding_free 与 packing 原理 - 2. packing 机制 - 图 02.excalidraw|800]]

> [!important]
> 
> **关键**：使用「样本边界 attention mask」以避免跨样本注意力交叉。Flash Attention 2 原生支持 `cu_seqlens` 参数。

---

## 3. 需要同时启用的启动参

```bash
swift sft \
  --padding_free true \
  --packing true \
  --attn_impl flash_attn \
  --max_length 4096 \
  ...
```

## 4. 实现骨架

```python
# train/packing.py
"""简化版 packing dataloader，说明拼接逻辑。"""
from dataclasses import dataclass
from typing import Iterator, List
import torch

@dataclass
class PackedSample:
    input_ids: torch.Tensor
    cu_seqlens: torch.Tensor  # (n_seq+1,)
    labels: torch.Tensor

def pack(samples: List[dict], max_len: int = 4096) -> Iterator[PackedSample]:
    buf_ids: List[int] = []
    buf_labels: List[int] = []
    cu = [0]
    for s in samples:
        ids = s['input_ids']
        lbl = s['labels']
        if len(buf_ids) + len(ids) > max_len:
            yield PackedSample(
                input_ids=torch.tensor(buf_ids),
                cu_seqlens=torch.tensor(cu),
                labels=torch.tensor(buf_labels),
            )
            buf_ids, buf_labels, cu = [], [], [0]
        buf_ids.extend(ids); buf_labels.extend(lbl)
        cu.append(len(buf_ids))
    if buf_ids:
        yield PackedSample(
            input_ids=torch.tensor(buf_ids),
            cu_seqlens=torch.tensor(cu),
            labels=torch.tensor(buf_labels),
        )
```
