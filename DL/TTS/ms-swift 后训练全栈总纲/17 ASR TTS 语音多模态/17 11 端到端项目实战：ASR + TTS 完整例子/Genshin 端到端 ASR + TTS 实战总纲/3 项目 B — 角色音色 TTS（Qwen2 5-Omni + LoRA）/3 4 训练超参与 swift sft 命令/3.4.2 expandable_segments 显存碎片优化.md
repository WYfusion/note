## 0. 定位

> 一句话：PyTorch CUDA 分配器默认使用 caching allocator，在变长序列 + packing 场景下产生极多碎片；expandable_segments 允许造能 block 多次扩展，减少 OOM。

---

## 1. 问题示例

不启 expandable_segments 时，泰 4096 token batch 越到 4097 则重新申请 block，原有 4096 的 block 作为碎片锁住：

```jsx
allocated: 70 GB / 80 GB
fragmented: 8 GB
free (uninhabitable): 2 GB → OOM
```

---

## 2. 使用方式

```bash
export PYTORCH_CUDA_ALLOC_CONF='expandable_segments:True'
python train.py ...
```

或在 Python 内。

```python
import torch
torch.cuda.memory._set_allocator_settings('expandable_segments:True')
```

---

## 3. 准则与限制

> [!important]
> 
> 1. **训练 packing/padding_free 场景优先启用**，可减少 ~30% OOM 频率。 
> 
> 1. **推理场景不需**，vLLM 已有 PagedAttention。 
> 
> 1. **不要与** `**max_split_size_mb**` **同时使用**，设置互斗。

## 4. 调试脚本

```python
# debug/mem_dump.py
import torch, json, time

def snapshot(label: str):
    stats = torch.cuda.memory_stats()
    out = {
        'label': label,
        'allocated_GB': stats['allocated_bytes.all.current'] / 2**30,
        'reserved_GB':  stats['reserved_bytes.all.current'] / 2**30,
        'active_GB':    stats['active_bytes.all.current'] / 2**30,
        'inactive_split_GB': stats['inactive_split_bytes.all.current'] / 2**30,
        'oom_count': stats['num_ooms'],
    }
    print(json.dumps(out, indent=2))
    return out

if __name__ == '__main__':
    snapshot('before_train')
    # 训练 N step 后再采样
    time.sleep(60)
    snapshot('after_60s')
    torch.cuda.memory._dump_snapshot('mem.pkl')  # 可可视化
```
