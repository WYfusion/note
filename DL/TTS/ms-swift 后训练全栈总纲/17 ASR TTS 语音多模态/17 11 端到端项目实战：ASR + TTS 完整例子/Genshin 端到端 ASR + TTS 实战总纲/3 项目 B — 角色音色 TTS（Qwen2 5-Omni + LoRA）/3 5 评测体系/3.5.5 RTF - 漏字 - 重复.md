## 0. 定位

> 一句话：RTF 衡量性能；漏字、重复、截断是序列生成常见故障。

---

## 1. RTF 定义

$$\text{RTF} = \frac{T_{\text{compute}}}{T_{\text{audio}}}$$

RTF < 1 为实时，TTS 项目目标 RTF < 0.3（含首包延迟除外）。

## 2. 漏字 / 重复检测

```python
# eval/quality_check.py
import pandas as pd, jiwer
from .cer_wer import normalize_text

def check_omission_repetition(ref: str, hyp: str, lang='zh') -> dict:
    ref_n, hyp_n = normalize_text(ref, lang), normalize_text(hyp, lang)
    diff = jiwer.process_characters(ref_n, hyp_n)
    return {
        'omission_rate':  diff.deletions / max(len(ref_n), 1),
        'repetition_rate': diff.insertions / max(len(ref_n), 1),
        'truncated': len(hyp_n) < len(ref_n) * 0.6,
    }
```

## 4. RTF 测试脚本

```python
# eval/rtf.py
import asyncio, time, pathlib, soundfile as sf, websockets, json

async def measure_rtf(uri, requests):
    rtfs = []
    async with websockets.connect(uri) as ws:
        for req in requests:
            t0 = time.time()
            await ws.send(json.dumps(req))
            audio_bytes = bytearray()
            while True:
                try:
                    chunk = await asyncio.wait_for(ws.recv(), timeout=10)
                    audio_bytes.extend(chunk)
                except asyncio.TimeoutError:
                    break
                except websockets.ConnectionClosed:
                    break
            t1 = time.time()
            wav_path = '/tmp/rtf.wav'
            pathlib.Path(wav_path).write_bytes(bytes(audio_bytes))
            wav, sr = sf.read(wav_path)
            audio_dur = len(wav) / sr
            rtfs.append((t1 - t0) / audio_dur)
    return rtfs
```
