## 0. 本页定位

> 给出**两个可直接复现**的 ms-swift 项目骨架：
> - **项目 A：领域客服 ASR**（基于 Qwen2-Audio-7B-Instruct 做语音→文本指令微调）
> - **项目 B：可控播报 TTS**（基于 Qwen2.5-Omni-7B 做文本→语音端到端微调）
> 每个项目都包含：**数据集 → 模型 → 训练命令 → 评测**四段闭环。

> [!important]
> 
> **先认清 ms-swift 在语音里的位置**：ms-swift 训练的是**语音多模态 LLM**（输入或输出包含音频 token 的 LLM），不是 Kaldi/FunASR 这类传统声学框架。如果只追求最低 WER 的纯 ASR，建议直接用 FunASR/Whisper 微调；ms-swift 的优势是**指令式 ASR、对话式 ASR、Speech-LLM 端到端训练**。

## 1. 项目 A — 领域客服 ASR（Qwen2-Audio-7B-Instruct + LoRA SFT）

### 1.1 业务场景

金融客服热线录音转写 + 关键意图识别（一次模型调用同时输出转写文本和槽位 JSON），用于替换「ASR + NLU」两段式 pipeline。

### 1.2 数据集

|来源|类型|规模|用途|
|---|---|---|---|
|**AISHELL-1**（`speech_asr/speech_asr_aishell1_trainsets`）|开源中文 ASR|~150h|预热,缓解灾难遗忘|
|**自有客服录音**|16kHz 单声道 wav,人工转写 + 槽位标注|~80h|领域适配主力|
|**MagicData-RAMC**|开源对话语音|~30h|对话风格鲁棒性|

**数据格式**（messages + audios，参考 17.5）：

```json
{"messages": [
  {"role": "user", "content": "<audio>请转写并提取意图、产品、金额"},
  {"role": "assistant", "content": "转写：我想咨询一下基金赎回流程。\nJSON：{\"intent\":\"赎回咨询\",\"product\":\"基金\",\"amount\":null}"}
], "audios": ["data/audio/cs_0001.wav"]}
```

**预处理要点**（详见 17.6）：

- 统一 16kHz / mono / PCM16

- VAD 切分长录音（>30s 必切，FunASR `fsmn-vad`）

- 构造**指令多样性**：转写、转写+JSON、纠错、关键词提取四类 prompt 各占 25%

- train/val/test = 90/5/5，test 集严格隔离不同坐席

### 1.3 模型

- **Base**：`Qwen/Qwen2-Audio-7B-Instruct`（已预训练好 audio encoder + LLM）

- **训练范式**：冻结 audio encoder，只训 LLM 侧 LoRA（rank=16, alpha=32, target=`all-linear`）

### 1.4 训练命令

```bash
NPROC_PER_NODE=4 \
CUDA_VISIBLE_DEVICES=0,1,2,3 \
swift sft \
  --model Qwen/Qwen2-Audio-7B-Instruct \
  --dataset data/cs_asr_train.jsonl \
            speech_asr/speech_asr_aishell1_trainsets:validation \
  --split_dataset_ratio 0.02 \
  --train_type lora \
  --lora_rank 16 --lora_alpha 32 \
  --target_modules all-linear \
  --freeze_vit true \
  --torch_dtype bfloat16 \
  --attn_impl flash_attn \
  --num_train_epochs 2 \
  --per_device_train_batch_size 1 \
  --gradient_accumulation_steps 8 \
  --learning_rate 1e-4 \
  --warmup_ratio 0.05 \
  --max_length 2048 \
  --eval_steps 200 --save_steps 200 \
  --output_dir out/cs-asr-qwen2audio
```

参考：ms-swift 多模态 audio 示例脚本 `examples/train/multimodal/audio.sh`。

### 1.5 评测

|指标|工具|目标|
|---|---|---|
|**字错率 CER / 词错率 WER**|jiwer，按 test 集分坐席统计|客服域 CER < 8%，AISHELL test ≤ 6%|
|**JSON 槽位 F1**|自定义脚本（intent/product/amount 三个键）|F1 > 0.85|
|**端到端业务完成率**|下游 RAG 准确回答率|+5pp vs 旧 pipeline|
|**鲁棒性切片**|SNR<10dB / 方言 / 长录音>20s|CER 下降幅度 < 30%|

**评测命令**（用 swift 部署 + 离线脚本评 CER）：

```bash
# 1. vLLM 部署微调后的模型
swift deploy \
  --model out/cs-asr-qwen2audio/checkpoint-final \
  --infer_backend vllm --port 8000 &

# 2. 跑批量推理 → 输出预测 jsonl
python scripts/infer_asr.py \
  --endpoint http://localhost:8000/v1 \
  --testset data/cs_asr_test.jsonl \
  --out preds.jsonl

# 3. 计算 CER / 槽位 F1
python scripts/eval_asr.py preds.jsonl --ref data/cs_asr_test.jsonl
```

## 2. 项目 B — 可控播报 TTS（Qwen2.5-Omni-7B + LoRA SFT）

### 2.1 业务场景

电商直播开场播报 + 商品讲解，要求**音色稳定、可控制语速 / 情感**，并能在 prompt 中切换「正式 / 亲切 / 兴奋」三种播报风格。

> [!important]
> 
> **为什么用 Qwen2.5-Omni 而不是直接 fine-tune CosyVoice？**
> 
> - CosyVoice 的训练流程（speech tokenizer + flow matching + vocoder）**不在 ms-swift 范围内**，要用 CosyVoice 官方 `examples/libritts/cosyvoice2` 脚本。[[1]](https://github.com/FunAudioLLM/CosyVoice/issues/1132)
> 
> - ms-swift 适合训练**带音频输出能力的 Omni LLM**（Qwen2.5-Omni 原生支持 text→speech token 输出），用于「指令式可控 TTS / 语音对话」场景。[[2]](https://swift.readthedocs.io/zh-cn/v3.10/BestPractices/MLLM-Registration.html)
> 
> - **生产折中方案**：ms-swift 微调中间 LLM（语义/风格/SSML），CosyVoice 负责声学合成。本节给出端到端方案，折中方案见 17.3。

### 2.2 数据集

|来源|类型|规模|用途|
|---|---|---|---|
|**AISHELL-3**|开源中文多说话人|~85h，218 人|多说话人预热|
|**自录主播语料**|24kHz 录音棚单说话人 + 风格标注|~10h（正式 4h / 亲切 3h / 兴奋 3h）|目标音色 + 风格控制|
|**LibriTTS-R**（可选）|开源英文高质量|~585h|跨语种泛化|

**数据格式**（指令式 TTS，messages 中标注风格，assistant 端为音频 token）：

```json
{"messages": [
  {"role": "system", "content": "你是直播主播，按指定风格朗读文本。"},
  {"role": "user", "content": "风格：兴奋\n语速：1.1\n文本：欢迎来到今晚的直播间，全场五折！"},
  {"role": "assistant", "content": "<audio>"}
], "audios": ["data/host/excited_001.wav"]}
```

**预处理要点**（详见 17.7）：

- 24kHz / mono，去静音首尾 / 削峰归一（峰值 -3dB）

- **ASR 反向校验**：用 Whisper 转写录音，与原文 CER 必须 < 3%，超过则丢弃

- 文本侧规范化：数字 / 货币 / 日期统一展开（「五折」保留口语形式，「50%」展开为「百分之五十」）

- 风格标签强制三选一，禁止空标签

### 2.3 模型

- **Base**：`Qwen/Qwen2.5-Omni-7B`（原生 text↔speech 双向）

- **训练范式**：LoRA on LLM + Talker（音频生成头），冻结 audio encoder 和 vision

### 2.4 训练命令

```bash
PYTORCH_CUDA_ALLOC_CONF='expandable_segments:True' \
NPROC_PER_NODE=4 \
CUDA_VISIBLE_DEVICES=0,1,2,3 \
swift sft \
  --model Qwen/Qwen2.5-Omni-7B \
  --dataset data/host_tts_train.jsonl \
            data/aishell3_train.jsonl \
  --split_dataset_ratio 0.01 \
  --train_type lora \
  --lora_rank 8 --lora_alpha 32 \
  --target_modules all-linear \
  --freeze_vit true \
  --torch_dtype bfloat16 \
  --attn_impl flash_attn \
  --padding_free true --packing true \
  --num_train_epochs 3 \
  --per_device_train_batch_size 1 \
  --gradient_accumulation_steps 16 \
  --learning_rate 1e-4 \
  --warmup_ratio 0.03 \
  --max_length 4096 \
  --output_dir out/host-tts-omni
```

参考 ms-swift Qwen2.5-Omni 注册与训练实践。[[2]](https://swift.readthedocs.io/zh-cn/v3.10/BestPractices/MLLM-Registration.html)

### 2.5 评测

TTS 评测必须**主观 + 客观双轨**：

|维度|指标|方法|目标|
|---|---|---|---|
|**可懂度**|WER（ASR 反推）|合成 → Paraformer 转写 → 与原文比 WER|WER < 5%|
|**音色相似度**|SECS|WeSpeaker / ERes2Net 提 embedding 算余弦|SECS > 0.75|
|**自然度**|MOS（5 分制）|≥10 人盲评，每条 ≥3 评分|MOS ≥ 4.0|
|**风格可控性**|风格分类准确率|训练一个 audio 风格分类器在合成集上跑|Acc > 0.85|
|**稳定性**|RTF / 漏字率 / 重复率|1000 条压测脚本|RTF<0.3，漏字<1%|

**评测脚本骨架**：

```bash
# 1. 离线合成测试集
swift infer \
  --model out/host-tts-omni/checkpoint-final \
  --infer_backend pt \
  --val_dataset data/host_tts_test.jsonl \
  --result_path syn/

# 2. WER（ASR 反推）
python scripts/eval_tts_wer.py syn/ --ref data/host_tts_test.jsonl

# 3. SECS（音色）
python scripts/eval_secs.py syn/ --ref data/host_ref/

# 4. 主观 MOS：导出到 webMUSHRA 平台收集人评
python scripts/build_mushra.py syn/ --out mushra_session/
```

## 3. 复现 Checklist

- [ ] 环境：`ms-swift>=3.9`，`transformers==4.48.*`（Qwen2-Audio）/ `transformers==4.57.*`（Qwen2.5-Omni），`flash-attn`，`vllm`

- [ ] 数据：jsonl 通过 17.5 的 schema 校验脚本（`REQUIRED_KEYS={'messages','audios'}`）

- [ ] 训练：先用 100 条小样本跑通 `--max_steps 20` 烟囱测试，再放大

- [ ] 评测：**train/val/test 严格不重叠**（按说话人 / 坐席划分，不能按 utterance 随机切）

- [ ] 部署：ASR 走 vLLM，TTS 走原生 PyTorch（音频生成 vLLM 支持有限）

- [ ] 灰度：固定声学层只切 LLM 5–10% 流量，回流 raw audio 监控（详见 17.10）
