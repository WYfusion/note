## 3. 案例：智能客服参考实现

```python
# 客服 pipeline 伪代码
import asyncio
from funasr import AutoModel as ASRModel
from openai import OpenAI

asr = ASRModel(model='paraformer-zh')
llm = OpenAI(base_url='http://localhost:8000/v1', api_key='dummy')

async def serve(audio_stream, tts_client):
    async for audio_chunk in audio_stream:
        # 1. ASR
        text = asr.generate(input=audio_chunk, hotword='公司名 产品A 产品B')[0]['text']
        if not text.strip():
            continue
        # 2. LLM (ms-swift deployed)
        resp = llm.chat.completions.create(
            model='customer-service-qwen',
            messages=[
                {'role': 'system', 'content': '你是客服助手，回答简短，调用工单工具。'},
                {'role': 'user', 'content': text},
            ],
            tools=load_tools(),
            tool_choice='auto',
        )
        msg = resp.choices[0].message
        if msg.tool_calls:
            tool_result = await dispatch_tools(msg.tool_calls)
            answer = summarize(tool_result)
        else:
            answer = msg.content
        # 3. TTS
        await tts_client.synth_stream(answer)
```

## 4. 落地路线（按优先级）

|优先级|路线|适用|
|---|---|---|
|**工业稳定优先**|VAD → ASR → LLM(ms-swift SFT/DPO/GRPO) → TTS → 流式部署|客服 / 教育 / 医疗|
|**体验自然优先**|Speech LLM/Omni → ms-swift SFT → 端到端部署|个性化助手 / 共情对话|
|**成本可控优先**|专用 ASR/TTS 不动，只用 ms-swift 微调中间 LLM|已有 ASR/TTS 平台|
|**领域知识优先**|ASR 热词 / 纠错 → LLM CPT/SFT → RAG → TTS|专业术语密集|
|**推理能力优先**|Speech/Text SFT → GRPO → 业务验证器评测|音频推理 / 数学口算|

## 5. 模块切分原则

> [!important]
> 
> - **声学层（VAD / 降噪 / ASR / TTS）** 用专用模型与服务，独立扩缩容。
> 
> - **语义层（LLM / Agent / RAG）** 用 ms-swift 训出的模型，OpenAI 兼容 API。
> 
> - **接口层（gRPC / WebSocket）** 流式打通声学层与语义层。
> 
> - **数据回流层** 把音频 + ASR 文本 + LLM 回复 + 标注一并落库，驱动迭代。

## 6. 流式 vs 非流式

|组件|非流式|流式|
|---|---|---|
|LLM|等 ASR 完整|partial transcript 触发预生成 + 修正|
|典型延迟|1-3s|200-600ms|
