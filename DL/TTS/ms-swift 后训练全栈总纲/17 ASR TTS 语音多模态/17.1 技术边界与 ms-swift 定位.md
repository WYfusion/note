## 0. 定位

> 一句话：ASR / TTS / Speech LLM 是三件事。ms-swift 的主战场是 **Speech LLM** 的 SFT / 偏好对齐 / GRPO，而不是底层声学。

---

## 1. 三类任务的边界

|类别|I/O|核心问题|典型框架|
|---|---|---|---|
|**ASR**|Speech → Text|声学建模 + 语言建模|FunASR / Whisper / Paraformer|
|**TTS**|Text → Speech|声学合成 + 声码器|CosyVoice / VITS / FastSpeech|
|**Speech LLM**|Speech/Text/Image/Video → Text/Speech|语音理解 + 多模态推理 + 对话|Qwen2-Audio / Qwen-Omni|

## 2. ms-swift 适合什么

> [!important]
> 
> **适合训练**：
> 
> - 语音指令跟随（Speech Instruction Following）
> 
> - 音频问答（Audio QA）
> 
> - 语音对话（Voice Chat）
> 
> - 语音情绪 / 事件理解
> 
> - 多模态语音 Agent
> 
> - Speech / Text 混合 SFT
> 
> - Speech / Text 偏好对齐（DPO / SimPO）
> 
> - Speech / Text GRPO（可验证 reward）

## 3. ms-swift 不适合什么

> [!important]
> 
> **不适合单独承担**：
> 
> - 超低延迟流式 ASR 内核
> 
> - 高保真 TTS 声码器训练
> 
> - 大规模音色克隆
> 
> - 说话人分离专用建模
> 
> - 声学前端算法优化（VAD / 降噪 / 回声消除）

## 5. 何时上端到端 Speech LLM

|诉求|选 Speech LLM / Omni|选级联|
|---|---|---|
|语气情绪保留|✅|❌（ASR 丢失）|
|多模态融合（看图听声）|✅|组合困难|
|评测可解释|❌|✅（每段独立评测）|
|低延迟流式|需 streaming Omni|专用 ASR/TTS 已成熟|
|工业稳定 / 易追责|❌|✅|

## 6. ms-swift 中的语音入口（命令模板）

```bash
# Speech LLM SFT（以 Qwen2-Audio 为例）
swift sft \
    --model Qwen/Qwen2-Audio-7B-Instruct \
    --dataset speech-instruct.jsonl \
    --train_type lora --lora_rank 16 \
    --target_modules all-linear \
    --freeze_vit true \
    --freeze_aligner false \
    --max_length 4096 \
    --output_dir out/qwen2-audio-sft
```

关键参数与 MLLM 章节一致：`freeze_vit/aligner/llm` 决定哪几部分参与训练；语音模型把 audio encoder 当作 vit。
