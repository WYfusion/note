## 0. 定位

> 一句话：本页梳理 ms-swift 对 **300+ 多模态大模型（MLLM）**的训练支持，重点在「**三部分独立控制**」（vit / aligner / llm）与**多模态 packing**等工程点。

---

## 1. MLLM 三部件架构

![[6. 多模态训练（MLLM） - 1. MLLM 三部件架构.excalidraw|800]]

|部件|职责|例子|
|---|---|---|
|**Visual Encoder (ViT)**|提取图像特征 patch embedding|SigLIP、Qwen2-VL ViT、InternViT|
|**Aligner**|将视觉特征映射到 LLM 词嵌入空间|MLP、Q-Former、Resampler|
|**LLM**|接收 vision token + text token 生成|Qwen3、LLaMA3、InternLM|

## 2. 冗余与三部件独立控制

ms-swift 可独立冻结 / 打开三部件训练：

```bash
swift sft --model Qwen/Qwen2-VL-7B-Instruct \
    --dataset my_mllm_data.jsonl \
    --train_type lora \
    --target_modules all-linear \
    --freeze_vit true \
    --freeze_aligner false \
    --freeze_llm false \
    --vit_lr 2e-6 \
    --aligner_lr 1e-5 \
    --learning_rate 1e-4
```

> [!important]
> 
> **工程默认推荐**：数据少时**冻结 ViT、只调 aligner + llm**；数据充足且需要领域视觉理解时才解冻 ViT，并应用**显著低于 LLM 的学习率**（一般 1/50）。

## 3. 多模态数据格式

```json
{
  "messages": [
    {"role": "user", "content": "<image>这是什么?"},
    {"role": "assistant", "content": "这是一只黑猫..."}
  ],
  "images": ["./imgs/cat.jpg"]
}
```

**关键点**：

- `<image>` / `<video>` / `<audio>` 占位符与 `images` / `videos` / `audios` 数组**一一对应**

- 路径应**稳定可访问**（本地绝对路径或 OSS URL）

- 同一样本可多图、多轮交错出现

---

## 4. 多模态 Packing

多模态训练的是「**不等长 vision token + text token**」。开启 packing 可**提升 30%-50% 吐吐量**，但需要使用 FlashAttention：

```bash
swift sft --model Qwen/Qwen2-VL-7B-Instruct \
    --packing true \
    --attn_impl flash_attn \
    --max_length 8192 \
    ...
```

> [!important]
> 
> **Packing 避坍**：不同样本的 vision token 不能跨样本注意，必须用带 attention mask 的 FlashAttention（否则引起 cross-sample leak）。

---

## 5. 多模态偏好 / GRPO

ms-swift 同样支持多模态的 DPO、GRPO 训练。典型应用：

- **多模态 DPO**：OCR 偏好、描述风格对齐

- **多模态 GRPO**：几何推理、图表问答、空间定位（需可验证奖励）

```bash
# 多模态 DPO
swift rlhf --rlhf_type dpo \
    --model Qwen/Qwen2-VL-7B-Instruct \
    --dataset my_vl_dpo.jsonl \
    --train_type lora
```

## 6. 常见误区

> [!important]
> 
> **误区 1**：训练时路径与推理时路径不一致 → 推理报错。**建议**：训练使用绝对路径或服务化路径。

> [!important]
> 
> **误区 2**：冻结 ViT 后仍将 vit_lr 设为非 0。**建议**：冻结时会被自动忽略，但明确设 `--vit_lr 0` 避免误会。

> [!important]
> 
> **误区 3**：高分辨率图像不限制尺寸 → vision token 爆炸。**建议**：设置 `--max_pixels` 控制处理后像素数。
