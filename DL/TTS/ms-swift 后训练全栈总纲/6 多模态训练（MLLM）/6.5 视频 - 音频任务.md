## 0. 定位

> 一句话：视频 / 音频 SFT 与图像类似，但需额外控制**采样频率与总 token 量**以避免 OOM。

---

## 1. 视频 token 预算

以 Qwen2.5-VL 为例，每帧经 vit + merger 后约占 196-256 token：

|视频|采样帧|token 量估计|上下文需求|
|---|---|---|---|
|10s，采 16 帧|16|~4k|OK|
|10min，采 64 帧|64|~16k|限制输出|

## 2. 视频 SFT 命令

```bash
swift sft \
    --model Qwen/Qwen2.5-VL-7B-Instruct \
    --dataset my-video-data \
    --train_type lora \
    --freeze_vit true --freeze_aligner true \
    --video_max_frames 16 \
    --video_min_frames 4 \
    --max_pixels 360000 \          # 限制单帧像素
    --max_length 8192 \
    --gradient_checkpointing true \
    --output_dir out/qwen-vl-video
```

## 3. Audio SFT

Qwen2-Audio / Qwen2.5-Omni 等多模态音频模型在 ms-swift 中训练路径：

```bash
swift sft \
    --model Qwen/Qwen2-Audio-7B-Instruct \
    --dataset my-asr-data \
    --train_type lora \
    --freeze_vit true --freeze_aligner true \   # audio_encoder 在这里也被冻结
    --audio_sample_rate 16000 \
    --max_length 4096 \
    --learning_rate 1e-4 \
    --output_dir out/qwen-audio-asr
```

## 4. Omni（多模态联合）

```jsx
# 同一个样本包含多种模态
{"messages":[
  {"role":"user",
   "content":"<image><audio>请根据这张图与这段讲解回答: 照片中的人在说什么?"},
  {"role":"assistant",
   "content":"他说 ..."}
], "images":["/data/img.png"], "audios":["/data/voice.wav"]}
```

```bash
swift sft \
    --model Qwen/Qwen2.5-Omni-7B \
    --dataset my-omni-data \
    --template qwen2_5_omni \
    --train_type lora \
    --target_modules all-linear \
    --freeze_vit true --freeze_aligner true \
    --max_pixels 360000 \
    --video_max_frames 16 \
    --audio_sample_rate 16000 \
    --max_length 8192 \
    --output_dir out/qwen-omni-sft
```

## 5. 资源控制实战

|瓶颈|解决办法|
|---|---|
|显存 OOM|开 `gradient_checkpointing`、降 batch、降 LoRA rank|
|多路模态杂烩|分阶段：先单模态调通，再混合训|

## 6. 完整示例：视频 VQA SFT

```bash
NPROC_PER_NODE=4 swift sft \
    --model Qwen/Qwen2.5-VL-7B-Instruct \
    --dataset my-video-qa \
    --train_type lora \
    --lora_rank 16 \
    --target_modules all-linear \
    --freeze_vit true --freeze_aligner true \
    --video_max_frames 16 \
    --video_min_frames 4 \
    --max_pixels 360000 \
    --max_length 8192 \
    --gradient_checkpointing true \
    --learning_rate 1e-4 \
    --output_dir out/qwen-vl-video-sft
```

## 7. 踩坑提示

> [!important]
> 
> - **视频帧采样均匀**：默认 ms-swift 采用均匀采样，关键事件可能漏掉，必要时自行预处理为关键帧。
> 
> - **音频采样率**：训练时与推理时必须一致（默认 16000）。
> 
> - **音频长度**：超过 30s 的音频建议切段，否则 token 数爆炸。
> 
> - **Omni 多模态混合训练**：建议先单模态各自跑通再混合，否则定位 bug 困难。
