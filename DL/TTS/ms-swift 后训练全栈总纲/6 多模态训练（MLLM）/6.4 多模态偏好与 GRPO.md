## 0. 定位

> 一句话：MLLM 同样可做 DPO / SimPO / GRPO，仅需**在原有偏好数据中附加 images / videos**。

---

## 1. 多模态 DPO

```jsx
# 数据格式
{"messages": [{"role":"user","content":"<image>这是什么鱼?"}],
 "chosen":   [{"role":"assistant","content":"这是一条 koi （锦鲪）..."}],
 "rejected": [{"role":"assistant","content":"这是一条金鱼。"}],
 "images":   ["/data/koi.jpg"]}
```

```bash
swift rlhf --rlhf_type dpo \
    --model Qwen/Qwen2.5-VL-7B-Instruct \
    --dataset my-vlm-dpo \
    --train_type lora \
    --target_modules all-linear \
    --freeze_vit true --freeze_aligner true \
    --beta 0.1 \
    --learning_rate 5e-6 \
    --max_length 4096 \
    --output_dir out/qwen-vl-dpo
```

## 2. 多模态 GRPO

```jsx
# 用于多模态推理的可验证数据
{"messages":[{"role":"user","content":"<image>表格中总计多少?"}],
 "solution": "42500",
 "images":["/data/finance_table.png"]}
```

```bash
swift rlhf --rlhf_type grpo \
    --model Qwen/Qwen2.5-VL-7B-Instruct \
    --dataset my-vlm-grpo \
    --reward_funcs accuracy format \
    --num_generations 4 \         # 比纯文本少，因为多模态 rollout 贵
    --max_completion_length 1024 \
    --temperature 1.0 \
    --train_type lora \
    --freeze_vit true --freeze_aligner true \
    --use_vllm true \
    --output_dir out/qwen-vl-grpo
```

## 3. 多模态 GRPO 的难点

|难点|原因|应对|
|---|---|---|
|vLLM 不支持某些 VL|需要适配|退连 transformers 后端|
|多图 / 视频 OOM|token 炸裂|限制帧数 / 像素|

## 4. 多模态 RM

训练多模态 RM 与文本 RM 一致，仅需提供 chosen/rejected 成对多模态数据：

```bash
swift rlhf --rlhf_type rm \
    --model Qwen/Qwen2.5-VL-7B-Instruct \
    --dataset my-vlm-pair \
    --train_type lora \
    --target_modules all-linear \
    --freeze_vit true --freeze_aligner true \
    --output_dir out/qwen-vl-rm
```

## 5. 与文本 RLHF 的差异调参

```python
# 多模态推荐起点
# - num_generations: 8 → 4。4 以内防 OOM
# - max_completion_length: 4096 → 1024-2048
# - lora_rank: 8 → 16，多模态参数量大，适当提高
# - learning_rate: 保持与文本一致
# - freeze_vit / aligner: 默认 true
```

## 6. 完整示例：多模态推理 GRPO（文档 VQA）

```bash
NPROC_PER_NODE=4 swift rlhf \
    --rlhf_type grpo \
    --model Qwen/Qwen2.5-VL-7B-Instruct \
    --dataset my-doc-vqa-data \
    --reward_funcs accuracy format \
    --reward_weights 1.0 0.3 \
    --num_generations 4 \
    --max_completion_length 1024 \
    --temperature 1.0 \
    --beta 0.0 \
    --train_type lora \
    --lora_rank 16 \
    --target_modules all-linear \
    --freeze_vit true --freeze_aligner true \
    --use_vllm true \
    --vllm_gpu_memory_utilization 0.4 \
    --learning_rate 1e-6 \
    --output_dir out/qwen-vl-doc-grpo
```

## 7. 踩坑提示

> [!important]
> 
> - **多模态 DPO 仅学文本偏好**：冻 vit 并不会限制表现，但部分 "看不准" 的问题仍需靠 SFT 解决。
> 
> - **负例质量关键**：负例太差（明显胡言乱语）会让 DPO 变成拒异常，学不到偏好。
> 
> - **vLLM rollout** ：部分 VL 模型 vLLM 还未完全支持，需要检查版本。
