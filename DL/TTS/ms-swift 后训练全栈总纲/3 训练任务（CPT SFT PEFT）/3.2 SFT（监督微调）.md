## 0. 定位

> 一句话：SFT 让模型从"补全器"变成"指令遵循者"，**只在 assistant 回答处计算 loss**。

---

## 1. SFT 与 CPT 对比

|维度|CPT|SFT|
|---|---|---|
|数据格式|纯文本|messages（多轮）|
|典型 LR|5e-6|1e-4 (LoRA) / 2e-5 (全参)|
|Loss 范围|所有 token|仅 assistant token|
|数据量|数十亿 token|1k - 10M 条|
|目标|知识注入|能力激发 + 指令对齐|

## 2. 损失公式

令样本对为 $(P, R)$，其中 $P$ 为 prompt token 序列，$R = (r_1, l\dots, r_T)$ 为 response：

$\mathcal{L}_{\mathrm{SFT}} = -\sum_{t=1}^{T} \log P_\theta(r_t \mid P, r_{<t})$

本质上是把 CPT 的 loss 在 prompt 段 mask 掉。

## 3. 最小可跑命令

```bash
# SFT-LoRA 单卡 7B（24GB 显存即可）
CUDA_VISIBLE_DEVICES=0 swift sft \
    --model Qwen/Qwen2.5-7B-Instruct \
    --dataset AI-ModelScope/alpaca-gpt4-data-zh#10000 \
              swift/self-cognition#500 \
    --train_type lora \
    --lora_rank 8 --lora_alpha 32 --target_modules all-linear \
    --num_train_epochs 2 \
    --per_device_train_batch_size 4 \
    --gradient_accumulation_steps 8 \
    --learning_rate 1e-4 \
    --warmup_ratio 0.05 \
    --eval_strategy steps --eval_steps 200 \
    --save_strategy epoch \
    --output_dir out/qwen-sft-lora
```

## 4. 完整 Python 调用 + 推理

```python
from swift.llm import TrainArguments, sft_main, InferArguments, infer_main

# === 训练 ===
sft_main(TrainArguments(
    model='Qwen/Qwen2.5-7B-Instruct',
    template='qwen',
    dataset=['AI-ModelScope/alpaca-gpt4-data-zh#10000'],
    train_type='lora',
    lora_rank=8, lora_alpha=32, target_modules='all-linear',
    num_train_epochs=2,
    per_device_train_batch_size=4,
    gradient_accumulation_steps=8,
    learning_rate=1e-4,
    warmup_ratio=0.05,
    output_dir='out/qwen-sft-lora',
))

# === 推理（加载 adapter）===
infer_main(InferArguments(
    adapters='out/qwen-sft-lora/checkpoint-final',
    infer_backend='pt',           # 或 'vllm'
    stream=True,
))
```

## 5. 关键超参选择

|超参|LoRA|全参|learning_rate|1e-4|2e-5|
|---|---|---|---|---|---|
|num_train_epochs|2-3|1-2（易过拟合）|batch_size × grad_accum|有效 32-128|有效 64-256|
|warmup_ratio|0.03-0.05|0.03|weight_decay|0.01|0.0-0.1|

## 6. SFT 数据混合配方

> [!important]
> 
> 一份能跑出像样效果的配方（适合 7B）：  
> - **通用对话**（alpaca-gpt4 / OpenHermes）：60-70%  
> - **领域数据**（你的目标场景）：20-30%  
> - **自我认知**（swift/self-cognition）：1-3%  
> - **安全 / 拒答**（少量 unsafe → safe pairs）：1-2%

```bash
swift sft --model Qwen/Qwen2.5-7B-Instruct \
    --dataset 'AI-ModelScope/alpaca-gpt4-data-zh#30000' \
              'my-finance-sft#10000' \
              'swift/self-cognition' \
    --dataset_ratio 1 1 0.05 \
    --train_type lora --lora_rank 16
```

## 7. 何时该选 LoRA / QLoRA / 全参

![[3.2 SFT（监督微调） - 7. 何时该选 LoRA QLoRA 全参.excalidraw|800]]

> [!important]
> 
> - **不要直接对 Instruct 模型大数据 SFT**：易破坏原有对齐，建议 LoRA 微调或小 LR 全参。
> 
> - **base 模型小数据 SFT** 容易冷启动失败，至少 5k-10k 高质量样本。
