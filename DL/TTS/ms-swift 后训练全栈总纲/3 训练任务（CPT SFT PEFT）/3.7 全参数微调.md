## 0. 定位

> 一句话：全参数微调 = 所有参数参与梯度更新，**训练上限最高**但显存代价最大，常与 DeepSpeed Z2/Z3 或 FSDP 配合。

---

## 1. 何时必须全参

|场景|全参 vs LoRA|大量领域 CPT (>1B token)|✅ 全参，LoRA 容量不够|
|---|---|---|---|
|SFT < 100k 样本|❌ LoRA 即可，全参易过拟合|SFT 数百万样本、同领域|✅ 全参效果明显领先|
|研究重要能力迁移（RLHF、Reasoning）|✅ 全参为主流 baseline|快速多任务 / 多领域切换|❌ LoRA adapter 体验更好|

## 2. 显存需求估算

7B 模型 bfloat16 全参训练起点需求（AdamW）：

|项|估算|权重|14 GB|
|---|---|---|---|
|梯度|14 GB|优化器 (m, v fp32)|56 GB|
|激活/中间量（bs=4、seq=4096）|10-20 GB|**合计**|**~95-105 GB**|

因此 7B 全参需要 80GB+ 单卡或多卡分片。

## 3. 多卡命令（DeepSpeed Z3）

```bash
# 4×80GB A100 全参训练 7B
NPROC_PER_NODE=4 swift sft \
    --model Qwen/Qwen2.5-7B-Instruct \
    --dataset AI-ModelScope/alpaca-gpt4-data-zh \
    --train_type full \
    --torch_dtype bfloat16 \
    --deepspeed zero3 \
    --num_train_epochs 2 \
    --per_device_train_batch_size 1 \
    --gradient_accumulation_steps 16 \
    --learning_rate 2e-5 \
    --warmup_ratio 0.03 \
    --gradient_checkpointing true \
    --attn_impl flash_attn \
    --save_strategy epoch \
    --output_dir out/qwen-full
```

## 5. 全参与 LoRA 的潜在差异

> [!important]
> 
> **辩证思考**：FLAN、OpenAssistant 等研究表明在中等规模 SFT（< 100k 条）上LoRA 与全参 benchmark 差异 < 1%。但在严谨领域任务、长上下文、跨领域迁移场景，全参仍明显领先。判断原则：_你是在"激发能力"还是"注入或重塑能力"？_

## 6. 超参参考

|超参|7B 全参 SFT|说明|
|---|---|---|
|warmup_ratio|0.03|防止初期崩坏预训练表示|
|weight_decay|0.0-0.1|领域微调 0.0；多任务联合 0.1|
|lr_scheduler_type|cosine|或 cosine_with_min_lr|

## 7. Python 调用 + 断点续训

```python
from swift.llm import TrainArguments, sft_main

sft_main(TrainArguments(
    model='Qwen/Qwen2.5-7B-Instruct',
    template='qwen',
    dataset=['AI-ModelScope/alpaca-gpt4-data-zh'],
    train_type='full',
    torch_dtype='bfloat16',
    deepspeed='zero3',
    num_train_epochs=2,
    per_device_train_batch_size=1,
    gradient_accumulation_steps=16,
    learning_rate=2e-5,
    warmup_ratio=0.03,
    gradient_checkpointing=True,
    attn_impl='flash_attn',
    save_strategy='epoch',
    save_total_limit=2,
    # 断点续训：传入以前的 checkpoint 路径
    resume_from_checkpoint='out/qwen-full/checkpoint-1000',
    output_dir='out/qwen-full',
))
```

> [!important]
> 
> - **全参训练后的 checkpoint 体积完整 = 模型参数 × 2-3**（7B → ~30-40GB/checkpoint），务必设 `--save_total_limit`。  
>     - **需要 attn_impl=flash_attn**，否则 7B 也会 OOM。  
>     - **DeepSpeed Z3 快照保存**需要所有 rank 同步，该过程占用额外内存。
