## 0. 定位

> 一句话：QLoRA = 把 base 权重以 **NF4 4-bit** 量化保存，反量化到 bf16 做前向，反向只更新 LoRA 旁路，**24GB 即可微调 65B**。

---

## 1. QLoRA 三大技术点

|技术|作用|**NF4 (4-bit NormalFloat)**|对正态分布权重 "信息论最优" 的 4bit 编码|
|---|---|---|---|
|**Double Quantization**|把量化的 scale 再量化一次，省 ~0.4 bit/参数|**Paged Optimizer**|优化器状态溢出时分页到 CPU，避免突发 OOM|

## 2. 显存收益估算

以 7B 模型为例（fp16 base ≈ 14GB）：

|模式|权重|梯度|优化器|合计估算|
|---|---|---|---|---|
|全参 fp16|14GB|14GB|56GB(AdamW)|~84GB|
|LoRA fp16|14GB|~50MB|~200MB|~15GB|
|**QLoRA 4bit**|~3.5GB|~50MB|~200MB|~5GB|

## 3. 命令模板

```bash
# 单张 24GB 显卡微调 14B 模型
swift sft \
    --model Qwen/Qwen2.5-14B-Instruct \
    --dataset AI-ModelScope/alpaca-gpt4-data-zh#10000 \
    --train_type lora \
    --quant_method bnb --quant_bits 4 \
    --bnb_4bit_quant_type nf4 \
    --bnb_4bit_use_double_quant true \
    --bnb_4bit_compute_dtype bfloat16 \
    --lora_rank 16 --lora_alpha 32 \
    --target_modules all-linear \
    --learning_rate 1e-4 \
    --gradient_checkpointing true \
    --output_dir out/qwen14b-qlora
```

## 5. NF4 量化原理（直观）

NF4 不是"等距"4-bit，而是按权重正态分布的**分位点**取 16 个值（`[-1, ..., 0, ..., +1]`），令量化误差最小。

```jsx
标准 INT4（等距）:    [-8 ... -1, 0, +1 ... +7] / 7
NF4（分位点）:        近似 [-1, -0.69, -0.52, ..., 0, ..., 0.52, 0.69, 1]
```

配合 group-wise scale（每 64 个权重共享一个 scale），既保留小数值精度也限制反量化误差。

## 6. FSDP-QLoRA：跨卡 QLoRA

标准 BitsAndBytes 不能与 FSDP/Zero3 直接组合（量化权重无法被分片），ms-swift 集成了 **FSDP-QLoRA** 解锁 70B+ 在多卡 4bit 微调：

```bash
# 4×80GB A100 微调 70B
swift sft \
    --model Qwen/Qwen2.5-72B-Instruct \
    --train_type lora \
    --quant_method bnb --quant_bits 4 \
    --bnb_4bit_quant_storage bfloat16 \
    --target_modules all-linear --lora_rank 16 \
    --gradient_checkpointing true \
    --fsdp 'full_shard auto_wrap' \
    --fsdp_config configs/fsdp_qlora.json \
    --learning_rate 1e-4 --output_dir out/qwen72b-qlora
```

关键点：`bnb_4bit_quant_storage=bfloat16` 让量化权重以 bf16 张量存储，FSDP 才能分片。

## 7. 经验与陷阱

> [!important]
> 
> QLoRA 在大多数 SFT 任务上与 LoRA 几乎等效（diff < 1%），但 **CPT 类任务** 上 4bit 误差累积明显，建议至少用 8bit 或全 fp 训练。

> [!important]
> 
> - QLoRA + DeepSpeed Z3 不兼容（用 FSDP-QLoRA 替代）。  
>     - 推理时若想保留 4bit 权重 + LoRA adapter，使用 `swift infer` 直接挂载；想合并到 fp16 base 用 `swift export --merge_lora`，会自动反量化。  
>     - bnb 量化在 CPU offload + 梯度检查点 + 流式数据三件套都开时偶发 NaN，建议至少关一项。
