## 0. 定位

> 一句话：LoRA 的几种**改良变体**，分别在初始化、表达力、长上下文、稳定性维度上改进，全部通过 `--train_type` 一键切换。

---

## 1. 变体对照

|方法|核心改动|解决什么|开关|
|---|---|---|---|
|**LoRA**|$\Delta W = BA$, $B=0$ 起步|baseline|`--train_type lora`|
|**LoRA+**|对 $B$ 用更大 LR|提升 LoRA 上限|`--lora_lr_ratio 16.0`|
|**DoRA**|分解为 "方向 + 模长" 两部分|表达力↑、收敛更稳|`--use_dora true`|
|**OLoRA**|用正交初始化|稳定性更好|`--init_lora_weights olora`|
|**PiSSA**|用 $W$ 的 SVD 主成分初始化 $A,B$|更快收敛、更高上限|`--init_lora_weights pissa`|
|**LongLoRA**|S²Attn + LoRA|长上下文（32k+）训练|`--train_type longlora`|
|**LoftQ**|量化感知初始化|QLoRA 起点更优|`--init_lora_weights loftq`|

## 2. DoRA：方向-模长分解

标准 LoRA：$W' = W + BA$

DoRA：把 $W' = m \cdot \dfrac{V}{|V|_c}$，其中 $V = W + BA$，$m$ 为可学模长向量，$|\cdot|_c$ 为列范数。

直观理解：分别学 "权重方向" 和 "权重模长"，更接近全参微调的更新模式。

```bash
swift sft --model Qwen/Qwen2.5-7B-Instruct \
    --dataset my-data --train_type lora \
    --use_dora true \
    --lora_rank 8 --lora_alpha 16 \
    --target_modules all-linear
```

## 3. PiSSA：SVD 主成分初始化

标准 LoRA $B=0$ 起步，前期 $\Delta W = 0$ 浪费 step。PiSSA：

1. 对 $W$ 做 SVD：$W = U \Sigma V^{\top}$

2. 取最大 $r$ 个奇异值，构造 $A_{init} = \sqrt{\Sigma_r}, V_r^{\top}$，$B_{init} = U_r \sqrt{\Sigma_r}$

3. 残差 $W_{res} = W - B_{init} A_{init}$ 作为冻结 base

这样 $\Delta W$ 起点就在主成分上，等价于继续训练 "主能量"。

```bash
swift sft --model Qwen/Qwen2.5-7B-Instruct \
    --dataset my-data --train_type lora \
    --init_lora_weights pissa \
    --lora_rank 16
```

## 4. LongLoRA：长上下文训练

通过 **Shifted Sparse Attention（S²Attn）** 把 attention 划块（如 4 块 8k=32k），各块内做 full attention，块间通过移位重叠交换信息：

![[3.5 进阶 PEFT（DoRA - PiSSA - LongLoRA - OLoRA） - 4. LongLoRA：长上下文训练 - 图 01.excalidraw|800]]

命令：

```bash
swift sft --model Qwen/Qwen2.5-7B-Instruct \
    --dataset long-context-dataset \
    --train_type longlora \
    --max_length 32768 \
    --rope_scaling linear \
    --rope_scaling_factor 4.0 \
    --gradient_checkpointing true
```

## 5. LoRA+：双学习率

经验发现 $B$ 矩阵需要比 $A$ 更大 LR（约 16x）才能完全发挥容量：

```bash
swift sft --train_type lora \
    --lora_rank 8 --lora_alpha 16 \
    --learning_rate 1e-4 \
    --lora_lr_ratio 16.0       # B 的 LR = 1e-4 × 16 = 1.6e-3
```

## 7. 经验小结

> [!important]
> 
> - 大多数 7B SFT 任务，标准 LoRA r=8 已经足够。
> 
> - DoRA 在小数据（< 5k 条）上提升明显，大数据基本持平。
> 
> - PiSSA 的好处主要在"前期收敛快"，最终上限与 LoRA 接近。
> 
> - LongLoRA 是"训练侧"长上下文方案；推理时仍需配合 RoPE scaling。
> 
> - LoRA+ 是几乎免费的提升（只多一个超参），值得默认尝试。
