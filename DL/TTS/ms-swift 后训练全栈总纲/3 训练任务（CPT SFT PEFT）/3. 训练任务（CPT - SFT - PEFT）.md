## 0. 定位

> 一句话：本页定义**监督学习类**三大任务的边界与选型——**继续预训练（CPT）**用于知识，**监督微调（SFT）**用于行为，**参数高效微调（PEFT）**用于资源。三者**正交**，可任意组合。

---

## 1. 三任务的核心区分

|任务|目标|损失|典型数据|典型 epoch|
|---|---|---|---|---|
|**CPT**（Continual Pre-Training）|注入领域知识、扩展上下文、新语言|所有 token 上的 next-token CE|无标签长文本|1-3|
|**SFT**（Supervised Fine-Tuning）|学习指令格式、对话能力、工具调用|**仅 assistant** token 上的 CE|messages 格式|2-5|
|**PEFT**（Parameter-Efficient Fine-Tuning）|**正交**：以上两者都可叠加 PEFT|同上，但只更新少量参数|同上|同上|

> [!important]
> 
> **关键洞察**：CPT 与 SFT 是**任务**，PEFT 是**手段**。「LoRA」不是任务，而是 SFT/CPT/DPO/GRPO 都可叠加的参数高效化技术。

---

## 2. 三种任务的数学差异（Loss Mask）

设 token 序列 $x = (x_1, x_2, \dots, x_T)$，模型预测分布为 $p_\theta$。

**CPT loss**：

$$\mathcal{L}_{\text{CPT}} = -\frac{1}{T-1} \sum_{t=1}^{T-1} \log p_\theta(x_{t+1} \mid x_{\le t})$$

**SFT loss**（带 mask $m_t \in \{0, 1\}$，仅 assistant 段为 1）：

$$\mathcal{L}_{\text{SFT}} = -\frac{1}{\sum_t m_t} \sum_{t=1}^{T-1} m_t \cdot \log p_\theta(x_{t+1} \mid x_{\le t})$$

二者的工程实现差异只是`**labels**` **中 system/user 段是否设为 -100**。

## 3. PEFT 方法选型矩阵

|方法|可训练参数比例|额外推理开销|典型场景|
|---|---|---|---|
|**LoRA**|~0.1% - 1%|0（merge 后）|通用首选|
|**QLoRA**|~0.1%|0（merge 后）|显存受限（24GB 训 70B）|
|**DoRA**|~0.1% - 1%|0（merge 后）|需略优于 LoRA 的精度|
|**LongLoRA**|~0.1%|训练时稀疏 attn|长上下文扩展|
|**GaLore**|**100%**（梯度低秩）|0|需要全参等效精度|
|**LLaMA-Pro**|新增层 100%|+若干层|完全不破坏原能力|
|**Adapter**|~1% - 5%|+若干 FFN|多任务切换|
|**Prefix Tuning**|~0.01%|+少量 KV|极低参数预算|

> [!important]
> 
> **优先级链**：先 **LoRA** → 显存不够换 **QLoRA** → 想要更高精度试 **DoRA** → 需要全参等效用 **GaLore**。其余方法仅在特定场景使用。

## 4. 三种任务的 ms-swift 命令对照

```bash
# CPT：只给文本，不区分角色
swift sft --model Qwen/Qwen3-8B \
    --dataset my_corpus.jsonl \
    --train_type lora \
    --use_chat_template false \
    --max_length 8192 \
    --learning_rate 5e-5 \
    --num_train_epochs 1

# SFT-LoRA：messages 格式，应用 chat template
swift sft --model Qwen/Qwen3-8B \
    --dataset my_sft.jsonl \
    --train_type lora \
    --target_modules all-linear \
    --learning_rate 1e-4 \
    --num_train_epochs 3

# QLoRA：低显存版 SFT
swift sft --model Qwen/Qwen3-8B \
    --dataset my_sft.jsonl \
    --train_type lora \
    --quant_method bnb --quant_bits 4 \
    --learning_rate 1e-4 \
    --gradient_checkpointing true
```

> [!important]
> 
> **CPT 的两个易错点**：(1) 必须 `--use_chat_template false`，否则 template 会强行包裹文本破坏分布；(2) learning_rate 应**显著低于 SFT**（一般 5e-5 vs 1e-4），否则容易遗忘。
