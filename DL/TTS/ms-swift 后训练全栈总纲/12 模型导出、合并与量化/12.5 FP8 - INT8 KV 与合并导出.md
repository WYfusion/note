## 0. 定位

> 一句话：FP8 适合 H100 原生加速，INT8 KV cache 减少长上下文显存；本节另补充合并与导出的完整流程。

---

## 1. FP8 二种格式

|格式|位宽|适用|
|---|---|---|
|**E5M2**|1 符号 + 5 指数 + 2 尾数|梯度|

H100 / Blackwell 原生支持 FP8 张量核心。

## 2. FP8 导出

```bash
swift export \
    --model out/qwen-sft \
    --quant_method fp8 \
    --quant_dataset my-calib \
    --quant_n_samples 256 \
    --output_dir merged/qwen-7b-fp8
```

部署：

```bash
swift deploy \
    --model merged/qwen-7b-fp8 \
    --infer_backend vllm \
    --quantization fp8 \
    --kv_cache_dtype fp8
```

## 3. INT8 KV cache

生产中上下文越长，KV cache 越费显存。用 INT8 KV 可减少 50%：

```bash
swift deploy \
    --model merged/qwen-7b-awq \
    --infer_backend vllm \
    --quantization awq \
    --kv_cache_dtype fp8 \           # vLLM 支持 fp8 KV
    --max_model_len 32768
```

LMDeploy 使用：

```bash
swift deploy \
    --infer_backend lmdeploy \
    --quant_policy 8                  # 8 = INT8 KV cache
```

## 4. LoRA 合并导出

```bash
# 单 LoRA 合并
swift export \
    --adapters out/qwen-lora \
    --merge_lora true \
    --output_dir merged/qwen-merged

# 多个 LoRA 不合并，仅动态加载
swift deploy \
    --model Qwen/Qwen2.5-7B-Instruct \
    --adapters style1=out/lora1 style2=out/lora2 \
    --enable_lora true
```

## 5. Megatron-LM 训练后的导出

```bash
# Step 1: 转 HF
swift export \
    --mcore_model out/qwen-72b-mg-sft \
    --to_hf true \
    --output_dir hf/qwen-72b-sft

# Step 2: 量化
swift export \
    --model hf/qwen-72b-sft \
    --quant_method awq --quant_bits 4 \
    --output_dir merged/qwen-72b-awq
```

## 7. 选型总结

> [!important]
> 
> - **A100/A6000**：AWQ 是首选。GPTQ 仅在需要额外 0.3% 质量时选。
> 
> - **H100**：FP8 首选（原生加速、不需校准集）。
> 
> - **超长上下文 (≥ 32k)**：加 INT8 / FP8 KV cache 减少显存。
> 
> - **资源受限 (≤ 24G GPU)**：AWQ 在 4-bit 下可跑 7B-13B；FSDP-QLoRA 训练 70B。

> [!important]
> 
> - **量化后评测不可跳过**：生产上线前用 benchmark 迎验，质量损失 > 2% 需重调。
> 
> - **不要在训练中途量化**：训练后一次量化是主流流程。
