## 0. 定位

> 一句话：GPTQ = 逐列 OBQ 量化，迭代补偿量化误差，是质量最高的推理量化之一，代价是量化过程较慢。

---

## 1. 原理

GPTQ 针对每个线性层的权重 $W in mathbb{R}^{d_{out}times d_{in}}$，逐列量化。在量化某列后，用二次估计补偿后面列的误差：

新公式 W_{:,j+1:} leftarrow W_{:,j+1:} - frac{(hat W_{:,j} - W_{:,j})}{H_{j,j}^{-1}} H_{j,j+1:}^{-1} 新公式

其中 $H = 2 X X^T$ 是逆 Hessian。这使量化后输入输出近似不变。

## 2. ms-swift 命令

```bash
swift export \
    --model out/qwen-sft \
    --quant_method gptq \
    --quant_bits 4 \
    --quant_dataset wikitext-2-raw-v1 \
    --quant_n_samples 1024 \
    --quant_seqlen 2048 \
    --quant_batch_size 1 \
    --output_dir merged/qwen-7b-gptq
```

## 3. 超参与资源

|参数|含义|推荐|`quant_bits`|位宽|4 (首选)|
|---|---|---|---|---|---|
|`group_size`|量化粒度|128|`quant_n_samples`|校准集样本数|512-1024|
|`quant_seqlen`|校准集序列长|2048|`damp_percent`|不逆保护|0.01|

## 4. 校准集选择

```python
import json, random
from datasets import load_dataset

# 首选：生产采样 + 通用文本混合
prod = [json.loads(l)['messages'][-1]['content']
        for l in open('production.jsonl')][:512]
wiki = load_dataset('wikitext', 'wikitext-2-raw-v1', split='train')
wiki_samples = [r['text'] for r in wiki if len(r['text']) > 200][:512]

random.seed(42)
calib = prod + wiki_samples
random.shuffle(calib)
open('calib.txt','w').write('\n\n'.join(calib[:1024]))
```

```bash
swift export ... \
    --quant_dataset calib.txt \
    --quant_n_samples 1024
```

## 5. 量化耗时

> [!important]
> 
> Qwen2.5-7B 量化 · 1×A100 80G
> 
> - 1024 样本 × 2048 token：约 30-60 分钟。
> 
> - 如果带上 70B 模型可能 6-12 小时。

## 6. 部署

```bash
swift deploy \
    --model merged/qwen-7b-gptq \
    --infer_backend vllm \
    --quantization gptq_marlin \          # vLLM 会自动推断
    --max_model_len 8192
```

## 7. 踩坑提示

> [!important]
> 
> - **校准集太小会过拟合**：需 ≥ 256 样本，推荐 1024+。
> 
> - **校准集与训练集重叠**：不会造成泄露，但该集应与生产分布一致。
> 
> - **group_size 128 是安全选择**：越小质量越高但出错量增加。
> 
> - **量化 LoRA 权重需合并**：不可直接量化 LoRA adapter。
