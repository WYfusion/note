## 0. 定位

> 一句话：BitsandBytes（bnb）提供动态 4/8 bit 量化权重，是 QLoRA 训练的底层依赖。

---

## 1. NF4 与双量化

![[12.4 BnB - QLoRA - 1. NF4 与双量化.excalidraw|800]]

NF4 (Normal Float 4) 是在高斯分布上优化的 4-bit 表示。双量化在 `quant_constants` 上再量化一次。

## 2. QLoRA 训练

```bash
swift sft \
    --model Qwen/Qwen2.5-7B-Instruct \
    --dataset alpaca \
    --train_type lora --lora_rank 16 \
    --target_modules all-linear \
    --quant_method bnb \
    --quant_bits 4 \
    --bnb_4bit_quant_type nf4 \
    --bnb_4bit_compute_dtype bfloat16 \
    --bnb_4bit_use_double_quant true \
    --gradient_checkpointing true \
    --learning_rate 1e-4 \
    --max_length 2048 \
    --output_dir out/qwen-qlora
```

## 3. 参数详解

|参数|含义|推荐|
|---|---|---|
|`bnb_4bit_quant_type`|nf4 / fp4|nf4|
|`bnb_4bit_use_double_quant`|双量化|true|

## 4. 8-bit 量化 ([LLM.int](http://LLM.int)8)

```bash
# 仅加载使用，不训练
swift infer \
    --model Qwen/Qwen2.5-7B-Instruct \
    --quant_method bnb \
    --quant_bits 8 \
    --infer_backend pt
```

8-bit 适合调试，生产推理首选 AWQ/GPTQ。

## 6. 与 AWQ/GPTQ 区别

|项|BnB / NF4|AWQ / GPTQ|定位|训练 (QLoRA)|推理|
|---|---|---|---|---|---|
|量化采样|动态，加载时|静态，预量化|量化耗时|几乎为零|数十分钟 - 几小时|
|推理速度|较慢|快|需校准集|不需|需|

## 7. 合并与部署

```bash
# 训练后合并 LoRA 为完整模型
swift export \
    --adapters out/qwen-qlora \
    --merge_lora true \
    --output_dir merged/qwen-qlora-merged

# 合并后可再 AWQ 量化为生产模型
swift export \
    --model merged/qwen-qlora-merged \
    --quant_method awq --quant_bits 4 \
    --output_dir merged/qwen-qlora-awq
```

> [!important]
> 
> - **BnB 量化不能直接部署生产**：BnB 4-bit 权重 依赖 dequant 计算，推理慢于 AWQ。
> 
> - **bnb_4bit_compute_dtype 需 bf16/fp16**：不要设 fp32。
> 
> - **合并后是完整 fp16 权重**：不是 4-bit。需重新 AWQ 量化才能生产部署。
