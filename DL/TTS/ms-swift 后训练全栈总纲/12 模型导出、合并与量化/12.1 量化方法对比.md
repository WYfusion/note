## 0. 定位

> 一句话：GPTQ / AWQ / BnB / FP8 是常见量化方法，定位不同：GPTQ 与 AWQ 面向推理，BnB 面向训练（QLoRA）。

---

## 1. 量化方法总览

|方法|位宽|训练 / 推理|需校准集|主要优势|
|---|---|---|---|---|
|**AWQ**|4 bit|推理|需|推理快|
|**FP8**|8 bit (E4M3)|训练 + 推理|不需|H100 原生加速|
|**HQQ**|2-8 bit|训练 + 推理|不需|不需校准|

## 5. ms-swift 量化命令总表

```bash
# AWQ
swift export --model Qwen/Qwen2.5-7B \
    --quant_method awq --quant_bits 4 \
    --quant_n_samples 256 \
    --output_dir merged/qwen-7b-awq

# GPTQ
swift export --model Qwen/Qwen2.5-7B \
    --quant_method gptq --quant_bits 4 \
    --quant_n_samples 1024 \
    --output_dir merged/qwen-7b-gptq

# BnB 量化加载（错误不会生成独立量化检点，主要于训练场景）
# 训练 QLoRA 时使用:
swift sft --model Qwen/Qwen2.5-7B \
    --quant_method bnb --quant_bits 4 \
    --bnb_4bit_quant_type nf4 \
    --train_type lora

# FP8 (H100)
swift export --model Qwen/Qwen2.5-7B \
    --quant_method fp8 \
    --output_dir merged/qwen-7b-fp8

# HQQ
swift export --model Qwen/Qwen2.5-7B \
    --quant_method hqq --quant_bits 4 \
    --output_dir merged/qwen-7b-hqq
```

## 6. 量化后验证质量

```bash
# 与原始模型在同一 benchmark 上对比
swift eval \
    --model merged/qwen-7b-awq \
    --eval_dataset mmlu ceval \
    --infer_backend vllm \
    --output_dir eval/qwen-7b-awq

# 与未量化主动代表间质量损失 < 1.5%。
```

## 7. 踩坑提示

> [!important]
> 
> - **量化后必须重测**：不能默认 "损失 ≈0"，用 benchmark 验证。
> 
> - **校准集要与生产分布一致**：用 wikitext 校准会造成领域上表现下降。采样生产 prompt 为校准集。
> 
> - **不要在已 LoRA 模型上量化**：先 `merge_lora` 后量化。
> 
> - **低于 4 bit 谨慎**：2-3 bit 需额外补偿手段，质量下降明显。
