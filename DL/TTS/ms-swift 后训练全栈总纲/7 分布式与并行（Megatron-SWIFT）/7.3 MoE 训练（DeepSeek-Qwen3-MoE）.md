## 0. 定位

> 一句话：MoE = sparse activation 的大模型架构，训练需**EP 专家并行 + 负载均衡损失 + token drop 控制**。

## 2. EP / ETP 详解

- **EP**（Expert Parallel）：不同 experts 分到不同 GPU。token 发给请求的 expert。

- **ETP**（Expert TP）：在单个 expert 内部分示 TP。

```bash
megatron sft \
    --expert_model_parallel_size 8 \        # 128 experts / 8 = 16 / GPU
    --expert_tensor_parallel_size 2 \        # 每 expert 再拆 2 卡
    --num_experts 128 \
    --moe_router_topk 8
```

## 3. 负载均衡损失

MoE 容易陷入 "热门 expert" 问题。Megatron-Core 提供辅助损失：

新公式 mathcal{L}_{text{aux}} = alpha cdot N sum_{i=1}^{N} f_i cdot P_i 新公式

其中 $f_i$ 是分配给 expert i 的比例，$P_i$ 是 router 平均概率。

```bash
megatron sft \
    --moe_aux_loss_coeff 0.01 \
    --moe_router_load_balancing_type aux_loss \  # 或 sinkhorn / none
    --moe_token_dispatcher_type alltoall_seq
```

## 4. token drop 与 capacity

当某 expert 超过 capacity 限制，token 会被 drop。Megatron 提供：

```bash
megatron sft \
    --moe_expert_capacity_factor 1.5 \        # 1.0=均分, 1.5=多甚 50%
    --moe_pad_expert_input_to_capacity true \  # padding 防变长
    --moe_token_drop_policy probs              # 或 position
```

## 5. 实战命令：DeepSeek-V3 SFT

```bash
# 32×H100 (4 到 8 机)
NNODES=4 NPROC_PER_NODE=8 \
megatron sft \
    --load mcore/deepseek-v3 \
    --dataset my-instruction-data \
    --tensor_model_parallel_size 2 \
    --pipeline_model_parallel_size 4 \
    --expert_model_parallel_size 4 \
    --expert_tensor_parallel_size 1 \
    --moe_router_topk 8 \
    --moe_router_load_balancing_type aux_loss \
    --moe_aux_loss_coeff 0.01 \
    --moe_expert_capacity_factor 1.5 \
    --sequence_parallel true \
    --train_type full \
    --bf16 true \
    --learning_rate 5e-6 \
    --save out/dsv3-sft
```

## 6. Qwen3-MoE（小参量 MoE）示例

Qwen3-MoE-30B-A3B 可在单机 8×H100 完成：

```bash
NPROC_PER_NODE=8 megatron sft \
    --load mcore/qwen3-moe-30b \
    --dataset alpaca \
    --tensor_model_parallel_size 2 \
    --pipeline_model_parallel_size 1 \
    --expert_model_parallel_size 4 \
    --moe_router_topk 8 \
    --moe_aux_loss_coeff 0.01 \
    --bf16 true \
    --train_type lora \
    --target_modules all-linear \
    --save out/qwen3moe-lora
```

## 7. 踩坑提示

> [!important]
> 
> - **EP 必须能整除 num_experts**：例如 256 experts 可设 EP=8/16/32，不能设 5。
> 
> - **aux_loss 太小会造成热门 expert**，太大会损害主损失，0.01 是常见起点。
> 
> - **token drop 过高** → 指标波动，提高 capacity_factor。
> 
> - **MoE 推理**：SGLang/vLLM 已支持主流 MoE，但要注意转 HF 后加载。
