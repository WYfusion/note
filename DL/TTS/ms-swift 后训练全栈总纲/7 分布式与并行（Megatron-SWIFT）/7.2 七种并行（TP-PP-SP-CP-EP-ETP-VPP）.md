## 0. 定位

> 一句话：Megatron 提供 7 种正交并行，乘积总路为全球 GPU 数；ms-swift 在此基础上提供友好参数。

---

## 1. 七维并行总览

|缩写|名称|切什么|作用|
|---|---|---|---|
|**TP**|Tensor Parallel|单个矩阵乘法|划分 attention/MLP 权重|
|**SP**|Sequence Parallel|序列 (LayerNorm/Dropout)|与 TP 配套，剩余内存|
|**EP**|Expert Parallel|MoE 专家|划分 experts|
|**VPP**|Virtual Pipeline|PP 的子划分|减小 bubble|

乘积：`DP×TP×PP×CP×EP = total_GPUs`。

## 3. 参数映射

```bash
# 32 GPU = DP×TP×PP×CP×EP
megatron sft \
    --tensor_model_parallel_size 4 \          # TP=4
    --pipeline_model_parallel_size 2 \        # PP=2
    --context_parallel_size 1 \                # CP=1
    --expert_model_parallel_size 4 \           # EP=4 (MoE 专用)
    --expert_tensor_parallel_size 1 \          # ETP=1
    --num_layers_per_virtual_pipeline_stage 2 \ # VPP
    --sequence_parallel true                   # SP=on (与 TP 配套)
    
    # 剩余为 DP：DP = 32 / (4 × 2 × 1 × 4) = 1 (需增加卡)
```

## 4. TP 详解

TP 在单个线性层内划分权重：

新公式 Y = X W,quad W = [W_1 W_2] Rightarrow Y = [X W_1 X W_2] 新公式

两卡各计算一半，最后 all-gather。适合单节点内 (NVLink 带宽 > 100GB/s)。

```python
# 伪代码
class ColumnParallelLinear:
    def __init__(self, in_features, out_features, tp_size):
        self.weight = Parameter(out_features // tp_size, in_features)
    def forward(self, x):
        local_y = x @ self.weight.T               # [B, T, h/tp]
        return all_gather(local_y, dim=-1)        # [B, T, h]
```

## 5. PP + VPP

PP 按层划分模型。VPP（虚拟流水线）进一步将 stage 划分成多个不连续的 chunk，用 1F1B 调度减小 bubble：

```jsx
原 PP=4（高 bubble）：
  GPU0: F1 F2 F3 F4 - - - B4 B3 B2 B1
  GPU1: -  F1 F2 F3 F4 - B4 B3 B2 B1
  ... 

VPP=2 (chunk 交错)：
  GPU0: F1a F2a F1b F2b ... B2b B1b B2a B1a
  bubble 明显变小
```

## 6. CP（Context Parallel）与超长上下文

CP 将**序列维度**拆到多卡，每卡只看到 1/CP 的 token。在 attention 处用 ring 或 ulysses 通信。

```bash
megatron sft \
    --context_parallel_size 4 \        # 64k 上下文可能需 CP=4
    --tensor_model_parallel_size 2 \
    --max_position_embeddings 65536 \
    --max_length 65536
```

## 7. 并行选择心法

> [!important]
> 
> **启动顺序**：TP ≥ 8 跳机会崩 → PP 逸机架→ EP 接主机架 → SP/CP。举例：
> 
> - 8 卡 70B SFT：TP=4, PP=2 (DP=1)
> 
> - 16 卡 70B SFT：TP=4, PP=2, DP=2
> 
> - 32 卡 DeepSeek-V3 MoE：TP=2, PP=4, EP=4, DP=1
> 
> - 8 卡 64k 上下文 7B SFT：TP=2, CP=4, DP=1

> [!important]
> 
> - TP 跨节点会被 NVLink/PCIe 带宽捏死，必须限制在单节点内。
> 
> - PP 跨节点可行但 bubble 变大，启 VPP。
> 
> - EP 需要 expert 数貃 EP 整除。
