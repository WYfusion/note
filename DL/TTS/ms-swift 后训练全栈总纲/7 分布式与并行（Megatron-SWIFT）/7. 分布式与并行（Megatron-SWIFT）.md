## 0. 定位

> 一句话：本页梳理从 **DDP → ZeRO → FSDP → Megatron-SWIFT** 的分布式训练选型谱系，重点拆解 **Megatron 七种并行**（TP/PP/SP/CP/ETP/EP/VPP）与 MoE 训练 **10x 加速**背后的工程选择。

---

## 1. 分布式策略谱系

|策略|原理|适用规模|显存收益|实现成本|
|---|---|---|---|---|
|**DDP**|数据并行，模型全复制|< 13B 且单卡能装下|无|最低|
|**DeepSpeed ZeRO-2**|序列化 optimizer state|13B-30B|2-3x|低|
|**DeepSpeed ZeRO-3 / FSDP**|双序列化参数 + grad + opt|30B-70B|4-8x|中|
|**Megatron-SWIFT**|3D 并行 (TP/PP/DP) + MoE 特化|≥ 70B 或 MoE|10x 主要在速度|高|

> [!important]
> 
> **关键边界**：< 13B 用 DDP / ZeRO，30-70B 用 FSDP，**MoE 或 ≥ 70B 上 Megatron**。

---

## 2. Megatron 七种并行详解

|缩写|全称|切分对象|典型取值|
|---|---|---|---|
|**TP**|Tensor Parallel|权重矩阵按列 / 行切|2 / 4 / 8|
|**PP**|Pipeline Parallel|按 transformer 层切|2 / 4 / 8 / 16|
|**SP**|Sequence Parallel|LayerNorm/Dropout 序列维切（与 TP 配套）|true/false|
|**CP**|Context Parallel|超长上下文按序列切|2 / 4 / 8|
|**EP**|Expert Parallel|MoE 专家跨卡切|2 / 4 / 8 / 16|
|**ETP**|Expert Tensor Parallel|专家内部 TP|1 / 2 / 4|
|**VPP**|Virtual Pipeline Parallel|PP 微批交错，减少 bubble|2 / 4|

## 3. Megatron-SWIFT 命令与 HF 转换

```bash
# Step 1: 将 HF 权重转换为 Megatron 格式
swift export \
    --model Qwen/Qwen3-30B-A3B \
    --to_mcore true \
    --output_dir Qwen3-30B-A3B-mcore

# Step 2: Megatron 训练 (8 卡示例，MoE 场景)
NPROC_PER_NODE=8 megatron sft \
    --load Qwen3-30B-A3B-mcore \
    --dataset my_sft.jsonl \
    --tensor_model_parallel_size 2 \
    --pipeline_model_parallel_size 2 \
    --expert_model_parallel_size 2 \
    --sequence_parallel true \
    --context_parallel_size 1 \
    --moe_grouped_gemm true \
    --micro_batch_size 1 \
    --global_batch_size 16 \
    --learning_rate 1e-5 \
    --save Qwen3-30B-A3B-sft

# Step 3: 训练后转回 HF
swift export \
    --mcore_model Qwen3-30B-A3B-sft \
    --to_hf true \
    --output_dir Qwen3-30B-A3B-sft-hf
```

> [!important]
> 
> **默认选型公式**：TP×PP×CP×DP = NPROC_TOTAL。优先提高 TP（4 为常见上限），再开 PP。CP 仅在序列 ≥ 16k 时启用。

## 4. MoE 专项：为什么快 10x

Megatron-SWIFT 在 MoE 上加速关键：

1. **Grouped GEMM**：将多个专家的 token 计算融合为一次 batched matmul

1. **Expert Parallel (EP)**：专家水平切到多卡，all-to-all 交换 token

1. **Token Drop 优化**：路由不均衡时丢弃超额 token 提升吞吞量

> [!important]
> 
> **实际提速**：Qwen3-30B-A3B / DeepSeek-V3 这类稀疏 MoE 上，相比 ZeRO3 实验可到 5-10x 吐吞量提升。

## 5. 序列并行：Ulysses + Ring-Attention

超长上下文（≥ 32k）下，单卡装不下激活，需序列维并行：

```bash
# Ulysses（多头跨卡） + Ring-Attention（序列环状传递 KV）
swift sft --model Qwen/Qwen3-32B \
    --sequence_parallel_size 4 \
    --max_length 65536 \
    --attn_impl flash_attn \
    --packing true
```

|方法|原理|优势|
|---|---|---|
|**Ulysses**|头维切分，all-to-all 序列 ↔ 头|通信量小|
|**Ring-Attention**|序列切分，KV 环形传递|支持超长序列|
|**Megatron CP**|集成二者优点|与 Megatron 生态兑备|
