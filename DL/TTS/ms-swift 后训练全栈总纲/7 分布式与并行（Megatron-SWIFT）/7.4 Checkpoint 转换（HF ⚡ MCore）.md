## 0. 定位

> 一句话：Megatron 训练前需将 HuggingFace 权重转为 Megatron-Core (MCore) 格式；训练后转回 HF 供推理使用。

## 2. HF ⚡ MCore

```bash
swift export \
    --model Qwen/Qwen2.5-72B-Instruct \
    --to_mcore true \
    --tensor_model_parallel_size 4 \
    --pipeline_model_parallel_size 2 \
    --target_dtype bf16 \
    --output_dir mcore/qwen2.5-72b-tp4pp2
```

转换后目录：

```jsx
mcore/qwen2.5-72b-tp4pp2/
  iter_0000001/
    mp_rank_00_000/  # TP rank 0, PP rank 0
    mp_rank_01_000/  # TP rank 1, PP rank 0
    mp_rank_02_000/
    mp_rank_03_000/
    mp_rank_00_001/  # TP rank 0, PP rank 1
    ...
  latest_checkpointed_iteration.txt
  args.json          # 记录 TP/PP/架构参数
```

## 3. MCore ⚡ HF

```bash
swift export \
    --mcore_model out/qwen-72b-mg-sft \
    --to_hf true \
    --target_dtype bf16 \
    --output_dir hf/qwen-72b-sft-merged
```

输出为标准 HF 格式：

```jsx
hf/qwen-72b-sft-merged/
  config.json
  model-00001-of-00030.safetensors
  model-00002-of-00030.safetensors
  ...
  tokenizer.json
  tokenizer_config.json
```

## 4. LoRA 训练后的处理

```bash
# 训练是 LoRA，需先合并 adapter 
swift export \
    --mcore_model out/qwen-72b-mg-lora \
    --merge_lora true \
    --to_hf true \
    --output_dir hf/qwen-72b-lora-merged
```

## 5. 验证转换正确性

```python
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

orig = AutoModelForCausalLM.from_pretrained(
    'Qwen/Qwen2.5-7B-Instruct', torch_dtype=torch.bfloat16, device_map='cuda')
conv = AutoModelForCausalLM.from_pretrained(
    'hf/qwen-7b-roundtrip', torch_dtype=torch.bfloat16, device_map='cuda')
tok  = AutoTokenizer.from_pretrained('Qwen/Qwen2.5-7B-Instruct')

inp = tok('Tell me a joke about cats.', return_tensors='pt').to('cuda')
with torch.no_grad():
    o1 = orig(**inp).logits
    o2 = conv(**inp).logits
# bf16 下可接受差异 ~ 1e-3
print('max abs diff:', (o1 - o2).abs().max().item())
```

> [!important]
> 
> 如果 diff > 0.1，说明转换有问题。常见原因：
> 
> 1. dtype 不一致（fp16/bf16/fp32 混使）
> 
> 1. RoPE 参数不匹配
> 
> 1. 词表不匹配。

## 6. 常见 TP/PP 重划分

如果需要从 TP=4 转为 TP=2（不同集群）：

```bash
# 方案 1：先合并为 HF，再重新拆分
swift export --mcore_model in_tp4pp2 --to_hf true --output_dir tmp_hf
swift export --model tmp_hf --to_mcore true \
    --tensor_model_parallel_size 2 --pipeline_model_parallel_size 4 \
    --output_dir out_tp2pp4
```

## 7. 踩坑提示

> [!important]
> 
> - **TP/PP 变更 → 重转**：训练中途改并行配置需重新转换权重。
> 
> - **磁盘空间**：72B bf16 需 ~144GB，TP=4 PP=2 会生成 8 份拷贝（实际划分后总量不变，但中间转换需临时空间）。
> 
> - **不同架构不存在互转**：LLaMA 与 Qwen 架构不同，不要试图互转权重。
> 
> - **转换错误多是版本不匹配**：ms-swift、Megatron-LM 、PyTorch 版本需严格对齐。
