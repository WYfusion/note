## 0. 定位

> 一句话：本节以 70B + DeepSeek-V3 为例给出实际可复现的训练脚本与资源预算。

## 1. 70B SFT (全参 vs LoRA)

|方案|最小 GPU|推荐并行|备注|70B 全参 SFT|16×H100|TP=4, PP=4|需 Megatron|
|---|---|---|---|---|---|---|---|
|70B LoRA (HF)|4×A100 80G|FSDP-QLoRA|不需 Megatron|70B LoRA (Megatron)|8×H100|TP=4, PP=2|更快|

## 2. 70B 全参 SFT 完整脚本

```bash
#!/bin/bash
# 16×H100 (2 机)。预计 ~24h 训 1 epoch alpaca-en (50k samples)

# 1\. 转 MCore
swift export \
    --model Qwen/Qwen2.5-72B-Instruct \
    --to_mcore true \
    --tensor_model_parallel_size 4 \
    --pipeline_model_parallel_size 4 \
    --output_dir mcore/qwen-72b

# 2\. 启动训练 
NNODES=2 NPROC_PER_NODE=8 \
MASTER_ADDR=$MASTER_ADDR MASTER_PORT=29500 \
megatron sft \
    --load mcore/qwen-72b \
    --dataset alpaca-en \
    --tensor_model_parallel_size 4 \
    --pipeline_model_parallel_size 4 \
    --sequence_parallel true \
    --num_layers_per_virtual_pipeline_stage 2 \   # VPP=2
    --recompute_granularity selective \
    --train_type full \
    --micro_batch_size 1 \
    --global_batch_size 32 \
    --bf16 true \
    --learning_rate 5e-6 \
    --min_lr 5e-7 \
    --weight_decay 0.1 \
    --warmup_iters 100 \
    --train_iters 5000 \
    --max_length 4096 \
    --eval_interval 500 \
    --save_interval 500 \
    --save out/qwen-72b-mg-sft

# 3\. 转 HF
swift export \
    --mcore_model out/qwen-72b-mg-sft \
    --to_hf true \
    --output_dir hf/qwen-72b-sft
```

## 3. 显存估算

70B bf16 训练 · 每卡显存 ≈ (参数 + 优化器 + 梯度) / TP / PP + activation。估算公式：

```jsx
params_mem    = 70e9 × 2 (bf16)            = 140 GB
optim_mem     = 70e9 × 8 (Adam fp32)        = 560 GB
grad_mem      = 70e9 × 2                    = 140 GB
total_static  ≈ 840 GB

TP=4, PP=4 → 每卡 ≈ 840 / 16 = 52.5 GB
+ activation · · · ≈ H100 80G 可育
```

## 4. CPT（继续预训练）

与 SFT 主要区别：使用 raw-text 数据 + 较大 batch：

```bash
megatron pt \
    --load mcore/qwen-72b \
    --dataset my-domain-corpus \           # 在预训练语料
    --tensor_model_parallel_size 4 \
    --pipeline_model_parallel_size 4 \
    --sequence_parallel true \
    --train_type full \
    --micro_batch_size 1 \
    --global_batch_size 256 \              # 预训练需大 batch
    --bf16 true \
    --learning_rate 1e-5 \
    --train_iters 100000 \
    --max_length 4096 \
    --save out/qwen-72b-domain-cpt
```

## 5. DeepSeek-V3 SFT 脚本骨架

```bash
# 4 机 32 GPU H100 (资源下限)
NNODES=4 NPROC_PER_NODE=8 \
megatron sft \
    --load mcore/deepseek-v3 \
    --dataset my-instruction-data \
    --tensor_model_parallel_size 2 \
    --pipeline_model_parallel_size 4 \
    --expert_model_parallel_size 4 \
    --moe_router_topk 8 \
    --moe_router_load_balancing_type aux_loss \
    --moe_aux_loss_coeff 0.01 \
    --moe_expert_capacity_factor 1.5 \
    --sequence_parallel true \
    --train_type full \
    --micro_batch_size 1 \
    --global_batch_size 16 \
    --bf16 true \
    --learning_rate 5e-6 \
    --train_iters 3000 \
    --save out/dsv3-sft
```

## 7. 实战建议

> [!important]
> 
> - **先拿小模型躚一遍**：用 Qwen-7B 跑通脚本，再上 70B。
> 
> - **启动 dry-run**：`--exit_on_missing_checkpoint true` 可在 1 迭代后退出，验证 IO。
> 
> - **save_interval 必须设置**：大模型训练多点设 checkpoint 防崩。
> 
> - **TP+PP 同机纪约**：TP 到占一机，跨机优先 PP/DP。

> [!important]
> 
> **70B 全参 SFT 不是首选**：除非你需要修改模型能力边界，否则在 70B 上 LoRA + DPO/GRPO 已足够，资源代价低很多。
