## 0. 定位

> 一句话：本页从「训练脱锉」走向「生产交付」，梳理**可复现 / 可比较 / 可回滚 / 可部署 / 可扩展 / 可治理**六大验收维度。

---

## 1. 六可验收模型

|维度|含义|关键物件|
|---|---|---|
|**可复现**|任何人获不到代码 + 数据 + 参数能重现结果|git commit、数据版本、随机种子、环境 lock|
|**可比较**|base / SFT / DPO / GRPO / 量化多版本统一评测|评测集冻结、推理参数一致|
|**可回滚**|adapter / merged / quantized 分层保存与版本号|checkpoint 命名规范、model card|
|**可部署**|OpenAI 兼容 API、压测通过、监控完备|vLLM/SGLang/LMDeploy + Prometheus|
|**可扩展**|自定义 dataset/template/metric/reward/callback|ms-swift 插件机制|
|**可治理**|数据合规、输出安全、日志审计|PII 脱敏、输出过滤、Trace ID|

## 2. 创建可复现环境

```bash
# 1. 使用 conda 锁定环境
conda create -n swift python=3.10
conda activate swift
pip install ms-swift==4.0.0  # 明确锁定版本
pip freeze > requirements.lock

# 2. 记录随机种子与训练环境
swift sft --model ... \
    --seed 42 \
    --data_seed 42 \
    --logging_dir tb_logs \
    --output_dir output/exp_v1

# 3. 一体化项目目录
project/
  data/v1/                       # 数据版本
configs/sft_v1.yaml            # 训练参数
scripts/train_v1.sh            # 启动脚本
  output/v1/checkpoint-xxx/      # 产出
MODELCARD.md                   # 模型卡
```

## 3. 产物命名规范

```jsx
{model_base}-{task}-{stage}-{quant}-{date}
例： Qwen3-8B-business-sft-dpo-awq-20251015
```

> [!important]
> 
> **细节**：同一实验多个阶段产物都**独立存储**：adapter / merged / awq。这样任意阶段出问题都能精准回滚，同时使用硬链接节省磁盘。

## 4. 生产部署压测清单

```python
# 伪代码：用 locust 压测 vLLM 部署
from locust import HttpUser, task, between

class LLMUser(HttpUser):
    wait_time = between(0.1, 0.5)
    
    @task
    def chat(self):
        self.client.post('/v1/chat/completions', json={
            'model': 'my-model',
            'messages': [{'role': 'user', 'content': '介绍一下 LoRA'}],
            'max_tokens': 256,
            'temperature': 0.7,
        })

# 压测指标需验收：
# - QPS / TPS（token/秒）
# - P50/P95/P99 廹尾延迟
# - 错误率 < 0.1%
# - 显存占用 < 90%
# - 超长上下文 (8k+) 不 503
```
