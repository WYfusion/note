## 0. 定位

> 一句话：训练崩了不要从头再来，用 `--resume_from_checkpoint` 续训。注意学习率、随机种子、数据顺序的可复现性。

---

## 1. 自动保存 ckpt

```bash
swift sft ... \
    --save_strategy steps \
    --save_steps 500 \
    --save_total_limit 5 \
    --save_only_model false \
    --output_dir out/qwen-sft
```

`save_only_model=false` 会同时保存 optimizer/scheduler/rng state，是续训的前提。

## 2. 续训

```bash
swift sft ... \
    --resume_from_checkpoint out/qwen-sft/checkpoint-2000 \
    --output_dir out/qwen-sft

# 或 resume_from_checkpoint=true 自动找最新
swift sft ... \
    --resume_from_checkpoint true \
    --output_dir out/qwen-sft
```

## 3. 续训需要保留

|组件|说明|
|---|---|
|optimizer state|`optimizer.pt`|
|rng state|`rng_state_*.pth`（每 rank 一份）|
|data|iterable dataset 需 stateful|

## 4. 数据顺序的可复现性

```python
# datasets 流式读取 + 跳过已处理样本
from datasets import load_dataset

def build_resumable(jsonl_path: str, skip: int = 0):
    ds = load_dataset('json', data_files=jsonl_path, split='train', streaming=False)
    ds = ds.shuffle(seed=42)        # 固定 seed，保证不同次运行顺序一致
    if skip > 0:
        ds = ds.skip(skip)
    return ds

# 续训时 skip = global_step * batch_size * grad_accum
```

## 6. ZeRO / FSDP 续训

```bash
# DeepSpeed ZeRO-3 ckpt 是分片的
out/qwen-sft/checkpoint-2000/
  global_step2000/
    bf16_zero_pp_rank_0_mp_rank_00_optim_states.pt

# 续训时配置必须一致：world_size / TP / PP / ZeRO stage
swift sft ... \
    --deepspeed configs/zero3.json \
    --resume_from_checkpoint out/qwen-sft/checkpoint-2000
```

## 7. 健壮的训练启动脚本

```bash
#!/usr/bin/env bash
set -euo pipefail

OUT=out/qwen-sft
MAX_RETRY=10

for i in $(seq 1 $MAX_RETRY); do
    echo "[$(date)] attempt $i"
    if swift sft \
        --model Qwen/Qwen2.5-7B-Instruct \
        --dataset alpaca \
        --train_type lora \
        --output_dir $OUT \
        --resume_from_checkpoint true \
        --save_steps 500 \
        --save_total_limit 5 \
        --num_train_epochs 3; then
        echo 'training finished'
        exit 0
    else
        echo "crashed, sleeping 60s before retry"
        sleep 60
    fi
done
echo "max retries exceeded" >&2
exit 1
```

> [!important]
> 
> - 多机训练崩溃后退出码非 0，外层脚本检测后重启即可。
> 
> - 配合 K8s Job / Slurm 自动重启更稳。
