## 0. 定位

> 一句话：单机多卡用 DDP/FSDP/ZeRO，多机多卡再加 NCCL 网络与启动脚本编排。

---

## 1. 单机多卡（torchrun）

```bash
# 4 卡 DDP
NPROC_PER_NODE=4 swift sft \
    --model Qwen/Qwen2.5-7B-Instruct \
    --dataset alpaca \
    --train_type lora \
    --per_device_train_batch_size 2 \
    --gradient_accumulation_steps 8 \
    --output_dir out/qwen-sft
```

## 2. 单机多卡（FSDP）

```bash
NPROC_PER_NODE=4 swift sft \
    --model Qwen/Qwen2.5-7B-Instruct \
    --train_type full \
    --use_fsdp true \
    --fsdp_config configs/fsdp.json \
    --output_dir out/qwen-sft
```

```json
// configs/fsdp.json
{
    "sharding_strategy": "FULL_SHARD",
    "backward_prefetch": "BACKWARD_PRE",
    "forward_prefetch": true,
    "limit_all_gathers": true,
    "cpu_offload": false,
    "activation_checkpointing": true
}
```

## 4. NCCL 网络调优

|变量|说明|典型值|
|---|---|---|
|`NCCL_IB_DISABLE`|禁用 InfiniBand|0（用 IB）/ 1（强制 TCP）|
|`NCCL_P2P_DISABLE`|禁用 P2P|0（默认）|
|`NCCL_NTHREADS`|每 GPU 线程数|auto|

```bash
# 健康检查
python -m torch.distributed.run --nproc_per_node 8 \
    -m torch.utils.collect_env

# all_reduce 带宽测试
NCCL_DEBUG=INFO mpirun -np 16 -hostfile hosts \
    /opt/nccl-tests/build/all_reduce_perf -b 1M -e 1G -f 2
```

## 6. 通信开销估算

```jsx
7B   全量 SFT  4 节点 32 卡
  fp16 梯度规模:  ~14 GB
  每步 all_reduce: ~14 GB
  100 GbE 带宽:   12.5 GB/s → 1.1s 通信开销
  200 Gb IB:      25 GB/s   → 0.6s
  400 Gb IB:      50 GB/s   → 0.3s
```

经验：每步 step time 中通信 < 30% 是合格水平。

## 7. 多机踩坑

> [!important]
> 
> - **MASTER_ADDR 写错** → 卡死。可用主机名或 IP，但所有节点必须能解析。
> 
> - **不同节点版本不一致** → 偶发 hang。要求 PyTorch / NCCL / CUDA 严格一致。
> 
> - **共享存储 IO 瓶颈**：dataloader 直接读 NFS 可能成为瓶颈。本地 SSD 缓存数据。
> 
> - **NCCL_DEBUG 写满磁盘**：调试结束后关掉。
> 
> - **RDMA / IB 没装好**：NCCL 退化到 TCP，慢 5-10x。
