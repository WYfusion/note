## 0. 定位

> 一句话：Embedding 负责粗排（快但粗略），Reranker 负责精排（慢但精准），二者联手是 RAG 检索的主流架构。

## 2. 两者对比

|维度|Embedding|Reranker|
|---|---|---|
|输入|query / doc 各自编码|(query, doc) 联合编码|
|检索速度|快（向量索引）|慢（需 forward）|
|适用|粗排阈 100k+ 文档|精排 Top-100|

## 3. ms-swift 支持的架构

|类型|代表模型|train_type|Embedding|BGE-M3, GTE, E5, Qwen3-Embedding|`embedding`|
|---|---|---|---|---|---|
|Reranker|BGE-Reranker, Qwen3-Reranker|`reranker`|Generative Reranker|RankGPT-style|`reranker` • LLM|

## 4. 快速入门

```bash
# Embedding 训练
swift sft \
    --model Alibaba-NLP/gte-Qwen2-7B-instruct \
    --task_type embedding \
    --train_type lora \
    --dataset my-pair-data \
    --loss_type infonce \
    --output_dir out/my-embedding

# Reranker 训练
swift sft \
    --model BAAI/bge-reranker-v2-m3 \
    --task_type reranker \
    --train_type lora \
    --dataset my-pair-data \
    --loss_type listwise \
    --output_dir out/my-reranker
```

## 5. 部署与使用

```python
from swift.llm import infer_main, InferArguments

# Embedding 推理
args = InferArguments(
    model='out/my-embedding',
    task_type='embedding',
    infer_backend='pt',
)
emb = infer_main(args, ['Notion 是什么?', '云端协作软件'])
print(emb.shape)   # [2, hidden_dim]

# Reranker 推理
args.model = 'out/my-reranker'
args.task_type = 'reranker'
scores = infer_main(args, [
    {'query': 'Notion 是什么?', 'doc': 'Notion 是云端协作软件...'},
    {'query': 'Notion 是什么?', 'doc': '今天天气不错...'},
])
print(scores)   # [0.95, 0.05]
```

## 6. 什么时候微调

> [!important]
> 
> - **领域词汇差异大**：医学、法律、内部术语。预训练 embedding 不认识这些词。
> 
> - **中文 / 多语混合**：多语言场景下，微调 embedding 收益明显。
> 
> - **检索作业与主流不同**：例如查找 "代码 + 函数名"，预训练 embedding 表现一般。
> 
> - **仅提高检索准确率**：调 prompt + 换模型后仍不够。

## 7. 踩坑提示

> [!important]
> 
> - **Embedding 不计 LLM 损失**：`task_type=embedding` 会取出 hidden_state 计算 InfoNCE，与生成任务不同。
> 
> - **训练与推理池化一致**：mean / cls / last token 需一致，错会造成评测下降。
> 
> - **量级误认**：embedding 训练需要数万到几十万可靠偏好对，不是几百。
