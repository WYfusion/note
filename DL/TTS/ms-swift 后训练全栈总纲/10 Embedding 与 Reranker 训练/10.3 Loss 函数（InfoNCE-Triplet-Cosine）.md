## 0. 定位

> 一句话：Embedding 主流损失是 InfoNCE，辅以 Triplet / Cosine；Reranker 多用 listwise / pointwise。

---

## 1. InfoNCE（首选）

对 batch 内每个 query $q$，有 1 个 pos 与 N-1 个 in-batch negs：

$$\mathcal{L}_{\text{InfoNCE}} = -\log \frac{\exp(\mathrm{sim}(q, p) / \tau)}{\sum_{d \in B} \exp(\mathrm{sim}(q, d) / \tau)}$$

其中 $\tau$ 是温度（0.05-0.1）。

```python
import torch
import torch.nn.functional as F

def info_nce(q, p, negs, tau=0.05):
    """q: [B, h], p: [B, h], negs: [B, K, h] (手工难负例)."""
    q = F.normalize(q, dim=-1)
    p = F.normalize(p, dim=-1)
    
    # 正例: [B, B] (in-batch + diag)
    sim_qp = q @ p.T / tau                 # [B, B]
    
    if negs is not None:
        negs = F.normalize(negs, dim=-1)
        sim_qn = torch.einsum('bh,bkh->bk', q, negs) / tau   # [B, K]
        sim    = torch.cat([sim_qp, sim_qn], dim=1)          # [B, B+K]
    else:
        sim = sim_qp
    
    labels = torch.arange(q.size(0), device=q.device)        # diag 为正例
    return F.cross_entropy(sim, labels)
```

## 2. Triplet Loss

$$\mathcal{L} = \max(0,\ \mathrm{sim}(q, n) - \mathrm{sim}(q, p) + m)$$

```python
def triplet(q, p, n, margin=0.2):
    return F.relu(F.cosine_similarity(q, n) - F.cosine_similarity(q, p) + margin).mean()
```

适用：负例质量高，但量少。

## 3. Cosine Embedding Loss

$$\mathcal{L} = \begin{cases} 1 - \cos(q, d) & \text{if label}=+1 \\ \max(0,\ \cos(q, d) - m) & \text{if label}=-1 \end{cases}$$

```python
def cosine_loss(q, d, label, margin=0.0):
    cs = F.cosine_similarity(q, d)
    pos_loss = (1 - cs)
    neg_loss = F.relu(cs - margin)
    return torch.where(label > 0, pos_loss, neg_loss).mean()
```

## 4. Reranker - Listwise Loss

对于同一 query、多个 docs 与打分：

$$\mathcal{L} = -\sum_i y_i \log \frac{\exp(s_i)}{\sum_j \exp(s_j)}$$

其中 $s_i$ 是模型输出，$y_i$ 是归一化后的 label。

## 5. Reranker - Pointwise (BCE) Loss

每个 (query, doc) 独立判断二分类：

$$\mathcal{L} = -y\log \sigma(s) - (1-y)\log(1 - \sigma(s))$$

适用：doc 质量不一 (二元 1/0)。

## 6. ms-swift 中选择

```bash
# Embedding

    --temperature 0.05 \
    --num_negatives 4

# Reranker
swift sft --task_type reranker \
    --loss_type listwise \              # listwise / pointwise
    ...
```

## 7. 实践建议

> [!important]
> 
> - **优先 InfoNCE**：主流、充分利用 batch、调参简单。
> 
> - **temperature 0.05 是合理起点**：调 0.02-0.1，太低梯度太空。
> 
> - **batch_size 越大越好**：InfoNCE 依赖 in-batch negs，增大 batch 可明显提升。
> 
> - **混合损失仅在调不动时考虑**：InfoNCE + Triplet 均可，但必要时才加。
