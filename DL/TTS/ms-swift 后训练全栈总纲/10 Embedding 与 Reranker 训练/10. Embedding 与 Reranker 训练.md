## 0. 定位

> 一句话：本页说明 ms-swift 对 **检索处理链路**中「**双塔 Embedding 模型**」与「**Reranker 交互模型**」训练的支持，定位于「**不同于生成模型 SFT**」的专项任务。

---

## 1. 与生成模型 SFT 的本质区别

|维度|生成 SFT|Embedding|Reranker|
|---|---|---|---|
|任务|生成 token|生成向量表示|输出相关性分数|
|主要 loss|CE|InfoNCE / Triplet|Pairwise / Listwise|
|输入|messages|(query, positive, negatives)|(query, doc) / pairs|
|推理输出|token 序列|D 维向量|标量分数|

---

## 2. InfoNCE Loss（Embedding 核心）

给定 query $q$、正例 $d^+$、负例集 $\{d_i^-\}$：

$$\mathcal{L} = -\log \frac{\exp(\text{sim}(q, d^+) / \tau)}{\exp(\text{sim}(q, d^+) / \tau) + \sum_i \exp(\text{sim}(q, d_i^-) / \tau)}$$

其中 $\tau$ 是温度参数，$\text{sim}$ 一般为 cosine。

## 3. ms-swift Embedding 训练示例

```bash
swift sft --task_type embedding \
    --model Qwen/Qwen3-Embedding-0.6B \
    --dataset my_embedding.jsonl \
    --train_type lora \
    --learning_rate 5e-5 \
    --temperature 0.05 \
    --hard_negatives 7
```

数据格式：

```json
{
  "query": "什么是 LoRA?",
  "positive": "LoRA 是低秩适配微调方法...",
  "negative": [
    "DPO 是偏好优化算法",
    "区块链是一种分布式存储...",
    "..."
  ]
}
```

> [!important]
> 
> **硬负例是 Embedding 训练质量的灵魂**：从现有检索系统 top-k 结果中挑「似是而非」的负例。随机负例在难例上不起作用。

---

## 4. Reranker 训练

Reranker 是「**cross-encoder**」：(query, doc) 拼接后过模型输出单个分数。

```bash
swift sft --task_type reranker \
    --model Qwen/Qwen3-Reranker-0.6B \
    --dataset my_rerank.jsonl \
    --train_type lora \
    --learning_rate 5e-5
```

> [!important]
> 
> **为什么 Reranker 需要 cross-encoder**：bi-encoder（Embedding）独立编码后点积，速度快但精度低；cross-encoder 让 query 与 doc 在所有 layer 交互注意，精度高但算量大。以“Embedding 粗排 + Reranker 精排”的两阶段架构是 RAG 最佳实践。

## 6. 常见误区

> [!important]
> 
> **误区 1**：用生成模型直接抽 hidden state 做检索。**反驳**：未训练的向量各向同性差，需过 InfoNCE 领域适配。

> [!important]
> 
> **误区 2**：被动快缓存训练集负例，不更新。**建议**：每个 epoch 用当前模型重新挑硬负例（hard negatives mining）。

> [!important]
> 
> **误区 3**：Reranker 训练遇到 negative 不足。**建议**：1:n 中 n 推荐 7–15，过少难区分边界。
