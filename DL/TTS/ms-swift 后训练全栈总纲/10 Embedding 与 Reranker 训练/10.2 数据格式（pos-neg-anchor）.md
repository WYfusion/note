## 0. 定位

> 一句话：Embedding 训练数据是**(query, positive, negatives)** 三元组，难负例是质量关键。

---

## 1. 三元组格式

```jsx
# Pair (query, pos_doc) - 需动态采集负例
{"query": "什么是 Notion?", "response": "Notion 是云端协作软件..."}

# Triplet (query, pos, neg)
{"query":     "什么是 Notion?",
 "response":  "Notion 是云端协作软件。",
 "rejected_response": "今天天气很好。"}

# Multi-Negatives (query, pos, neg_1, neg_2, ..., neg_k)
{"query": "什么是 Notion?",
 "response": "Notion 是云端协作软件。",
 "rejected_response": [
   "今天天气很好。",
   "请处理 PDF 文件。",
   "Excel 是表格软件。"
 ]}
```

## 2. 难负例挖掘

难负例（hard negative）= 看起来似是但实际不是答案的文档。是提升训练效果的关键。

```python
import torch
from torch.nn.functional import cosine_similarity

def mine_hard_negatives(query_emb, doc_embs, doc_texts, pos_idx, k=4):
    """从调用 doc 中选中似但不是 pos 的倒为难负例."""
    sims = cosine_similarity(query_emb.unsqueeze(0), doc_embs)
    sims[pos_idx] = -1                     # 排除 pos
    top_k = sims.topk(k).indices
    return [doc_texts[i] for i in top_k]

# 在训练前用原始模型推理一遍，为每个 query 选 4 个难负例
```

ms-swift 提供内置：

```bash
swift sft --task_type embedding \
    --hard_negatives_mining true \
    --num_hard_negatives 4 \
    --hard_negative_score_threshold 0.7
```

## 3. 在批 negatives（In-Batch）

InfoNCE 默认将同批其他样本的 pos 作为负例，是免费的难负例源。

|batch_size|有效 negatives|备注|32|31 + 手工负例|一般|
|---|---|---|---|---|---|
|128|127 + 手工负例|推荐|512+|极多|劲需 cross-batch 技巧|

## 4. Multi-Negatives 批

如果手工提供 k 个难负例，ms-swift 会拼接为 [pos, neg_1, ..., neg_k]，InfoNCE 计算。

## 5. Reranker 数据格式

Reranker 使用 listwise 训练：

```jsx
{"query": "什么是 Notion?",
 "docs": [
   {"text": "Notion 是云端协作软件。", "label": 1.0},
   {"text": "Notion 可以做任务管理。", "label": 0.7},
   {"text": "今天天气很好。",      "label": 0.0},
   {"text": "Excel 是表格软件。",   "label": 0.1}
 ]}
```

## 7. 踩坑提示

> [!important]
> 
> - **负例太粗会崩 InfoNCE**：随机选负例过于容易，模型学不到东西。首选 hard negatives。
> 
> - **难负例太难是假 negatives**：阈 0.85+ 可能本身是同义 pos，应设阈。
> 
> - **中文场景需中文负例**：勿访问跨语言检索，训练与推理语言保持一致。
