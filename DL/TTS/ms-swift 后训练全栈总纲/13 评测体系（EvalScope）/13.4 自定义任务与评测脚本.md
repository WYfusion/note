## 0. 定位

> 一句话：业务上线前必须有 **业务自定义评测集**，用 EvalScope 注册或写独立脚本。

## 1. 业务集设计原则

> [!important]
> 
> - **覆盖关键场景**：每个核心场景至少 50 条。
> 
> - **难度分层**：容易 / 中等 / 困难三档。
> 
> - **冻结**：评测集一旦冻结，下次发版前不动。
> 
> - **版本化**：v1, v2 …，每次扩充另存版本。
> 
> - **去污染**：必须与训练集做 hash 比对。

## 3. 验证器实现

```python
import re, json
from typing import Callable

VERIFIERS: dict[str, Callable] = {}

def register(name):
    def deco(fn):
        VERIFIERS[name] = fn
        return fn
    return deco

@register('exact')
def _exact(pred: str, item: dict) -> float:
    return 1.0 if pred.strip() == item['reference'].strip() else 0.0

@register('contains')
def _contains(pred: str, item: dict) -> float:
    kws = item.get('keywords', [item['reference']])
    return 1.0 if all(k in pred for k in kws) else 0.0

@register('regex')
def _regex(pred: str, item: dict) -> float:
    return 1.0 if re.search(item['pattern'], pred) else 0.0

@register('llm_judge')
def _llm_judge(pred: str, item: dict) -> float:
    from openai import OpenAI
    judge = OpenAI(base_url='http://judge:8001/v1', api_key='x')
    prompt = (
        f"问题：{item['input']}\n"
        f"参考答案：{item['reference']}\n"
        f"评分细则：{item['rubric']}\n"
        f"模型回复：{pred}\n\n"
        "请给出 0 到 1 的分数，只输出数字。"
    )
    r = judge.chat.completions.create(
        model='judge-llm',
        messages=[{'role': 'user', 'content': prompt}],
        temperature=0.0,
    )
    try:
        return float(r.choices[0].message.content.strip())
    except ValueError:
        return 0.0
```

## 4. 评测主循环

```python
import json, statistics
from collections import defaultdict
from openai import OpenAI

def evaluate(model_url: str, model_name: str, dataset_path: str):
    client = OpenAI(base_url=model_url, api_key='dummy')
    by_scene = defaultdict(list)
    by_diff = defaultdict(list)
    total = []

    for line in open(dataset_path):
        item = json.loads(line)
        resp = client.chat.completions.create(
            model=model_name,
            messages=[{'role': 'user', 'content': item['input']}],
            temperature=0.0,
            max_tokens=512,
        )
        pred = resp.choices[0].message.content
        score = VERIFIERS[item['verifier']](pred, item)
        total.append(score)
        by_scene[item['scene']].append(score)
        by_diff[item['difficulty']].append(score)

    print(f'总分: {statistics.mean(total):.4f}  (n={len(total)})')
    print('--- 按场景 ---')
    for k, v in by_scene.items():
        print(f'  {k}: {statistics.mean(v):.4f}  n={len(v)}')
    print('--- 按难度 ---')
    for k in ['easy', 'medium', 'hard']:
        v = by_diff.get(k, [])
        if v:
            print(f'  {k}: {statistics.mean(v):.4f}  n={len(v)}')
```

## 5. 注册到 EvalScope

```python
# 把上面的 evaluate 包成 EvalScope 任务
from evalscope.benchmarks import DataAdapter
from evalscope.utils import register_benchmark

@register_benchmark('my-business')
class MyBusinessAdapter(DataAdapter):
    def load(self, **kw):
        return [json.loads(l) for l in open('biz_eval.jsonl')]

    def gen_prompt(self, item, **kw):
        return item['input']

    def match(self, pred, item):
        return VERIFIERS[item['verifier']](pred, item)
```

## 7. 踩坑提示

> [!important]
> 
> - **LLM-as-Judge 不稳定**：温度务必设 0，judge 模型固定；多次平均。
> 
> - **判分细则要明确**：模糊 rubric 会导致评分方差大。
> 
> - **业务集不要太小**：< 50 条，单条波动太大无意义。
> 
> - **样本要审核**：脏 / 模糊 / 多解的样本必须人工把关。
