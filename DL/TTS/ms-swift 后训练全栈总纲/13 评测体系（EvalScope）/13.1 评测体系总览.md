|---|---|---|---|

## 2. ms-swift 评测入口

```bash
# 标准 benchmark
swift eval \
    --model out/qwen-sft \
    --eval_dataset mmlu ceval gsm8k humaneval \
    --infer_backend vllm \
    --eval_batch_size 8 \
    --output_dir eval/qwen-sft

# 自定义 benchmark（需 EvalScope 注册）
swift eval \
    --model out/qwen-sft \
    --eval_dataset my-custom-bench \
    --custom_eval_config configs/my_eval.json
```

## 5. 评测频率

> [!important]
> 
> - **训练中**：每 N 步小评测（GSM8K-100 子集）作为早停信号。
> 
> - **训练后**：完整 benchmark 一次。
> 
> - **上线前**：业务集 + 生产 SLO 压测。
> 
> - **上线后**：日级回归（采样生产流量评测）。

## 6. 与训练联动

```python
# 在训练 callback 中调用 eval
from swift.llm import get_model_tokenizer
from transformers import TrainerCallback

class EvalCallback(TrainerCallback):
    def __init__(self, eval_dataset='gsm8k', subset_size=100):
        self.dataset = eval_dataset
        self.subset = subset_size

    def on_evaluate(self, args, state, control, **kwargs):
        if state.global_step % 500 != 0:
            return
        # 调 swift eval 子进程，结果回写到 metric
        import subprocess, json
        out = subprocess.check_output([
            'swift', 'eval', '--model', args.output_dir,
            '--eval_dataset', self.dataset,
            '--eval_limit', str(self.subset),
        ])
        score = json.loads(out)['score']
        state.log_history.append({'eval_gsm8k': score, 'step': state.global_step})
```

## 7. 常见误区

> [!important]
> 
> - **只看一两个 benchmark 就发版**：覆盖不全，业务上可能崩盘。
> 
> - **benchmark 污染**：训练数据混入测试集 → 分数虚高。需做 decontamination。
> 
> - **prompt 不一致**：不同评测代码 prompt 模板差异 → 分数不可比。统一一份 ms-swift / EvalScope 即可。
> 
> - **批量大小影响**：某些任务 batch_size 改变会影响 padding，建议固定。
