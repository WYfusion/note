## 1. 三种 Judge 模式

|模式|说明|典型用途|**Single Score**|给一个回答打 1-10 分|整体质量|
|---|---|---|---|---|---|
|**Pairwise**|A vs B 选优|A/B 测试 / Arena|**Reference-based**|给参考答案，判对错|有标准答案的题|

## 2. Single Score Prompt 模板

```python
SINGLE_PROMPT = '''你是严格的评分员。请按以下细则给回复打分。

## 任务
{question}

## 模型回复
{answer}

## 评分细则（满分 10）
1. 准确性（0-4）：是否正确回答了问题
2. 完整性（0-3）：是否覆盖关键信息
3. 清晰度（0-2）：表达是否清晰
4. 礼貌度（0-1）：语气是否得体

请先简要分析，最后输出 JSON：
"accuracy": int, "completeness": int, "clarity": int, "politeness": int, "total": int
'''

import re, json
from openai import OpenAI

judge = OpenAI()  # 假设走 OpenAI / 或本地 72B

def judge_single(question: str, answer: str) -> dict:
    r = judge.chat.completions.create(
        model='gpt-4o',
        messages=[{'role': 'user', 'content': SINGLE_PROMPT.format(question=question, answer=answer)}],
        temperature=0.0,
    )
    txt = r.choices[0].message.content
    m = re.search(r'\{[^{}]+\}', txt[::-1])  # 找最后一个 JSON
    return json.loads(m.group(0)[::-1]) if m else {'total': 0}
```

## 3. Pairwise Prompt（去位置偏置）

```python
PAIR_PROMPT = '''请比较两个回答，选出更好的一个。

## 问题
{question}

## 回答 A
{a}

## 回答 B
{b}

请综合考虑准确性、完整性、清晰度。最后输出 JSON：
"winner": "A" | "B" | "tie", "reason": "..."
'''

def judge_pairwise(question: str, ans1: str, ans2: str) -> str:
    # 双向各跑一次去位置偏置
    out_ab = call(PAIR_PROMPT.format(question=question, a=ans1, b=ans2))
    out_ba = call(PAIR_PROMPT.format(question=question, a=ans2, b=ans1))
    w_ab = parse_winner(out_ab)  # A/B/tie
    w_ba = parse_winner(out_ba)  # A/B/tie
    # AB 中 A 胜 = ans1 胜；BA 中 B 胜 = ans1 胜
    score1 = (w_ab == 'A') + (w_ba == 'B')
    score2 = (w_ab == 'B') + (w_ba == 'A')
    if score1 > score2:
        return 'ans1'
    if score2 > score1:
        return 'ans2'
    return 'tie'
```

## 4. 已知偏置

|偏置|缓解|**位置偏置**：偏向 A 或 B 第一位|双向各跑一次|
|---|---|---|---|
|**长度偏置**：偏好长答案|规则限长 + 在 rubric 显式约束|**自我偏置**：judge 偏好同家族模型|用第三方 judge（gpt-4o 评 Qwen）|
|**列表偏置**：偏好结构化列表|rubric 指出格式不计分|**奉承偏置**：偏好阿谀奉承的回答|明确禁止把礼貌作为加分项|

## 5. 与人工评测的相关性

```python
import pandas as pd
from scipy.stats import pearsonr, spearmanr, kendalltau

# df: 每行一个样本，含 human_score, judge_score
df = pd.read_csv('judge_vs_human.csv')
print('Pearson :', pearsonr(df['human_score'], df['judge_score']))
print('Spearman:', spearmanr(df['human_score'], df['judge_score']))
print('Kendall :', kendalltau(df['human_score'], df['judge_score']))
```

经验：好的 Judge 与人工 Spearman ≥ 0.7。低于 0.5 说明 rubric 或 judge 模型不合适。

## 7. 上线建议

> [!important]
> 
> - **小规模人工 + 大规模 Judge 校准**：100-200 人工标注样本作为校准集，定期跑 Judge 校验相关性。
> 
> - **Judge 也是版本管理**：换模型、换 prompt 都要打版本，结果不可比。
> 
> - **temperature=0**：保证可复现。
> 
> - **多次采样取众数 / 平均**：高分歧样本更稳。

> [!important]
> 
> - 不要用 7B Judge 评 70B 模型，结果可能反向。
> 
> - Judge 要看 prompt 缓存命中率（同一 system prompt 重复利用）。
