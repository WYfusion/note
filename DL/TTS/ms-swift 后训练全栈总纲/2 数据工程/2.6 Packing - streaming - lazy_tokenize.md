## 0. 定位

> 一句话：三种数据输送模式，应对**短样本多**（packing）、**数据极大**（streaming）、**冷启动慢**（lazy）三类问题。

---

## 1. 三模式对照

|模式|触发参数|核心做法|适用|
|---|---|---|---|
|**Streaming**|`--streaming true`|边读边训练，不落盘|数据 ≥ TB 级|
|**全量预编码**|默认|启动时一次性 encode 并缓存|常规场景，最快|

## 2. Packing 工作原理

![[2.6 Packing - streaming - lazy_tokenize - 2. Packing 工作原理 - 图 01.excalidraw|800]]

关键点：

1. 拼接后用 **document_ids** 标记每段来源，attention 不跨段（或用 FlashAttn 变长 mask）。

1. 各段的 loss mask 独立，互不影响。

1. 单 batch 训练 token 数显著提高，等效 batch 增大。

## 3. Packing 命令

```bash
swift sft \
    --model Qwen/Qwen2.5-7B-Instruct \
    --dataset AI-ModelScope/alpaca-gpt4-data-zh \
    --max_length 4096 \
    --packing true \
    --attn_impl flash_attn \
    --train_type lora \
    --output_dir out/packed-lora
```

> [!important]
> 
> **经验**：当数据集平均长度 ≪ max_length（如 alpaca 类指令数据 < 1k tokens vs max_length=4096）时，开启 packing 可让吞吐 2-5x。

## 4. Streaming 命令

```bash
# 数据集太大无法预 encode，用 streaming 边读边训
swift sft \
    --model Qwen/Qwen2.5-7B-Instruct \
    --dataset HuggingFaceFW/fineweb-edu:CC-MAIN-2024-10 \
    --streaming true \
    --max_steps 10000 \
    --train_type full \
    --gradient_checkpointing true
```

注意点：

- streaming 不知道总长度，必须用 `--max_steps` 而非 `--num_train_epochs`。

- shuffle 用 buffer-based，buffer 大小 `--streaming_buffer_size`。

- 配合 `--packing` 实现连续 token 流。

## 5. Lazy tokenize 命令

```bash
# 数据规模中等，但 encode 太慢/磁盘紧张：每个 batch 现编码
swift sft \
    --model Qwen/Qwen2.5-7B-Instruct \
    --dataset my-30M-sample-data \
    --lazy_tokenize true \
    --num_proc 0 \
    --train_type lora
```

## 7. 常见组合

|场景|组合|
|---|---|
|预训练大语料|`--streaming true --packing true --max_steps N`|
|本地 GPU 调试|`--dataset xxx#1000` • 默认|

> [!important]
> 
> packing 在多模态 / 含图像视频的样本上需要谨慎：图像 token 与文本 token 拼接逻辑不同，建议先用单卡 small batch 跑通再放量。
