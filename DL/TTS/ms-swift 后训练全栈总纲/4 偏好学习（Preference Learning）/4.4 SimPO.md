## 0. 定位

> 一句话：SimPO = **去掉 reference model** + **序列长度归一化** + **target reward margin**，有时超过 DPO，且训练显存减半。

---

## 1. SimPO 与 DPO 的差异

|维度|DPO|SimPO|是否需 reference|需要|不需要|
|---|---|---|---|---|---|
|logp 是否除以长度|否|是|是否有 margin|否|是（`gamma`）|
|训练显存|~2x SFT|~1x SFT|调参复杂度|beta|beta + gamma|

## 2. 损失公式

$$\mathcal{L}_{\text{SimPO}} = -\log \sigma\Big( \dfrac{\beta}{|y_w|}\log \pi_\theta(y_w \mid x) - \dfrac{\beta}{|y_l|}\log \pi_\theta(y_l \mid x) - \gamma \Big)$$

三个关键点：

1. **去掉 ref**：不再计算 参考模型的 logp，训练需要的显存 ÷ 2。

1. **长度归一化**：除以 token 数，避免长回答 "不公平地" 拿到更高分。

1. **target margin** $\gamma$：要求 chosen 比 rejected 的 reward 高出 $\gamma$，防止“勉强差异”。

## 4. 超参选择经验

|超参|推荐范围|作用|`beta`|2.0 - 10.0（远大于 DPO 的 0.1）|另一个量纲，因为 logp/T 本身小|
|---|---|---|---|---|---|
|`simpo_gamma`|0.5 - 2.0|margin；与 beta 联动，常取 beta × 0.5-1.0|`learning_rate`|5e-7 – 1e-6|与 DPO 一致，偏小|

## 5. 手写 SimPO Loss

```python
import torch.nn.functional as F

def simpo_loss(policy_chosen_logps, policy_rejected_logps,
               chosen_lengths, rejected_lengths,
               beta=2.0, gamma=1.4):
    """logp 是未归一化的序列累加 log概率."""
    # 长度归一化后乘以 beta
    chosen_reward   = beta * policy_chosen_logps   / chosen_lengths
    rejected_reward = beta * policy_rejected_logps / rejected_lengths
    
    # 加 target margin gamma
    logits = chosen_reward - rejected_reward - gamma
    loss = -F.logsigmoid(logits).mean()
    
    accuracy = (chosen_reward > rejected_reward).float().mean()
    return loss, accuracy
```
