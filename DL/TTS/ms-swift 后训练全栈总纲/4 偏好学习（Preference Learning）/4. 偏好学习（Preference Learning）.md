## 1. 偏好学习的本质问题

SFT 教会模型「**说什么**」（imitation），但教不会「**说哪个更好**」。当存在 $(x, y_w, y_l)$ 形式的偏好数据时，我们希望模型对 $y_w$（chosen）的概率高于 $y_l$（rejected）。

所有偏好算法都在解决同一目标的不同近似：

$$\max_\theta \; \mathbb{E}_{(x, y_w, y_l)} \big[ r(x, y_w) - r(x, y_l) \big] - \beta \cdot \text{KL}(\pi_\theta \| \pi_{\text{ref}})$$

其中 $r$ 是奖励函数，$\pi_{\text{ref}}$ 是 reference model（防止分布漂移）。

## 3. 主流算法一览

|算法|是否需要 RM|是否需要 ref model|数据形式|优势|劣势|
|---|---|---|---|---|---|
|**PPO**|✅|✅|$(x, y_w, y_l)$ → RM → on-policy rollout|上限高、可在线学习|4 模型 + 复杂|
|**DPO**|❌|✅|$(x, y_w, y_l)$|简单稳定、效果好|off-policy、需 ref|
|**SimPO**|❌|**❌**|$(x, y_w, y_l)$|显存省、长度归一化|对超参敏感|
|**CPO**|❌|**❌**|$(x, y_w, y_l)$|NLL + 偏好双损失|—|
|**ORPO**|❌|**❌**|$(x, y_w, y_l)$|**省略 SFT 阶段**|需充分多样数据|
|**KTO**|❌|✅|$(x, y, \text{label})$|单边偏好，标注成本低|理论较新|

## 4. ms-swift 命令对照

ms-swift 用统一的 `swift rlhf` 命令，通过 `--rlhf_type` 切换算法：

```bash
# DPO
swift rlhf --rlhf_type dpo \
    --model output/sft_model \
    --dataset DPO/dpo_zh.jsonl \
    --train_type lora \
    --beta 0.1 \
    --learning_rate 5e-6

# SimPO（无 ref model，省一份显存）
swift rlhf --rlhf_type simpo \
    --model output/sft_model \
    --dataset DPO/dpo_zh.jsonl \
    --beta 2.0 --simpo_gamma 1.0 \
    --learning_rate 5e-6

# KTO（单边偏好数据）
swift rlhf --rlhf_type kto \
    --model output/sft_model \
    --dataset KTO/kto_zh.jsonl \
    --desirable_weight 1.0 --undesirable_weight 1.0

# 经典 RLHF：第一步训练 RM
swift rlhf --rlhf_type rm \
    --model output/sft_model \
    --dataset DPO/dpo_zh.jsonl \
    --train_type full --num_train_epochs 1

# 第二步：PPO
swift rlhf --rlhf_type ppo \
    --model output/sft_model \
    --reward_model output/rm_model \
    --dataset PPO/prompts.jsonl \
    --learning_rate 1e-6
```

> [!important]
> 
> **β 的常用区间**：DPO `beta=0.1-0.5`，SimPO `beta=2.0-2.5`（SimPO 因无 ref，β 量级不同）。盲目套 DPO 的 β 到 SimPO 会得到很差结果。

## 6. 常见误区

> [!important]
> 
> **误区 1**：DPO 前 SFT 不充分，直接 DPO。**反驳**：DPO 数据通常远少于 SFT，没有强 SFT 锚定，DPO 后模型容易输出退化（loss 下降但生成变差）。

> [!important]
> 
> **误区 2**：DPO 学习率沿用 SFT 的 1e-4。**反驳**：DPO 是「微调对齐」，应使用 **5e-7 至 5e-6**，否则模型崩塌。

> [!important]
> 
> **误区 3**：把 ORPO 当作万能替代 SFT+DPO 的方案。**反驳**：ORPO 在数据多样性不足时效果显著差于 SFT+DPO，仅在偏好数据本身覆盖了 SFT 行为时才适用。
