## 0. 定位

> 一句话：DPO **绕过 RM 与强化学习**，直接从成对偏好数据学习策略，是当前中小规模偏好对齐的 baseline。

---

## 1. 核心思想

DPO 发现：在 RLHF 的 KL 约束下，**最优策略** $\pi^*$ 与 参考策略 $\pi_{\text{ref}}$ 之间存在封闭形联系，可以反推出隐含 reward：

$$r(x, y) = \beta \log \dfrac{\pi^*(y \mid x)}{\pi_{\text{ref}}(y \mid x)} + Z(x)$$

并且常数 $Z(x)$ 在 pairwise 差分中被消掉，得到：

$$\mathcal{L}_{\text{DPO}} = -\mathbb{E}_{(x, y_w, y_l)} \Big[ \log \sigma\big( \beta \log\dfrac{\pi_\theta(y_w \mid x)}{\pi_{\text{ref}}(y_w \mid x)} - \beta \log\dfrac{\pi_\theta(y_l \mid x)}{\pi_{\text{ref}}(y_l \mid x)} \big) \Big]$$

本质是**让 chosen 的相对似然大于 rejected**。

    --output_dir out/qwen-dpo

## 4. 关键超参

|超参|常用值|作用|`beta`|0.05 - 0.5|KL 强度；偏小更能偏离 ref，偏大更保守|
|---|---|---|---|---|---|
|`learning_rate`|5e-7 – 5e-6|比 SFT 小一个量级|`num_train_epochs`|1|>1 常过拟合|
|`rpo_alpha`|0 / 1.0|0=纯 DPO；1.0=DPO+SFT 联合，防遗忘|`loss_type`|sigmoid / hinge / ipo|多种变体|

## 5. 手写 DPO Loss（理解用）

```python
import torch.nn.functional as F

def dpo_loss(policy_chosen_logps, policy_rejected_logps,
             ref_chosen_logps,    ref_rejected_logps,
             beta=0.1):
    """所有输入是序列累加 logprob (sum over response tokens)."""
    pi_logratios  = policy_chosen_logps - policy_rejected_logps
    ref_logratios = ref_chosen_logps - ref_rejected_logps
    logits = beta * (pi_logratios - ref_logratios)   # 即文中的差分项
    
    loss = -F.logsigmoid(logits).mean()
    
    # 隐含 reward（训练中监控）
    chosen_reward   = beta * (policy_chosen_logps   - ref_chosen_logps).detach()
    rejected_reward = beta * (policy_rejected_logps - ref_rejected_logps).detach()
    accuracy = (chosen_reward > rejected_reward).float().mean()
    return loss, accuracy
```

## 6. 常见变体（一行切换）

|变体|`--loss_type`|与 DPO 区别|
|---|---|---|
|**Hinge**|`hinge`|类 SVM hinge，对 outlier 更鲁棒|
|**加 SFT**|`--rpo_alpha 1.0`|DPO + 在 chosen 上加一个 SFT 损失，防调高 chosen logp 后遗忘|

## 7. 踩坑提示

> [!important]
> 
> - **DPO 前 SFT 必须够好**：DPO 是"偏好微调"，不是"能力训练"。若 SFT 版本指令遵循不够，DPO 会放大问题。  
>     - **偶发 chosen logp 下降**：beta 太小 / 数据哪里有问题。可加 `rpo_alpha=1.0` 缓解。  
>     - **数据 prompt 必须严格一致**：chosen 和 rejected 使用同一轮 prompt，避免引入虚假 reward。  
>     - **用 LoRA 训练 ref 与 policy 共享 base**：mem 需求 ≈ 1.5x SFT-LoRA，可接受。
