## 0. 定位

> 一句话：KTO **不需要 chosen/rejected 成对**，仅需“样本是否受欢迎”的布尔标签，代价是効率略低。

---

## 1. 何时用 KTO

|场景|是否适用 KTO|
|---|---|
|仅有用户“满意度”二值标签|✅ KTO|
|可验证任务|❌ 用 GRPO|

## 2. 损失公式

设 $r_\theta(x, y) = \beta \log \dfrac{\pi_\theta(y \mid x)}{\pi_{\text{ref}}(y \mid x)}$，设参考点 $z_0 = \mathbb{E}_{x'}[\mathrm{KL}(\pi_\theta(\cdot \mid x') \| \pi_{\text{ref}}(\cdot \mid x'))]$：

$$v(x, y) = \begin{cases} \lambda_D\, \sigma\big( r_\theta(x,y) - z_0 \big), & y\ \text{desirable} \\ \lambda_U\, \sigma\big( z_0 - r_\theta(x,y) \big), & y\ \text{undesirable} \end{cases}$$

$$\mathcal{L}_{\text{KTO}} = \mathbb{E}\big[ \lambda_y - v(x, y) \big]$$

關鍵欲望：

- 偏离参考点 $z_0$ 远越多，损失越难以减小 → 需要准确区分 desirable / undesirable。

- $\lambda_D \ne \lambda_U$：允许对 desirable / undesirable 赋予不同权重，应对不平衡数据。

## 5. 关键超参

|超参|常用值|说明|
|---|---|---|
|`desirable_weight` ($\lambda_D$)|1.0|desirable 样本权重|
|desirable:undesirable 比例|1:1|严重不平衡时调权重|

> [!important]
> 
> **推荐设置**：设 $\lambda_U \cdot n_U \approx \lambda_D \cdot n_D$，以平衡两类梯度贡献。

## 6. 手写 KTO Loss

```python
import torch, torch.nn.functional as F

def kto_loss(policy_chosen_logps, policy_rejected_logps,
             ref_chosen_logps, ref_rejected_logps,
             chosen_kl, rejected_kl,                # 用于估计 z0
             beta=0.1, lambda_d=1.0, lambda_u=1.0):
    chosen_reward   = beta * (policy_chosen_logps   - ref_chosen_logps)
    rejected_reward = beta * (policy_rejected_logps - ref_rejected_logps)
    
    # z0：反向 KL 估计（实现粗略化）
    kl = (chosen_kl.mean() + rejected_kl.mean()) / 2
    z0 = beta * kl.detach()
    
    chosen_loss   = lambda_d * (1 - F.sigmoid(chosen_reward   - z0))   # desirable: 要让 reward > z0
    rejected_loss = lambda_u * (1 - F.sigmoid(z0 - rejected_reward))    # undesirable: 要让 reward < z0
    
    loss = (chosen_loss.mean() + rejected_loss.mean()) / 2
    return loss
```

## 7. KTO 与其他偏好方法对比

|方法|数据形式|逆境表现|DPO|(prompt, chosen, rejected)|chosen logp 偶发下降|
|---|---|---|---|---|---|
|SimPO|(prompt, chosen, rejected)|length bias 调节|KTO|(prompt, response, label:bool)|desirable/undesirable 比例失衡时需调权重|

> [!important]
> 
> - **KTO 効率不如 DPO**：同样样本数量，KTO 提升一般比 DPO 小 10-30%。  
>     - **最适合产品环境**：你拿不到 chosen-rejected 成对，只有线上点赞/点踩记录。  
>     - **数据最低阈值**：建议 ≥5k 条（含两类标签）。
