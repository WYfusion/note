## 0. 定位

> 一句话：PPO = 经典 RLHF 中的策略梯度算法，用 **clipped surrogate objective + KL 约束** 让模型在 RM 指引下优化，不偏离初始策略太远。

## 2. 损失公式

$$\mathcal{L}_{\text{PPO}} = -\mathbb{E}\Big[ \min\big( \rho_t \hat{A}_t,\ \mathrm{clip}(\rho_t,\ 1-\epsilon,\ 1+\epsilon)\hat{A}_t \big) \Big] + \beta\, \mathrm{KL}(\pi_\theta \| \pi_{\text{ref}}) - c\,\mathcal{L}_V$$

- $\rho_t = \dfrac{\pi_\theta(a_t \mid s_t)}{\pi_{\text{old}}(a_t \mid s_t)}$：重要性采样比

- $\hat{A}_t$：GAE 估计的优势

- $\epsilon \approx 0.2$：clip 阈值

- $\beta$：KL 惩罚系数

- $\mathcal{L}_V = (V_\theta(s_t) - R_t)^2$：价值函数损失

## 3. 命令骨架

```bash
# 4×A100，PPO 训练 7B
NPROC_PER_NODE=4 swift rlhf \
    --rlhf_type ppo \
    --model out/qwen-sft/checkpoint-final \
    --reward_model out/qwen-rm/checkpoint-final \
    --dataset AI-ModelScope/alpaca-gpt4-data-zh \
    --train_type lora \
    --lora_rank 8 --target_modules all-linear \
    --num_train_epochs 1 \
    --learning_rate 1e-5 \
    --per_device_train_batch_size 1 \
    --gradient_accumulation_steps 8 \
    --max_new_tokens 512 \
    --kl_coef 0.05 \
    --cliprange 0.2 \
    --gamma 1.0 --gae_lambda 0.95 \
    --whiten_advantages true \
    --use_vllm true \
    --output_dir out/qwen-ppo
```

## 4. 关键超参

|超参|常用值|作用|`kl_coef`（β）|0.02 - 0.2|限制偶 SFT 漂移。偏小更探索，偏大更保守|
|---|---|---|---|---|---|
|`cliprange`（ε）|0.2|限制重要性采样比 jump|`learning_rate`|5e-7 – 1e-5|远小于 SFT|
|`num_generations`/rollout|1-4|每 prompt 采样次数|`max_new_tokens`|256-1024|rollout 长度|
|`gae_lambda`|0.95|GAE 衰减，稳定优势估计|`whiten_advantages`|true|优势标准化，提高稳定性|

## 5. PPO 循环伪代码

```python
for epoch in range(num_epochs):
    for prompts in dataloader:
        # ① Rollout: 用当前策略采样
        with torch.no_grad():
            responses = policy.generate(prompts, max_new_tokens=512)
            ref_logprobs = ref_policy.logprob(prompts, responses)
            old_logprobs = policy.logprob(prompts, responses)
            rewards = reward_model(prompts, responses)
            values  = value_net(prompts, responses)
        
        # ② 计算 advantage、return
        advantages, returns = compute_gae(
            rewards, values, gamma=1.0, lam=0.95)
        advantages = (advantages - advantages.mean()) / (advantages.std() + 1e-8)
        
        # ③ 多轮优化
        for _ in range(num_inner_epochs):
            new_logprobs = policy.logprob(prompts, responses)
            ratio = (new_logprobs - old_logprobs).exp()
            
            unclipped = ratio * advantages
            clipped   = ratio.clamp(1-0.2, 1+0.2) * advantages
            policy_loss = -torch.min(unclipped, clipped).mean()
            
            kl = (new_logprobs - ref_logprobs).mean()
            value_loss = (value_net(prompts, responses) - returns).pow(2).mean()
            
            loss = policy_loss + kl_coef * kl + 0.5 * value_loss
            loss.backward()
            optimizer.step(); optimizer.zero_grad()
```

ms-swift `--rlhf_type ppo` 使用 TRL `PPOTrainer` 底层，集成 vLLM 加速 rollout。

![[4.2 PPO（近端策略优化） - 6. PPO vs DPO GRPO 折中选择 - 图 02.excalidraw|800]]

> [!important]
> 
> **PPO 调参是夸张的坑**：4 套参数（策略/价值/RM/参考）+ rollout 成本 + reward hacking 防范。当代推荐优先使用 DPO 或 GRPO，除非你重点复现 InstructGPT/ChatGPT 这类经典 RLHF 管线。

## 7. ms-swift PPO 特色

- 内置 **vLLM rollout**，加速采样（最耗时环节）。

- 支持 **value-head 共享主体**（省显存）或独立 value network。

- 提供 `--ref_model_init` 选择 "reuse policy" / "separate copy" 。
