## 0. 定位

> 一句话：GSPO = 阐释 **序列级重要性采样比** 取代 token 级 ratio，理论上修正了 GRPO 的偏差问题。

---

## 1. 问题背景

GRPO 使用 token 级重要性采样比：

$$\rho_{i,t} = \frac{\pi_\theta(y_{i,t} \mid x, y_{i,<t})}{\pi_{\mathrm{old}}(y_{i,t} \mid x, y_{i,<t})}$$

但优势 $\hat A_i$ 是**序列级**的（整个回答一个分）。token 级 ratio 与序列级 advantage 错配，在长序列上偏差放大。

## 2. GSPO 修正

GSPO 使用 序列级几何平均 ratio：

$$\rho_i = \Big( \frac{\pi_\theta(y_i \mid x)}{\pi_{\mathrm{old}}(y_i \mid x)} \Big)^{1/|y_i|} = \exp\Big( \frac{1}{|y_i|}\sum_t \log\frac{\pi_\theta}{\pi_{\mathrm{old}}} \Big)$$

损失：

$$\mathcal{L}_{\mathrm{GSPO}} = -\mathbb{E}\Big[ \min\big(\rho_i \hat{A}_i,\ \mathrm{clip}(\rho_i,\ 1-\epsilon,\ 1+\epsilon)\hat{A}_i\big) \Big]$$

本质：把 token 级 ratio 变成序列级 "平均密度比"，使优势与 ratio 都是序列级。

## 4. 命令

```bash
swift rlhf \
    --rlhf_type grpo \
    --model Qwen/Qwen2.5-Math-7B \
    --dataset AI-MO/NuminaMath-CoT \
    --train_type lora \
    --importance_sampling_level sequence \
    --num_generations 8 \
    --reward_funcs accuracy format \
    --max_completion_length 4096 \
    --beta 0.0 \
    --learning_rate 1e-6 \
    --use_vllm true \
    --output_dir out/qwen-gspo
```

关键参数：`--importance_sampling_level sequence`（默认 `token`）。

## 5. 手写序列级 ratio

```python
import torch

def sequence_ratio(new_logp, old_logp, mask):
    """new_logp / old_logp / mask: [B, T]。返回 [B] 的序列级 ratio."""
    log_ratio = (new_logp - old_logp) * mask          # [B, T]
    seq_len   = mask.sum(dim=1)                       # [B]
    avg_log_ratio = log_ratio.sum(dim=1) / seq_len    # 几何平均 in log
    return avg_log_ratio.exp()

# clipped surrogate
def gspo_loss(seq_ratio, advantages, eps=0.2):
    unclipped = seq_ratio * advantages
    clipped   = seq_ratio.clamp(1-eps, 1+eps) * advantages
    return -torch.min(unclipped, clipped).mean()
```

## 6. 何时选 GSPO

> [!important]
> 
> - 训练长推理链（> 2k tokens）且出现**训练不稳定 / loss 振荡**。
> 
> - 同 prompt 不同采样之间长度差异大。
> 
> - 如果 DAPO 已足够稳定，不必叠加 GSPO。

## 7. 可叠加 DAPO

```bash
# DAPO + GSPO 叠加
swift rlhf --rlhf_type grpo \
    --importance_sampling_level sequence \   # GSPO
    --epsilon_high 0.28 \                     # DAPO clip-higher
    --loss_type token_mean \                  # DAPO token-mean
    --dynamic_sampling true \                 # DAPO dynamic
    ...
```
