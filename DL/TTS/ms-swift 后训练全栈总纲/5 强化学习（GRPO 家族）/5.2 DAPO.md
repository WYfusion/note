## 0. 定位

> 一句话：DAPO = 字节在 GRPO 上提出的四点改进（Clip-Higher / Dynamic Sampling / Token-Level Loss / Overlong Reward），在 AIME 2024 达到 50 分。

---

## 1. 四点改进总览

|改动|原 GRPO|DAPO|解决什么|
|---|---|---|---|
|**Dynamic Sampling**|固定 G|过滤全对/全错的组|去掉 0 梯度样本|
|**Overlong Reward Shaping**|超长直接截断|软惩罚|减少超长噪声|

## 2. Clip-Higher 公式

$$\rho_{i,t} \cdot \hat{A}_i \quad \xrightarrow{\text{clip}} \quad \mathrm{clip}\!\left(\rho_{i,t},\, 1-\epsilon_{\text{low}},\, 1+\epsilon_{\text{high}}\right) \cdot \hat{A}_i$$

论文取 $\epsilon_{\text{low}}=0.2,\ \epsilon_{\text{high}}=0.28$：允许上调探索者概率更多，限制下调同原 GRPO。

## 3. Token-Level Loss

原 GRPO：对每样本平均 token loss 后再对 batch 平均。问题：长回答（含多推理步骤）被拉与短回答同权重。

DAPO：直接在 batch 中所有 token 上均匀平均。

```python
# 原 GRPO（伪代码）
loss = (per_token_loss * mask).sum(dim=1) / mask.sum(dim=1)   # 样本级
loss = loss.mean()                                            # batch

# DAPO token-level
loss = (per_token_loss * mask).sum() / mask.sum()             # 直接平均 token
```

## 4. Dynamic Sampling

如果一个 prompt 的 G 个采样全对或全错，组内优势均为 0 → 梯度为 0，这个 prompt 这轮费了。

DAPO 动态采样过滤这类 prompt，只保留有梯度贡献的样本，提高采样效率。

## 5. 命令

```bash
swift rlhf \
    --rlhf_type grpo \
    --model Qwen/Qwen2.5-Math-7B \
    --dataset AI-MO/NuminaMath-CoT \
    --train_type lora \
    --num_generations 16 \
    --reward_funcs accuracy format \
    --epsilon 0.2 \
    --epsilon_high 0.28 \
    --loss_type token_mean \
    --dynamic_sampling true \
    --overlong_filter true \
    --max_completion_length 4096 \
    --beta 0.0 \
    --learning_rate 1e-6 \
    --use_vllm true \
    --output_dir out/qwen-dapo
```

## 6. 关键超参对比

|超参|原 GRPO|DAPO 推荐|`epsilon` / `epsilon_high`|0.2 / 0.2|0.2 / 0.28|
|---|---|---|---|---|---|
|`loss_type`|`grpo`|`token_mean`|`dynamic_sampling`|false|true|
|`beta` (KL)|0.04|0.0|`num_generations`|8|16|

> [!important]
> 
> DAPO 在 AIME 2024 数据集上将 Qwen-32B 推高到 50 分，超过 GPT-4 。在 Qwen-2.5-Math-7B 上从 35 提升至 ~45。
