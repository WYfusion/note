## 0. 定位

> 一句话：GRPO 家族还有多个变体在不同维度作改进，ms-swift 均已集成，一行参数切换。

---

## 1. 变体总览

|变体|改进点|开关|
|---|---|---|
|**CISPO**|序列级 IS 修正 + 处罚高偏差 token|`importance_sampling_level=sequence_token`|
|**SAPO**|Self-Adaptive RL，动态调 KL/clip|`sapo_*` 参数组|
|**Reinforce++**|随机梯度估计，无价值|`advantage_estimator=reinforce_plus_plus`|

## 2. CISPO（Sequence-Token Mixed IS）

思路：hybrid 雨 GSPO 。ratio 在序列级计算，但在 token 级 weight 上加一个 clip，防止某个 token 偏差放大 ratio。

```bash
swift rlhf --rlhf_type grpo \
    --importance_sampling_level sequence_token \
    --num_generations 8 ...
```

## 3. CHORD（SFT 与 RL 联合）

CHORD 在 GRPO 训练过程中周期性插入 SFT step，防止能力遗忘：

![[5.4 其他变体（CISPO-CHORD-SAPO-RLOO） - 3. CHORD（SFT 与 RL 联合） - 图 01.excalidraw|800]]

```bash
swift rlhf --rlhf_type grpo \
    --chord_enable true \
    --chord_sft_iter_count 50 \         # 每 50 step 插入 1 step SFT
    --chord_sft_dataset my-demo-data
```

## 4. SAPO（Self-Adaptive）

根据训练过程中的 KL / loss 动态调整 beta 与 clip range，避免手动调参：

- KL 过高 → 提高 beta。

- KL 过低 → 降低 beta（鼓励探索）。

## 5. RLOO（Leave-One-Out）

原 GRPO 优势：

$$\hat{A}_i = \frac{r_i - \bar{r}}{\mathrm{std}(r) + \varepsilon}$$

RLOO 改为：求则 $i$ 的 baseline 时排除 $i$ 自己：

$$\hat{A}_i = r_i - \frac{1}{G-1}\sum_{j \ne i} r_j$$

减少 baseline 与当前样本的相关偏差。

```bash
swift rlhf --rlhf_type grpo \
    --advantage_estimator rloo \
    --num_generations 4
```

## 6. Reinforce++

一个轻量 RL baseline：不训价值，不要求可验证 reward，在较少维度上调参：

```bash
swift rlhf --rlhf_type grpo \
    --advantage_estimator reinforce_plus_plus \
    --reward_funcs my_reward
```
