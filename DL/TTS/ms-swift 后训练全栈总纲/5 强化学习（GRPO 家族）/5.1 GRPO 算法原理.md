## 0. 定位

> 一句话：GRPO = 去掉价值网络、用**同 prompt 多采样的组内相对优势**取代 GAE，是 DeepSeek-R1 背后的核心 RL 算法。

---

## 1. PPO 与 GRPO 对比

|维度|PPO|GRPO|
|---|---|---|
|优势估计|GAE|组内标准化 reward|
|适用任务|一般|可验证奖励（数学/代码）|

## 2. 损失公式

对每个 prompt 采样 G 个输出，得到 reward 序列 $r_1, \dots, r_G$。组内优势：

$$\hat{A}_i = \frac{r_i - \mathrm{mean}(r)}{\mathrm{std}(r) + \varepsilon}$$

GRPO 损失（PPO 风格 clipped + KL）：

$$\mathcal{L} = -\mathbb{E}\Big[ \min\big(\rho_t \hat{A}_i,\ \mathrm{clip}(\rho_t,\ 1-\epsilon,\ 1+\epsilon)\hat{A}_i\big) - \beta\, \mathrm{KL}(\pi_\theta \| \pi_{\mathrm{ref}}) \Big]$$

## 4. 命令骨架

```bash
NPROC_PER_NODE=4 swift rlhf \
    --rlhf_type grpo \
    --model Qwen/Qwen2.5-Math-7B \
    --dataset AI-MO/NuminaMath-CoT \
    --train_type lora --lora_rank 8 --target_modules all-linear \
    --reward_funcs accuracy format \
    --num_generations 8 \
    --max_completion_length 1024 \
    --temperature 1.0 \
    --beta 0.04 \
    --learning_rate 1e-6 \
    --use_vllm true \
    --output_dir out/qwen-math-grpo
```

## 5. 关键超参

|超参|常用值|作用|`num_generations`|4-16|组内采样量|
|---|---|---|---|---|---|
|`temperature`|1.0|探索|`beta`|0-0.05|KL 系数|
|`max_completion_length`|512-4096|rollout 长度|`learning_rate`|1e-6 - 1e-5|偏小|

## 6. 手写组内优势

```python
import torch

def group_advantages(rewards):
    """rewards: [B, G] → [B, G] 优势."""
    mean = rewards.mean(dim=1, keepdim=True)
    std  = rewards.std(dim=1, keepdim=True) + 1e-8
    return (rewards - mean) / std

# 使用
rewards = torch.tensor([[1.0, 0.0, 1.0, 0.0],   # prompt 1: 2/4 对
                        [1.0, 1.0, 1.0, 0.0]])  # prompt 2: 3/4 对
adv = group_advantages(rewards)
print(adv)
# tensor([[ 1.0,-1.0, 1.0,-1.0],
#         [ 0.58,0.58,0.58,-1.73]])
```

## 7. 适用边界

> [!important]
> 
> - **需要可验证奖励**：数学题有标准答、代码过不过测试。纯开放式的主观任务不适。
> 
> - **rollout 是主要耗时**：不开 vLLM/SGLang 加速项目会面临 10x 变慢。
> 
> - **组内多样性**：如果 G 个采样全对或全错，优势为 0，梯度失效。
