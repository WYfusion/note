## 0. 定位

## 2. 高频参数速查表

|分组|参数|典型值|
|---|---|---|
|数据|`--dataset` / `--max_length` / `--split_dataset_ratio`|HF/MS id 或本地路径 / 4096 / 0.05|
|LoRA|`--lora_rank` / `--lora_alpha` / `--target_modules`|8 / 32 / all-linear|
|显存|`--gradient_checkpointing` / `--deepspeed` / `--torch_dtype`|true / zero3.json / bfloat16|
|推理|`--infer_backend` / `--max_new_tokens`|vllm\|sglang\|pt / 2048|

## 3. yaml 配置 + 多环境覆盖

```yaml
# configs/base.yaml
model: Qwen/Qwen2.5-7B-Instruct
template: qwen
dataset:
  - AI-ModelScope/alpaca-gpt4-data-zh
max_length: 4096
train_type: lora
lora_rank: 8
lora_alpha: 32
target_modules: all-linear
output_dir: out/baseline
```

```yaml
# configs/dev.yaml
extends: base.yaml      # 概念示意，实际通过 shell 拼接
learning_rate: 5.0e-5
num_train_epochs: 1
per_device_train_batch_size: 2
```

```bash
# 训练脚本：base + dev 两层配置 + 命令行兜底
swift sft \
    --config configs/base.yaml \
    --config configs/dev.yaml \
    --output_dir out/dev-run-$(date +%s)
```

## 4. 在 Python 里以编程方式构造 Args

```python
from swift.llm import TrainArguments, sft_main

args = TrainArguments(
    model='Qwen/Qwen2.5-7B-Instruct',
    template='qwen',
    dataset=['AI-ModelScope/alpaca-gpt4-data-zh'],
    train_type='lora',
    lora_rank=8, lora_alpha=32, target_modules='all-linear',
    num_train_epochs=1,
    per_device_train_batch_size=4,
    gradient_accumulation_steps=8,
    learning_rate=1e-4,
    warmup_ratio=0.05,
    save_strategy='epoch',
    output_dir='out/programmatic',
)
sft_main(args)
```

> [!important]
> 
> **工程实践**：把 `model/template/dataset` 这类「与运行环境无关」的字段写在 `base.yaml`，把 `lr/bs/epoch/output_dir` 写在 `dev.yaml`/`prod.yaml`，差异化覆盖。

## 5. 常见踩坑

> [!important]
> 
> - **template 和 model 不匹配**：传错 `--template` 会导致 chat 拼接错位、loss 异常。
> 
> - **target_modules=all-linear 在 MoE 上慎用**：可能命中 router，建议显式列出。
> 
> - `**--config**` **多次传入时后者覆盖前者**：调试时确认顺序。
