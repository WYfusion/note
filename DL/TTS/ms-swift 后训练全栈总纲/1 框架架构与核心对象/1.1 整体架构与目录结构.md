## 1. 仓库物理结构

|目录|职责|典型入口|
|---|---|---|
|`swift/llm/`|LLM/MLLM 训练核心|`train/sft.py`、`infer/`、`export/`|
|`swift/llm/template/`|chat template 注册表|`register_template`|
|`swift/llm/dataset/`|数据集加载与预处理|`load_dataset`|
|`swift/megatron/`|Megatron-SWIFT 集成|`megatron sft`|
|`swift/cli/`|命令行入口分发|`main.py`|
|`swift/llm/argument/`|参数定义（dataclass）|`TrainArguments`、`RLHFArguments`|
|`swift/llm/model/`|模型加载与 patch|`get_model_tokenizer`|
|`swift/trainers/`|各类 Trainer（SFT/DPO/GRPO 等）|`Seq2SeqTrainer` 派生|
|`swift/plugin/`|用户扩展点（reward/loss/callback）|`orms`、`losses`|

## 3. 极简执行链（伪代码）

```python
# swift/cli/main.py 分发逻辑（精简版）
from swift.llm import sft_main, pt_main, rlhf_main, infer_main, export_main

ROUTES = {
    'sft':    sft_main,    # 监督微调 / PEFT
    'pt':     pt_main,     # 继续预训练
    'rlhf':   rlhf_main,   # DPO/PPO/GRPO/KTO/...
    'infer':  infer_main,  # 推理
    'export': export_main, # 合并/量化
    'eval':   eval_main,   # 评测
}

def main():
    cmd = sys.argv[1]            # e.g. 'sft'
    rest = sys.argv[2:]
    ROUTES[cmd](rest)            # 路由到具体 main

# swift/llm/train/sft.py（伪代码）
def sft_main(argv):
    args = TrainArguments.from_argv(argv)         # ① 参数
    model, tokenizer = get_model_tokenizer(args)  # ② 模型
    template = get_template(args.template, tokenizer)  # ③ 模板
    train_ds, val_ds = load_dataset(args, template)    # ④ 数据
    trainer = Seq2SeqTrainer(                     # ⑤ Trainer
        model=model, args=args.training_args,
        train_dataset=train_ds, eval_dataset=val_ds,
        data_collator=template.data_collator,
        compute_loss_func=args.loss_func,         # 可被 plugin 替换
    )
    trainer.train()
    trainer.save_model(args.output_dir)
```

> [!important]
> 
> **理解锚点**：所有训练命令最终汇聚到 `Seq2SeqTrainer`（或其派生），区别只在 _args 解析_、_Loss 计算_ 和 _Data Collator_ 三处。
