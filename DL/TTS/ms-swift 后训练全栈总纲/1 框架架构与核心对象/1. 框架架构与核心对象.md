## 2. 九大核心对象一览

|对象|核心职责|对应 CLI 参数|所属子页面|
|---|---|---|---|
|**Model**|基座模型加载、ModelType 注册|`--model`|1.1|
|**Tokenizer / Processor**|文本/多模态输入编码|自动随 Model 加载|1.2|
|**Template**|对话模板、loss mask、tool schema|`--template`、`--system`|1.3|
|**Dataset**|数据加载、schema 转换、预处理|`--dataset`、`--val_dataset`|1.4|
|**Trainer**|训练循环：SFT/DPO/GRPO/RM/PPO|子命令 `swift sft\|rlhf\|...`|1.5|
|**Tuner**|PEFT 适配器注入|`--train_type`、`--target_modules`|1.6|
|**Infer Engine**|统一推理后端抽象|`--infer_backend`|1.7|
|**Eval / Export**|评测与产物导出|`swift eval`、`swift export`|1.8|
|**Agent Template**|跨模型工具调用统一格式|`--agent_template`|1.9|

## 3. 一行命令背后的对象协作

以最常见的 SFT-LoRA 命令为例：

```bash
swift sft \
    --model Qwen/Qwen3-8B \
    --dataset AI-ModelScope/alpaca-gpt4-data-zh \
    --train_type lora \
    --target_modules all-linear \
    --learning_rate 1e-4 \
    --max_length 2048 \
    --output_dir output
```

背后的对象组装顺序：

```python
# 伪代码：展示 ms-swift 内部的对象协作（简化版）
from swift.llm import (
    SftArguments, get_model_tokenizer, get_template,
    get_dataset, prepare_model, Seq2SeqTrainer,
)

# 1. 解析参数（含校验、推断默认值）
args = SftArguments(model='Qwen/Qwen3-8B', dataset=['...'],
                    train_type='lora', target_modules='all-linear')

# 2. 加载 Model + Tokenizer（含 Processor，多模态时自动启用）
model, tokenizer = get_model_tokenizer(args.model, args.torch_dtype)

# 3. 加载 Template（控制 prompt 拼接 + loss mask）
template = get_template(args.template, tokenizer,
                        default_system=args.system,
                        max_length=args.max_length)

# 4. 加载 Dataset 并按 Template 编码（messages → input_ids/labels）
train_ds, val_ds = get_dataset(args.dataset, args.val_dataset,
                               template=template, num_proc=8)

# 5. 注入 Tuner（LoRA 在此处把 nn.Linear 替换为 LoRA 包装层）
model = prepare_model(model, args)  # 内部走 PEFT/SwiftTuners

# 6. 构造 Trainer 并训练
trainer = Seq2SeqTrainer(model=model, args=args.training_args,
                         train_dataset=train_ds, eval_dataset=val_ds,
                         tokenizer=tokenizer, template=template)
trainer.train()
trainer.save_state()  # 保存 adapter
```

> [!important]
> 
> **工程判断**：99% 的 ms-swift 用户只用 CLI 即可。只有当你需要**自定义 Trainer 子类、注入自定义 callback、或集成到现有训练系统**时，才需要走 Python API（如上）。

## 4. 与同类框架的横向对比

|维度|ms-swift 4.x|LLaMA-Factory|TRL|Axolotl|
|---|---|---|---|---|
|模型支持数|**600+ LLM / 300+ MLLM**|~200|HF 全部|~150|
|Megatron 后端|✅ 集成（10x MoE 加速）|❌|❌|❌|
|GRPO 家族算法|✅ GRPO/DAPO/GSPO/SAPO/CISPO/CHORD/RLOO|仅 GRPO|仅 GRPO|❌|
|多模态训练|✅ 系统支持（图/视频/音频）|部分|❌|有限|
|序列并行|✅ Ulysses + Ring-Attention + Megatron CP|第三方实现|❌|❌|
|Web-UI|✅|✅|❌|❌|

> [!important]
> 
> **何时选 ms-swift**：需要 Megatron 大模型训练 / GRPO 家族最新算法 / 多模态训练 / 中文社区支持。**何时选其他**：纯研究复现选 TRL（最贴近论文），重 community ecosystem 选 Axolotl。

## 5. 模块边界与扩展点

ms-swift 的模块化让以下扩展**无需改框架代码**：

- 自定义 **Model**：在 `swift/llm/model/register.py` 注册新 ModelType（详见 1.1.1）

- 自定义 **Template**：继承 `Template` 类（详见 1.3.1）

- 自定义 **Dataset**：通过 `register_dataset` 注册（详见 1.4）

- 自定义 **Tuner**：实现 `SwiftTuners` 接口（详见 1.6.1）

- 自定义 **Reward**：在 GRPO 训练时通过 `--reward_funcs` 加载插件（详见 5.1.3）

- 自定义 **Agent Template**：继承 `BaseAgentTemplate`（详见 1.9）
