## 0. 定位

> 一句话：训练慢 = GPU / IO / 通信 / 数据 之一打满或没打满。先看 SM 占用，再分类排查。

---

## 1. 三类瓶颈

|瓶颈|征兆|对策|**GPU 算力**|SM 100% / Tensor Core 高|已经打满，可换更大 GPU 或更高效 kernel|
|---|---|---|---|---|---|
|**IO / 数据**|SM < 60%，CPU 高|workers ↑ / 缓存数据 / 排序分桶|**通信**|多机 step time 远超单机 ×N|NCCL 调优 / 减 grad_sync 频率|

## 2. 快速诊断

```bash
# A. SM 占用
nvidia-smi dmon -s u -c 60        # 60 秒
# B. nsys 抓取 30 秒
nsys profile -d 30 -o train.nsys-rep python train.py
# C. PyTorch Profiler
export TORCH_PROFILER_ENABLED=1
swift sft ... --max_steps 50
```

## 3. PyTorch Profiler

```python
import torch
from torch.profiler import profile, ProfilerActivity, schedule

with profile(
    activities=[ProfilerActivity.CPU, ProfilerActivity.CUDA],
    schedule=schedule(wait=2, warmup=2, active=6, repeat=1),
    on_trace_ready=torch.profiler.tensorboard_trace_handler('logs/prof'),
    record_shapes=True,
    with_stack=True,
) as prof:
    for step, batch in enumerate(loader):
        loss = model(**batch).loss
        loss.backward()
        opt.step(); opt.zero_grad()
        prof.step()
        if step >= 10:
            break

print(prof.key_averages().table(sort_by='cuda_time_total', row_limit=20))
```

## 4. dataloader 调优

```bash
swift sft ... \
    --dataloader_num_workers 8 \
    --dataloader_pin_memory true \
    --dataloader_persistent_workers true \
    --preprocess_num_proc 16
```

> [!important]
> 
> **经验**：单机 8 卡训练 dataloader_num_workers 不要少于 4×卡数。CPU 不够时考虑预处理后落 parquet 缓存。

## 5. 数据缓存

```python
# 把数据集预处理一次落 parquet，下次直接读
from datasets import load_dataset

ds = load_dataset('json', data_files='train.jsonl', split='train')
ds = ds.map(preprocess, num_proc=16)
ds.save_to_disk('cache/train.parquet')

# 训练时
from datasets import load_from_disk
ds = load_from_disk('cache/train.parquet')
```

## 6. 通信优化

```bash
# 1. 增大 grad_accum，减少 all_reduce 频率
swift sft ... --gradient_accumulation_steps 16

# 2. ZeRO-3 + bucketing
# 在 ds_config.json 里：
# "reduce_bucket_size": 5e8,
# "stage3_prefetch_bucket_size": 5e8

# 3. 用 BF16 (大多数 NCCL 已支持 bf16 all_reduce)
swift sft ... --bf16 true

# 4. 看 NCCL 是否走 IB
NCCL_DEBUG=INFO swift sft ... 2>&1 | grep 'NET/'
```

## 7. 性能基线参考

|设置|tokens/s/GPU|SM 占用|
|---|---|---|
|7B QLoRA · 1×A100 · seq 4k · BS 4|~3500-5000|60-75%|
|70B Megatron · TP=8 PP=2 · seq 4k|~700-1000|70-85%|

> [!important]
> 
> - **吞吐低于基线 30%** → 排查 dataloader / 通信 / GPU 散热。
> 
> - **温度 > 80°C** → 节流，吞吐下降，先解决散热。
> 
> - **PCIe Gen3 节点**：8 卡 ZeRO-3 的通信开销可能成主导。
