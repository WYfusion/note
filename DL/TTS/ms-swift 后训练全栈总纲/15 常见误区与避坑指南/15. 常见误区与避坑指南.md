## 0. 定位

> 一句话：本页集中梳理 ms-swift 使用中的**12 个高频踩坑**，按「数据 / 训练 / 对齐 / 部署 / 工程」五类列出「错误 → 反驳 → 修正」。

---

## 1. 数据类误区

> [!important]
> 
> **1. 混淆 Base 与 Instruct 模型**。Base 模型未经 SFT/RLHF，直接调词会输出乱码；Instruct 模型继续 SFT 需保护原有对齐。**修正**：明确起点，领域专家走 Base + CPT + SFT，轻量领域适配走 Instruct + LoRA。

> [!important]
> 
> **2. chat template 错位**。`--template` 与模型不匹配，训练后推理表现与 loss 指标不一致。**修正**：使用默认 template，不到过于确信不手动指定。

> [!important]
> 
> **3. 训练集污染评测集**。评测分数被「数据泄漏」严重夸大。**修正**：N-gram 检测，13-gram 重合率 < 0.5%。

> [!important]
> 
> **4. 用低质合成数据做大规模训练**。效果反降，甚至不如原始模型。**修正**：合成数据必须人工抽检样本，业务数据优先，合成补长尾。

---

## 2. 训练类误区

> [!important]
> 
> **5. 只看 loss 不看任务指标**。loss 下降不代表任务提升，有时反而过拟合。**修正**：每 1k 步跑小型 benchmark（抽 100题 GSM8K），同步贴 prompt采样阅读。

> [!important]
> 
> **6. learning_rate 随便拍**。LoRA SFT 用 1e-3（过高）、全参用 1e-4（过高）、DPO 用 1e-4（过高，崩坏）。**修正**：LoRA SFT 1e-4 / 5e-5，全参 SFT 5e-6 / 1e-5，DPO 5e-7 / 5e-6。

> [!important]
> 
> **7. QLoRA 训练后期待生产高吞吐**。QLoRA 训练产出仍需 merge + 优化量化才能生产。**修正**：merge 后采用 AWQ/GPTQ 正规量化 → vLLM/LMDeploy 部署。

---

## 3. 对齐类误区

> [!important]
> 
> **8. DPO 前 SFT 不充分**。DPO 是「对齐微调」而非能力训练。**修正**：先 SFT 达到业务下限再 DPO。

> [!important]
> 
> **9. GRPO 用于开放式生成**。GRPO 需要**可验证奖励**。创意写作 / 对话 / 闲聊不适用。**修正**：开放式任务走 SFT + DPO，仅有可验证奖励任务上 GRPO。

---

## 4. 部署类误区

> [!important]
> 
> **10. 量化后不重测**。AWQ/GPTQ 可能在某些任务上损失 5%+。**修正**：量化后用与训练一致的评测集重测，接受临界 < 3%。

> [!important]
> 
> **11. 部署后不压测长上下文与高并发**。测试环境表现与生产负载差距巨大。**修正**：Locust 压测，覆盖长 prompt + 多轮 + 突发并发。

---

## 5. 工程类误区

> [!important]
> 
> **12. adapter / merged / quantized 版本管理混乱**。出问题定位不到是哪阶段。**修正**：严格命名规范，三件产物独立存储，同一 model card 记录详细谱系。
