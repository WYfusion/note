## 0. 定位

> 一句话：Agent SFT 与普通 SFT 的差异在于**多轮 + 工具调用与环境 token 不计 loss**，需使用 loss_scale 与多轮拼接。

---

## 1. 如何考虑 Agent

Agent 轨迹是一个多轮对话：

```jsx
user → assistant(think+tool_call) → tool(observation) → assistant(think+tool_call) → ... → assistant(answer)
```

在 SFT 中只能**学 assistant 轮**，**user/tool 轮 不作为训练目标**。

## 3. ms-swift 默认行为

```bash
swift sft \
    --model Qwen/Qwen2.5-7B-Instruct \
    --dataset my-agent-data \
    --train_type lora \
    --target_modules all-linear \
    --max_length 8192 \
    --output_dir out/agent-sft
```

默认会自动对 user/tool 轮打 -100，仅 assistant 轮计损失。

## 4. loss_scale：细粒度控制

在 think / tool_call / answer 几个部分上可以分别加权：

```bash
swift sft ... \
    --loss_scale react \           # 预设：react / agentflan / qwen / hermes / default
```

常见预设：

|loss_scale|think|tool_call|answer|适用|
|---|---|---|---|---|
|react|1.0|2.0|1.0|强化工具调用|
|qwen|1.0|1.0|1.0|Qwen 原服|

## 5. 手写 loss mask

```python
def build_labels(input_ids, role_spans, mask_roles=('user', 'tool', 'system')):
    """role_spans: [(role, start, end)] 按 token 范围；mask_roles 不参与 loss."""
    labels = input_ids.clone()
    for role, s, e in role_spans:
        if role in mask_roles:
            labels[s:e] = -100
    return labels
```

## 6. Packing 下的 mask

多个样本 packing 进同一序列时，需加上 `attention_mask` 不跨越样本：

```bash
swift sft ... \
    --packing true \
    --attn_impl flash_attn   # FA varlen 支持 packing
```

ms-swift 默认处理 packing 与 loss mask，无需手写。

## 7. 踩坑提示

> [!important]
> 
> - **system token 是否计 loss**：默认不计。如果需要调整 system，手动设 `system_in_loss=true`。
> 
> - **工具 observation 轮必须是 tool role**：如果你写成 user role，ms-swift 会当作训练输入。
> 
> - **多轮拼接阀会被裁剪**：如果总长超过 max_length，ms-swift 默认从左侧裁剪（保留最后几轮）。
