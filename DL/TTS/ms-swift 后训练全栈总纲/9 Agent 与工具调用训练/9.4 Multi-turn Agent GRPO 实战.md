## 2. 环境函数设计

三个关键点：

1. **返回下一轮消息**（tool/user role）或 None（结束）。

2. **错误不中断 rollout**：工具报错也要返 observation（`Error: ...`）。

3. **限制轮数** `max_turns` 以防死循环。

```python
from swift.plugin import register_multi_turn
import json, re

MAX_DEPTH = 5

@register_multi_turn('search_calc')
def env(messages, **kwargs):
    last = messages[-1]
    if last['role'] != 'assistant':
        return None
    
    m = re.search(r'<tool_call>(.+?)</tool_call>', last.get('content', ''), re.S)
    if not m:
        return None    # 没有调用，结束
    
    try:
        call = json.loads(m.group(1))
    except json.JSONDecodeError:
        return {'role': 'tool', 'content': 'Error: invalid JSON'}
    
    name = call['name']
    args = call['arguments']
    
    if name == 'search':
        result = fake_search(args['query'])
    elif name == 'calc':
        try:
            result = str(eval(args['expr'], {'__builtins__': {}}, {}))
        except Exception as e:
            result = f'Error: {e}'
    else:
        result = f'Error: unknown tool {name}'
    
    return {'role': 'tool', 'name': name, 'content': result}

def fake_search(q):
    return f'[mock results for: {q}]'
```

## 4. Reward 设计示例

```python
@register_orm('agent_outcome')
def agent_outcome_reward(completion, solution=None, **kwargs):
    """检查最终答案是否包含 solution."""
    if not solution:
        return 0.0
    last_answer_match = re.search(r'<answer>(.+?)</answer>', completion, re.S)
    if last_answer_match:
        ans = last_answer_match.group(1).strip()
        return 1.0 if solution.strip() in ans else 0.0
    return 0.0

@register_orm('agent_efficiency')
def tool_call_count_penalty(completion, **kwargs):
    """调用越少越好（限制在 5 以内）."""
    n = len(re.findall(r'<tool_call>', completion))
    return max(0.0, 1.0 - 0.1 * n)
```

## 5. 资源设计

> [!important]
> 
> Multi-turn GRPO rollout 是双重加倍：每个 prompt 要走 G × max_turns 轮推理。G=4, max_turns=5 意味着 20 轮推理 / prompt。
> 
> **估算公式**： `rollout time ≈ G × avg_turns × single_turn_time`。在 7B 模型 + 4×A100 + vLLM 上，500 个 prompt 一轮训练 ≈ 30-60 分钟。
