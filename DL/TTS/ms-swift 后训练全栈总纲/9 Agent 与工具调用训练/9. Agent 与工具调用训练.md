## 0. 定位

> 一句话：本页梳理 ms-swift 对 **Agent / Function Calling** 训练的支持——从数据 schema 、Agent Template、Loss Mask 设计到常见 Agent benchmark 训练套路。

---

## 1. Agent 训练与普通 SFT 的差异

> [!important]
> 
> 普通 SFT 只训练「user 问 → assistant 答」。Agent 训练要训练「**多轮决策**：读取 tool 定义 → 决策调用哪个 tool → 读取 tool 结果 → 进一步推理」。这要求三件事：(1) 数据含 `tools` 与 tool 消息；(2) Template 能处理 tool 消息；(3) loss mask 设在同时包含 assistant 的 thought、action、最终回复。

---

## 2. Agent 数据格式

```json
{
  "messages": [
    {"role": "system", "content": "You are a helpful agent."},
    {"role": "user", "content": "深圳现在温度多少?"},
    {"role": "assistant", "content": null, "tool_calls": [{"type": "function", "function": {"name": "get_weather", "arguments": "{\"city\": \"shenzhen\"}"}}]},
    {"role": "tool", "content": "{\"temp\": 28}"},
    {"role": "assistant", "content": "深圳现在温度 28°C。"}
  ],
  "tools": [{
    "type": "function",
    "function": {
      "name": "get_weather",
      "description": "获取某城市实时天气",
      "parameters": {"type": "object", "properties": {"city": {"type": "string"}}, "required": ["city"]}
    }
  }]
}
```

## 3. Agent Template 横向对照

不同模型家族的 tool 调用格式不同，ms-swift 提供统一抽象：

|Agent Template|典型模型|tool_call 格式|
|---|---|---|
|**hermes**|Qwen2.5/3, Hermes|`<tool_call>{json}</tool_call>`|
|**toolbench**|ToolLLaMA|thought + action + observation|
|**glm4**|GLM-4 Series|官方专用格式|
|**react**|通用|ReAct 经典 thought/action/observation|

```bash
swift sft --model Qwen/Qwen3-8B \
    --dataset my_agent.jsonl \
    --agent_template hermes \
    --loss_scale all \
    --train_type lora
```

> [!important]
> 
> **工程推荐**：Qwen2.5+ 选 hermes；ToolLLaMA 生态用 toolbench；GLM 模型专用 glm4；其余应能在用 react 作为通用 fallback。

---

## 4. Loss Scale：Agent 专属限定词

```bash
# 默认 last_round：仅最后一轮 assistant 计 loss
--loss_scale last_round

# all ：所有 assistant 中间说话都计 loss（推荐用于 Agent）
--loss_scale all

# react ：对 thought / action / observation 加权
--loss_scale react
```

> [!important]
> 
> **误区**：Agent 训练使用 last_round → 如果轨迹中间几轮错误决策模型不会学习。**必须** `**--loss_scale all**`。

---

## 5. 完整训练 → 推理示例

```bash
# 训练
swift sft --model Qwen/Qwen3-8B \
    --dataset AI-ModelScope/MS-Agent-V2 \
    --train_type lora \
    --target_modules all-linear \
    --agent_template hermes \
    --loss_scale all \
    --learning_rate 1e-4 \
    --output_dir output/agent

# 部署为 OpenAI 兼容 API（含 function call）
swift deploy --model output/agent/checkpoint-xxx \
    --infer_backend vllm \
    --merge_lora true \
    --served_model_name my-agent
```

## 6. Agent + GRPO

Agent 任务是 GRPO 的天然套用场景（工具调用成功与否是可验证奖励）。ms-swift 支持 multi-turn agent rollout：

```bash
swift rlhf --rlhf_type grpo \
    --model output/agent_sft \
    --dataset agent_grpo.jsonl \
    --reward_funcs tool_use_success format \
    --num_generations 8 \
    --max_completion_length 4096
```
