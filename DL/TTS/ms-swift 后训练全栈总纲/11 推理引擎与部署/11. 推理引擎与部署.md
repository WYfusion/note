## 0. 定位

> 一句话：本页梳理 ms-swift 支持的四种推理后端——`transformers` / **vLLM** / **SGLang** / **LMDeploy**——从 **PagedAttention / RadixAttention / Speculative Decoding** 的加速原理到部署套路。

---

## 1. 四种后端一览

|后端|关键技术|适用场景|启动参数|
|---|---|---|---|
|**transformers**|HF 原生，无优化|调试、少量推理|`--infer_backend transformers`|
|**vLLM**|PagedAttention、continuous batching|**高并发**在线服务|`--infer_backend vllm`|
|**SGLang**|RadixAttention prefix 缓存|**多轮对话**、Agent、同 prompt 多 sample|`--infer_backend sglang`|
|**LMDeploy**|TurboMind、原生 W4A16|**高吞吐低延迟**、AWQ 量化部署|`--infer_backend lmdeploy`|

---

## 2. 三大加速技术原理

![[11. 推理引擎与部署 - 2. 三大加速技术原理.excalidraw|800]]

> [!important]
> 
> **为什么 SGLang 在 Agent 场景赢**：Agent 轮间输入共享 system + history。RadixAttention 按 prefix tree 存 KV cache，**重复 prefix 的 KV 仅计一次**，加速几乎与轮数成正比。

## 3. 不同后端启动示例

```bash
# 1. transformers 快验证
swift infer --model output/sft/checkpoint-xxx \
    --infer_backend transformers \
    --temperature 0.7 --max_new_tokens 512

# 2. vLLM 高并发 OpenAI 兼容 API
swift deploy --model output/sft/checkpoint-xxx \
    --merge_lora true \
    --infer_backend vllm \
    --tensor_parallel_size 2 \
    --gpu_memory_utilization 0.9 \
    --max_model_len 8192 \
    --served_model_name my-model

# 3. SGLang 多轮优化
swift deploy --model output/sft/checkpoint-xxx \
    --merge_lora true \
    --infer_backend sglang \
    --enable_prefix_caching true

# 4. LMDeploy 低延迟（含 AWQ 量化）
swift deploy --model output/sft-awq \
    --infer_backend lmdeploy \
    --quant_policy 4
```

## 4. 调用 OpenAI 兼容 API

ms-swift `swift deploy` 启动后提供标准 OpenAI 接口：

```python
# OpenAI Python SDK 直接调用
from openai import OpenAI

client = OpenAI(
    api_key='EMPTY',
    base_url='http://localhost:8000/v1',
)

resp = client.chat.completions.create(
    model='my-model',
    messages=[
        {'role': 'system', 'content': 'You are a helpful assistant.'},
        {'role': 'user', 'content': '解释 LoRA 原理'},
    ],
    temperature=0.7,
    max_tokens=512,
    stream=False,
)
print(resp.choices[0].message.content)
```

## 5. 选型决策表

|应用类型|首选后端|原因|
|---|---|---|
|在线中高并发 API|**vLLM**|PagedAttention + continuous batching 绿本优势|
|多轮对话 / Agent|**SGLang**|RadixAttention 加速 prefix|
|快调试 / 个人使用|**transformers**|零配置|
|AWQ / GPTQ 量化低延迟|**LMDeploy**|TurboMind 原生 W4A16|

## 6. Speculative Decoding与推理压缩

ms-swift 在 vLLM/SGLang 中支持 Speculative Decoding：用小 draft 模型预测 N 个 token，大模型一次验证：

```bash
swift deploy --model Qwen/Qwen3-32B \
    --infer_backend vllm \
    --speculative_model Qwen/Qwen3-1.7B \
    --num_speculative_tokens 5
```

> [!important]
> 
> **适用判据**：draft 与 target 需**同词表**，预测接受率越高加速越明显，常见 1.5-2.5x 提速。
