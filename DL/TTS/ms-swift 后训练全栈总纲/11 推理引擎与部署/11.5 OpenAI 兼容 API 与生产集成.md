## 0. 定位

> 一句话：ms-swift `deploy` 启动后提供与 OpenAI 完全兼容的 REST API，可直接接入 LangChain / LlamaIndex 等生态。

---

## 1. 支持的端点

|端点|说明|`POST /v1/chat/completions`|对话接口|
|---|---|---|---|
|`POST /v1/completions`|补全接口|`POST /v1/embeddings`|Embedding（task_type=embedding）|
|`POST /v1/rerank`|仅 reranker 模式下可用|`GET /v1/models`|查看模型列表（含 LoRA）|

## 2. Function Calling

```python
from openai import OpenAI
client = OpenAI(base_url='http://localhost:8000/v1', api_key='dummy')

tools = [{
    'type': 'function',
    'function': {
        'name': 'search',
        'parameters': {'type':'object','properties':{'query':{'type':'string'}}},
    },
}]
resp = client.chat.completions.create(
    model='my-qwen',
    messages=[{'role':'user','content':'查下北京今天天气'}],
    tools=tools,
    tool_choice='auto',
)
print(resp.choices[0].message.tool_calls)
```

需在启动时使用 `--enable_auto_tool_choice` 与匹配的 `--tool_call_parser`：

```bash
swift deploy ... \
    --enable_auto_tool_choice true \
    --tool_call_parser hermes
```

## 3. LangChain 集成

```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(
    base_url='http://localhost:8000/v1',
    api_key='dummy',
    model='my-qwen',
    temperature=0.7,
)
print(llm.invoke('Hello').content)
```

## 4. LlamaIndex 集成

```python
from llama_index.llms.openai_like import OpenAILike

llm = OpenAILike(
    api_base='http://localhost:8000/v1',
    api_key='dummy',
    model='my-qwen',
    is_chat_model=True,
)
print(llm.complete('Hello').text)
```

关键考虑：

1. **负载均衡**：多个复本后 Nginx 轮询即可。

1. **健康检查**：调 `GET /v1/models` 验证服务可用。

1. **限流**：在路由层 (如 Envoy) 加 rate limit。

- `vllm_num_requests_running` / `_waiting`

## 7. 踩坑提示

> [!important]
> 
> - **API key 必须传**：OpenAI SDK 检查，传个 `dummy` 即可。生产中才在路由层验证。
> 
> - **负载均衡 sticky session**：SGLang 的 RadixAttention 仅在同节点缓存，有效路由需 sticky。
> 
> - **多模型部署**：一个 `swift deploy` 仅能服务一个主模型 + 多个 LoRA。混部不同主模型要多起进程。
