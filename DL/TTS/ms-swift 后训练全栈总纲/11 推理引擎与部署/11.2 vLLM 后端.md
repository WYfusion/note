## 0. 定位

> 一句话：vLLM = PagedAttention + Continuous Batching，主流首选，在同一机器上吞吐远超 transformers。

---

## 1. 核心机制

|机制|作用|**PagedAttention**|KV cache 分页存储，避免碎片|
|---|---|---|---|
|**Continuous Batching**|动态拼接多请求，打满 GPU|**CUDA Graph**|减少 launch 开销|
|**Speculative Decoding**|小模型猜大模型验证|**Chunked Prefill**|长 prompt 分块预填充避免狭路|

## 2. 启动服务

```bash
swift deploy \
    --model out/qwen-sft \
    --infer_backend vllm \
    --port 8000 \
    --max_model_len 8192 \
    --tensor_parallel_size 2 \
    --gpu_memory_utilization 0.9 \
    --max_num_seqs 256 \
    --enable_chunked_prefill true \
    --max_num_batched_tokens 4096
```

## 3. 常用参数详解

|参数|含义|推荐|
|---|---|---|
|`tensor_parallel_size`|TP 并行数|1 / 2 / 4 / 8|
|`max_num_seqs`|并发序列上限|128-512|
|`enforce_eager`|禁用 CUDA Graph|false (生产)|

## 4. 调用示例

```python
from openai import OpenAI

client = OpenAI(base_url='http://localhost:8000/v1', api_key='dummy')

# 流式
stream = client.chat.completions.create(
    model='my-qwen',
    messages=[{'role':'user','content':'写一首诗'}],
    stream=True,
)
for chunk in stream:
    delta = chunk.choices[0].delta.content or ''
    print(delta, end='', flush=True)
```

## 5. 压测

```bash
# vllm-benchmark 脚本
python -m vllm.entrypoints.benchmarks.benchmark_serving \
    --model my-qwen \
    --dataset-name sharegpt \
    --dataset-path ShareGPT_V3_unfiltered_cleaned_split.json \
    --num-prompts 1000 \
    --request-rate 10 \
    --base-url http://localhost:8000
```

关键指标：

- **TTFT** (Time To First Token)：首 token 延迟。

- **TPOT** (Time Per Output Token)：生成中每 token 耗时。

- **Throughput**：tokens/s。

## 6. 资源估算

> [!important]
> 
> **经验公式**：Qwen2.5-7B + 8k 上下文 + 1×A100 80G：
> 
> - max_num_seqs ≈ 64-128
> 
> - 吞吐量 ≈ 2000-3000 tokens/s (生成)
> 
> - TTFT ≈ 50-100 ms (空载) / 200-500 ms (高负载)

## 7. 踩坑提示

> [!important]
> 
> - **启动慢**：首次启动需 build CUDA graph，可能 1-3 分钟。
> 
> - **OOM 调低 gpu_memory_utilization**：默认 0.9。如果遇到 OOM 先试 0.85。
> 
> - **技能不被支持会退连 transformers**：部分新模型 vLLM 未适配，会报警告。
> 
> - **AWQ/GPTQ 量化**：加 `--quantization awq` 或 `gptq`，vLLM 会自动识别。
