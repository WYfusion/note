## 0. 定位

> 一句话：ms-swift 提供三个推理后端（vLLM / SGLang / LMDeploy）与一个 OpenAI 兼容服务入口，调参一致。

---

## 1. 三个后端对比

|后端|作者|优势|适用|
|---|---|---|---|
|**SGLang**|LMSys|RadixAttention，多轮/Agent 快|Agent / RAG|
|**transformers**|HF|通用，但慢|调试|

## 2. swift infer：脱机推理

```bash
# 一次性推理 (调试用)
swift infer \
    --model out/qwen-sft \
    --infer_backend vllm \
    --max_model_len 8192 \
    --temperature 0.7 \
    --top_p 0.9 \
    --max_new_tokens 1024
```

交互式下会提示输入。批量推理可加 `--val_dataset my-test-data`。

## 3. swift deploy：起服务

```bash
swift deploy \
    --model out/qwen-sft \
    --infer_backend vllm \
    --port 8000 \
    --host 0.0.0.0 \
    --max_model_len 8192 \
    --tensor_parallel_size 2 \
    --gpu_memory_utilization 0.9 \
    --served_model_name my-qwen
```

启动后提供标准 OpenAI 兼容 API。

## 4. 参数与后端映射

|ms-swift 参数|含义|后端实现|
|---|---|---|
|`--max_model_len`|最大序列长度|vLLM `max_model_len`|
|`--gpu_memory_utilization`|GPU 显存使用比|vLLM 同名|
|`--max_num_seqs`|最大并发序列|vLLM 同名|

## 5. 使用 OpenAI SDK 访问

```python
from openai import OpenAI

client = OpenAI(base_url='http://localhost:8000/v1', api_key='dummy')

resp = client.chat.completions.create(
    model='my-qwen',
    messages=[
        {'role': 'system', 'content': 'You are a helpful assistant.'},
        {'role': 'user',   'content': 'Hello!'},
    ],
    temperature=0.7,
    max_tokens=512,
)
print(resp.choices[0].message.content)
```

## 7. LoRA 热据载

vLLM 支持动态加载 LoRA：

```bash
swift deploy \
    --model Qwen/Qwen2.5-7B-Instruct \
    --adapters my-lora-1=out/lora1 my-lora-2=out/lora2 \
    --infer_backend vllm \
    --enable_lora true \
    --max_loras 4 --max_lora_rank 16
```

调用时使用 `model='my-lora-1'` 动态选择。

> [!important]
> 
> - **不同后端不能混用**：vLLM 与 SGLang 是互斥进程。
> 
> - **检点路径需是 HF 格式**：Megatron 训练后需先 `swift export --to_hf true`。
> 
> - **AWQ/GPTQ 量化模型**：需要后端支持 (vLLM 均支持)。
