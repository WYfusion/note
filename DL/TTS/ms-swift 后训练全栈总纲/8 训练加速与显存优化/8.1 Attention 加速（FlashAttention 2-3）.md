## 0. 定位

> 一句话：FlashAttention = IO-aware 的 attention 实现，**O(N) 内存 + 2-3x 提速**。仅加一个参数。

---

## 1. 原理背景

传统 attention 内存复杂度 O(N²)：需保存 attention scores 矩阵 [N, N]。FA 采用 tiling + recomputation，避免在 HBM 中位变 [N, N] 矩阵：

## 5. 验证

```python
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

tok = AutoTokenizer.from_pretrained('Qwen/Qwen2.5-7B-Instruct')
model = AutoModelForCausalLM.from_pretrained(
    'Qwen/Qwen2.5-7B-Instruct',
    attn_implementation='flash_attention_2',
    torch_dtype=torch.bfloat16,
    device_map='cuda',
)
print(model.config._attn_implementation)   # flash_attention_2

x = tok('hello', return_tensors='pt').to('cuda')
with torch.no_grad():
    y = model(**x)
print(y.logits.shape)
```

## 6. 常见问题

> [!important]
> 
> - **fp32 不支持**：FA2/FA3 仅支持 fp16/bf16。训练中不能使用 fp32 attention。
> 
> - **head_dim 限制**：FA2 支持 head_dim ≤ 256；FA3 仅支持 ≤ 128（部分架构需 padding）。
> 
> - **变长序列**：使用 `flash_attn_varlen_func` 或试 padding-free。
> 
> - **编译失败**：需 CUDA 与 PyTorch 版本匹配，考虑预编译 wheel。

## 7. 选择推荐

> [!important]
> 
> - **A100**：FA2（默认）。
> 
> - **H100**：FA3 试验。FA3 在超长序列上增益更明显（> 4k）。
> 
> - **其他 GPU**（未支持 FA 的）：SDPA 允价。
> 
> - **不要用 eager**：eager 是参考实现，仅用于调试。
