## 0. 定位

> 一句话：本页梳理「**让训练更快 / 更省显存**」的五类手段：**混精度数值**、**FlashAttention**、**Liger Kernel**、**梯度检查点**、**量化加载（QLoRA/HQQ/AQLM）**。

---

## 1. 五类手段一览

|手段|加速贡献|显存贡献|代价|
|---|---|---|---|
|**bf16 / fp16**|2-3x|~50%|fp16 需 loss scale|
|**FlashAttention 2/3**|1.5-3x attn|O(N) attn 内存|需 GPU 支持|
|**Liger Kernel**|吞吞量 +20%|峰值 -60%|只适部分模型|
|**Gradient Checkpointing**|-20-30%|**-50% 激活**|反向重计算|
|**QLoRA / HQQ / AQLM**|~同 LoRA|**4 位加载**，可 -75%|量化误差|

---

## 2. Liger Kernel：一行开启的 +20% / -60%

Liger Kernel 是 LinkedIn 开源的 Triton kernel 库，重写了 RMSNorm/RoPE/SwiGLU/CrossEntropy/FusedLinearCE等热点算子，**与 HF Trainer 无缝集成**。

```bash
swift sft --model Qwen/Qwen3-8B \
    --use_liger_kernel true \
    --packing true \
    --attn_impl flash_attn \
    ...
```

> [!important]
> 
> **为什么 -60% 显存**：FusedLinearCrossEntropy 避免了 [B,T,V] 的 logits 全实体化，直接在 softmax 后等价地计算 loss—— vocabulary 越大提升越明显。

---

## 3. Gradient Checkpointing例例必选

```bash
swift sft --gradient_checkpointing true \
    --gradient_checkpointing_kwargs '{"use_reentrant": false}' ...
```

**原理**：反向传播时**重新计算**中间激活，用 ~25% 训练时间换 ~50% 激活显存。**低显存场景例行开启**。

---

## 4. 量化训练方法对比

|方法|位宽|质量|训练速度|
|---|---|---|---|
|**BNB (NF4)**|4|中|中|
|**HQQ**|2/3/4/8|高 (同位宽)|高|
|**AQLM**|2|极高|低（压缩贵）|
|**EETQ**|8|高|高|

```bash
# QLoRA 标准配置
swift sft --model Qwen/Qwen3-32B \
    --train_type lora \
    --quant_method bnb --quant_bits 4 \
    --bnb_4bit_quant_type nf4 \
    --bnb_4bit_use_double_quant true \
    --gradient_checkpointing true
```

---

## 5. FSDP-QLoRA：70B 模型单机 4×A100 训练

2024 年 [Answer.AI](http://Answer.AI) 提出的 FSDP-QLoRA 让「70B 模型在 4×80GB A100」成为可能：

```bash
NPROC_PER_NODE=4 swift sft \
    --model Qwen/Qwen3-72B \
    --train_type lora \
    --quant_method bnb --quant_bits 4 \
    --fsdp full_shard auto_wrap \
    --fsdp_config conf/fsdp_qlora.json \
    --gradient_checkpointing true
```
