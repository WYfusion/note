## 前置知识

- [[8. 训练加速与显存优化]]

## 0. 定位

> 一句话：Unsloth = LoRA 专用加速框架，在单卡上能提供 2-5x 加速 + 显著节省显存；ms-swift 提供存入点。

---

## 1. 能力与范围

|项|会|不会|
|---|---|---|
|GPU|单卡 (Ada / Hopper / Ampere)|多卡 (受限)|
|量化|bnb 4-bit|fp8/int8|

## 2. 原理总览

Unsloth 采用三点优化：

1. **手写 Triton kernel**：attention、MLP、RoPE 等全部重实现。

2. **manual backward**：在 LoRA 上手写 backward pass，比 autograd 快。

3. **动态 chunk**：控动 cross_entropy 与 LoRA 设越拼块大小。

## 3. ms-swift 上启用

```bash
swift sft \
    --model Qwen/Qwen2.5-7B-Instruct \
    --dataset alpaca \
    --train_type lora \
    --tuner_backend unsloth \         # 开启 Unsloth
    --quant_method bnb \
    --quant_bits 4 \
    --max_length 4096 \
    --learning_rate 1e-4 \
    --output_dir out/unsloth-qwen
```

## 4. Python 接口

```python
from swift.llm import sft_main, TrainArguments

sft_main(TrainArguments(
    model='Qwen/Qwen2.5-7B-Instruct',
    dataset=['alpaca'],
    train_type='lora',
    tuner_backend='unsloth',
    quant_method='bnb',
    quant_bits=4,
    max_length=4096,
    output_dir='out/unsloth-qwen',
))
```

## 5. 提速示例

```jsx
Qwen2.5-7B + LoRA + alpaca · 1×RTX 4090
                    显存       iter/s
  HF + FA2:         18 GB      1.0
  HF + FA2 + Liger: 14 GB      1.4
  Unsloth:           9 GB      2.5
```

## 6. 区别与选择

![[8.5 Unsloth - 6. 区别与选择.excalidraw|800]]

> [!important]
> 
> - **单卡 LoRA 首选 Unsloth**，资源受限是主要场景。
> 
> - **多卡 LoRA 不选 Unsloth**，多卡支持不如原生 FSDP/DeepSpeed。
> 
> - **不要与 Liger 同启用**：Unsloth 已覆盖 Liger 能力范围。

## 7. 踩坑提示

> [!important]
> 
> - **monkey-patch 可能冲突版本**：Unsloth 会覆盖 transformers 内部函数，transformers 升级后可能不佃谐。调试不颗代，退连 HF 后端调查。
> 
> - **仅上传原 LoRA**：训练后上传调用 `swift export --merge_lora true` 合并后以 HF 格式发布。
> 
> - **全参训练不可用**：仅限 LoRA / QLoRA。
