## 0. 定位

> 一句话：FSDP-QLoRA 是 [Answer.AI](http://Answer.AI) 提出的组合，允许在**两张 24G GPU 上训 70B 全参 LoRA**，ms-swift 一行开启。

---

## 1. 原理合并
![[8.4 FSDP-QLoRA 详解 - 1. 原理合并.excalidraw|800]]
```mermaid
flowchart LR
    M["原始 70B fp16 (140GB)"] --> Q["4-bit 量化\n· QLoRA\n(35GB)"]
    Q --> Sh["FSDP 分片\n2卡· 17.5GB · 卡"]
    Sh --> Lora["+ LoRA adapter\n·1.5GB"]
    Lora --> Ok[" ~24GB / GPU · 实现训练"]
```
关键点：

1. **底层 4-bit 量化**（NF4）压缩原始权重 4x。

2. **FSDP 分片** 将量化后的权重跨卡分片。

3. **LoRA adapter** 在在 fp16/bf16 训练。

4. **梯度 / 优化器状态** 仅对 LoRA 参数。

## 2. 与 DeepSpeed-Z3+QLoRA 区别

|项|FSDP+QLoRA|DeepSpeed Z3+QLoRA|
|---|---|---|
|跨卡分片|FSDP native|Z3|
|适用场景|资源受限 (2-4×24G)|2-4×80G|

## 3. 资源预算

70B FSDP-QLoRA：

```jsx
底层 NF4 4-bit:    70B · 0.5 = 35 GB
FSDP shard × 2:     35 / 2 = 17.5 GB
LoRA fp16:           ~ 1.5 GB (rank=64)
activation:          变动（与上下文相关）
----------------------------·
合计 ~ 22-24 GB / GPU
```

## 4. 命令

```bash
# 2×RTX 4090 (24G) 训 70B
NPROC_PER_NODE=2 swift sft \
    --model Qwen/Qwen2.5-72B-Instruct \
    --dataset alpaca-zh \
    --train_type lora \
    --lora_rank 64 \
    --target_modules all-linear \
    --quant_method bnb \
    --quant_bits 4 \
    --bnb_4bit_quant_type nf4 \
    --bnb_4bit_compute_dtype bfloat16 \
    --bnb_4bit_use_double_quant true \
    --use_fsdp_qlora true \         # 关键开关
    --fsdp_config_file fsdp_qlora.json \
    --gradient_checkpointing true \
    --learning_rate 1e-4 \
    --max_length 2048 \
    --output_dir out/qwen-72b-fsdp-qlora
```

## 5. fsdp_qlora.json 示例

```jsx
{
  "compute_environment": "LOCAL_MACHINE",
  "distributed_type": "FSDP",
  "fsdp_config": {
    "fsdp_auto_wrap_policy": "TRANSFORMER_BASED_WRAP",
    "fsdp_backward_prefetch": "BACKWARD_PRE",
    "fsdp_state_dict_type": "SHARDED_STATE_DICT",
    "fsdp_offload_params": false,
    "fsdp_sharding_strategy": "FULL_SHARD",
    "fsdp_use_orig_params": false
  },
  "mixed_precision": "bf16"
}
```

## 6. 合并 LoRA

```bash
# 合并后仅是 70B 4-bit 模型，加载到一卡进行推理即可
swift export \
    --adapters out/qwen-72b-fsdp-qlora \
    --merge_lora true \
    --output_dir merged/qwen-72b-q4-lora
```

推理：

```bash
swift infer --model merged/qwen-72b-q4-lora \
    --infer_backend vllm --max_model_len 4096
```

## 7. 踩坑提示

> [!important]
> 
> - **PyTorch 版本 ≥ 2.2**：FSDP 与 bitsandbytes 佃谐需 PyTorch 2.2+。
> 
> - **不能与 DeepSpeed 组合**：FSDP-QLoRA 与 DeepSpeed Z3 互斥。
> 
> - **检点是 sharded**：FSDP SHARDED_STATE_DICT 需同样 GPU 数才能加载。启用 `--save_only_model true` 只保存 LoRA 部分。
> 
> - **OOM 问题**：如果仍 OOM，调低 `lora_rank`、`max_length`，启用 `--gradient_checkpointing true`。
