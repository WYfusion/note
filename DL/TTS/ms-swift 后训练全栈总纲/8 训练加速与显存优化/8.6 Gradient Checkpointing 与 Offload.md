## 0. 定位

> 一句话：Gradient Checkpointing 用计算换显存；Offload 用 CPU/NVMe 换显存。二者是 OOM 时的最后手段。

---

## 1. Gradient Checkpointing

原理：反向传播时不保留中间 activation，而重新计算 forward：

![[8.6 Gradient Checkpointing 与 Offload - 1. Gradient Checkpointing - 图 01.excalidraw|800]]

增益：显存 × 0.5；代价：训练速度 × 0.7-0.8。

## 2. 启用

```bash
swift sft \
    --gradient_checkpointing true \
    --gradient_checkpointing_kwargs '{"use_reentrant": false}' \
    ...
```

## 3. selective recompute (Megatron)

Megatron-SWIFT 提供更细粒度：

```bash
megatron sft \
    --recompute_granularity selective \   # full / selective
    --recompute_method block \             # block / uniform
    --recompute_num_layers 1 \
    ...
```

`selective` 仅重计算 attention（费镂大部分显存但计算多）。`full` 重算所有 transformer block。

## 4. CPU Offload

DeepSpeed Z3 与 FSDP 都提供：

```jsx
// ds_z3_offload.json
{
  "zero_optimization": {
    "stage": 3,
    "offload_optimizer": {"device": "cpu", "pin_memory": true},
    "offload_param":     {"device": "cpu", "pin_memory": true}
  }
}
```

```bash
swift sft --deepspeed ds_z3_offload.json ...
```

FSDP：

```jsx
{"fsdp_config": {"fsdp_offload_params": true}}
```

## 5. NVMe Offload

在极端资源下，可将优化器状态进一步放到 NVMe：

```jsx
{
  "zero_optimization": {
    "stage": 3,
    "offload_optimizer": {
      "device": "nvme",
      "nvme_path": "/mnt/nvme/zero_offload",
      "buffer_count": 4,
      "buffer_size": 1e9
    }
  }
}
```

## 6. 代价对比

|手段|显存节省|速度损失|适用|
|---|---|---|---|
|FA2/3|activation 纯下降|加速|首选|
|NVMe offload|~95%|80-90%|实验、不宜生产|

## 7. 选择顺序

![[8.6 Gradient Checkpointing 与 Offload - 7. 选择顺序 - 图 02.excalidraw|800]]

> [!important]
> 
> - **CPU offload 是最后手段**：训练变很慢（CPU-GPU 带宽低）。能加 GPU 就加 GPU。
> 
> - **NVMe offload 不宜生产**：除非你运营发展体验。常见 NVMe IO 逻辑不佃谐锐。
> 
> - **械选 selective recompute**：在 Megatron 中优先 selective，比 full 提速 30%+。
