## 0. 定位

> 一句话：Liger Kernel = LinkedIn 开源的一组 Triton kernel，将 LayerNorm/RMSNorm/RoPE/CrossEntropy/SwiGLU 等点名为 fused kernel，节省 ~20% 显存 + 提速 10-30%。

---

## 1. 启用

```bash
swift sft \
    --model Qwen/Qwen2.5-7B-Instruct \
    --dataset alpaca \
    --use_liger_kernel true \         # 一个开关
    --attn_impl flash_attn ...
```

## 2. Liger 覆盖的算子

|算子|优化内容|预期增益|RMSNorm|fused fwd/bwd|显存 -10% · 速度 +5%|
|---|---|---|---|---|---|
|RoPE|fused apply|显存 ≈ 0 · 速度 +3%|SwiGLU|fused SiLU·gate|显存 -5% · 速度 +5%|
|**CrossEntropy**|chunk + log-softmax|显存 -50%(LLM head)|FusedLinearCE|合并 lm_head + CE|提速明显|

## 3. 原理：FusedLinearCrossEntropy

常规 LM 训练中`lm_head` 输出 logits [B, T, V]，V (vocab) 可达 152k。存储 logits 需 ~B×T×V×2 bytes，极费显存。

Liger 将 `lm_head` 与 CE 跨越为单一 kernel，在 chunk 上计算 logits 后直接出损失，logits 不位变 HBM。

```python
# 伪代码
import torch

def fused_linear_ce(hidden, weight, labels, chunk_size=1024):
    """hidden: [B*T, h], weight: [V, h], labels: [B*T]."""
    total_loss, n = 0.0, 0
    for i in range(0, hidden.shape[0], chunk_size):
        h_chunk = hidden[i:i+chunk_size]                  # [c, h]
        l_chunk = labels[i:i+chunk_size]
        logits  = h_chunk @ weight.T                       # [c, V]  (这里仅 chunk)
        loss    = torch.nn.functional.cross_entropy(
                      logits, l_chunk, reduction='sum')
        total_loss += loss
        n += (l_chunk != -100).sum()
    return total_loss / n
```

## 4. 与 FA 组合

Liger 与 FA 是互补的：

- FA 改善 attention（增益随序列长度增加）。

- Liger 改善其他层（增益随 vocab/hidden 并增加）。

合一使用，在 7B + 4k 上下文可减显存 30%+。

## 6. 踩坑提示

> [!important]
> 
> - **仅在 Triton 可用 GPU**：Triton 需 NVIDIA Compute 能力 ≥ 7.0。Volta+ 都可。
> 
> - **模型架构需适配**：Liger 仅对主流架构（LLaMA / Qwen / Mistral / Gemma）提供 kernel，不在表中的架构上不生效。
> 
> - **与某些 custom op 冲突**：如遇到报错，试设 `--use_liger_kernel false` 缩小问题。
