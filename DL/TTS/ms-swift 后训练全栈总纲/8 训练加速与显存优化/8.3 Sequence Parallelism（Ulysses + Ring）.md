## 0. 定位

> 一句话：序列并行将 token 维度拆到多卡，是训练 64k+ 上下文的必选项；ms-swift 提供 Ulysses、Ring、以及两者混合。

---

## 1. 为什么需要

attention 计算量 × N²。在 64k 上下文下：

- 逻辑 attention compute：64000² = 4.1B ·· ·运算 / token。

- KV cache：64k × d × layers × 2，超过单卡能力。

解决：拆分序列。

## 2. Ulysses (DeepSpeed)

![[8.3 Sequence Parallelism（Ulysses + Ring） - 2. Ulysses (DeepSpeed).excalidraw|800]]

原理：拆 N (序列) → 多卡计算路索 → all-to-all → 拆 head 后多卡计算 attention。

## 3. Ring Attention

不同于 Ulysses 的 all-to-all，Ring Attention 采用圈 K/V：

```jsx
GPU0  GPU1  GPU2  GPU3
Q0    Q1    Q2    Q3   (各卡保留自己的 Q)
K0    K1    K2    K3   (依次传递)
V0    V1    V2    V3

step 0: GPU0 计 Q0 · K0ス
step 1: GPU0 接受 K1V1, 计 Q0 · K1
· · ·
step 3: GPU0 接受 K3V3, 计 Q0 · K3
  ····迭代后梦合
```

优点：通信量低于 Ulysses，不需 all-to-all。

## 4. ms-swift 启用

```bash
# Ulysses 
swift sft \
    --model Qwen/Qwen2.5-7B-Instruct \
    --dataset my-long-context \
    --max_length 65536 \
    --attn_impl flash_attn \
    --sequence_parallel_size 4 \         # SP=4
    --sp_type ulysses \                   # ulysses / ring / unified
    --train_type lora ...

# Ring Attention (需 zigzag 模式避免负载不均)
swift sft ... \
    --sequence_parallel_size 4 \
    --sp_type ring
```

## 5. unified SP（Ulysses+Ring）

Unified SP 代码的是 Ulysses×Ring 混合，可在超多卡上拓展 SP 跨节点：

```bash
swift sft \
    --max_length 131072 \
    --sequence_parallel_size 16 \        # SP=16
    --sp_type unified \
    --sp_ulysses_degree 4 \               # Ulysses=4
    --sp_ring_degree 4 \                  # Ring=4 (4×4=16=SP)
    ...
```

## 6. 实际选型表

|上下文|SP 推荐|类型|≤ 8k|不需|-|
|---|---|---|---|---|---|
|16-32k|SP=2|Ulysses|64k|SP=4|Ulysses 或 Ring|
|128k|SP=8-16|Unified|1M|SP=64+|Unified · Ring 为主|

## 7. 踩坑提示

> [!important]
> 
> - **SP 与 DP 乘积为总卡数**：SP=4、DP=2 需 8 卡，不能重复。
> 
> - **序列长度需能被 SP 整除**：如果 N=131073, SP=4，会出错。
> 
> - **跨节点 SP 带宽决定能不能跨**：Ring 跨节点需 InfiniBand。
> 
> - **与 TP 克制**： Megatron 中 SP 是 TP 的似拼，含义不同。 ms-swift 默认 SP 必须是 Ulysses/Ring。
