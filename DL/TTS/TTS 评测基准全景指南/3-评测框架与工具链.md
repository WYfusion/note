## 一、Seed-TTS-Eval 官方脚本


📌**最轻量的零样本 TTS 评测工具**

- **来源**：ByteDance 随 Seed-TTS 论文发布
- **GitHub**：[BytedanceSpeech/seed-tts-eval](https://github.com/BytedanceSpeech/seed-tts-eval)
- **评测指标**：WER + SIM
- **依赖模型**：Whisper-large-v3（EN ASR）、Paraformer-zh（ZH ASR）、WavLM Speaker Encoder


**目录结构**：

```
seed-tts-eval/
├── en/
│   ├── meta.lst              # 英文零样本 TTS
│   └── non_para_reconstruct_meta.lst  # 英文 VC
├── zh/
│   ├── meta.lst              # 中文零样本 TTS
│   ├── hardcase.lst          # 中文困难样本
│   └── non_para_reconstruct_meta.lst  # 中文 VC
├── eval_metrics.py           # WER + SIM 计算
└── requirements.txt
```

**典型调用流程**：

```python
# 1. 批量推理生成合成语音
# 2. 运行评测脚本
python eval_metrics.py \
    --meta_file zh/meta.lst \
    --gen_dir ./generated_wavs/ \
    --asr_model paraformer-zh \
    --spk_model wavlm-speaker
```

**优点**：轻量、标准化、业界公认

**局限**：仅支持 WER + SIM 两个指标，不含 MOS/PESQ 等

## 二、UltraEval-Audio


⭐**首个同时支持语音理解与语音生成评测的开源框架**

- **来源**：OpenBMB
- **GitHub**：[OpenBMB/UltraEval-Audio](https://github.com/OpenBMB/UltraEval-Audio)
- **覆盖**：34 个权威基准，4 大领域（语音、声音、医疗、音乐），10 种语言，12 个任务类别


### 2.1 核心特性

| **特性** | **说明** |
| --- | --- |
| 一键 Benchmark 管理 | 自动下载和预处理数据集（LibriSpeech、Seed-TTS-Eval 等） |
| 内置评测工具 | 绑定官方评测方法（WER、WER-ZH、BLEU、G-Eval 等） |
| 复现支持 | 提供开源模型的复现文档和命令 |
| 断点续传 | 支持预览测试、随机采样、错误重试、断点恢复 |
| 自定义数据集 | 支持快速接入私有数据集 |

### 2.2 架构概览

```mermaid
graph LR
    A["配置文件<br>YAML"] --> B["数据管理器<br>自动下载/预处理"]
    B --> C["推理引擎<br>模型适配器"]
    C --> D["评测管道<br>WER/SIM/MOS.."]
    D --> E["结果聚合<br>JSON/CSV"]
```

### 2.3 快速上手

```python
# 安装
pip install ultraeval-audio

# 单个基准评测
ultraeval-audio eval \
    --model qwen3-tts \
    --benchmark seed-tts-eval \
    --output results/

# 全量评测
ultraeval-audio eval \
    --model qwen3-tts \
    --benchmark all \
    --output results/
```

## 三、TTSDS2（Text-to-Speech Distribution Score 2）


🆕**分布式语音质量评估框架（2025）**

- **核心思想**：将 TTS 评估视为**分布相似性**任务，比较合成语音集合与真实语音集合的特征分布
- **论文**：[arXiv:2506.19441](https://arxiv.org/abs/2506.19441)


### 3.1 方法原理

传统方法逐样本评分，TTSDS2 则：

1. 提取合成语音集合的特征分布 $\hat{P}(X|D_{syn})$
2. 提取真实语音参考集合的特征分布 $\hat{P}(X|D_{real})$
3. 确保与噪声分布 $\hat{P}(X|D_{noise})$ 保持距离
4. 在多个感知维度上计算综合得分

### 3.2 评估维度

- **韵律**（Prosody）：F0、能量、语速
- **说话人**（Speaker）：音色一致性
- **环境**（Environment）：背景噪声、混响
- **整体质量**（Overall）：综合自然度

### 3.3 优势

- **无需参考音频配对**：只需真实语音集合作参考
- **更鲁棒**：避免单样本评分的随机性
- **跨域泛化**：在 out-of-domain 场景下仍保持与人工评分的高相关性

## 四、F5-TTS 评测模块


📌**随 F5-TTS 开源的完整评测 Pipeline**

- **GitHub**：[SWivid/F5-TTS](https://github.com/SWivid/F5-TTS)
- **覆盖**：Seed-TTS Eval + LibriSpeech-PC test-clean
- **指标**：WER + SIM


**使用流程**：

```bash
# 1. 安装评测依赖
pip install -e .[eval]

# 2. 下载评测模型
# - Paraformer-zh (中文 ASR)
# - Faster-Whisper-large-v3 (英文 ASR)
# - WavLM Speaker Model

# 3. 批量推理
accelerate config
bash src/f5_tts/eval/eval_infer_batch.sh

# 4. 评测 Seed-TTS
python src/f5_tts/eval/eval_seedtts_testset.py

# 5. 评测 LibriSpeech-PC
python src/f5_tts/eval/eval_librispeech_test_clean.py
```

## 五、ESPnet TTS 评测

- **来源**：ESPnet 端到端语音处理工具包
- **特色**：与训练 Pipeline 深度集成，支持 MOS 预测（UTMOS）、PESQ、STOI 等
- **适用**：研究型场景，需要完整训练-评测闭环

## 六、QualiSpeech


🆕**语音质量评估 + 自然语言推理（2025）**

- **创新点**：不仅给出 MOS 分数，还提供**自然语言描述**解释质量问题
- **数据来源**：Blizzard Challenges + VCC Challenges + 10 个最新开源 TTS 模型
- **用途**：深入理解合成语音的具体缺陷

## 七、框架选型指南

| **场景** | **推荐框架** | **理由** |
| --- | --- | --- |
| 零样本 TTS 快速对比 | Seed-TTS-Eval 脚本 | 业界标准，轻量易用 |
| 全面系统评测 | UltraEval-Audio | 34 基准聚合，一键管理 |
| 无参考分布式评估 | TTSDS2 | 更鲁棒，跨域泛化好 |
| Flow Matching TTS 开发 | F5-TTS 评测模块 | 与 F5-TTS 深度集成 |
| 研究型训练-评测闭环 | ESPnet | 完整 Pipeline |
| 质量缺陷分析 | QualiSpeech | NL 推理 + MOS |
