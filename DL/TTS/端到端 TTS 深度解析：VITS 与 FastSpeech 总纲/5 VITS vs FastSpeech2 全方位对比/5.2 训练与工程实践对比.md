## 定位

> 训练复杂度、数据需求、调参难度、部署考量的工程视角对比

---

## 1. 训练复杂度对比

|**维度**|**FastSpeech2 + HiFi-GAN**|**VITS**|
|---|---|---|
|训练步骤|两阶段：先声学后声码器|一阶段：端到端|
|损失函数|简单（Mel MSE + GAN）|复杂（5 个损失项 + 权重调节）|
|外部依赖|MFA + pyworld|无（MAS 自动对齐）|
|调参难度|低|中-高（GAN 平衡、KL 权重）|
|训练时间 (LJSpeech)|~1 天（声学）+ ~1 天（声码器）|~2 天（端到端）|
|GPU 显存|~8GB|~16GB|

```python
# FastSpeech2 训练：简单直接
loss = mel_l1_loss + duration_mse + pitch_mse + energy_mse

# VITS 训练：需要精心平衡
loss_g = recon_loss * 45 + kl_loss * 1 + dur_loss * 1 + adv_loss_g * 1 + fm_loss * 2
loss_d = adv_loss_d  # 判别器单独更新
# 还需要 GAN 训练的技巧：梯度裁剪、学习率调度、判别器延迟启动等
```

> [!important]
> 
> **思辨：VITS 虽然是「一阶段」训练，但并不一定更简单。** 5 个损失项的权重平衡、GAN 训练的不稳定性、KL 坍缩问题等都需要经验。FastSpeech2 虽然是两阶段，但每阶段都很直观。**端到端不等于简单**——这是很多初学者的误区。
